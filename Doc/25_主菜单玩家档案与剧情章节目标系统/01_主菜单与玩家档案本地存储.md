# 25-01 主菜单与玩家档案本地存储

> 目标：实现游戏启动后的主菜单、用户名输入、玩家档案创建和跨平台本地保存。

## 1. 主菜单界面

主菜单只保留两个主按钮：

```text
开始游戏
设置
```

推荐布局：

- 背景：404 哨所地下入口或地表废土远景。
- 标题：`废土生存法则`
- 主按钮：`开始游戏`
- 次按钮：`设置`
- 右下角显示版本号。

## 2. 开始游戏流程

```text
点击开始游戏
-> 检查是否已有玩家档案
-> 无档案：打开用户名输入弹窗
-> 有档案：进入局外主界面或最近保存位置
```

首次创建档案：

```text
输入用户名
-> 校验
-> 创建 PlayerProfile
-> 保存本地
-> 播放开场视频剧情，可跳过
-> 进入第一段“回到404哨所与背景故事”剧情对白
-> 进入序章夜晚出发入口，可查看基础仓库，研究所和高级制造所保持未解锁
```

## 3. 用户名规则

建议规则：

- 长度 2-12 个字符。
- 允许中文、英文、数字、下划线。
- 去除前后空格。
- 禁止空名。
- 本地单机不需要唯一性校验。

失败提示：

```text
请输入 2-12 个字符的用户名
```

## 4. 玩家档案字段

```gdscript
{
    "profile_id": "local_profile_001",
    "username": "玩家名",
    "created_at_unix": 0,
    "last_played_at_unix": 0,
    "current_chapter": 1,
    "surface_day": 0,
    "intro_cinematic_seen": false,
    "world_intro_dialogue_seen": false,
    "first_departure_outpost_dialogue_seen": false,
    "second_day_black_tide_reveal_seen": false,
    "first_return_dialogue_seen": false,
    "shop_loop_unlocked": false,
    "research_station_unlocked": false,
    "advanced_manufacturing_station_unlocked": false,
    "chapter_1_completed": false,
    "currencies": {
        "mine_coin": 0
    }
}
```

`surface_day` 用于统计“用了多少天解锁制造所”。新档初始为 0；每次 loading 完成后玩家按任意按钮正式进入一局地面探索时 +1，Debug 重开不计入。

`shop_loop_unlocked` 和 `research_station_unlocked` 新档初始为 `false`。玩家第一次成功撤离并播放首次返回剧情后，两者一起设置为 `true`，此时才进入正式“白天准备 -> 开店营业 -> 夜晚出发”的经营循环。旧档兼容：如果存档缺少这些字段但 `first_return_dialogue_seen == true`，读取时按 `true` 补齐并保存。`advanced_manufacturing_station_unlocked` 新档初始为 `false`，通过 5000 矿币目标解锁。

## 5. 存储方案

所有平台使用统一接口：

```gdscript
class_name SaveStorageAdapter

func save_profile(profile: Dictionary) -> Dictionary
func load_profile() -> Dictionary
func has_profile() -> bool
func delete_profile_debug_only() -> Dictionary
```

桌面版推荐：

```text
user://profile/profile.json
```

Web 版推荐：

- 仍优先通过 Godot 的 `user://` 抽象写入。
- 如果目标 Web 导出环境无法稳定持久化 `user://`，使用 JS bridge 接 IndexedDB。
- `localStorage` 只作为轻量 fallback，不保存大型仓库数据。

实现时不要在业务代码里判断平台路径，全部交给 `SaveStorageAdapter`。

## 6. 设置按钮

V0.1 设置页可先包含：

- 音量。
- 全屏/窗口。
- 语言预留。
- 返回。

设置数据单独保存：

```text
user://settings/settings.json
```

## 7. AI/MCP 开工提示词

```text
请实现主菜单、用户名输入和玩家档案本地存储。

要求：
1. 启动后进入 MainMenuScene，包含“开始游戏”和“设置”两个按钮。
2. 点击开始游戏后检查本地 PlayerProfile。
3. 无档案时弹出用户名输入，校验 2-12 字符，允许中文/英文/数字/下划线。
4. 创建 PlayerProfile，字段包含 username、current_chapter、surface_day、intro_cinematic_seen、world_intro_dialogue_seen、first_departure_outpost_dialogue_seen、second_day_black_tide_reveal_seen、first_return_dialogue_seen、shop_loop_unlocked、research_station_unlocked、advanced_manufacturing_station_unlocked、chapter_1_completed、currencies。
5. 保存通过 SaveStorageAdapter，桌面默认 user://profile/profile.json。
6. Web 版通过同一接口适配；优先 user://，必要时接 IndexedDB，不在 UI 中写平台判断。
7. 档案创建成功后播放开场视频剧情；视频可跳过，结束或跳过后进入第一段“回到404哨所与背景故事”剧情对白。
8. 第一段“回到404哨所与背景故事”剧情对白完成后进入序章夜晚出发入口；研究所和高级制造所保持锁定，直到首次成功返回剧情结束后开启正式白天经营循环和研究所。
```

## 8. 验收标准

- 第一次启动能输入用户名并创建档案。
- 重启后能读到用户名和章节进度。
- 空用户名不能创建档案。
- UI 不直接读写文件。
- Web 存储路径有适配层，不写死桌面路径。
