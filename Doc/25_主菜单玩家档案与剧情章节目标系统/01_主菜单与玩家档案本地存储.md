# 25-01 主菜单与玩家档案本地存储

> 目标：实现游戏启动后的主菜单、用户名输入、玩家档案创建和跨平台本地保存，并让档案字段与最新昼夜经营流程一致。

## 1. 主菜单界面

主菜单只保留两个主按钮：

```text
开始游戏
设置
```

推荐布局：

- 背景：404 哨所地下入口或地表废土远景。
- 标题：`废土生存法则`
- 主按钮：`开始游戏`
- 次按钮：`设置`
- 右下角显示版本号。

## 2. 开始游戏流程

```text
点击开始游戏
-> 检查是否已有玩家档案
-> 无档案：打开用户名输入弹窗
-> 有档案：进入局外主界面或最近保存位置
```

首次创建档案：

```text
输入用户名
-> 校验
-> 创建 PlayerProfile
-> 保存本地
-> 播放开场视频剧情，可跳过
-> 进入第一段“回到404哨所与背景故事”剧情对白
-> 进入序章夜晚出发入口，可查看基础仓库
```

注意：

- 制造所是默认存在的局外设施，但新档序章第一次出发前不进入白天经营循环。
- 序章第一次出发前，不展示“开店营业”主流程，也不出现旧独立交易页签。
- 不存在额外制造设施的新手目标字段。

## 3. 用户名规则

建议规则：

- 长度 2-12 个字符。
- 允许中文、英文、数字、下划线。
- 去除前后空格。
- 禁止空名。
- 本地单机不需要唯一性校验。

失败提示：

```text
请输入 2-12 个字符的用户名
```

## 4. 玩家档案字段

```gdscript
{
    "profile_id": "local_profile_001",
    "username": "玩家名",
    "created_at_unix": 0,
    "last_played_at_unix": 0,
    "current_chapter": 1,
    "surface_day": 0,
    "intro_cinematic_seen": false,
    "world_intro_dialogue_seen": false,
    "first_departure_outpost_dialogue_seen": false,
    "second_day_black_tide_reveal_seen": false,
    "first_return_dialogue_seen": false,
    "shop_loop_unlocked": false,
    "research_station_unlocked": false,
    "starter_shop_supply_granted": false,
    "first_shop_tutorial_completed": false,
    "chapter_1_goal_active": false,
    "chapter_1_completed": false,
    "currencies": {
        "mine_coin": 0
    }
}
```

字段说明：

- `surface_day`：统计玩家正式进入地表探索的次数。新档初始为 0；每次 loading 完成后玩家按任意按钮正式进入一局地面探索时 +1，Debug 重开不计入。
- `shop_loop_unlocked`：新档初始为 `false`。序章探索结束并播放返回剧情后，无论撤离成功或失败，设置为 `true`，进入正式“白天准备 -> 开店营业 -> 夜晚出发”的经营循环。
- `research_station_unlocked`：新档初始为 `false`。序章返回剧情结束后开启，用于承接局外成长。
- `starter_shop_supply_granted`：新档初始为 `false`。序章返回剧情发放一次性开店周转物资后设置为 `true`。该字段防止重复领取。
- `first_shop_tutorial_completed`：玩家完成第一次制造、上架和开店结算后设置为 `true`。
- `chapter_1_goal_active`：序章返回剧情结束后设置为 `true`，用于展示第一章经营目标。

旧档兼容：

- 如果旧档缺少 `starter_shop_supply_granted`，读取时补 `false`。
- 如果旧档缺少 `first_shop_tutorial_completed`，读取时补 `false`。
- 如果旧档存在 `advanced_manufacturing_station_unlocked`，新流程不再读取它作为章节目标依据；迁移时可保留字段但不在 UI 中展示。
- 如果旧档 `first_return_dialogue_seen == true` 但 `shop_loop_unlocked` 缺失，读取时按 `true` 补齐并保存。

## 5. 存储方案

所有平台使用统一接口：

```gdscript
class_name SaveStorageAdapter

func save_profile(profile: Dictionary) -> Dictionary
func load_profile() -> Dictionary
func has_profile() -> bool
func delete_profile_debug_only() -> Dictionary
```

桌面版推荐：

```text
user://profile/profile.json
```

Web 版推荐：

- 仍优先通过 Godot 的 `user://` 抽象写入。
- 如果目标 Web 导出环境无法稳定持久化 `user://`，使用 JS bridge 接 IndexedDB。
- `localStorage` 只作为轻量 fallback，不保存大型仓库数据。

实现时不要在业务代码里判断平台路径，全部交给 `SaveStorageAdapter`。

- 音量。

## 7. AI/MCP 开工提示词

```text
请实现主菜单、用户名输入和玩家档案本地存储。

要求：
1. 启动后进入 MainMenuScene，包含“开始游戏”和“设置”两个按钮。
2. 点击开始游戏后检查本地 PlayerProfile。
3. 无档案时弹出用户名输入，校验 2-12 字符，允许中文/英文/数字/下划线。
4. 创建 PlayerProfile，字段包含 username、current_chapter、surface_day、intro_cinematic_seen、world_intro_dialogue_seen、first_departure_outpost_dialogue_seen、second_day_black_tide_reveal_seen、first_return_dialogue_seen、shop_loop_unlocked、research_station_unlocked、starter_shop_supply_granted、first_shop_tutorial_completed、chapter_1_goal_active、chapter_1_completed、currencies。
5. 保存通过 SaveStorageAdapter，桌面默认 user://profile/profile.json。
6. Web 版通过同一接口适配；优先 user://，必要时接 IndexedDB，不在 UI 中写平台判断。
7. 档案创建成功后播放开场视频剧情；视频可跳过，结束或跳过后进入第一段“回到404哨所与背景故事”剧情对白。
8. 第一段“回到404哨所与背景故事”剧情对白完成后进入序章夜晚出发入口；此时还没有正式进入白天经营循环。
9. 序章探索结束后，无论撤离成功或失败，返回局外时都要播放序章返回剧情，并在剧情中发放一次性开店周转物资。
10. 不实现旧独立交易入口，不读取旧制造设施分层字段作为新手目标。
```

## 8. 验收标准

- 第一次启动能输入用户名并创建档案。
- 重启后能读到用户名和章节进度。
- 空用户名不能创建档案。
- UI 不直接读写文件。
- Web 存储路径有适配层，不写死桌面路径。
- 新档字段不包含新流程会使用的额外制造设施解锁目标。
- 序章失败返回后仍能通过 `starter_shop_supply_granted` 获得首次开店所需周转物资。
