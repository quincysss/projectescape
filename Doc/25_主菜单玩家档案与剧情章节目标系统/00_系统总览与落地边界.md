# 25-00 主菜单、玩家档案、剧情章节目标系统总览

> 关联文档：`09_局外仓库系统`、`10_研究所与制作所系统_修订版_废土生存法则.md`、`20_UI视觉规范与界面资源Sheet提示词_修订版_废土生存法则.md`、`23_世界观包装_废土生存法则.md`、`32_局外白天经营流程`
> 目标：补齐进入游戏后的主菜单、玩家用户名档案、本地/网页版存档、开场视频、两段新手剧情、序章返回剧情、保底开店周转物资、第一章开店目标和昼夜循环入口。

## 1. 功能闭环

```text
打开游戏
-> 主菜单
-> 点击开始游戏
-> 输入用户名
-> 创建本地玩家档案
-> 播放开场视频剧情，可跳过
-> 第一段剧情对白：回到404哨所与背景故事
-> 进入序章夜晚出发入口，可查看基础仓库
-> 玩家点击出发探索
-> 第二段剧情对白：前哨站、通道损坏、撤离重新激活
-> 出发进入对局加载界面
-> 地图/容器/前哨/撤离/HUD 初始化完成
-> Loading 显示“按下任意按钮继续”
-> 玩家按任意按钮
-> 若为第二日首次进入局内，播放暗潮物质视频与主角独白，期间局内计时冻结
-> 进入可操作局内
-> 玩家撤离成功或撤离失败，进入结算
-> 返回哨所 loading
-> Loading 显示“按下任意按钮继续”
-> 玩家按任意按钮
-> 序章返回剧情
-> 无论序章是否成功带回物资，剧情发放一次性开店周转物资
-> 开启正式白天经营循环和研究所
-> 第一章目标：重启杂货店，完成制造、上架、首次营业结算
-> 白天开店前：制造所加工可售物资、管理仓库、研究升级、查看当日需求榜
-> 白天开店中：将可售物资摆上货台并等待难民购买
-> 白天开店结算
-> 夜晚出发准备
-> 选择角色、地点、装备和消耗品
-> 确认出发，进入下一次地表探索
```

## 2. 系统边界

本系统负责：

- 主菜单入口。
- 用户名输入与玩家档案创建。
- 本地存档抽象。
- Web 版本存档适配方案。
- 开场视频播放和跳过后的流程衔接。
- 剧情对白触发条件。
- 出发进入对局加载进度条和局内初始化编排。
- 序章返回剧情与一次性开店周转物资发放。
- 第一章目标状态。
- 昼夜循环入口解锁。

本系统不负责：

- 仓库物品细节。
- 白天货台成交算法。
- 制造所配方细节。
- 局内战利品和撤离结算。
- 具体装备属性和战斗数值。

## 4. 关键术语

| 术语 | 说明 |
| --- | --- |
| 玩家档案 | 包含用户名、创建时间、章节进度、局外货币、剧情 flag 的本地数据 |
| 暗潮 | 核战后地表无处不在的异常物质，会侵蚀人类 |
| 404 哨所 | 玩家所属地下哨所 |
| 杂货店 | 玩家在避难所内经营的店铺，白天通过售卖制造物资获得矿币 |
| 制造所 | 默认开放的局外加工设施；系统实现可沿用现有“制作所”模块，用于把局内带回的道具加工为可售物资 |
| 周转物资 | 序章结束后剧情发放的一次性保底材料，确保玩家即使撤离失败也能完成首次制造和开店教学 |
| 白天准备阶段 | 玩家制造物资、管理仓库、研究升级、查看当日需求榜的阶段；没有出发探索按钮 |
| 开店阶段 | 玩家把可售物资摆上货台并等待难民购买的阶段，持续 1 分钟，可提前结束 |
| 夜晚出发阶段 | 玩家选择角色、地点、装备和消耗品后进入地表探索 |
| 第一章 | 目标为重启杂货店，完成制造、上架、首次营业结算，并建立“探索 -> 制造 -> 售卖 -> 升级 -> 再探索”的循环 |
| 出发加载 | 玩家确认出发后、局内可操作前的进度条和资源初始化流程 |
| 对白原画 | 剧情对白中随 speaker_id 显示的角色半身原画，不等同于局内 HUD 头像 |

## 5. 硬约束

- 开场视频在用户名创建后播放，可跳过；跳过视频后仍必须进入第一段“回到404哨所与背景故事”对白。
- 第一段“回到404哨所与背景故事”对白在新档创建后播放一次，必须在自然对话中交代 2050 核战后地下生活、暗潮、玩家能短暂上地表和 404 的生存压力，结束后进入序章夜晚出发入口。
- 制造所从新档开始就是默认开放的局外设施，不存在额外制造设施作为第一章目标或固定矿币解锁目标。
- 新档序章首次进入局外后，只开放基础仓库和夜晚出发入口；玩家还没有进入正式白天经营循环。
- 第二段首次出发任务对白只在第一次点击出发探索后播放一次。
- 序章探索结束后，无论玩家撤离成功、撤离失败、是否带回物资，都必须触发序章返回剧情。
- 序章返回剧情结束时必须发放一次性开店周转物资；该奖励只发一次，不能被反复领取。
- 周转物资必须通过剧情包装发放，不能表现为无来源的系统补偿。包装方向：404 哨所把仓底材料、居民寄卖的旧料和玩家带回/没带回的残余资源凑成第一天开店本钱。
- 序章返回剧情结束时开启正式白天经营循环和研究所，再显示“前往制造所/查看需求/开始营业”指引。
- 第一章目标不再是“赚取固定数额矿币”或“解锁额外制造设施”，而是完成首次经营闭环：制造可售物资、上架货台、完成第一次开店结算。
- 白天准备阶段没有“出发探索”按钮，只有“开店营业”按钮。
- 开店阶段玩家可以继续管理货台；若货台无货且仓库没有可上架物资，必须提供“提前结束营业”按钮并进入当日开店结算。
- 出发加载完成且玩家按任意按钮提交进入地表日后，才允许 `surface_day +1`；如果随后触发第二日暗潮物质剧情，计时和操作仍要等剧情结束后才启动。
- 出发加载到 100% 后必须停留在 loading 界面，显示“按下任意按钮继续”，不能自动进局。
- 第二日暗潮物质剧情只在 `run_day_index == 2` 且未播放时触发；剧情期间必须冻结对局倒计时、稳定值、视野、容器、材料、前哨、撤离等所有局内计时，对白结束后才启动本局玩法。
- 出发加载失败时必须回滚待出发装备，不能消耗局外资源。
- 存档读写必须通过统一 SaveStorageAdapter，不允许 UI 直接写文件。
- Web 版本不能依赖桌面绝对路径，必须走 Web 存储适配。
- 剧情对白必须支持主角和 404 哨所管理员原画；对白正文来自 `setting/dialogues.tab`，角色展示信息来自 `setting/dialogue_speakers.tab`。
- 剧情对白人物立绘必须统一放在屏幕最左侧并左对齐，不使用主角右侧、管理员左侧的左右分站位。
- 对白左侧主立绘槽位中心固定为 `PORTRAIT_SLOT_CENTER_X_RATIO := 0.26`，后续实现、重构或 AI 重写不得随意调整。
