# 25-00 主菜单、玩家档案、剧情章节目标系统总览

> 关联文档：`09_局外仓库系统`、`10_研究所与制作所系统_修订版_废土生存法则.md`、`20_UI视觉规范与界面资源Sheet提示词_修订版_废土生存法则.md`、`23_世界观包装_废土生存法则.md`
> 目标：补齐进入游戏后的主菜单、玩家用户名档案、本地/网页版存档、开场视频、两段新手剧情、首次返回剧情、第一章目标和制造所解锁闭环。

## 1. 功能闭环

```text
打开游戏
-> 主菜单
-> 点击开始游戏
-> 输入用户名
-> 创建本地玩家档案
-> 播放开场视频剧情，可跳过
-> 第一段剧情对白：回到404哨所与背景故事
-> 进入局外仓库/商人/出发准备界面
-> 玩家点击出发探索
-> 第二段剧情对白：前哨站、通道损坏、撤离重新激活
-> 出发进入对局加载界面
-> 地图/容器/前哨/撤离/HUD 初始化完成
-> Loading 显示“按下任意按钮继续”
-> 玩家按任意按钮
-> 进入可操作局内
-> 修复两处前哨并撤离
-> 首次返回剧情对白
-> 第一章目标：赚取 5000 矿币解锁制造所
-> 解锁制造所
-> 弹窗显示用了多少天
-> 第一章结束
```

## 2. 系统边界

本系统负责：

- 主菜单入口。
- 用户名输入与玩家档案创建。
- 本地存档抽象。
- Web 版本存档适配方案。
- 开场视频播放和跳过后的流程衔接。
- 剧情对白触发条件。
- 出发进入对局加载进度条和局内初始化编排。
- 第一章目标状态。
- 制造所解锁条件和章节结束弹窗。

本系统不负责：

- 仓库物品细节。
- 商人出售算法。
- 制作所配方细节。
- 局内战利品和撤离结算。

## 4. 关键术语

| 术语 | 说明 |
| --- | --- |
| 玩家档案 | 包含用户名、创建时间、章节进度、局外货币、剧情 flag 的本地数据 |
| 暗潮 | 核战后地表无处不在的异常物质，会侵蚀人类 |
| 404 哨所 | 玩家所属地下哨所 |
| 制造所 | 剧情文案称“制造所”，系统实现可沿用现有“制作所”模块 |

| 出发加载 | 玩家确认出发后、局内可操作前的进度条和资源初始化流程 |

## 5. 硬约束

- 开场视频在用户名创建后播放，可跳过；跳过视频后仍必须进入第一段“回到404哨所与背景故事”对白。
- 第一段“回到404哨所与背景故事”对白在新档创建后播放一次，必须在自然对话中交代 2050 核战后地下生活、暗潮、玩家能短暂上地表和 404 的生存压力，结束后进入局外界面。
- 第二段首次出发任务对白只在第一次点击出发探索后播放一次。
- 首次返回剧情只在第一次成功撤离回家后播放。
- 制造所解锁花费暂定 5000 矿币。
- 章节结束弹窗必须显示玩家用了多少天。
- 出发加载完成并正式进入可操作局内后，才允许 `surface_day +1`。
- 出发加载到 100% 后必须停留在 loading 界面，显示“按下任意按钮继续”，不能自动进局。
- 出发加载失败时必须回滚待出发装备，不能消耗局外资源。
- 存档读写必须通过统一 SaveStorageAdapter，不允许 UI 直接写文件。
- Web 版本不能依赖桌面绝对路径，必须走 Web 存储适配。
