# 25-07 出发进入对局加载进度条

> 目标：在玩家确认出发后、局内正式可操作前，增加一段带进度条的加载界面，用来承接地图、容器、前哨、撤离、背包快照、音频和首帧渲染等局内初始化工作。

## 1. 功能定位

出发加载界面不是装饰性转场，而是一次正式的“局内启动编排”。

同一套 loading 条机制也允许被结算返回哨所流程复用，但两者必须使用不同 mode，避免提交逻辑混用。

```text
ENTER_RUN：出发进入对局，loading 完成后按任意按钮进入局内。
RETURN_TO_BASE：结算后返回哨所，loading 完成后按任意按钮回到局外。
```

它需要解决三件事：

- 让玩家明确知道“已经出发，正在进入地面”。
- 给 Godot 局内场景、数据和渲染初始化留出安全窗口。
- 防止玩家在地图、容器、前哨或 HUD 尚未完成时提前获得输入控制。

推荐流程：

```text
玩家点击确认出发
-> LoadoutManager 生成待出发装备/背包快照
-> 若首次出发任务对白未播放，则先播放 first_departure_outpost_dialogue
-> RunLoadingScreen 打开
-> RunLoadingController 逐步加载局内资源和运行状态
-> 局内首帧渲染完成
-> Loading 进度到 100%
-> 显示“按下任意按钮继续”
-> 玩家按任意按钮
-> 提交正式进入对局状态
-> 若 run_day_index == 2 且第二日暗潮物质剧情未播放，则进入 RunScene 后先播放视频与主角独白，期间局内计时冻结
-> 关闭加载界面，进入可操作局内
```

## 2. 触发时机

触发入口：

```text
夜晚出发准备界面
-> 玩家点击“确认出发”
-> 系统校验装备、负重、背包空间
-> 若 first_departure_outpost_dialogue_seen=false，播放首次出发任务对白
-> 进入出发加载界面
```

首次新档特殊规则：

```text
开场视频和第一段“回到404哨所与背景故事”对白发生在新档创建后。
这里不再播放第一段剧情对白。
这里仅处理首次出发任务对白 first_departure_outpost_dialogue。
```

`surface_day` 计数规则：

- 只有加载成功、玩家按任意按钮、正式提交进入地表日时，`surface_day +1`。
- 如果提交后触发第二日暗潮物质剧情，`surface_day` 已经计入第二日，但本局倒计时、稳定值、视野和交互计时仍要等剧情结束后才开始。
- Debug 重开、加载失败、玩家在加载前取消，都不增加 `surface_day`。
- 第一章结束弹窗读取的天数仍以 `surface_day` 为准。

## 3. 加载阶段与进度权重

进度条采用“任务权重 + 显示平滑”的方式。不要做纯假的倒计时，也不要把进度直接绑死到单个资源加载百分比。

| 阶段 | 进度区间 | 玩家可见文案 | 系统任务 |
| --- | ---: | --- | --- |
| lock_departure | 0%-5% | 校验出发装备... | 锁定输入，生成待出发 loadout_snapshot |
| build_run_context | 5%-15% | 升降通道启动中... | 生成 run_id、run_seed、难度、场景随机事件、时间、角色出生点 |
| load_map | 15%-30% | 同步地表地图... | 加载 RunScene、地图、TileMap、碰撞、导航区域 |
| load_player | 30%-42% | 整备随身物资... | 实例化玩家、背包、装备、负重、快捷栏 |
| init_systems | 42%-58% | 暗潮读数稳定中... | 初始化稳定值、实际对局时长、视野、输入锁、局内状态机 |
| init_outposts | 58%-70% | 扫描前哨信号... | 初始化前哨站、前哨材料需求、撤离解锁状态 |
| init_loot | 70%-82% | 标记容器与资源信号... | 初始化地图容器清单、战利品表和稀有物资权重 |
| preload_ui_audio | 82%-90% | 接入作战界面... | 预加载 RunHUD、目标链 HUD、背包面板、音频、常用 VFX |
| prewarm_frame | 90%-98% | 渲染地表首帧... | 等待资源实例化完成，至少等待 1 个 process_frame 和 1 个 physics_frame |
| ready_to_continue | 98%-100% | 按下任意按钮继续 | 首帧预热完成，停留在 loading 界面，等待玩家输入 |

显示进度只允许追赶真实目标进度：

```text
target_progress = 当前已完成任务权重总和
display_progress 平滑追赶 target_progress
真实任务未完成时，display_progress 不能到 100%
```

## 4. 局内加载内容清单

必须放入加载流程的内容：

- `RunScene` 主场景。
- 地图 TileMap、地表装饰、碰撞层、导航层。
- 玩家场景、摄像机、输入控制器。
- 局外带入装备和背包快照。
- 局内背包容量、负重、装备属性加成。
- 场景随机事件结果。
- 稳定值/暗潮倒计时/局内实际时间。
- 家中撤离点、前哨站、撤离解锁状态。
- 前哨材料需求和目标链 HUD 初始状态。
- 地图容器清单、容器内容、稀有度表和 S 级稀有物资权重结果。
- 怪物点位抽取结果、随机障碍点位抽取结果。
- 交互描边服务、可交互对象注册表。
- RunHUD、背包 UI、容器 UI、撤离读条 UI。
- 局内环境音、警告音、打开容器音效、撤离音效。

可以延迟到进入局内后再懒加载的内容：

- 未靠近区域的远端装饰音效。
- 非首屏可见的容器打开特效。
- 后续章节才出现的特殊事件 UI。
- Debug 专用覆盖层。

## 5. UI 排版规则

界面结构：

```text
全屏暗色背景
顶部/中上：正在前往地面
中部：当前阶段文案
下方：横向进度条
进度条下：百分比或“按下任意按钮继续”
左下/右下：操作提示
底部：稳定值/视野机制提示
```

V0.1 推荐文案：

```text
标题：正在前往地面
阶段：同步地表地图...
进度：42%
提示：前哨材料以菱形标记，收集足够后优先修复前哨站。
```

加载完成态文案：

```text
标题：正在前往地面
阶段：按下任意按钮继续
进度：100%
操作：WASD 移动 / Tab 背包 / F 拾取与交互 / E 撤离
提示：暗潮会压缩视野，稳定值会持续下降；回到基地后稳定值会恢复。
```

操作提示必须出现在 loading 界面上：

| 按键 | 文案 | 说明 |
| --- | --- | --- |
| `WASD` | 移动 | 玩家基础移动 |
| `Tab` | 背包 | 打开/关闭背包界面 |
| `F` | 拾取/交互 | 拾取物品、打开容器、提交前哨材料等 |
| `E` | 撤离 | 撤离点激活后按住或确认撤离 |

机制提示推荐轮播或分行显示：

```text
暗潮会逐渐压缩你的视野。
在地表停留越久，稳定值下降越多。
稳定值过低时，请优先寻找撤离机会。
成功回到基地后，稳定值会恢复。
先找菱形修复件，修复前哨站，再撤离。
```

视觉要求：

- 使用暗灰、黑色、脏金属、旧纸片和手绘粗线框。
- 进度条主色可用低饱和黄/橙，危险阶段可短暂使用红色脉冲。
- 文字必须由 Godot UI 渲染，不烘焙进 PNG。
- 进度条不要做现代科技蓝光风格。
- 背景可以是升降通道、地表闸门或 404 哨所通道的静态/轻动画画面。
- 不显示调试任务名、资源路径、报错堆栈给普通玩家。
- 操作提示可以使用小键帽图标，但文字必须由 Godot UI 渲染。

交互规则：

- 加载中不显示跳过按钮。
- 加载中屏蔽移动、背包、攻击、交互和暂停以外的输入。
- `Esc` 可打开最小暂停菜单，但 V0.1 可先不做。
- 加载完成后不能自动进入局内。
- 加载完成后阶段文本改为 `按下任意按钮继续`。
- 任意键/鼠标/手柄确认输入都可继续，但 Debug 覆盖层输入不应误触发。
- 玩家按任意按钮后，才关闭 loading、提交进入对局并启用控制。
- 如果加载时间短于 1 秒，仍保留最短 1 秒显示时间，避免闪屏。

返回哨所 loading 交互规则：

- 由结算界面的 `返回哨所` 按钮触发。
- 不显示 WASD/Tab/F/E 操作教学。
- 不增加 `surface_day`。
- 不提交出发装备。
- 不启用局内控制。
- loading 到 100% 后同样停留，显示 `按下任意按钮继续`。
- 玩家按任意按钮后，切回局外仓库/哨所界面。

## 6. 失败与回滚规则

加载失败分两类。

可恢复失败：

```text
地图资源缺失
容器配置校验失败
玩家场景实例化失败
HUD 场景加载失败
```

处理：

- 显示失败弹窗：`地表通道同步失败，请返回哨所重试。`
- 返回局外出发准备界面。
- 已配置装备恢复为出发前状态。
- 不增加 `surface_day`。
- 不写入正式 `run_start` 存档。
- Debug 模式下记录详细错误。

不可恢复失败：

```text
存档写入失败
核心配置表无法读取
玩家档案损坏
```

处理：

- 显示系统错误弹窗。
- 返回主菜单或 BootScene。
- 不允许进入局内。
- Debug 模式输出错误来源。

## 8. 与出发准备系统的交接

`LoadoutManager` 不应该直接切到 `RunScene`。它只负责生成待出发快照并交给 `RunLoadingController`。

推荐交接：

```text
LoadoutManager.confirm_departure()
-> validate_loadout()
-> play first_departure_outpost_dialogue if needed
-> create_loadout_snapshot()
-> mark_items_pending_run()
-> GameFlowController.request_enter_run(loadout_snapshot)
-> RunLoadingController.begin_run_loading(loadout_snapshot, profile)
```

提交时机：

```text
加载成功且首帧准备完成
-> RunLoadingScreen 显示“按下任意按钮继续”
-> 玩家按任意按钮
-> LoadoutManager.commit_pending_run()
-> ProfileService.increment_surface_day()
-> SaveService.save_run_start(run_context)
-> GameFlowController.switch_to_run_scene()
-> RunStoryGateController.try_play_run_entry_story(run_context)
-> 若无入局剧情或入局剧情结束，RunSimulationPauseService 释放 story_pause_token
-> RunTimerController / StabilityDrain / VisionShrink / 交互计时正式启动
```

失败回滚：

```text
RunLoadingController 失败
-> LoadoutManager.rollback_pending_run()
-> GameFlowController.return_to_loadout()
```

## 11. 验收标准

- 点击确认出发后先进入加载界面，而不是直接进局。
- 加载界面显示标题、阶段文案、进度条、百分比和提示。
- 加载界面显示 WASD、Tab、F、E 操作提示。
- 加载界面显示视野收缩、稳定值下降、回基地恢复稳定值的简短说明。
- 地图、玩家、背包、前哨、容器、撤离、HUD 完成初始化后才允许操作。
- 场景随机事件完成解析后才允许进入局内。
- 首次出发任务对白播放完成后才进入加载界面。
- 加载到 100% 后不自动进局，必须显示“按下任意按钮继续”。
- 玩家按任意按钮进入局内后，`surface_day +1`。
- 第 2 日首次进入局内会播放暗潮物质剧情，剧情期间倒计时和所有交互计时冻结，结束后才正式开始。
- 加载失败不增加 `surface_day`，装备回滚到出发前状态。
- 进度条不会在真实任务未完成前到 100%。
- 加载完成后局内首帧不黑屏、不缺 HUD、不缺碰撞。
- Debug 模式可模拟慢加载和加载失败。
- 结算返回哨所 loading 不增加 `surface_day`，且按任意按钮后回到局外。
