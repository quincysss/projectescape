# 25-04 高级制造所解锁与第一章结束

> 目标：定义第一章最终目的：为了救出妹妹，花费 5000 矿币解锁高级制造所，并显示用了多少天。

## 1. 解锁条件

```text
chapter_1_goal_active == true
advanced_manufacturing_station_unlocked == false
mine_coin >= 5000
```

高级制造所解锁费用：

```text
5000 mine_coin / 5000 矿币
```

矿币来源：
- 白天开店营业的货台成交。
- 系统奖励或剧情奖励，后续扩展。

不得作为矿币来源：
- 仓库原材料直接变现。
- 结算界面直接变现。
- 旧独立交易入口。

## 2. UI 状态

制造所入口在未解锁时仍可进入：

```text
基础手作台：可用，用于制造 C 级可售物资
高级制造所：未解锁
需要矿币：current / 5000
```

按钮：

```text
解锁高级制造所
```

按钮状态：

| 条件 | 状态 |
| --- | --- |
| 矿币不足 | 置灰，显示缺少矿币 |
| 矿币足够 | 可点击 |
| 已解锁 | 高级制造所分区正常可进入 |

## 3. 解锁流程

```text
点击解锁高级制造所
-> 二次确认
-> 扣除 5000 矿币
-> advanced_manufacturing_station_unlocked = true
-> chapter_1_completed = true
-> 保存档案和货币
-> 显示第一章结束弹窗
```

确认弹窗：

```text
确认解锁高级制造所？
将消耗 5000 矿币。

取消 / 确认解锁
```

## 4. 第一章结束弹窗

标题：

```text
第一章结束
```

正文：

```text
你用了 {surface_day} 天，终于让高级制造所重新运转。
也许，救出妹妹的路终于有了第一盏灯。

第一章节结束，后续章节开发中
```

按钮：

```text
继续游戏
重新开始
```

## 5. 天数统计

`surface_day` 统计规则：

- 每次正式进入地面探索时 +1。
- Debug 进入不计。
- 失败局也计入天数，因为玩家确实消耗了一天地面机会。
- 弹窗使用解锁瞬间的 `surface_day`。

## 6. 失败与原子性

扣币和解锁必须原子化：

```text
检查矿币
-> 扣除 5000
-> 写入高级制造所解锁 flag
-> 保存
```

如果保存失败：
- 不显示章节结束成功。
- 回滚或提示保存失败。
- 不得出现扣币成功但功能未解锁的状态。

## 7. AI/MCP 开工提示词

```text
请实现第一章高级制造所解锁和章节结束。
要求：
1. 第一章目标文案为“救出妹妹”，当前功能目标为解锁高级制造所，费用 5000 mine_coin。
2. 基础手作台默认可用，高级制造所未解锁时显示锁定状态和矿币 current / 5000。
3. 矿币不足时按钮置灰；矿币足够时可点击。
4. 点击后显示确认弹窗，确认后原子化扣除 5000 矿币并设置 advanced_manufacturing_station_unlocked=true、chapter_1_completed=true。
5. 解锁成功后弹窗显示“第一章结束”，正文包含 surface_day。
6. 弹窗按钮包含“继续游戏”和“重新开始”。
7. 解锁和扣币必须通过 CurrencyWallet/ChapterProgressService，不允许 UI 直接改 currencies。
8. 不实现仓库原材料直接变现，不实现旧独立交易入口。
```

## 8. 验收标准

- 5000 矿币不足不能解锁。
- 5000 矿币足够时能解锁。
- 解锁后扣除矿币并保存。
- 第一章结束弹窗显示正确天数。
- 第一章结束弹窗能承接“救出妹妹”的章节动机。
- 重启后高级制造所仍保持解锁。
- 基础手作台在解锁前仍可制造 C 级可售物资。
