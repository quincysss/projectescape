# 25-08 剧情对白角色原画显示规则

> 目标：让剧情对白在角色说话时显示对应角色原画，先支持主角和 404 哨所管理员，后续可扩展更多 NPC。

## 1. 实现目标

建立剧情对白通用角色原画系统。

V0.1 必须支持：

- 主角说话时显示主角原画。
- 404 哨所管理员说话时显示管理员原画。
- 两人对话时，当前说话者高亮，未说话者弱化。
- 玩家独白时，只显示主角原画。
- 第二日暗潮物质遭遇剧情属于玩家独白，只显示主角原画，不显示管理员。
- 白幕/黑幕/视频结束后的对白都使用同一套 DialoguePanel。

该功能用于剧情对白，不用于局内 HUD 头像，也不用于背包角色预览。

## 2. 角色定义

V0.1 固定两名说话角色：

| speaker_id | 显示名 | 定位 | 默认站位 |
| --- | --- | --- | --- |
| `player` | 玩家用户名 | 404 哨所成员，能短暂抵御暗潮的地表探索者 | left |
| `operator_404` | 404 哨所管理员 | 哨所调度、任务说明、资源压力传达者 | left |

兼容说明：

- 现有 `dialogues.tab` 中的 `operator_404 / 404 哨所调度员` 不需要立刻改 ID。
- 运行时显示名优先使用 `dialogue_speakers.tab.display_name`，可将 `operator_404` 显示为 `404 哨所管理员`。
- `speaker_id == player` 时，名称优先显示玩家输入的用户名；如果用户名为空，显示 `你`。

## 3. 数据表设计

新增：

```text
setting/dialogue_speakers.tab
```

表头：

```text
speaker_id	display_name	portrait_path	portrait_side	nameplate_color	enabled_version	notes
```

示例：

```text
player	玩家	res://assets/characters/dialogue/player/player_dialogue_bust_01.png	left	#c8d4dc	v0.1	主角默认对白半身原画
operator_404	404 哨所管理员	res://assets/characters/dialogue/operator_404/operator_404_dialogue_bust_01.png	left	#d6c07a	v0.1	兼容 dialogues.tab 中的 operator_404
```

读取规则：

- `dialogues.tab` 仍然是对白正文唯一来源。
- `dialogue_speakers.tab` 只存说话人展示信息。
- `DialogueService` 按 `speaker_id` 关联 `dialogue_speakers.tab`。
- 如果找不到 speaker 配置，仍可用 `dialogues.tab.speaker_name` 和默认剪影占位图播放对白。
- `portrait_path` 缺失时，Debug 模式打印 warning，正式模式显示默认剪影，不阻断剧情流程。

## 4. 资源路径

推荐：

```text
assets/characters/dialogue/player/player_dialogue_bust_01.png
assets/characters/dialogue/operator_404/operator_404_dialogue_bust_01.png
assets/characters/dialogue/common/dialogue_portrait_placeholder.png
```

原画规格：

| 项 | 要求 |
| --- | --- |
| 输出尺寸 | 1024x1024 或 1024x1536 |
| 内容 | 半身或大半身立绘，腰部以上优先 |
| 背景 | 透明 PNG |
| 构图 | 角色主体完整，不裁头，不裁肩 |
| 视角 | 正面或 3/4 正面 |
| 风格 | 灰黑手绘漫画感，废土质感，小面积蓝紫/暖黄点缀 |
| 边缘 | 干净透明边缘，适合 Godot UI 叠放 |

不要把角色原画烘焙进对白框背景图。角色原画必须是可独立替换的 PNG。

## 5. 视觉布局

DialoguePanel 推荐结构：

```text
DialoguePanel
  BackgroundDim
  PortraitLayer
    LeftPortraitCurrent
    LeftPortraitInactive
  DialogueBox
    SpeakerName
    DialogueText
    NextIndicator
  SkipButton
```

布局规则：

- 所有角色原画统一放在屏幕最左侧，使用左边缘锚点左对齐。
- 不再使用“主角右侧、管理员左侧”的对称站位。
- 当前说话者显示在左侧主立绘位，`anchor_left = 0`，距离屏幕左边缘建议 16-32 px。
- 左侧主立绘槽位中心使用固定常量 `PORTRAIT_SLOT_CENTER_X_RATIO := 0.26`；这是已人工校准的视觉参数，后续实现、重构、响应式适配或 AI 重写时不得改成 0.25、边距推导值或其他自适应猜测值。
- 非当前说话者如需保留，可显示在同一左侧立绘区域的后层或轻微右移 24-48 px，并明显弱化；V0.1 也允许直接隐藏非当前说话者，只显示当前说话者。
- 当前说话者 alpha = 1.0，轻微提亮。
- 未说话者 alpha = 0.35-0.5，降低饱和度。
- 当前说话者可以有 4-8 px 的轻微描边或背光。
- 对白框位于屏幕底部，占屏幕高度约 24%-30%。
- 角色原画底部可被对白框遮住一小部分，但不能遮住脸和主要表情。
- 720p 下原画显示高度建议 430-520 px；1080p 下 620-780 px。
- 移动端或窄屏只显示当前说话者，避免双人立绘挤压文本。

玩家独白：

- 只显示主角左侧左对齐原画。
- 不显示管理员原画。
- 背景根据剧情发生地点暗化处理；局外独白可使用仓库/住处，第二日暗潮物质独白使用局内首帧暗化，不需要额外 NPC。

## 6. 交互规则

- 每切换一句对白时，根据当前 `speaker_id` 切换当前说话者。
- 同一角色连续说话时，不重复播放大幅入场动画。
- 角色第一次出现在当前对白段时，可做 0.12-0.18 秒淡入。
- 切换说话者时，只做亮度/alpha 切换，不要让立绘来回大幅滑动。
- 点击/按键推进对白时，先补完逐字文本；文本已完成时进入下一句。
- 跳过当前对白段时，隐藏 DialoguePanel 和所有原画。
- 播放开场视频时不显示原画；视频结束或跳过后进入 DialoguePanel 才显示。

## 7. 美术口径

主角原画：

```text
404 哨所地表探索者，20-30 岁，长期地下生活后的疲惫感，穿旧防护外套和轻量背包，胸前有简易呼吸过滤装置或旧式防护围巾，衣物有灰尘、磨损和修补痕迹，眼神警觉但克制，少量蓝紫电子模块，废土手绘漫画风，透明背景，半身立绘。
```

404 哨所管理员原画：

```text
404 哨所管理员/调度员，长期负责地下哨所通讯和物资调度，神情克制、可靠、略显疲惫，穿旧制服或维修背心，肩部可有 404 标识，耳机/通讯器/旧终端配件，暖黄色工作灯反光，灰黑废土手绘漫画风，透明背景，半身立绘。
```

禁止：

- 过度明亮的赛博朋克霓虹。
- 干净崭新的军装。
- 夸张二次元大表情。
- 现代校园或都市服装。
- 与局内俯视 sprite 混用。

## 8. Godot 落地范围

推荐新增：

```text
systems/dialogue/dialogue_speaker_registry.gd
systems/dialogue/dialogue_speaker_definition.gd
systems/dialogue/dialogue_portrait_controller.gd
scenes/ui/dialogue/DialoguePanel.tscn
scenes/ui/dialogue/DialoguePortraitView.tscn
```

`DialogueSpeakerRegistry`：

- 读取 `setting/dialogue_speakers.tab`。
- 提供 `get_speaker(speaker_id)`。
- 对 `player` 返回玩家用户名和主角原画。
- 缺配置时返回 placeholder。

`DialoguePortraitController`：

- 根据当前对白行 `speaker_id` 更新左侧原画。
- 管理当前说话者高亮和非说话者弱化。
- 处理首次淡入、隐藏、跳过。
- 不负责读取对白正文。

`DialoguePanel`：

- 展示背景暗化、角色原画、说话人名、对白正文、继续提示和跳过按钮。
- 不硬编码 `player` 或 `operator_404` 的资源路径。
- 保留 `PORTRAIT_SLOT_CENTER_X_RATIO := 0.26` 作为左侧主立绘槽位中心，常规 UI 调整不得修改该常量。

## 10. 验收标准

| 编号 | 标准 |
| --- | --- |
| A1 | 第一段剧情中 `operator_404` 说话时显示 404 哨所管理员原画 |
| A2 | 第一段剧情中 `player` 说话时显示主角原画 |
| A3 | 双人对白时当前说话者高亮，另一方弱化 |
| A4 | 首次返回玩家独白只显示主角原画 |
| A5 | 第二日暗潮物质遭遇独白只显示主角原画，并使用局内首帧暗化背景 |
| A6 | `player` 名称能显示玩家输入的用户名 |
| A7 | `dialogues.tab` 仍然只存对白正文，不硬编码资源 |
| A8 | `dialogue_speakers.tab` 缺资源时使用 placeholder，不阻断剧情 |
| A9 | 原画不遮挡对白正文、跳过按钮和继续提示 |
| A10 | 跳过对白或对白结束后，原画正确隐藏 |
| A11 | 所有对白立绘都位于屏幕最左侧并左对齐，不出现主角右侧站位 |
| A12 | `PORTRAIT_SLOT_CENTER_X_RATIO` 保持 `0.26`，常规实现和重构不改动该值 |
| A13 | 窄屏下不出现双立绘挤压文本 |

## 11. 暂不实现

- 表情差分。
- 嘴型动画。
- 多套服装。
- 角色好感度演出。
- 多主角选择。
- 全身 Live2D。
