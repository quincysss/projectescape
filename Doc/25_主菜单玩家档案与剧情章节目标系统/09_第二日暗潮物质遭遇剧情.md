# 25-09 第二日暗潮物质遭遇剧情

> 目标：玩家第二日首次进入局内时，播放一段可跳过的视频动画，并在视频后播放主角独白，让玩家第一次明确看到“暗潮物质”已经出现在场景中。整个演出期间局内玩法计时冻结，对白结束后才正式开始本局行动。

## 1. 功能定位

这是一次性局内前置剧情，用于把第二日的地图变化包装成叙事节点。

它不是场景随机事件：

- 不改变 `26_怪物与场景随机事件系统` 中“第 1/2 日不自然触发随机事件”的规则。
- 不开启怪物、超级时间、超短时间或随机障碍事件池。
- 只在第二日首次进入局内时触发一次。
- 暗潮物质可以作为剧情指定的场景物件出现，但不走随机事件抽取。

## 2. 触发条件

触发条件：

```text
PlayerProfile.second_day_black_tide_reveal_seen == false
RunContext.run_day_index == 2
RunLoadingController mode == ENTER_RUN
RunLoadingScreen 已到 100%
玩家已按下任意按钮继续
RunScene 首帧已完成渲染
```

不触发：

- 第 1 日。
- 第 3 日及以后。
- 结算返回哨所 loading。
- 出发加载失败。
- Debug 重开但未走正式 ENTER_RUN 提交流程。
- `second_day_black_tide_reveal_seen == true` 的旧档或已播放档。

## 3. 流程顺序

推荐流程：

```text
RunLoadingScreen 显示 100%
-> 玩家按下任意按钮继续
-> 提交 pending_run，surface_day +1，保存 run_start
-> 切入 RunScene
-> RunStoryGateController 判断 run_day_index == 2 且未播放
-> 创建 story_pause_token
-> 冻结局内计时、输入和交互
-> 播放第二日暗潮物质视频，可跳过
-> 视频结束或跳过后，播放 player 独白对白
-> 对白结束后写入 second_day_black_tide_reveal_seen = true
-> 保存档案
-> 释放 story_pause_token
-> 启用玩家控制、局内倒计时、稳定值衰减和所有交互计时
```

关键点：

- `surface_day +1` 仍发生在玩家按下 loading 完成态的任意按钮之后。
- 剧情只冻结本局玩法开始，不回滚已经提交的出发状态。
- 剧情播放期间玩家已经“进入第二日地表”，但尚未获得行动权。
- 如果视频被跳过，仍然必须播放独白。

## 4. 视频动画

推荐配置：

```text
cinematic_id: second_day_black_tide_reveal
resource_path: res://assets/cinematics/source/second_day_black_tide_reveal_720p.mp4
fallback_resource_path: res://assets/cinematics/second_day_black_tide_reveal_720p.ogv
skip_label: 跳过
next_flow: second_day_black_tide_reveal_dialogue
```

视频内容方向：

- 镜头从玩家落地后的街道低角度扫过。
- 地面裂缝、墙角、废车底部出现黑色潮状附着物。
- 暗潮物质不是明亮魔法特效，而是像油污、烟雾、凝结阴影混合的异常残留。
- 可以有轻微脉动、拉丝、吞光感。
- 不要提前展示怪物，不要做战斗演出。

视频可跳过，但跳过只跳过视频，不跳过后续独白。

## 5. 独白对白

对白 ID：

```text
second_day_black_tide_reveal_dialogue
```

说话人：

```text
player
```

演出形式：

- 使用 `DialoguePanel`。
- 只显示主角原画。
- 不显示 404 哨所管理员原画。
- 背景保持当前 RunScene 暗化，暗潮物质在场景中可见。
- 对白期间玩家不可移动，不可拾取，不可打开背包。

润色后的对白正文：

```text
[player]
这是……

[player]
这些就是暗潮物质？

[player]
昨天还只是读数异常，今天已经爬到街面上来了。

[player]
暗潮压得越来越近。今天的搜集不会像上次那么顺。

[player]
别碰它们，绕开走。能带回去的，才算真的到手。
```

设计意图：

- 第一、二句保留玩家第一次目击的停顿和确认感。
- 第三句说明第二日相比第一日的变化。
- 第四句把气氛转化成玩法压力：搜集难度提高。
- 第五句给出明确行动原则：避开暗潮物质。

## 6. 计时冻结规则

剧情期间必须冻结：

- 对局总倒计时。
- 稳定值下降计时。
- 视野收缩计时。
- 容器打开读条和冷却。
- 前哨材料拾取/提交读条。
- 前哨修复读条。
- 撤离读条。
- 怪物巡逻、警戒、冲撞和补刷计时，如果 Debug 强制让第二日有怪物。
- 随机事件内部计时，如果 Debug 强制注入。
- 任何会消耗玩家局内时间或资源的交互计时。

表现规则：

- 顶部倒计时可显示初始时间，但数字不跳动。
- 稳定值和视野不变化。
- 容器、材料、前哨、撤离交互按钮不响应。
- 玩家移动、背包、拾取、交互、撤离输入全部屏蔽。
- 只允许剧情推进、视频跳过、对白推进/跳过输入。

推荐实现：

```text
RunStoryGateController
-> RunSimulationPauseService.acquire("second_day_black_tide_reveal")
-> play cinematic/dialogue
-> RunSimulationPauseService.release("second_day_black_tide_reveal")
```

所有局内计时器必须读取统一暂停状态，不能只暂停 UI 倒计时。

## 7. 暗潮物质场景规则

第二日剧情中可以强制布置少量暗潮物质点位。

规则：

- 暗潮物质必须在 RunScene 首帧后可见，便于视频和对白承接。
- 暗潮物质应放在玩家出生点附近能看见、但不会立刻碰撞的位置。
- 不得覆盖出生点、前哨站交互区、初始必经拾取点。
- 玩家必须能通过绕行避开。
- 第二日暗潮物质不是随机障碍，不计入 `road_obstacle` 事件。
- 如果暗潮物质具备伤害、减稳定值或阻挡效果，必须在剧情结束后才启用碰撞/伤害检测。

V0.1 如果暗潮物质实体玩法尚未完成，可以先只作为可见场景危险物出现；对白仍然成立。

## 8. 存档字段

新增：

```gdscript
{
    "second_day_black_tide_reveal_seen": false
}
```

写入时机：

```text
second_day_black_tide_reveal_dialogue 播放完成或被跳过
-> second_day_black_tide_reveal_seen = true
-> SaveService.save_profile()
```

如果视频播放失败：

- 记录 warning。
- 直接进入独白对白。
- 不因为视频资源缺失阻断玩家进入第二日。

## 10. 验收标准

| 编号 | 标准 |
| --- | --- |
| A1 | 第 1 日进入局内不播放该剧情 |
| A2 | 第 2 日首次进入局内播放视频动画 |
| A3 | 视频可跳过，跳过后仍播放主角独白 |
| A4 | 独白只显示主角原画 |
| A5 | 视频和对白期间对局倒计时不减少 |
| A6 | 视频和对白期间稳定值、视野、容器、材料、前哨、撤离计时全部冻结 |
| A7 | 视频和对白期间玩家不能移动、拾取、打开背包或撤离 |
| A8 | 对白结束后才开始局内倒计时和交互计时 |
| A9 | 播放完成后 `second_day_black_tide_reveal_seen=true`，后续不重复 |
| A10 | 第 2 日剧情不启用第 3 日才开始的场景随机事件池 |
| A11 | 视频资源缺失不阻断流程，会进入独白并记录 warning |
