# 25-02 开档剧情与首次出发剧情对白

> 目标：把新档进入游戏后的叙事流程拆成“开场视频、第一段回到404哨所与背景故事对白、首次出发任务对白”三段。第一段必须在首次进入对局前交代背景故事，但不能像设定说明书一样硬讲；它要以“你回来了”开场，确认玩家本来就是 404 的一员，并用自然对话讲出 2050 核战、地下哨所、暗潮和 404 的生存压力。

## 1. 最新历程

```text
输入用户名
-> 创建并保存 PlayerProfile
-> 播放开场视频剧情
-> 玩家可跳过视频
-> 视频结束或跳过后，进入第一段剧情对白
-> 第一段对白以“你回来了”开场，确认玩家本来就是 404 哨所成员，并自然交代 2050 核战后地下生活、暗潮、少数上行者和哨所资源压力
-> 进入仓库/商人/研究所/出发准备等局外界面
-> 玩家点击出发探索
-> 触发第二段剧情对白
-> 第二段对白交代通道暂时损坏、收集菱形前哨修复件、修复两处前哨站、重新激活撤离
-> 进入出发加载界面
-> 加载完成后停留在 loading 界面，提示“按下任意按钮继续”
-> 玩家按任意按钮
-> 正式进入可操作局内
```

## 2. 档案 flag

新档案推荐使用三个独立标记：

```gdscript
{
    "intro_cinematic_seen": false,
    "world_intro_dialogue_seen": false,
    "first_departure_outpost_dialogue_seen": false
}
```

兼容规则：

- 旧字段 `first_intro_dialogue_seen` 不再作为唯一判断。
- 如果旧档只有 `first_intro_dialogue_seen=true`，迁移时可视为 `world_intro_dialogue_seen=true`。
- `first_departure_outpost_dialogue_seen` 必须单独判断，避免玩家没看过前哨站说明。

## 3. 开场视频剧情

触发条件：

```text
PlayerProfile.intro_cinematic_seen == false
且刚完成用户名创建
```

播放完成或跳过后：

```text
intro_cinematic_seen = true
保存档案
进入第一段“回到404哨所与背景故事”对白
```

视频定位：

- 用镜头和字幕建立时代氛围，不承担具体玩法教学；背景故事由后续第一段对白自然承接。
- 可以出现核爆残影、地下哨所通道、地表废墟、暗潮涌动、404 哨所门牌。
- 视频可以跳过，但跳过视频后仍必须进入第一段“回到404哨所与背景故事”对白。
- 跳过按钮文案：`跳过影像`
- 运行时优先播放 MP4：`res://assets/cinematics/source/opening_intro_cinematic_720p.mp4`。
- MP4 播放依赖 `res://addons/ffmpeg/ffmpeg.gdextension`；旧 `.ogv` 可作为兼容回退。

## 4. 第一段剧情：你回来了

触发条件：

```text
PlayerProfile.world_intro_dialogue_seen == false
且开场视频已结束或被跳过
```

播放完成或跳过后：

```text
world_intro_dialogue_seen = true
保存档案
进入局外仓库/出发准备界面
商人和研究所保持锁定
```

剧情定位：

- 玩家不是刚被带到 404 的新人，而是 404 哨所的一员。
- 第一段必须交代背景故事，但不能用“先说清楚世界”这种硬讲法。
- 通过调度员和玩家熟人式对话，让玩家知道：2050 核战后人类退入地下、地表被暗潮占据、普通人无法长时间暴露、玩家属于少数能上去的人、404 需要地表物资才能继续维持。
- 具体“菱形修复件 -> 修复前哨 -> 解锁撤离”的规则仍放到首次出发前讲。
- 道具名只能引用 `setting/items.tab` 中已有条目；本文当前引用：`脏布条`、`废金属`、`旧电池`、`安定糖`、`临时绷带`、`旧线圈`、`工具零件`、`前哨保险丝`、`净化滤芯`。

建议说话人：

| ID | 名称 | 口吻 |
| --- | --- | --- |
| `operator_404` | 404 哨所管理员 | 熟悉、克制、少废话 |
| `player` | 玩家 | 已属于 404，知道处境但仍有压力 |

角色原画规则：

- `operator_404` 说话时显示 404 哨所管理员原画，默认位于左侧。
- `player` 说话时显示主角原画，默认位于右侧，并优先使用玩家输入的用户名作为说话人名。
- 两人同屏时当前说话者高亮，未说话者弱化。
- 原画资源和站位来自 `setting/dialogue_speakers.tab`，详细规则见 `08_剧情对白角色原画显示规则.md`。

对白正文：

```text
[operator_404]
你回来了。

[player]
巡检记录已经交到终端。下层风道还是有杂音。

[operator_404]
2050年那场核战以后，能听见风道响，反而算好消息。

[operator_404]
活下来的人往地下退，矿井、旧地铁、避难层，能封住门的地方都成了家。404 也是这么留下来的。

[player]
地下待久了，大家都这么安慰自己。

[operator_404]
404 哨所不是靠安慰撑到今天的。一切不起眼的物资，却撑着我们的生存。


[operator_404]
仓库又压到警戒线了。

[player]
所以今天轮到我上去？

[operator_404]
不是轮到。是还能在暗潮里撑住的人，已经没几个了。

[operator_404]
普通人上去，几分钟就会被暗潮拖垮。你至少还能走一段。

[player]
别说得像什么好事。

[operator_404]
不是好事。只是 404 还想活下去，也还想有一天回到地面。

[operator_404]
地下已经没什么物资了，如果你真的还需要矿币的话...

[player]
说吧，这次要带什么回来？

[operator_404]
你看到一切物资都别放过。

[operator_404]
但，上去之后别逞强。暗潮一压近，人的判断会先出问题。

[player]
知道。带得回来才算数。

[operator_404]
嗯，准备一下，出发吧。
```

这段对白必须让玩家理解：

- 玩家已经属于 `404 哨所`，不是刚被介绍进来的新人。
- 背景故事仍然必须在首次进入对局前交代。
- 2050 核战后，人类退入地下哨所生活。
- `暗潮` 是地表探索的主要危险，普通人难以在其中行动。
- 哨所当前物资紧张，地表探索是维持 404 运转的日常任务。
- 玩家能短暂承受暗潮影响，所以会承担地表探索。
- 道具引用必须来自 `setting/items.tab`，不能临时创造“剧情道具”。
- 具体前哨和撤离规则暂时不在本段灌输。

## 5. 第二段剧情：首次出发任务说明

触发条件：

```text
PlayerProfile.first_departure_outpost_dialogue_seen == false
且玩家第一次点击“出发探索/确认出发”
且出发校验已通过
```

播放完成或跳过后：

```text
first_departure_outpost_dialogue_seen = true
保存档案
进入 RunLoadingScreen
```

对白正文：

```text
[operator_404]
出发前最后确认一次任务目标。

[operator_404]
地表的旧通道已经不能稳定往返。你上去以后，升降通道会被暗潮反冲干扰，回来的路会暂时损坏。

[player]
也就是说，我不能直接回来？

[operator_404]
不能。至少不能立刻回来。

[operator_404]
你需要在地表找到前哨修复件。我们已经标成菱形信号，靠近后会更容易识别。

[operator_404]
收集足够修复件后，修复两处前哨站。前哨站会重新建立地面信号链。

[operator_404]
两处前哨都修好后，通道才能被重新激活，撤离信号才会亮起。

[player]
如果没来得及修好呢？

[operator_404]
暗潮会不断压缩你的视野，也会让你的稳定值持续下降。

[operator_404]
稳定值太低时，你会越来越难判断周围情况。不要贪，物资再多也得活着带回来。

[operator_404]
记住顺序：先找菱形修复件，修复前哨站，再撤离。

[operator_404]
回到基地后，你的稳定值会恢复。那里也是你整理物资、出售战利品和准备下一次探索的地方。

[player]
收到。先找修复件，修前哨，再回来。

[operator_404]
祝你好运。404 哨所等你回来。
```

这段对白必须让玩家理解：

- 点击出发后才讲具体任务规则。
- 出发后通道会暂时损坏，撤离不是开局可用。
- 玩家需要先收集菱形前哨修复件，当前文案示例为 `前哨保险丝`、`净化滤芯`。
- 修复两处前哨站后，撤离才会重新激活。
- 暗潮会压缩视野。
- 稳定值会随在外时间下降。
- 回到基地后稳定值能恢复。

## 6. UI/交互规则

- 开场视频可跳过，跳过后进入第一段“你回来了”对白。
- 第一段和第二段对白支持点击/按键推进。
- V0.1 可以保留对白跳过，但跳过只跳过当前对白，不跳过后续流程。
- 每句最长不超过两行。
- 对白 UI 需要显示当前说话者原画；第一段和第二段为主角/管理员双人立绘切换。
- 重要词高亮：`暗潮`、`404 哨所`、`菱形信号`、`前哨保险丝`、`净化滤芯`、`两处前哨站`、`撤离信号`、`稳定值`。
- 第一段对白结束后进入局外界面，但只开放仓库和出发准备；商人和研究所保持锁定，等首次成功返回剧情结束后再解锁。
- 第二段对白结束后进入 loading。
- 第二段对白只在第一次出发探索时播放一次。

## 8. 验收标准

- 新档输入用户名后播放开场视频。
- 跳过视频后仍进入第一段“你回来了”背景故事对白。
- 第一段对白结束后进入局外仓库等界面，而不是直接进局；此时商人和研究所仍未解锁。
- 玩家第一次点击出发探索后播放第二段任务对白。
- 第二段对白结束后进入 loading。
- 两段对白不会重复播放。
- 玩家能从第二段对白理解“菱形前哨修复件 -> 修复前哨站 -> 解锁撤离”的顺序。
- 主角和 404 哨所管理员说话时显示对应原画，切换说话者时高亮正确。
