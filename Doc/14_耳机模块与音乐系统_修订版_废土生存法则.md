---

## 1. 本文档一句话说明

耳机模块是玩家出发前选择的“局内音频装备”，用于决定本局可播放的音乐、信号提示能力和少量稳定值修正。

---

## 2. 系统定位

耳机系统不应只是播放器。

它需要承担三类功能：

```text
氛围：让局内探索有更强的孤独感、节奏感和安全感。
策略：玩家可选择不同耳机模块获得不同信号提示或稳定值修正。
养成：耳机模块可通过研究、制作、掉落逐步解锁。
```

V0.3 之前可以不做完整耳机系统。

V0.3 第一版建议只做：

```text
出发前选择 1 个耳机模块。
耳机模块决定本局可用歌单。
音乐对稳定值消耗提供轻微修正。
耳机可在局内给容器异常或前哨方向提供弱提示。
```

---

## 3. 不做什么

早期不建议做：

```text
复杂音游玩法。
玩家自定义导入音乐。
按节拍战斗。
大量电台频道。
歌词系统。
耳机耐久和电量双资源。
过强的透视雷达。
```

核心原则：

```text
耳机帮助玩家做判断，但不能替玩家做决定。
音乐缓解压力，但不能取消黑暗压力。
```

## 4. 耳机模块规则

## 4.1 基础字段

耳机模块建议作为装备或特殊模块处理。

字段：

```text
id
name
quality
weight
signal_range_bonus
stability_drain_modifier
playlist_id
noise_filter_level
unlock_source
icon
description
```

说明：

```text
signal_range_bonus：增强信号提示距离。
stability_drain_modifier：稳定值消耗倍率修正，低于 1 表示减缓消耗。
playlist_id：绑定歌单。
noise_filter_level：降低异常噪音干扰的等级。
unlock_source：研究、制作、掉落或初始赠送。
```

---

## 4.2 装备限制

规则：

```text
每次出发最多携带 1 个耳机模块。
未装备耳机时，本局没有主动音乐播放功能。
耳机模块不占普通消耗品栏。
耳机模块可占 SPECIAL 装备槽，或独立 EarphoneSlot。
```

建议：

```text
V0.3 使用独立 EarphoneSlot。
后续如果装备槽系统成熟，再并入 SPECIAL。
```

---

## 5. 音乐播放规则

## 5.1 播放触发

音乐播放来源：

```text
局外基地背景音乐。
局内耳机歌单。
低稳定值危险氛围层。
撤离成功音乐。
失败结算音乐。
```

局内耳机音乐规则：

```text
进入局内后，如果装备耳机，自动播放耳机歌单。
玩家可以开关耳机音乐。
关闭音乐后，耳机的信号提示仍可保留。
音乐不会暂停对局时间。
打开容器界面时音乐继续播放。
撤离读条时音乐继续播放，但可叠加紧张音效。
```

---

## 5.2 稳定值影响

耳机音乐可以轻微影响稳定值消耗。

推荐规则：

```text
无耳机：稳定值消耗倍率 1.00
普通耳机：稳定值消耗倍率 0.97
优质耳机：稳定值消耗倍率 0.94
稀有耳机：稳定值消耗倍率 0.90
```

限制：

```text
最低只能降到 0.90。
不能通过耳机抵消黑暗、雨夜、异常区域的全部压力。
耳机效果与消耗品效果相乘后仍需要全局下限。
```

建议：

```text
V0.3 第一版只做 0.97。
确认体验后再开放更高品质耳机。
```

## 6. 信号提示规则

## 6.1 提示类型

耳机可提供弱信号：

```text
附近有稀有容器。
附近有前哨候选点。
附近有异常天气或危险区域。
撤离点方向存在噪声变化。
```

表现方式：

```text
轻微音效。
耳机 UI 波形闪动。
屏幕边缘方向性提示。
不显示精确距离。
不显示完整路径。
```

---

## 6.2 信号强度

信号强度公式：

```text
signal_strength = base_signal * earphone_quality_multiplier * distance_falloff * noise_filter
```

推荐：

```text
base_signal = 1.0
earphone_quality_multiplier：C=1.0，B=1.2，A=1.5，S=2.0
distance_falloff：随距离线性或曲线衰减
noise_filter：天气或异常区域修正
```

规则：

```text
信号只做提示，不直接标记地图。
信号强度低于阈值时不显示。
多个信号重叠时优先显示离玩家最近或稀有度最高的来源。
```

## 8. Godot 实现建议

推荐模块：

```text
EarphoneManager
EarphoneDefinition
PlaylistDefinition
MusicTrackDefinition
MusicPlaybackService
SignalHintService
AudioBusManager
EarphoneHUD
```

职责：

```text
EarphoneManager：管理出发前选择和局内状态。
MusicPlaybackService：播放、切歌、淡入淡出、暂停。
SignalHintService：根据局内对象生成弱提示。
AudioBusManager：管理音乐、环境音、UI 音效音量。
EarphoneHUD：显示波形、信号强度和当前播放状态。
```

## 9. UI 规则

局外：

```text
出发准备界面显示耳机槽。
未解锁时显示锁定。
鼠标悬停显示稳定值修正和信号能力。
```

局内：

```text
HUD 显示小型耳机图标。
播放中显示轻微波形。
有信号时图标闪烁或显示方向边缘提示。
不要用大弹窗打断探索。
```

## 10. 验证清单

```text
未装备耳机时不播放耳机歌单。
装备耳机后进入局内自动播放绑定歌单。
关闭音乐不会关闭信号提示。
稳定值消耗倍率正确应用。
低稳定值危险氛围可覆盖或叠加耳机音乐。
信号提示不会暴露精确坐标。
耳机模块可由研究或制作解锁。
```

## 11. 当前设计体检与建议

合理点：

```text
耳机把音乐从纯表现变成轻量策略。
稳定值修正幅度很小，不会破坏核心压力。
信号提示可增强探索欲，但不直接给答案。
```

风险：

```text
提示过强会削弱玩家观察地图的乐趣。
音乐收益过强会让耳机变成必带。
音频资源制作成本可能高于预期。
```

建议：

```text
先做 1 个基础耳机和 1 个歌单。
信号提示先只服务稀有容器或前哨。
音乐收益先控制在 3% 稳定值消耗减免以内。
```

---

## 12. 本文档结论

耳机模块应作为 V0.3 之后的氛围与轻策略系统。

第一版只需要验证：

```text
玩家愿不愿意为了音乐和弱提示携带耳机。
音乐是否能缓解探索疲劳。
信号是否能增加“再往前一步”的诱惑。
```

如果这三点成立，再扩展歌单、品质、信号类型和研究制作链。
