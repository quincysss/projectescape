# 13_街道区块基底与区块资源规格

> 来源补充：根据落地制作反馈与参考图一、图二、图三修订。  
> 目标：把城市地图的地基从“楼房围路”改为“街道可通行 + 区块不可通行 + 楼房放置在区块上”的可施工结构。

---

## 1. 实现目标

V0.1 地图地基只由两类玩法面构成：

```text
街道：可通行。
区块：不可通行。
```

楼房、树、围栏、施工物、屋顶设备等都不是地图主碰撞来源，而是放置在区块上的视觉、遮挡和局部补充物。

执行目标：

```text
先用街道和区块画出城市关卡骨架。
再把楼房合理放到区块上。
最后用装饰和局部碰撞补细节。
```

---

## 2. 设计依据

参考图中，街道不是由楼房不规则边缘“挤出来”的，而是由规则区块之间的负空间形成。

正确层级：

```text
第 1 层：街道地面，可通行。
第 2 层：区块地基，不可通行。
第 3 层：楼房与装饰，放在区块内部。
第 4 层：路灯、花坛、围栏、车辆、容器等局部物件。
```

错误做法：

```text
直接摆不规则楼房。
让楼房外轮廓决定道路宽度。
用楼房透明像素或复杂多边形作为主要碰撞。
最后再试图修出可走街道。
```

这会导致：

```text
道路宽窄不可控。
玩家容易卡边。
点位可达性难验证。
街区视觉杂乱。
后续多人、敌人、寻路和事件封锁都难扩展。
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置规则 | 先完成街道/区块基底，再摆楼房 |
| 通行规则 | 街道 TileMap/区域为可通行，区块 TileMap/区域为不可通行 |
| 碰撞规则 | 区块提供主碰撞；楼房默认不再作为主道路边界碰撞 |
| 楼房规则 | 楼房必须完全或主要落在区块内部，不得伸入街道压缩通行宽度 |
| 边缘规则 | 区块边缘应形成清晰 curb/sidewalk/台阶边界，方便玩家读图 |
| 点位规则 | 容器和材料点应贴近街道边缘或区块入口，但交互半径必须在街道可达 |
| 调试规则 | Debug Overlay 必须能分别显示街道可通行区、区块不可通行区、楼房视觉层 |
| 失败规则 | 如果移除楼房后街道拓扑不成立，说明地图基底失败 |

## 5. 区块资源规格

区块资源对应参考图三红框中的“城市地基/街区地砖”资源。

区块不是楼房贴图，而是用于铺出不可通行街区底座的地砖纹理。

### 5.1 推荐 Sheet 内容

首批区块 Sheet 建议命名：

```text
block_district_tiles_sheet_01.png
```

建议包含：

| 类型 | 建议单元尺寸 | 用途 |
|---|---:|---|
| `block_fill_clean_01` | 256×256 | 区块内部基础水泥地砖，可平铺 |
| `block_fill_cracked_01` | 256×256 | 破损区块内部，可平铺 |
| `block_fill_dirty_01` | 256×256 | 脏污区块内部，可平铺 |
| `block_edge_straight_01` | 256×256 | 区块直边，带 curb/边框 |
| `block_corner_outer_01` | 256×256 | 区块外角 |
| `block_corner_inner_01` | 256×256 | 区块内凹角 |
| `block_alley_cut_01` | 256×256 | 区块中开小巷缺口 |
| `block_driveway_cut_01` | 256×256 | 入口/车道缺口 |
| `block_plaza_fill_01` | 256×256 | 可用于广场或较干净地面 |
| `block_decals_overlay_01` | 256×256 透明 | 裂纹、污渍、杂草、排水口等覆盖层 |

### 5.2 视觉规格

```text
俯视或轻微斜俯视。
灰黑脏旧水泥地面。
边缘有清晰 curb、砖线、台阶或破损边。
内部有可平铺的裂纹、地砖线、污渍。
不要自带楼房。
不要自带完整街道。
不要有不可控大投影。
不要有文字水印。
```

区块填充图必须满足：

```text
四边可无缝平铺。
重复铺 4×4 后不能出现明显接缝。
不应出现强方向性大图案。
亮暗变化不能过强，否则拉大面积会像花纹墙纸。
```

### 5.3 是否可以做成可拉伸地砖

可以，但推荐方式不是“直接拉伸一张不重复图片”，而是：

```text
首选：TileMap 逐格铺设 block_fill + block_edge + block_corner。
次选：使用可无缝平铺的 block_fill 纹理做可重复填充。
边缘：仍用 edge/corner tile 或独立 curb 资源收边。
```

Godot 中可选落地方式：

```text
方案 A：TileMap / TileMapLayer
  用 Terrain/Pattern 画区块，最稳定，适合长期制作。

方案 B：BlockArea 场景
  用 Polygon2D 或 Sprite2D 的 repeat texture 填充大矩形/多边形，
  再用 CollisionPolygon2D 表示不可通行区块。

方案 C：快速原型
  用 ColorRect/Polygon2D 灰色块 + CollisionShape2D 先跑玩法，
  后续替换为区块 TileSet。
```

禁止：

```text
把一张有明显边框和裂纹方向的 512×512 图直接缩放成 20×12 单位。
让缩放后的纹理变糊或变形。
用楼房资源的轮廓作为区块边界。
```

## 6. 区块尺寸建议

区块是城市街区的基本不可通行面。

V0.1 推荐：

| 区块类型 | 建议尺寸 | 用途 |
|---|---:|---|
| 小区块 | 8×6 到 12×8 单位 | 小楼、商铺、容器门口 |
| 标准区块 | 14×10 到 20×14 单位 | 常规楼群、路线分割 |
| 长条区块 | 20×6 到 32×10 单位 | 长街边界、市场/工厂边 |
| 大区块 | 24×16 到 36×24 单位 | 外层大型障碍、施工区 |
| 特殊区块 | 按需求 | 家、前哨、广场、施工地 |

施工规则：

```text
区块边缘到对面区块边缘的距离 = 实际街道宽度。
楼房可小于区块，区块边缘保留 1 - 2 单位人行道/空地视觉边。
楼房可以压住区块内部，但不能改变街道可通行宽度。
```

---

## 7. 楼房放置规则

楼房放置在区块上，应遵守：

```text
楼房视觉轮廓不得伸入街道可通行区域。
楼房底部离区块边缘建议保留 0.5 - 1.5 单位视觉余量。
大型楼房可以占满区块内部，但区块边缘仍要可读。
楼房碰撞默认只用于局部补充，不再决定整条街道边界。
```

允许楼房有不规则外形，但不规则性只影响视觉，不影响主通行网格。

调试检查：

```text
关闭 VisualBuildings 后，街道与区块碰撞仍然成立。
关闭 Collision_BlockSolid 后，玩家才能穿过区块。
关闭建筑局部碰撞后，玩家仍不能走进区块。
```

## 8. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 地图可在不摆楼房的情况下，仅靠街道/区块基底跑通 |
| A2 | 玩家不能进入区块 |
| A3 | 楼房移除后，道路宽度和通行拓扑不改变 |
| A4 | 每个容器/材料/前哨点的交互半径位于可通行街道侧 |
| A5 | 区块边缘清晰可读，玩家能看出哪里不能走 |
| A6 | Debug Overlay 能区分街道、区块、楼房、点位 |
| A7 | 区块资源 4×4 平铺无明显接缝 |
