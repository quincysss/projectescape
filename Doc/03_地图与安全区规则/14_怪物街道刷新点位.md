# 14_怪物街道刷新点位

> 对应系统：`Doc/26_怪物与场景随机事件系统`  
> 目标：在地图层明确怪物点位布置规范，保证怪物只在街道生成，并可被怪物事件系统读取。

## 1. 实现目标

在 TownMap 中布置 10 个怪物刷新点：

```text
monster_spawn_street_001
...
monster_spawn_street_010
```

所有点位加入 group：

```text
monster_spawn_street
```

## 2. 设计依据

怪物系统规则：

```text
怪物第 3 日开始出现。
怪物事件命中后，从 10 个点中抽 4 个生成怪物。
怪物只在街道上生成。
怪物只在出生点附近街道巡逻。
```

## 3. 布置规则

怪物点位必须：

- 中心点位于 `StreetWalkable`。
- 中心点不在 `BlockSolid`。
- 不在家安全区。
- 不在前哨安全区。
- 不在撤离点正上方。
- 不与容器、材料点位重叠。
- 点位附近至少有一段可巡逻街道。

建议分布：

| 区域 | 数量 |
| --- | ---: |
| 中层街道 | 3 |
| 外层街道 | 4 |
| 最外层街道 | 3 |

不建议放在离家最近的内层入口，避免第三日怪物直接卡住玩家出门。

## 4. 数据字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `point_id` | `StringName` | `monster_spawn_street_001` |
| `point_type` | `StringName` | `monster` |
| `ring` | `StringName` | middle/outer/far_outer |
| `patrol_radius` | `float` | 默认 180 |
| `tags` | `Array[StringName]` | plaza/alley/street_edge |
| `is_enabled` | `bool` | 是否启用 |

## 5. 与点位池文档的关系

该点位池独立于：

```text
container_spawn_*
outpost_material_spawn_*
anomaly_spawn
outpost_candidate_*
```

怪物不能复用容器或材料点位。

## 6. 验证规则

MapPointValidator 需要新增：

- 怪物点位数量是否为 10。
- 怪物点位是否全部在街道可通行区。
- 怪物点位是否进入安全区。
- 怪物点位是否与容器/材料/撤离点距离过近。
- 怪物点位周围是否有可巡逻空间。

## 7. AI/MCP 开工提示词

```text

读取 Doc/03_地图与安全区规则/14_怪物街道刷新点位.md 和 Doc/26_怪物与场景随机事件系统/02_怪物街道点位与生成规则.md。
在 TownMap 中新增 10 个 monster_spawn_street 点位，命名 monster_spawn_street_001 到 010。
点位必须位于 StreetWalkable，不能在 BlockSolid、家安全区、前哨安全区、撤离点或容器/材料点正上方。
更新 MapPointValidator，让它检查怪物点位数量、街道合法性和与其他点位的距离。
禁止实现怪物 AI 和生成逻辑。

```

## 8. 验收标准

- 有 10 个怪物街道刷新点。
- 全部加入 `monster_spawn_street` group。
- 所有点位在街道可通行区。
- 没有点位落入建筑区块或安全区。
- Validator 能报告怪物点位合法性。

```text

    └── Patrol_02
```

编辑规则：

- 移动 `Patrol_01`、`Patrol_02` 就是在调整该怪物默认巡逻线路。
- 子点必须仍在街道可行走区域内。
- `map_layout_point.gd` 会在编辑器里绘制橙色巡逻线，方便直接摆放。
- `map_spawn_point_sanity_check.gd` 会检查子点是否误落进街区块。
