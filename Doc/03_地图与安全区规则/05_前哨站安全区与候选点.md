# 05_前哨站安全区与候选点

> 对应母版章节：7、8、14.1、14.2  
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“前哨站安全区与候选点”小节。

---

## 1. 实现目标

把前哨站落实为“策划控制下的随机支点”：地图上预先放置 3 个第一前哨候选点和 4 个第二前哨候选点；每局随机选中后，未修复时只显示需求，修复后激活较小安全区。

---

## 2. 设计依据

前哨站定位：

```text
本局临时安全点
外层探索支点
负重中转点
路线规划节点
```

候选点数量：

```text
第一前哨站：3 个候选点中随机 1 个
第二前哨站：4 个候选点中随机 1 个
```

距离要求：

| 前哨 | 距家时间 | 路径长度 |
|---|---:|---:|
| 第一前哨 | 10 - 18 秒 | 60 - 108 单位 |
| 第二前哨 | 25 - 40 秒 | 150 - 240 单位 |

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置条件 | 候选点必须可达，且加入对应 group |
| 执行规则 | 第一前哨在内层/中层交界，第二前哨在外层靠近最外层入口 |
| 互斥规则 | 第一前哨在左侧时，第二前哨优先从中/右侧抽取 |
| 未修复规则 | 不停止稳定值、不回复、不存储，只显示修复需求 |
| 已修复规则 | 停止稳定值下降、以 8/秒回复、开启前哨存储或预留 |
| 描边规则 | 前哨站可交互反馈走统一 Shader 描边，不为前哨单独制作固定 PNG 边框 |
| 限制规则 | 前哨站不提供完整全图观察，V0.1 推荐局部舒适视野 |
| 失败规则 | 候选点不可达、数量不足、路径时间越界必须报错或警告 |
| 区块规则 | 前哨候选点应落在街道可达的区块边缘或小广场旁；前哨建筑放在区块上，交互半径必须在街道侧可触达 |

---

## 3.1 前哨站可交互描边

前哨站的候选点和实体只负责提供位置、修复状态和安全区范围；可交互描边由前哨系统或 `InteractionHighlightService` 统一管理。

```text
未修复前哨：可显示偏冷灰/蓝紫描边，强调“可修复但未安全”。
材料满足或可部分提交：进入交付触发器后描边增强，并显示交付提示。
修复读条/交付中：描边可轻微脉冲。
已修复前哨：描边降低攻击性，偏安全白灰/高饱和点缀色。
不可交互或已移除：关闭描边。
```

美术拆件可以提供前哨修复前、修复后、家的建筑 Sprite，但不要额外生产完整建筑形状的固定描边 PNG。描边默认由 Shader 按 Sprite alpha 轮廓自动生成。

## 4. Godot 落地范围

| 路径 | 职责 |
|---|---|
| `res://scripts/map/OutpostCandidatePoint.gd` | 候选点元数据 |
| `res://scenes/maps/OutpostCandidatePoint.tscn` | 候选点 prefab，可选 |
| `res://scripts/map/OutpostSafeArea.gd` | 修复后安全区 |
| `res://scripts/debug/OutpostCandidateValidator.gd` | 候选点数量、距离、互斥检查 |
| `res://scripts/interaction/InteractionHighlightService.gd` | 前哨站交互描边状态输出 |

---

## 5. 数据结构

`OutpostCandidatePoint` 字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `candidate_id` | `StringName` | 如 `outpost_a_candidate_001` |
| `outpost_tier` | `int` | 1 或 2 |
| `map_side` | `StringName` | left / center / right |
| `ring` | `StringName` | inner / middle / outer / far_outer |
| `expected_path_seconds_from_home` | `float` | 路径估算时间 |
| `has_two_routes` | `bool` | 是否至少两条可达路线 |
| `safe_area_size` | `Vector2` | 默认小于家 |

---

## 6. 核心流程

```text
1. 在 TownMap 中放置 3 个 outpost_candidate_a。
2. 放置 4 个 outpost_candidate_b。
3. 每个候选点设置 map_side、ring、预估路径时间。
4. 运行验证工具检查候选点数量。
5. 运行通行验证检查候选点可达。
6. 新局初始化时由 02 的 OutpostSelector 读取 group 抽取。
7. 修复后激活该候选点绑定的 OutpostSafeArea。
```

---

## 7. 信号与接口

提供给 02/06：

```text
OutpostSelector：读取候选点 group 并抽取
OutpostStation：使用候选点位置创建/激活前哨
StabilityComponent：已修复前哨进入时停止下降并以 8/秒回复
SignalDisplay：家中显示前哨位置和状态
```
