# 08_镜头模式与地图边界

> 对应母版章节：11、6.3、6.4、7.4  
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“镜头模式与地图边界”小节。

---

## 1. 实现目标

建立 V0.1 至少两种镜头模式：家中全图观察 `HOME_OVERVIEW` 和离家玩家跟随 `PLAYER_FOLLOW`。镜头必须受地图边界限制，不能看到地图外空白。

---

## 2. 设计依据

镜头模式：

```text
HOME_OVERVIEW：家中全图观察模式
PLAYER_FOLLOW：离家玩家跟随模式
OUTPOST_REST：可选，前哨站休整模式
```

离家跟随建议：

| 项目 | 建议值 |
|---|---:|
| 拉近过渡时间 | 0.5 秒 |
| 跟随平滑时间 | 0.1 - 0.2 秒 |
| 初始舒适视野半径 | 20 单位 |
| 最低视野半径 | 3 单位 |

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 触发条件 | 进入家切 `HOME_OVERVIEW`，离家切 `PLAYER_FOLLOW` |
| 执行规则 | 全图模式优先显示家到当前前哨目标区域 |
| 执行规则 | 跟随模式使用同一 Camera2D 调整 zoom 与 target |
| 边界规则 | Camera2D 设置 limit，不能显示地图外空白 |
| 失败规则 | 如果全图装不下整张地图，不允许把信号缩到不可读 |
| 反馈规则 | 调试 HUD 显示 camera mode、zoom、limit、target |
| 兼容规则 | 前哨站内不提供完整全图观察，最多使用 `OUTPOST_REST` 局部舒适视野 |

---

## 4. Godot 落地范围

| 路径 | 职责 |
|---|---|
| `res://scripts/camera/MapCameraController.gd` | 镜头模式、zoom、target、limit |
| `res://scenes/maps/CameraRig.tscn` | Camera2D 容器 |
| `res://scripts/debug/CameraDebugOverlay.gd` | 镜头调试 |

---

## 5. 数据结构

```gdscript
enum MapCameraMode {
    HOME_OVERVIEW,
    PLAYER_FOLLOW,
    OUTPOST_REST
}
```

字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `current_mode` | `MapCameraMode` | 当前模式 |
| `map_bounds` | `Rect2` | 地图边界 |
| `home_overview_zoom` | `Vector2` | 家中 zoom |
| `player_follow_zoom` | `Vector2` | 离家 zoom |
| `transition_duration` | `float` | 默认 0.5 |
| `follow_smoothing` | `float` | 0.1 - 0.2 |

---

## 6. 核心流程

```text
1. 初始化 Camera2D limit 为地图边界。
2. 玩家进入家，CameraController.switch_to_home_overview()。
3. 根据地图尺寸和当前目标区域计算 zoom。
4. 玩家离家，CameraController.switch_to_player_follow(player)。
5. 0.5 秒过渡到跟随 zoom。
6. 稳定值阶段变化交给 VisionController 调整遮罩半径，不直接改地图边界。
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 家中镜头能观察主要地图和信号 |
| A2 | 离家后镜头跟随玩家 |
| A3 | 切换有过渡，不瞬间跳变 |
| A4 | 镜头不会显示地图外空白 |
| A5 | 地图过大时优先显示家到当前目标区域 |
| A6 | 前哨站不提供完整全图观察 |
