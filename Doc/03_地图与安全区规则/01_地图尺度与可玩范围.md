# 01_地图尺度与可玩范围

> 对应母版章节：2、3、16  
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“地图尺度与可玩范围”小节。

---

## 1. 实现目标

建立 V0.1 第一张小镇地图的空间基准：Tile 单位、地图总尺寸、可玩半径、圈层、家位置、玩家尺度和基础速度换算。该文档用于指导 `TownMap.tscn` 的画布大小、街道/区块 TileMap 范围、点位圈层与路径时间验证。

---

## 2. 设计依据

| 项目 | 规则 |
|---|---|
| 单位 | `1 地图单位 = 1 Tile` |
| 原型 Tile | `64 × 64 px` |
| 可玩半径 | 约 `130` 单位 |
| 安全余量 | `10 - 20` 单位 |
| 总尺寸 | `280 × 220` 到 `320 × 260` 单位 |
| 玩家速度参考 | `6 单位/秒` |
| 玩家视觉尺寸 | `2 × 4` 单位 |
| 玩家碰撞尺寸 | `1.2 × 1.2` 单位 |
| 玩家交互半径 | `1.5 - 2` 单位 |
| 容器识别半径 | `3 - 5` 单位 |
| 街道/区块基底 | 街道可通行，区块不可通行 |

圈层：

| 圈层 | 半径范围 |
|---|---:|
| 内层 | 0 - 20 |
| 中层 | 20 - 60 |
| 外层 | 60 - 100 |
| 最外层 | 100 - 130 |

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置条件 | 地图必须有家中心点或地图原点参考，圈层以家为中心计算 |
| 执行规则 | 所有点位记录与家中心的距离圈层，不能只靠人工命名 |
| 执行规则 | 地图不要第一版做得超过 `320 × 260`，除非同步调整稳定值与速度 |
| 失败规则 | 家到最外层主目标的实际路程超过 45 秒时，必须标记为风险 |
| 反馈规则 | 调试模式显示家中心、圈层边界、地图边界、可玩范围 |
| 边界规则 | 家不应绝对正中心，也不能贴近边界；至少连接 3 条可选主路 |
| 基底规则 | 先确定街道和区块，不允许先用不规则楼房反推道路 |

## 6. 核心流程

```text
1. 创建 TownMapConfig。
2. 在 TownMap.tscn 中确定 HomeArea 中心点。
3. 以 HomeArea 中心点为圈层原点。
4. 创建地图边界 Collision。
5. 先规划街道可通行层和区块不可通行层。
6. 在调试 Overlay 中绘制内层/中层/外层/最外层参考线。
7. 放置道路和点位时，同步记录所属圈层。
8. 用玩家速度 6 单位/秒估算关键路径时间。
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 地图配置能明确 Tile 大小和地图尺寸 |
| A2 | 家中心可被脚本读取 |
| A3 | 调试模式能显示 4 个圈层 |
| A4 | 地图边界阻止玩家走出范围 |
| A5 | 家至少连接 3 条可选主路的规划空间 |
| A6 | 路径时间估算使用 6 单位/秒作为参考 |
| A7 | TownMap 预留街道可通行层和区块不可通行层 |
