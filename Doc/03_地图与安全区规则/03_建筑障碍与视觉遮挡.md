# 03_建筑障碍与视觉遮挡

> 对应母版章节：5、13.1  
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“建筑障碍与视觉遮挡”小节。

---

## 1. 实现目标

建立 V0.1 建筑与障碍的关卡规则：大多数建筑不可进入，只作为道路边界、视觉识别点和未来扩展入口；家与前哨站是例外的可交互建筑。碰撞必须明确，遮挡先用简单 Y 排序和层级解决。

---

## 2. 设计依据

建筑作用：

```text
场景障碍
道路边界
视觉识别点
未来可扩展室内入口
前哨站候选外观
```

V0.1 可进入或交互：

```text
家
未修复前哨站
已修复前哨站
```

建筑尺寸建议：

| 建筑类型 | 建议尺寸 |
|---|---:|
| 小房屋 | 8 × 6 |
| 标准房屋 | 10 × 8 |
| 长条商铺 | 16 × 6 |
| 公寓楼 | 12 × 12 |
| 工厂/仓库 | 18 × 12 |
| 施工场地 | 16 × 16 |
| 前哨站建筑 | 10 × 8 或 12 × 8 |
| 家 | 10 × 8 |

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置条件 | 建筑必须有明确碰撞，不依赖贴图透明区域 |
| 执行规则 | 普通建筑不可进入，家和前哨站通过安全区范围表达可交互 |
| 执行规则 | 建筑用于塑造路线，不能把关键点位完全遮挡 |
| 视觉规则 | V0.1 不做复杂遮挡层，使用 Y 排序和绘制层 |
| 失败规则 | 玩家被建筑边缘卡住、不可见碰撞挡路、点位被遮挡，都必须修正 |
| 反馈规则 | 调试模式显示建筑碰撞矩形和不可通行区域 |
| 边界规则 | 建筑边角不要形成小于 2 单位的夹缝 |

## 4. Godot 落地范围

| 路径 | 职责 |
|---|---|
| `res://scenes/maps/buildings/BuildingBlock.tscn` | 可复用建筑障碍基类，可选 |
| `res://scripts/map/BuildingObstacle.gd` | 建筑类型、尺寸、碰撞调试 |
| `res://scenes/maps/TownMap.tscn` | 建筑和障碍节点 |
| `res://scripts/debug/MapDebugOverlay.gd` | 显示碰撞与遮挡风险 |

---

## 5. 数据结构

`BuildingObstacle` 字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `building_id` | `StringName` | 建筑 ID |
| `building_type` | `StringName` | house/shop/apartment/factory/outpost/home |
| `size_units` | `Vector2` | 玩法尺寸 |
| `is_enterable` | `bool` | V0.1 普通建筑为 false |
| `has_collision` | `bool` | 必须为 true |
| `blocks_signal_visibility` | `bool` | V0.1 默认 false，不做真实遮挡 |

---

## 6. 核心流程

```text
1. 先放置道路骨架。
2. 用建筑模块填充道路边界。
3. 给每个建筑添加 StaticBody2D + CollisionShape2D。
4. 避免建筑贴近形成小于 2 单位夹缝。
5. 家和前哨站不靠建筑内部碰撞表达交互，而使用独立 SafeArea。
6. 打开调试碰撞检查玩家是否会卡边。
```

---

## 7. 信号与接口

建筑本身不主动发信号。地图验证工具读取：

```text
building_obstacle
collision_shape
walkable_area
spawn_points
```

用于检查：

```text
点位是否被建筑覆盖
道路是否被碰撞切断
建筑间夹缝是否过窄
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 普通建筑不可进入 |
| A2 | 建筑碰撞不依赖贴图透明像素 |
| A3 | 玩家不会被建筑边缘卡住 |
| A4 | 家和前哨站通过安全区触发交互 |
| A5 | 关键点位不被建筑遮挡到不可读 |
| A6 | 调试模式可显示建筑碰撞范围 |
