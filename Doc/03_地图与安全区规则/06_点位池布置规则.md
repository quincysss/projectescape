# 06_点位池布置规则

> 对应母版章节：9、13.2、13.3、16  
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“点位池布置规则”小节。

---

## 1. 实现目标

建立可被脚本读取的地图点位池：容器点、前哨材料点、异常预兆点、前哨候选点、出生点、撤离点。点位池必须拆分，不允许所有资源共用一套刷新点。

---

## 2. 设计依据

必须拆分：

```text
容器点位池
前哨材料点位池
异常预兆点位池
前哨站候选点池
玩家出生点
撤离点
```

V0.1 建议数量：

| 点位类型 | 建议数量 |
|---|---:|
| 内层容器点 | 4 - 6 |
| 中层容器点 | 6 - 8 |
| 外层容器点 | 6 - 8 |
| 最外层容器点 | 4 - 6 |
| 第一前哨材料点 | 4 - 6 |
| 第二前哨材料点 | 5 - 8 |
| 异常预兆点 | 3 - 5 |
| 第一前哨候选点 | 3 |
| 第二前哨候选点 | 4 |

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置条件 | 点位必须在可通行区域附近，玩家可交互半径能触达 |
| 容器点 | 放道路边缘、广场角落、建筑门口、废弃车旁、小巷末端 |
| 禁止容器点 | 道路正中央、安全区内部、地图边界外、前哨修复点正上方 |
| 材料点 | 更偏可达与可读，不能过深死胡同，不能被建筑遮挡 |
| 异常点 | 可偏高风险区域，但不能阻塞主流程 |
| 数据规则 | 所有点位命名可解析并加入 group |
| 失败规则 | group 缺失、命名错误、点位不可达必须被验证工具报出 |

## 4. Godot 落地范围

| 路径 | 职责 |
|---|---|
| `res://scripts/map/MapPoint.gd` | 点位基础数据 |
| `res://scripts/map/ContainerSpawnPoint.gd` | 容器点位 |
| `res://scripts/map/MaterialSpawnPoint.gd` | 前哨材料点位 |
| `res://scripts/map/AnomalySpawnPoint.gd` | 异常预兆点 |
| `res://scripts/debug/MapPointValidator.gd` | 命名、分组、数量、可达性检查 |

---

## 5. 数据结构

`MapPoint` 通用字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `point_id` | `StringName` | 可解析 ID |
| `point_type` | `StringName` | container/material/anomaly/outpost/spawn/extract |
| `ring` | `StringName` | inner/middle/outer/far_outer |
| `map_side` | `StringName` | left/center/right |
| `is_occupied` | `bool` | 当前是否占用 |
| `tags` | `Array[StringName]` | road_edge/plaza/alley/dead_end 等 |
| `validation_note` | `String` | 策划备注 |

---

## 6. 核心流程

```text
1. 在 TownMap 中建立 ContainerSpawnPoints 分层节点。
2. 按内/中/外/最外层放置容器点。
3. 建立 OutpostMaterialSpawnPoints，拆分 A/B 两组。
4. 建立 AnomalySpawnPoints。
5. 确认 PlayerSpawn 和 ExtractPoint。
6. 给每个点位设置 point_id、ring、map_side、tags。
7. 运行 MapPointValidator 检查数量、命名、group、可达性。
```

---

## 7. 信号与接口

点位本身不需要发信号，提供给系统读取：

```text
ResourceRefreshDirector：读取容器/材料/异常点
OutpostSelector：读取前哨候选点
RunInitializer：读取出生点
ExtractionController：读取撤离点
SignalDisplay：读取可见信号源位置
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 容器点位不少于 20 个 |
| A2 | 前哨材料点位不少于 10 个 |
| A3 | 异常预兆点位不少于 3 个 |
| A4 | 点位命名符合规范 |
| A5 | 点位加入正确 group |
| A6 | 所有点位可达或明确标记为未启用 |
| A7 | 安全区内部没有普通容器点 |

## 2026-05-07 V0.1 容器与材料点位校验规则

容器和前哨材料点位属于“可拾取/可交互刷新点”，不等同于前哨站本体，默认不享受街区开洞规则。

布置规则：

- 容器点位应放在街道边缘、广场角落、建筑入口旁、废弃物/车辆旁、小巷末端等可触达位置。
- 前哨材料点位应放在所选前哨附近的可达道路或街区边缘，避免放进街区块内部。
- 点位中心必须位于 `StreetWalkable` 覆盖范围内。
- 点位中心不应落入 `BlockSolid` 的街区碰撞内部。
- 如果视觉上希望材料贴近建筑或街区边，可以放在街区边缘外侧，但中心点仍要留在道路侧。

当前回归检查：

```text
res://tests/map_spawn_point_sanity_check.gd
```

该检查会报出：

- 容器/材料点位超出街道可走范围；
- 容器/材料点位落入街区块内部；
- 前哨候选点没有被纳入可进入例外区；
- 前哨候选点中心仍被生成后的街区碰撞阻挡。
