# 09_随机事件预留与扩展点

> 对应母版章节：12  
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“随机事件预留与扩展点”小节。

---

## 1. 实现目标

V0.1 不实现复杂随机事件，但地图结构需要预留事件点和规则边界，避免后续加入道路封锁、污染区域、异常风暴时破坏主流程。

---

## 2. 设计依据

后续事件包括：

```text
道路临时封锁
污染区域
好天气
异常风暴
临时高价值容器群
前哨站周边干扰
```

道路封锁必须：

```text
不能完全堵死主流程路线
每个前哨站至少保留一条可达路线
封锁路段应在家中全图观察时可见
封锁带来的绕路成本应与奖励权重提升相关
```

污染区域必须：

```text
在家中可见大致范围
边界有视觉提示
区域内奖励权重提高
不要把关键前哨材料强制刷在重污染中心
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置条件 | 事件点只预留，不影响 V0.1 核心闭环 |
| 道路封锁 | 只能标记可封锁路段，不能默认生效 |
| 污染区域 | 只能放置预留区域，不默认增加稳定值衰减 |
| 异常事件 | 使用独立异常点位池，不和容器/材料共用 |
| 失败规则 | 预留点不能阻挡玩家移动，不能覆盖关键材料点 |
| 反馈规则 | 调试模式显示事件预留区和风险说明 |
| 边界规则 | 任何封锁组合都必须保留主目标可达路线 |

---

## 4. Godot 落地范围

| 路径 | 职责 |
|---|---|
| `res://scripts/map/EventReservePoint.gd` | 事件预留点 |
| `res://scripts/map/EventReserveArea.gd` | 污染/风暴等区域预留 |
| `res://scripts/debug/EventReserveValidator.gd` | 预留事件不会破坏通行 |

---

## 5. 数据结构

字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `reserve_id` | `StringName` | 事件预留 ID |
| `reserve_type` | `StringName` | road_block/pollution/anomaly/weather |
| `is_enabled_in_v01` | `bool` | 默认 false |
| `risk_level` | `int` | 预留风险 |
| `affects_routes` | `Array[StringName]` | 可能影响的路线 |
| `must_keep_passage` | `bool` | 是否必须留通路 |

---

## 6. 核心流程

```text
1. 在地图中标记少量道路封锁预留点。
2. 在外层/最外层标记污染区域预留范围。
3. 在异常点位池中标记 3-5 个异常预兆点。
4. 默认不启用事件效果。
5. 验证预留区域不覆盖关键材料点和必经路线。
```

---

## 7. 信号与接口

V0.1 不向核心系统发事件信号，只提供未来接口：

```text
get_event_reserve_points(type)
get_event_reserve_areas(type)
validate_route_after_event(event_id)
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 至少有道路封锁预留点 |
| A2 | 至少有污染区域预留范围 |
| A3 | 至少有 3 个异常预兆点 |
| A4 | 所有预留默认不启用 |
| A5 | 预留点不覆盖关键材料点 |
| A6 | 预留点不破坏 V0.1 通行验证 |
