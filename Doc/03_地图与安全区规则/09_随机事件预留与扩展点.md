> 对应母版章节：12  
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“随机事件预留与扩展点”小节。

## 1. 实现目标

地图结构需要预留事件点和规则边界，避免后续加入怪物、随机障碍、污染区域、天气事件时破坏主流程。

当前正式接入的场景随机事件框架见 `Doc/26_怪物与场景随机事件系统`。本文件负责地图层预留和通行约束。

---

## 2. 当前事件与地图需求

| 事件 | 地图需求 | 说明 |
|---|---|---|
| 怪物出现 | `monster_spawn_street` | 10 个街道点，事件命中后抽 4 个生成怪物 |
| 随机障碍 | `random_obstacle_spawn` | 10 个街道点，事件命中后抽 2-4 个生成障碍 |
| 超级时间 | 无地图点位 | 本局倒计时改为 8 分钟 |
| 超短时间 | 无地图点位 | 本局倒计时改为 3 分 30 秒 |

后续预留事件包括：

```text
下雨
污染区域
异常风暴
临时高价值容器群
前哨站周边干扰
```

## 3. 随机障碍与道路封锁边界

随机障碍是当前道路变化类事件的第一版。

它和未来“道路临时封锁”的区别：

| 类型 | V0.1 是否真实启用 | 规则 |
|---|---|---|
| 随机障碍 | 是，可由 `road_obstacle` 事件触发 | 生成 2-4 个障碍，阻挡玩家但不阻挡怪物 |
| 道路临时封锁 | 否，仍为预留 | 更大规模地改变路网，后续再做 |

随机障碍必须：

```text
不能完全堵死主流程路线
每个前哨站至少保留一条可达路线
撤离点必须仍可达
材料点必须仍可达
障碍点位应放在可绕行或可产生路线选择的位置
```

## 4. 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置条件 | 第 3 日后场景随机事件池启用，地图点位才会被事件读取 |
| 怪物事件 | 使用独立怪物街道点位池，只在街道生成 |
| 随机障碍事件 | 使用独立障碍候选点位池，只在街道生成 |
| 时间事件 | 不读取地图点位，只改写本局倒计时 |
| 道路封锁预留 | 只能标记可封锁路段，不能默认生效 |
| 污染区域预留 | 只能放置预留区域，不默认增加稳定值衰减 |
| 异常事件 | 使用独立异常点位池，不和容器/材料共用 |
| 失败规则 | 事件点不能覆盖关键材料点、前哨交互区或撤离点 |
| 反馈规则 | 调试模式显示事件预留区、怪物点、障碍点和风险说明 |
| 边界规则 | 任何障碍组合都必须保留主目标可达路线 |

## 6. 数据结构

事件预留字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `reserve_id` | `StringName` | 事件预留 ID |
| `reserve_type` | `StringName` | road_block/pollution/anomaly/weather |
| `is_enabled_in_v01` | `bool` | 默认 false |
| `risk_level` | `int` | 预留风险 |
| `affects_routes` | `Array[StringName]` | 可能影响的路线 |
| `must_keep_passage` | `bool` | 是否必须留通路 |

点位 group：

```text
monster_spawn_street
random_obstacle_spawn
anomaly_spawn
```

---

## 7. 核心流程

```text
1. 在街道上布置 10 个怪物刷新点。
2. 在街道上布置 10 个随机障碍候选点。
3. 在外层/最外层标记污染区域预留范围。
4. 在异常点位池中标记 3-5 个异常预兆点。
5. 道路封锁/污染/天气默认不启用事件效果。
6. 怪物和随机障碍由 26 系统控制，从第 3 日开始纳入抽取。
7. 验证预留区域不覆盖关键材料点和必经路线。
8. 模拟障碍生成，验证家到前哨、材料点、撤离点仍可达。
```
