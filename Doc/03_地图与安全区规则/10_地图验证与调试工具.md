# 10_地图验证与调试工具

> 对应母版章节：14、15、16  
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“地图验证与调试工具”小节。

---

## 1. 实现目标

为每张 V0.1 地图建立验证工具和调试显示，确保地图不是“看起来能玩”，而是可达、可读、时间压力合理、信号信息层级正确、安全区不会抖动。

---

## 2. 设计依据

必须验证：

```text
通行验证
时间验证
信号验证
压迫感验证
```

V0.1 最小交付需要：

```text
1个家
1个玩家出生点
1个撤离点
3个第一前哨站候选点
4个第二前哨站候选点
20个以上容器刷新点
10个以上前哨材料刷新点
3个以上异常预兆预留点
完整道路碰撞
完整建筑障碍
镜头边界
安全区触发范围
调试显示模式
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 通行验证 | 家到所有点位可达；无地图边界漏洞；无不可见碰撞卡路 |
| 时间验证 | 按玩家速度 6 单位/秒估算关键路线 |
| 信号验证 | 家中可见机会信号，离家视野外隐藏 |
| 压迫感验证 | 中层容器往返有压力但不应必死；外层无前哨明显危险 |
| 失败规则 | 验证失败必须输出点位 ID、位置、原因 |
| 反馈规则 | 调试 Overlay 显示点位、路线、圈层、碰撞、安全区 |
| 边界规则 | 验证工具不应依赖正式 UI 或美术 |

## 4. Godot 落地范围

| 路径 | 职责 |
|---|---|
| `res://scripts/debug/MapValidationRunner.gd` | 总验证入口 |
| `res://scripts/debug/MapDebugOverlay.gd` | 可视化圈层/点位/碰撞/路线 |
| `res://scripts/debug/MapPointValidator.gd` | 点位数量、命名、group、可达 |
| `res://scripts/debug/RouteTimeValidator.gd` | 路线时间估算 |
| `res://scenes/debug/DebugMapValidation.tscn` | 地图验证场景 |

---

## 5. 数据结构

验证结果：

| 字段 | 类型 | 说明 |
|---|---|---|
| `check_id` | `StringName` | 验证项 |
| `status` | `StringName` | pass/warn/fail |
| `target_id` | `StringName` | 点位或路线 |
| `message` | `String` | 说明 |
| `position` | `Vector2` | 问题位置 |
| `suggested_fix` | `String` | 建议修复 |

---

## 6. 核心流程

```text
1. 打开 DebugMapValidation.tscn。
2. MapValidationRunner 读取 TownMap。
3. 检查必需节点和 group 数量。
4. 检查点位命名。
5. 检查所有点位是否可达。
6. 按参考速度估算关键路线时间。
7. 检查信号可见性规则。
8. 输出 pass/warn/fail 列表。
9. MapDebugOverlay 高亮问题点。
```

## 8. AI/MCP 开工提示词

```text
请实现“地图验证与调试工具”的最小版本。
读取 Doc/03_地图与安全区规则/10_地图验证与调试工具.md 和 12 中对应小节。
新增 MapValidationRunner、MapPointValidator、RouteTimeValidator、MapDebugOverlay。
验证至少包括：必需 group 数量、点位命名、玩家出生点到点位可达、关键路线时间估算、地图边界存在。
禁止实现正式编辑器插件和复杂自动寻路优化。
完成后运行验证并报告 pass/warn/fail 列表。
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 能检查 V0.1 必需点位数量 |
| A2 | 能检查命名和 group 缺失 |
| A3 | 能报告不可达点位 |
| A4 | 能估算关键路径时间 |
| A5 | 能显示圈层、安全区、点位 |
| A6 | 验证结果包含 pass/warn/fail |
