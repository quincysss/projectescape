# 15_随机障碍点位

> 对应系统：`Doc/26_怪物与场景随机事件系统/07_随机障碍事件规则.md`  
> 目标：在地图层明确随机障碍候选点位，让障碍可以随机改变道路通行，但不会破坏主流程可达性。

## 1. 实现目标

在 TownMap 中布置 10 个随机障碍候选点：

```text
obstacle_spawn_001
...
obstacle_spawn_010
```

所有点位加入 group：

```text
random_obstacle_spawn
```

当 `road_obstacle` 事件命中后，系统从候选点中抽取 2-4 个生成障碍。

## 2. 布置规则

障碍点位必须：

- 位于 `StreetWalkable`。
- 不位于 `BlockSolid`。
- 不在家安全区。
- 不在前哨安全区。
- 不在撤离点正上方。
- 不覆盖容器、前哨材料、怪物刷新点。
- 放置障碍后不会让主目标路线完全不可达。

建议放在：

- 有替代路线的路口。
- 可绕行的街道边缘。
- 广场入口一侧。
- 能制造路线选择，但不会形成软锁的位置。

不建议放在：

- 家门口第一段必经路。
- 唯一通往前哨的窄路。
- 前哨修复交互半径内。
- 撤离触发区内。

## 3. 建议分布

| 区域 | 数量 |
| --- | ---: |
| 中层街道 | 3 |
| 外层街道 | 4 |
| 最外层街道 | 3 |

障碍事件可以和怪物事件同时出现，因此障碍点位不要全部贴近怪物点位。

## 4. 数据字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `point_id` | `StringName` | `obstacle_spawn_001` |
| `point_type` | `StringName` | `obstacle` |
| `ring` | `StringName` | middle/outer/far_outer |
| `allowed_assets` | `Array[StringName]` | 可用障碍资源 |
| `block_shape` | `StringName` | small/line/wide |
| `requires_path_validation` | `bool` | 默认 true |
| `tags` | `Array[StringName]` | plaza/crossroad/street_edge/detour |
| `is_enabled` | `bool` | 是否启用 |

## 5. 可用资源

```text
res://assets/map/decals/overlays/ground_decal_overlay_002.png
res://assets/map/props/placement/road_cone_01.png
res://assets/map/props/placement/broken_signboard_blank_01.png
```

资源规则：

- `ground_decal_overlay_002.png` 必须配合不可见碰撞体。
- `road_cone_01.png` 可以组合成小型阻挡。
- `broken_signboard_blank_01.png` 适合横向阻挡。

## 6. 验证规则

MapPointValidator 需要新增：

- 障碍点位数量是否为 10。
- 障碍点位是否全部在街道可通行区。
- 障碍点位是否进入安全区或建筑碰撞。
- 障碍点位是否覆盖撤离、前哨材料、容器或怪物点位。
- 模拟生成障碍后，家到前哨、材料点、撤离点是否仍可达。

## 8. 验收标准

| 编号 | 标准 |
| --- | --- |
| A1 | 有 10 个随机障碍候选点 |
| A2 | 全部加入 `random_obstacle_spawn` group |
| A3 | 所有点位位于街道可通行区 |
| A4 | 没有点位落入建筑区块或安全区 |
| A5 | 点位不会覆盖容器、材料、怪物、撤离交互区 |
| A6 | Validator 能模拟障碍并报告主流程可达性 |
