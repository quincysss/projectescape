# 00_拆分规范与落地文档写法

> 来源文档：`../03_地图与安全区规则_修订版_废土生存法则.md`  
> 参考规范：`../02_核心循环规则/00_拆分规范与落地文档写法.md`  
> 目标：把 03 从设计母版拆成可用于 Godot 关卡搭建、点位布置、碰撞配置、信号调试和 AI/MCP 执行的施工文档。

---

## 0. 执行总则

本目录所有拆分文档必须遵守：

```text
Doc/00_落地执行文档总则.md
```

地图文档必须优先写成可施工的节点、碰撞、点位、分组和验证工具说明；美术氛围描述只能作为补充。V0.1 地图基底统一采用“街道可通行 + 区块不可通行 + 楼房放在区块上”的结构。

---

## 1. 拆分原则

03 的落地文档不是重新复述“地图应该是什么感觉”，而是把地图规则拆成可以直接做场景、节点、碰撞、分组、点位、验证工具的任务。

每份文档必须满足：

| 要求 | 说明 |
|---|---|
| 关卡可施工 | 能指导创建 `TownMap.tscn`、TileMap、碰撞、点位 Node、分组 |
| 基底可控 | 先施工街道和区块，再摆楼房；区块是主碰撞，不由楼房轮廓围路 |
| 规则可验证 | 有通行、距离、时间、信号、镜头、安全区的验收标准 |
| 数据可读取 | 点位命名、group、字段、圈层、标签必须可被脚本读取 |
| 不抢系统文档 | 容器掉落、稳定值细节、背包仓库、结算只引用对应系统，不在 03 深做 |
| AI 可开工 | 每个模块要有独立可复制的 AI/MCP 提示词 |

---

## 2. 推荐文档结构

每份落地文档使用：

```text
# 文档名

## 1. 实现目标
本模块最终要在 Godot 里产生什么可见或可验证结果。

## 2. 设计依据
来自母版文档的关键约束。

## 3. 功能规则细化
触发、前置、执行、中断、失败、反馈、边界。

## 4. Godot 落地范围
建议新增/修改 scenes、scripts、resources、groups。

## 5. 数据结构
节点字段、配置项、命名规则、group、枚举。

## 6. 核心流程
搭建步骤、运行流程、验证流程。

## 7. 信号与接口
与 02 核心循环、04 视野、05 容器、06 前哨材料等系统如何衔接。

## 8. AI/MCP 开工提示词
可直接复制给执行代理。

## 9. 验收标准
可人工或工具验证的结果。

## 10. 暂不实现
避免扩散到后续版本或其他系统。
```

## 4. Godot 路径约定

在实际工程结构确定前，03 统一使用以下建议路径：

| 类型 | 建议路径 |
|---|---|
| 主地图场景 | `res://scenes/maps/TownMap.tscn` |
| 地图配置资源 | `res://resources/maps/TownMapConfig.tres` |
| 街道 TileMap | `TownMap/BaseGround/TileMap_StreetWalkable` |
| 区块 TileMap | `TownMap/BaseGround/TileMap_BlockSolid` |
| 区块边缘 TileMap | `TownMap/BaseGround/TileMap_BlockEdge` |
| 道路标线 TileMap | `TownMap/BaseGround/TileMap_RoadMarking` |
| 区块碰撞 | `TownMap/Collision_BlockSolid` |
| 建筑视觉层 | `TownMap/VisualBuildings/BuildingRoot` |
| 点位基础脚本 | `res://scripts/map/MapPoint.gd` |
| 安全区脚本 | `res://scripts/map/SafeArea2D.gd` |
| 前哨候选点脚本 | `res://scripts/map/OutpostCandidatePoint.gd` |
| 容器点位脚本 | `res://scripts/map/ContainerSpawnPoint.gd` |
| 材料点位脚本 | `res://scripts/map/MaterialSpawnPoint.gd` |
| 地图验证脚本 | `res://scripts/debug/MapValidationRunner.gd` |
| 地图调试 HUD | `res://scripts/debug/MapDebugOverlay.gd` |

如果现有项目路径不同，执行时优先贴合现有结构，但职责边界保持不变。

## 6. AI/MCP 提示词质量底线

每份提示词必须包含：

```text
任务目标
必须读取的文档
允许新增/修改的文件范围
禁止实现的范围
完成标准
验证方式
最终报告格式
```

禁止模糊提示：

```text
帮我搭地图
把安全区做好
优化路线
按文档实现
```

合格提示示例：

```text
请实现 TownMap.tscn 的点位池基础结构。
读取 06_点位池布置规则.md 和 12 中的对应小节。
新增 MapPoint、ContainerSpawnPoint、MaterialSpawnPoint、OutpostCandidatePoint。
在调试场景中创建至少 20 个容器点、10 个材料点、7 个前哨候选点、3 个异常点。
所有点位必须命名可解析并加入对应 group。
禁止实现容器掉落和材料刷新逻辑。
完成后运行 MapValidationRunner，报告点位数量、命名错误、group 缺失。
```
