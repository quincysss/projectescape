# 12_模块功能规则细化与AI开工提示词

> 来源文档：`../03_地图与安全区规则_修订版_废土生存法则.md`  
> 用途：补足 03 拆分文档中的施工细节与可直接复制给 AI/MCP 的开工提示词。  
> 使用方式：执行某个地图模块时，先读对应模块文档，再读本文件同名小节。

## 1. 使用规则

本文件是 03 的执行增强层。它不替代单篇模块文档，而是给执行代理补足：

```text
触发/前置/执行/失败/反馈/边界规则
允许改什么
禁止做什么
如何验证
如何汇报
```

任何执行代理完成后必须按本文件末尾“执行代理最终报告格式”汇报。

## 2. 地图尺度与可玩范围

对应文档：`01_地图尺度与可玩范围.md`

### 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置条件 | 必须存在家中心参考点，圈层以家为中心而不是地图几何中心 |
| 执行规则 | 使用 `1 地图单位 = 1 Tile`，原型默认 `1 Tile = 64px` |
| 执行规则 | 可玩半径约 130，地图总尺寸控制在 280×220 到 320×260 |
| 失败规则 | 地图过大导致家中信号不可读时，优先缩小地图或调整视野，不盲目缩小信号 |
| 反馈规则 | 调试 Overlay 显示地图边界、家中心、四个圈层 |
| 边界规则 | 家不能贴边，至少连接 3 条可选主路 |

### AI/MCP 开工提示词

```text
请实现 03 地图系统的“地图尺度与可玩范围”基础。

必须读取：
- Doc/03_地图与安全区规则/00_拆分规范与落地文档写法.md
- Doc/03_地图与安全区规则/01_地图尺度与可玩范围.md
- Doc/03_地图与安全区规则/12_模块功能规则细化与AI开工提示词.md 的“地图尺度与可玩范围”小节

实现范围：
- 新增 TownMapConfig 或等价配置。
- 在 TownMap.tscn 中建立地图边界、家中心参考点、圈层调试显示。
- 新增 MapDebugOverlay 的最小版本，用于显示边界和圈层。

禁止范围：
- 不做正式美术。
- 不做程序化地图生成。
- 不做资源刷新和掉落。

完成标准：
- 可读取 Tile 大小、地图尺寸、可玩半径、圈层半径。
- 调试模式可显示内层/中层/外层/最外层。
- 地图边界存在并可用于 Camera2D limit。

验证方式：
- 打开调试场景或运行可用检查，确认配置和 Overlay 正常。
- 汇报地图尺寸、家中心、圈层半径、边界范围。
```

## 2.5 街道区块基底与区块资源规格

对应文档：`13_街道区块基底与区块资源规格.md`

### 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置条件 | 地图施工必须先有街道可通行层和区块不可通行层 |
| 执行规则 | 区块提供主碰撞，楼房放在区块上 |
| 执行规则 | 楼房隐藏后，街道拓扑仍必须成立 |
| 资源规则 | 区块填充纹理必须可平铺，边缘使用 edge/corner 收边 |
| 失败规则 | 用不规则楼房轮廓围路、道路宽度被楼房压缩、区块纹理拉伸变形都必须修正 |
| 反馈规则 | Debug Overlay 分别显示街道、区块、楼房视觉层 |

### AI/MCP 开工提示词

```text
请实现“街道区块基底与区块资源规格”的原型结构。

必须读取：
- Doc/03_地图与安全区规则/13_街道区块基底与区块资源规格.md
- Doc/03_地图与安全区规则/12_模块功能规则细化与AI开工提示词.md 的“街道区块基底与区块资源规格”小节

实现范围：
- 在 TownMap.tscn 中建立 BaseGround/TileMap_StreetWalkable。
- 建立 BaseGround/TileMap_BlockSolid 和 Collision_BlockSolid。
- 建立 VisualBuildings/BuildingRoot，楼房只能放在区块上。
- Debug Overlay 能显示街道层、区块层、建筑视觉层。

禁止范围：
- 不用楼房不规则轮廓围出街道。
- 不把建筑透明像素作为主碰撞。
- 不实现正式区块美术 Sheet，只预留资源路径和节点。

完成标准：
- 玩家不能进入区块。
- 隐藏 VisualBuildings 后，路线仍能跑通。
- 区块层和街道层可被验证工具读取。

验证方式：
- 用调试玩家测试街道和区块边界。
- 关闭建筑视觉层后重新跑路线验证。
- 汇报街道层节点、区块层节点、区块碰撞来源。
```

## 3. 道路拓扑与通行碰撞

对应文档：`02_道路拓扑与通行碰撞.md`
补充依赖：`13_街道区块基底与区块资源规格.md`

### 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置条件 | 玩家碰撞体约 1.2×1.2，道路宽度不得低于 2 单位 |
| 执行规则 | 主环路 + 支路 + 小巷捷径，家附近至少 3 个出口方向 |
| 执行规则 | 街道可通行层和区块不可通行层先于建筑摆放完成 |
| 执行规则 | 道路宽度由区块边缘决定，不由楼房不规则轮廓决定 |
| 死胡同规则 | 长度 15-20 单位内可接受，超过必须绑定高价值点或特殊信号 |
| 失败规则 | 点位不可达、道路被碰撞切断、地图边界泄漏必须 fail |
| 反馈规则 | 调试 Overlay 高亮可通行区、阻挡区、不可达点 |
| 边界规则 | 第二前哨候选点不能全部在死胡同 |

### AI/MCP 开工提示词

```text
请实现“道路拓扑与通行碰撞”的原型结构。

必须读取：
- Doc/03_地图与安全区规则/02_道路拓扑与通行碰撞.md
- Doc/03_地图与安全区规则/13_街道区块基底与区块资源规格.md
- Doc/03_地图与安全区规则/12_模块功能规则细化与AI开工提示词.md 的“道路拓扑与通行碰撞”小节

实现范围：
- 在 TownMap.tscn 中建立主干道、次级道路、小巷、广场节点的最小可通行结构。
- 建立街道可通行层、区块不可通行层和地图边界。
- 楼房后续只能放在区块上，不得用楼房轮廓围路。
- 新增或扩展 MapValidationRunner，能检查玩家出生点到所有点位是否可达。

禁止范围：
- 不做正式地图美术。
- 不做敌人寻路。
- 不做动态道路封锁。

完成标准：
- 玩家不能穿墙或走出地图。
- 玩家不能进入区块。
- 玩家可从家到达所有启用点位。
- 隐藏建筑视觉层后，道路拓扑仍成立。
- 验证工具能报告不可达点和路线时间风险。

验证方式：
- 使用调试玩家或验证工具跑通家到容器、材料、前哨候选点。
- 汇报道路类型、不可达点、关键路线时间。
```

## 4. 建筑障碍与视觉遮挡

对应文档：`03_建筑障碍与视觉遮挡.md`
补充依赖：`13_街道区块基底与区块资源规格.md`

### 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置条件 | 必须先有区块不可通行层，建筑放在区块内部 |
| 前置条件 | 普通建筑不依赖贴图透明像素，也不承担主道路边界 |
| 执行规则 | 家和前哨站用独立安全区表达可交互，不做室内 |
| 执行规则 | 建筑用于强化区块识别，不能伸入街道压缩通行宽度 |
| 执行规则 | 建筑碰撞只做局部补充；区块碰撞负责主不可通行区域 |
| 失败规则 | 小于 2 单位夹缝、不可见碰撞卡路、点位被覆盖必须修复 |
| 反馈规则 | 调试模式显示建筑碰撞矩形 |
| 边界规则 | V0.1 不做复杂遮挡层，优先 Y 排序 |

### AI/MCP 开工提示词

```text
请实现“建筑障碍与视觉遮挡”的原型规则。

必须读取：
- Doc/03_地图与安全区规则/03_建筑障碍与视觉遮挡.md
- Doc/03_地图与安全区规则/13_街道区块基底与区块资源规格.md
- Doc/03_地图与安全区规则/12_模块功能规则细化与AI开工提示词.md 的“建筑障碍与视觉遮挡”小节

实现范围：
- 新增 BuildingObstacle 或等价脚本。
- 建筑必须挂到区块或记录 host_block_id。
- 主碰撞来自区块，建筑只在必要时配置局部 StaticBody2D/CollisionShape2D。
- 调试模式可显示建筑碰撞范围。

禁止范围：
- 不做室内探索。
- 不做建筑透明化遮挡。
- 不做可破坏建筑。

完成标准：
- 普通建筑不可进入。
- 玩家不会被建筑边缘卡住。
- 隐藏建筑后，区块碰撞仍能阻止玩家进入街区。
- 建筑不会伸入街道改变通行宽度。
- 关键点位不被建筑遮挡到不可读。

验证方式：
- 用调试玩家沿建筑边缘移动，检查卡边和夹缝。
- 汇报建筑类型、碰撞配置、风险点。
```

## 5. 家安全区落地规则

对应文档：`04_家安全区落地规则.md`

### 功能规则细化

| 分类 | 规则 |
|---|---|
| 触发条件 | 玩家进入/离开 HomeArea 的 SafeArea2D |
| 前置条件 | HomeArea 默认激活 |
| 执行规则 | 进入家停止稳定值下降、回复、遮罩淡出、镜头全图、显示信号 |
| 执行规则 | 离家镜头拉近、遮罩淡入、稳定值下降、全图信号关闭 |
| 缓冲规则 | 离开边界加 0.2-0.3 秒防抖 |
| 失败规则 | 玩家在门口抖动导致事件刷屏必须修复 |
| 边界规则 | 家门口过渡区宽度 ≥3 单位，家至少 3 条出口 |

### AI/MCP 开工提示词

```text
请实现“家安全区落地规则”。

必须读取：
- Doc/03_地图与安全区规则/04_家安全区落地规则.md
- Doc/03_地图与安全区规则/12_模块功能规则细化与AI开工提示词.md 的“家安全区落地规则”小节

实现范围：
- 创建 HomeArea 节点结构：SafeArea2D、DoorBufferArea2D、PlayerSpawn、StoragePoint、ExtractPoint。
- SafeArea2D 进入/离开必须发信号。
- 离开家加入 0.2-0.3 秒缓冲，避免门口抖动。

禁止范围：

完成标准：
- 玩家出生在家。

验证方式：
- 玩家进出家 3 次，确认事件和状态无重复错乱。
- 汇报 HomeArea 节点结构、信号、验证结果。
```

## 6. 前哨站安全区与候选点

对应文档：`05_前哨站安全区与候选点.md`

### 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置条件 | 必须有 3 个 A 候选点和 4 个 B 候选点 |
| 执行规则 | A 在内层/中层交界，B 在外层靠近最外层入口 |
| 路线规则 | A 距家 10-18 秒，B 距家 25-40 秒，至少可达 |
| 互斥规则 | 候选点带 left/center/right 标签，支持简单互斥 |
| 未修复规则 | 不停止稳定值、不回复、不存储 |
| 已修复规则 | 激活较小安全区，以 8/秒回复稳定值 |
| 描边规则 | 前哨站只提供可交互状态，描边由统一 Shader / InteractionHighlightService 表现 |
| 边界规则 | 前哨不提供完整全图观察 |

### AI/MCP 开工提示词

```text
请实现“前哨站安全区与候选点”。

必须读取：
- Doc/03_地图与安全区规则/05_前哨站安全区与候选点.md
- Doc/03_地图与安全区规则/12_模块功能规则细化与AI开工提示词.md 的“前哨站安全区与候选点”小节

实现范围：
- 新增 OutpostCandidatePoint 和 OutpostSafeArea。
- 在 TownMap.tscn 中配置 3 个 outpost_candidate_a 和 4 个 outpost_candidate_b。
- 每个候选点必须有 candidate_id、tier、ring、map_side、路径时间估算。
- 实现候选点验证：数量、可达、路径时间、互斥标签。
- 前哨站可交互反馈接入统一描边服务，使用 Shader 按 Sprite alpha 自动描边，不制作固定 PNG 边框。

禁止范围：
- 不做正式前哨修复 UI。
- 不做材料消耗。
- 不做永久前哨建设。
- 不为前哨站制作独立固定描边边框图。

完成标准：
- 候选点数量正确。
- 所有候选点可达。
- 路径时间大致符合母版要求。
- 修复后安全区小于家，且不提供全图观察。
- 前哨站进入交互范围时有贴合 Sprite 轮廓的可交互描边。

验证方式：
- 运行候选点验证并报告 pass/warn/fail。
- 汇报候选点列表、路径时间、互斥标签。
```

## 10. 随机事件预留与扩展点

对应文档：`09_随机事件预留与扩展点.md`

### 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置条件 | V0.1 只预留，不启用真实效果 |
| 道路封锁 | 标记可封锁路段，但默认不阻挡 |
| 污染区域 | 标记区域，但默认不加速稳定值衰减 |
| 异常点 | 使用独立异常点池 |
| 失败规则 | 预留点不能覆盖关键材料点或破坏通行 |
| 反馈规则 | 调试模式显示事件预留区 |

### AI/MCP 开工提示词

```text
请实现“随机事件预留与扩展点”。

必须读取：
- Doc/03_地图与安全区规则/09_随机事件预留与扩展点.md
- Doc/03_地图与安全区规则/12_模块功能规则细化与AI开工提示词.md 的“随机事件预留与扩展点”小节

实现范围：
- 新增 EventReservePoint 和 EventReserveArea。
- 放置道路封锁、污染区域、异常事件预留点。
- 所有预留默认不启用，不影响 V0.1 核心闭环。

禁止范围：
- 不实现真实封锁。
- 不实现污染伤害或稳定值加速。
- 不实现异常奖励。

完成标准：
- 预留点数量可读取。
- 预留点不阻挡路线。
- 预留点不覆盖关键材料点。

验证方式：
- 运行地图通行验证，确认预留不影响核心路线。
- 汇报预留点类型、数量、风险说明。
```
