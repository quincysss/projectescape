# 02_道路拓扑与通行碰撞

> 对应母版章节：4、5.3、14.1、14.2  
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“道路拓扑与通行碰撞”小节。

---

## 1. 实现目标

建立小镇地图的可通行空间：街道、广场、家、已修复前哨站可走；区块、地图边界和局部障碍不可走。道路结构必须支持“家中规划、离家执行、前哨推进、回收撤离”的多路线往返。

---

## 2. 设计依据

可通行：

```text
道路
广场/空地
家内部安全区
已修复前哨站内部安全区
```

不可通行：

```text
区块上的建筑局部碰撞
普通废墟
树木/围栏/水池/施工架
地图边界
区块
```

道路宽度：

| 道路类型 | 建议宽度 |
|---|---:|
| 主干道 | 5 - 7 单位 |
| 次级道路 | 3 - 4 单位 |
| 小巷 | 2 - 3 单位 |
| 门口过渡区 | ≥3 单位 |
| 广场/节点 | 8 × 8 以上 |

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 前置条件 | 玩家碰撞体约 `1.2 × 1.2` 单位，道路宽度不得低于 2 单位 |
| 执行规则 | 道路采用“主环路 + 支路 + 小巷捷径”结构 |
| 执行规则 | 道路由街道 TileMap/区域与区块边缘共同定义，不由楼房轮廓定义 |
| 执行规则 | 家附近必须有内层小环路，中层至少两条主路，外层有绕行路 |
| 死胡同规则 | 死胡同长度建议不超过 15 - 20 单位，超过必须有高价值点或特殊信号 |
| 失败规则 | 前哨候选点、材料点、容器点不可被放在不可达区域 |
| 反馈规则 | 调试模式高亮可通行区、阻挡区、死胡同端点 |
| 边界规则 | 最外层不能只有单一路径；第二前哨候选点不能全在死胡同 |

## 4. Godot 落地范围

| 路径 | 职责 |
|---|---|
| `res://scenes/maps/TownMap.tscn` | 道路 TileMap、碰撞层 |
| `res://scripts/map/WalkableAreaTag.gd` | 标记可通行区域 |
| `res://scripts/map/MapBlockArea.gd` | 标记不可通行区块 |
| `res://scripts/debug/MapValidationRunner.gd` | 通行和路径时间验证 |
| `res://scripts/debug/DeadEndMarker.gd` | 死胡同调试标记，可选 |

Godot 原型建议：

```text
街道：TileMapLayer 或 NavigationRegion2D
区块：TileMapLayer / CollisionPolygon2D / StaticBody2D
建筑/障碍：只做局部补充碰撞，不承担主道路边界
玩家：CharacterBody2D
```

## 5. 数据结构

道路标签：

| 字段 | 说明 |
|---|---|
| `road_type` | `main / secondary / alley / plaza / transition` |
| `width_units` | 实际通行宽度 |
| `ring` | 所在圈层 |
| `connects_to` | 连接的关键节点 ID |
| `is_dead_end` | 是否死胡同 |
| `expected_reward_tag` | 死胡同是否有奖励用途 |
| `adjacent_block_ids` | 邻接区块 ID，用于验证街道边界 |

## 6. 核心流程

```text
1. 先铺主环路，保证家附近有至少 3 个出口方向。
2. 铺中层主路，连接第一前哨候选区。
3. 铺外层主路和绕行路，连接第二前哨候选区。
4. 添加小巷捷径和风险死胡同。
5. 铺区块不可通行层，用区块边缘确定道路宽度。
6. 将楼房放到区块内部。
7. 使用验证工具检查所有点位可达。
8. 用玩家速度估算关键路线时间。
```

## 8. AI/MCP 开工提示词

```text
请实现“道路拓扑与通行碰撞”的原型结构。
读取 Doc/03_地图与安全区规则/02_道路拓扑与通行碰撞.md 和 12 中对应小节。
在 TownMap.tscn 中建立主干道、次级道路、小巷、广场/节点的最小可通行结构，并为建筑/边界添加明确碰撞。
道路必须由街道可通行层和区块不可通行层组成，禁止用不规则楼房轮廓直接围出道路。

禁止实现正式地图美术、敌人、战斗、程序化生成。
完成后报告道路类型、不可达点、关键路线估算时间。
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 玩家不能穿过建筑、废墟、地图边界 |
| A2 | 玩家能从家到达所有容器点位 |
| A3 | 玩家能从家到达所有前哨候选点 |
| A4 | 第一前哨到第二前哨候选区域有可行路线 |
| A5 | 死胡同超过 20 单位时有奖励用途标记 |
| A6 | 关键路线时间符合母版建议范围或给出风险 |
| A7 | 隐藏楼房视觉层后，道路拓扑仍然成立 |
| A8 | 区块层是主要不可通行来源 |
