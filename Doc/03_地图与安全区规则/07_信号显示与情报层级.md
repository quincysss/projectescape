# 07_信号显示与情报层级

> 对应母版章节：10、15.2 风险一  
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“信号显示与情报层级”小节。

---

## 1. 实现目标

定义地图信号的“可见但不透明”规则：玩家在家中能据此规划路线，但不能提前知道容器内部奖励；离家后只显示视野范围内的局部信号。

---

## 2. 设计依据

信号类型：

```text
容器信号
前哨材料信号
前哨站信号
异常预兆信号
撤离信号
```

家中可显示：

| 信号 | 显示程度 |
|---|---|
| 容器 | 大致位置、轮廓/类型倾向、信号强弱 |
| 前哨材料 | 大致位置、材料类别或需求关联 |
| 未修复前哨站 | 明确位置和所需材料 |
| 已修复前哨站 | 明确位置和激活状态 |
| 异常预兆 | 区域性强信号，不显示具体奖励 |

不可显示：

```text
容器内部具体物品
容器内部品质
容器内部数量
容器是否必出任务条件物
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 触发条件 | 玩家在家中进入 HOME_OVERVIEW 时显示全图信号 |
| 执行规则 | 离家后只显示当前视野范围内信号 |
| 情报规则 | 容器只显示机会，不显示完整奖励 |
| 情报规则 | 前哨材料比容器更明确，服务主流程推进 |
| 异常规则 | 异常预兆显示区域性强信号，不显示具体奖励 |
| 失败规则 | 信号满屏或奖励过透明都属于设计失败 |
| 反馈规则 | 调试模式可显示信号真实类型和隐藏信息，正式模式隐藏 |

---

## 4. Godot 落地范围

| 路径 | 职责 |
|---|---|
| `res://scripts/map/SignalSource.gd` | 信号源基础数据 |
| `res://scripts/map/SignalVisibilityController.gd` | 家中/离家可见规则 |
| `res://scripts/ui/MapSignalLayer.gd` | 信号绘制层 |
| `res://scripts/debug/SignalDebugOverlay.gd` | 调试显示真实信息 |

---

## 5. 数据结构

信号字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `signal_id` | `StringName` | 信号 ID |
| `signal_type` | `StringName` | container/material/outpost/anomaly/extract |
| `world_position` | `Vector2` | 地图位置 |
| `signal_strength` | `float` | 强弱 |
| `home_visibility_level` | `int` | 家中显示级别 |
| `field_visibility_level` | `int` | 离家显示级别 |
| `hidden_payload` | `Dictionary` | 不给玩家直接看的真实内容 |

---

## 6. 核心流程

```text
1. 容器、材料、前哨、异常点注册 SignalSource。
2. 玩家在家时 SignalVisibilityController 切到 HOME_OVERVIEW。
3. MapSignalLayer 显示所有允许的全图信号。
4. 玩家离家后切到 FIELD_LOCAL。
5. 只显示视野半径内的 SignalSource。
6. 调试模式可显示隐藏 payload，正式模式不可显示。
```

---

## 7. 信号与接口

依赖：

```text
CameraController：当前镜头模式
VisionController：当前视野范围
ResourceRefreshDirector：活跃容器/材料/异常
OutpostSystem：前哨状态
```

输出：

```text
signal_visibility_changed(signal_id, is_visible, level)
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 家中能看到容器机会信号 |
| A2 | 家中看不到容器具体奖励 |
| A3 | 前哨材料信号比容器更明确 |
| A4 | 离家后视野外信号隐藏 |
| A5 | 已修复前哨状态在家中可识别 |
| A6 | 调试模式可查看真实信号信息 |
