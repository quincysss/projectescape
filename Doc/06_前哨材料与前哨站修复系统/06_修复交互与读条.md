# 06-06 修复交互与读条

> 来源文档：`06_前哨材料与前哨站修复系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

实现玩家靠近前哨站、查看需求、提交材料、长按修复、读条完成激活的交互流程。

## 2. 交互条件

玩家可交互条件：

- 在交互范围内。
- 玩家存活。
- 前哨站状态为 `DAMAGED` 或 `REPAIRABLE`。
- 没有正在进行其他互斥读条。
- UI 未被强制锁定。

可修复条件：

- 所有材料需求满足。
- 前哨站状态为 `REPAIRABLE`。
- 玩家仍在交互范围内。

## 3. 描边与提示

前哨站可见提示分三层：

| 状态 | 表现 |
| --- | --- |
| 可查看需求 | 弱描边或图标 |
| 可提交材料 | 材料提示高亮 |
| 可修复 | 强描边，显示修复按键 |

不要用大段教学文字塞满画面。

## 4. 材料提交

提交规则：

- 可单项提交。
- 可一键提交当前背包中可用材料。
- 提交后从背包移除。
- 提交进度写入前哨站需求。
- 提交后不可从前哨站取回，V0.1 如需撤销应不实现。

## 5. 修复读条

建议：

```gdscript
@export var repair_seconds: float = 5.0
```

中断条件：

- 玩家离开范围。
- 玩家移动。
- 玩家受击或稳定值耗尽。
- 玩家松开交互键。
- 前哨站状态改变。

中断后：

- 状态回到 `REPAIRABLE`。
- V0.1 读条进度重置。
- 已提交材料不退回。

## 6. AI/MCP 开工提示词

```text
请实现前哨站修复交互与读条。

要求：
1. 玩家靠近前哨站可查看材料需求。
2. 支持单项提交和一键提交背包中的前哨材料。
3. 材料提交后从背包移除，并写入前哨需求进度。
4. 需求满足后前哨站进入 REPAIRABLE。
5. 玩家长按交互键开始修复读条。
6. 离开范围、移动、松开按键、受击或稳定值耗尽会中断。
7. 中断后读条重置，材料不退回。
8. 读条完成后激活前哨站。
```

## 7. 验收标准

- 材料不足时只能查看/提交，不能修复。
- 材料满足后能开始修复。
- 中断不会激活前哨站。
- 完成读条后只激活一次。

## 8. 暂不实现

- 修复小游戏。
- 多人协作加速读条。
- 材料退回。
