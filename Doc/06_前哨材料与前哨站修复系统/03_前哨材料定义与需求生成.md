# 06-03 前哨材料定义与需求生成

> 来源文档：`06_前哨材料与前哨站修复系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

把前哨站修复材料做成明确的任务材料：它们可以占背包、可拾取、可提交，但不能主要依赖容器随机掉落。

## 2. 材料定位

前哨材料应满足：

- 用于修复前哨站。
- 是局内目标物。
- 主要由专用材料点刷新。
- 可以在地图上通过提示识别。
- 重量适中，促使玩家做路线选择。

不应满足：

- 变成普通容器掉落垃圾。
- 完全随机不可控。
- 单个材料过重导致玩家无法探索。

## 3. 配置属性

前哨修复材料不再放入 `items.tab`，而是放入 `setting/repairmaterial.tab`。它们没有品质，不可带出局外，不可售卖，不可作为研究所材料。

配置字段：

| 字段 | 规则 |
| --- | --- |
| `id` | 前哨修复材料 id，不得与 `items.tab` 重复 |
| `display_name` | 显示名 |
| `weight` | 单个材料重量提示 |
| `icon` | 背包/结算图标 |
| `description` | 描述 |
| `color_hex` | 菱形拾取物颜色 |
| `enabled` | 是否进入随机需求池 |
| `enabled_version` | 启用版本 |

## 4. 需求生成规则

每局开始时，每个被选中的前哨站从 `repairmaterial.tab` 的 6 个启用材料中随机抽取 2 种，单个前哨站内不重复，每种数量为 1。

需求结构：

```gdscript
{
    "outpost_fuse": {
        "display_name": "前哨保险丝",
        "amount": 1,
        "weight": 0.15
    },
    "outpost_filter": {
        "display_name": "净化滤芯",
        "amount": 1,
        "weight": 0.35
    }
}
```

## 5. 显示规则

前哨站交互 UI 必须显示：

- 所需材料。
- 已提交数量。
- 背包中可提交数量。
- 缺少材料的来源提示。

材料世界物提示：

- 名称。
- 前哨材料图标。
- 重量。
- 目标前哨阶段或用途。

## 6. AI/MCP 开工提示词

```text
请实现前哨材料定义与需求生成。

要求：
1. 使用 setting/repairmaterial.tab 定义前哨修复材料。
2. 前哨材料必须通过 repair_material_id 被背包和修复系统识别。
3. 每个前哨站从启用材料池随机抽取 2 种，各 1 个，单站内不重复。
4. 需求结构包含 material_id、display_name、amount、weight。
5. 前哨站交互 UI 能显示需求、已提交、背包可提交数量。
6. 不要把前哨材料主来源做成容器随机掉落。
```

## 7. 验收标准

- 每个前哨站能生成明确需求。
- 背包中材料能被识别为可提交。
- UI 能显示缺少哪些材料。
- 材料重量不会严重压垮背包体验。

## 8. 暂不实现

- 前哨材料品质系统。
- 材料替代配方。
