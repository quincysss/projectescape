# 06-13 V0.1 最小交付清单

> 来源文档：`06_前哨材料与前哨站修复系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md`。

## 1. 最小闭环

```text
本局选择前哨站候选点
-> 生成损坏前哨站
-> 生成修复需求
-> 地图刷新所需材料
-> 玩家拾取并回到前哨站提交
-> 长按修复
-> 前哨站 ACTIVE
-> 安全区/临时存储/撤离联动生效
```

## 2. 必须交付

- 2 个前哨站阶段配置。
- 候选点随机与互斥。
- 前哨材料定义。
- 固定需求表。
- 材料刷新和 90 秒保底。
- 前哨站状态机。
- 材料提交 UI。
- 修复读条和中断。
- ACTIVE 后安全区。
- ACTIVE 后临时存储。
- 撤离条件联动接口。
- Debug 面板。

## 3. 可以延后

- 多人协作修复。
- 前哨升级。
- 前哨被攻击。
- 材料替代配方。
- 局外永久前哨成长。

## 4. 推荐施工顺序

1. OutpostDefinition 和阶段配置。
2. 候选点选择。
3. OutpostEntity 状态机。
4. 需求生成。
5. 材料刷新与拾取。
6. 材料提交。
7. 修复读条。
8. 安全区注册。
9. 前哨存储适配。
10. 撤离结算接口。
11. Debug 验证。

## 5. AI/MCP 开工提示词

```text
请根据 06_前哨材料与前哨站修复系统拆分文档，在 Godot 中完成 V0.1 最小闭环。

优先实现：
1. 两个前哨站阶段配置。
2. 候选点随机与互斥。
3. 前哨站状态机。
4. 固定修复需求表。
5. 前哨材料刷新和 90 秒保底。
6. 材料提交与修复读条。
7. ACTIVE 后安全区和临时存储。
8. 撤离/死亡结算接口。
9. Debug 面板和验证清单。

约束：
- 不要删除原始 06 文档。
- 前哨站不是永久家园。
- 前哨材料主来源不要做成容器随机掉落。
- 存储逻辑必须调用 07 背包/存储接口。
- 多人只预留字段。
```

## 6. 完成定义

- 玩家可以在本局找到前哨站。
- 玩家知道缺哪些材料。
- 材料能在地图上出现。
- 玩家能提交材料并修复。
- 修复后安全区和存储生效。
- 撤离条件能响应前哨站状态。
