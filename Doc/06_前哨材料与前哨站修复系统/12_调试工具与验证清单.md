## 1. Debug 面板字段

| 字段 | 用途 |
| --- | --- |
| selected_outpost_id | 当前选中前哨站 |
| state | 当前状态 |
| candidate_point_id | 被选中的候选点 |
| repair_requirements | 需求和提交进度 |
| missing_materials | 当前缺失材料 |
| guarantee_timer | 90 秒保底计时 |
| safe_zone_enabled | 安全区是否启用 |
| storage_item_count | 临时存储物品数量 |

## 2. Debug 按钮

- Force Select Candidates
- Spawn Required Material
- Trigger 90s Guarantee
- Add Materials To Backpack
- Submit All Materials
- Start Repair
- Complete Repair
- Reset Outpost
- Show Candidate Points
- Show Material Points

## 3. 验证清单

### 候选点

- [ ] 每局能选择前哨候选点。
- [ ] 互斥组不会重复。
- [ ] 非法点会显示拒绝原因。

### 材料

- [ ] 需求表正确生成。
- [ ] 所需材料会刷在合法点。
- [ ] 90 秒保底能触发。
- [ ] 拾取后 UI 更新。

### 修复

- [ ] 材料不足不能修复。
- [ ] 材料满足后进入 REPAIRABLE。
- [ ] 修复读条可中断。
- [ ] 完成后进入 ACTIVE。

### 安全区与存储

- [ ] ACTIVE 后注册安全区。
- [ ] 容器不会刷进前哨安全区。
- [ ] ACTIVE 后可打开临时存储。
- [ ] 死亡和撤离能处理存储内容。

## 4. AI/MCP 开工提示词

```text
请实现 06 前哨站系统 Debug 面板和验证辅助。

要求：
1. 面板显示前哨站状态、候选点、需求进度、缺失材料、保底计时、安全区状态和存储数量。
2. 提供强制选点、生成所需材料、触发 90 秒保底、添加材料到背包、提交全部材料、完成修复等按钮。
3. 可视化候选点和材料点位，并显示拒绝原因。
4. 所有状态转换、材料刷新、提交、修复完成写结构化日志。
5. 输出一份可在编辑器中逐项执行的验证步骤。
```

## 5. 验收标准

- Debug 工具能快速跑完整修复闭环。
- 候选点和材料问题能定位。
- 验证清单能覆盖 V0.1 主要风险。
