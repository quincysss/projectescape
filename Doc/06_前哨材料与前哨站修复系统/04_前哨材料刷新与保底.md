# 06-04 前哨材料刷新与保底

> 来源文档：`06_前哨材料与前哨站修复系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

实现前哨材料的专用刷新逻辑，让玩家通过探索和风险路线获得修复材料，同时用 120 秒存在时间换点刷新和 90 秒保底避免因为随机性卡死。

## 2. 点位池规则

材料点位来自地图配置：

```gdscript
{
    "point_id": "material_spawn_01",
    "position": Vector2.ZERO,
    "stage_group": "stage_1",
    "material_tags": ["basic_repair", "electric"],
    "risk_level": 1,
    "weight": 10,
    "cooldown_seconds": 0.0
}
```

点位筛选：

- 匹配当前前哨站阶段。
- 匹配缺失材料标签。
- 不在安全区内。
- 不与容器或关键交互重叠。
- 可通行或可到达。

## 3. 材料存在时间规则

V0.1 推荐材料生成后存在 120 秒。

120 秒内未被拾取时，旧材料点消失，并从同一前哨站可用材料点位池中立刻选择另一个点位重刷。

注意：

- 存在时间较长，点位仍应放在风险路线。
- 不要让材料全刷在家门附近。
- 拾取后是否再刷新由阶段需求决定。
- 过期换点重刷时，优先避开刚刚消失的原点位。

## 4. 90 秒保底规则

如果玩家在 90 秒内没有获得当前前哨站仍缺失的关键材料，系统可以提高相关材料刷新概率或强制刷新一个材料点。

实现建议：

```gdscript
const GUARANTEE_SECONDS := 90.0
var seconds_since_last_required_material_found := 0.0
```

触发保底时：

```text
选择缺失材料 -> 找合法点 -> 生成材料 -> 打 Debug 日志
```

保底不是直接送到玩家脚下，仍要出现在合理探索区域。

## 5. 拾取后处理

材料被拾取后：

- 标记该物已拾取。
- 通知前哨需求 UI 更新。
- 如果材料属于当前缺失项，重置 90 秒计时。
- 不自动提交，玩家仍需回前哨站。

## 6. 信号与接口

```gdscript
signal outpost_material_spawned(material_id: String, point_id: String)
signal outpost_material_picked(material_id: String, player_id: String)
signal outpost_material_guarantee_triggered(material_id: String)
```

## 7. AI/MCP 开工提示词

```text
请实现前哨材料刷新与 90 秒保底。

要求：
1. 从地图材料点位池读取 stage_group 和 material_tags。
2. 按当前前哨站缺失需求选择可生成材料。
3. 材料点不能位于安全区、不可通行区或关键交互点。
4. V0.1 材料生成后存在 120 秒；过期未拾取时移除并立刻换点重刷。
5. 如果 90 秒没有获得所需关键材料，触发保底刷新，但不能刷在玩家脚下或家门口。
6. 拾取材料后更新前哨需求 UI，但不自动提交。
```

## 8. 验收标准

- 缺失材料会在合理点位生成。
- 材料不会卡死不出现。
- 材料 120 秒过期后会换到另一个合法点位重刷。
- 保底刷新有 Debug 日志。
- 拾取材料后需求提示能更新。

## 9. 暂不实现

- 材料动态搬运 NPC。
- 材料争夺事件。
- 多人拾取冲突。
