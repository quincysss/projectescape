# 06-00 拆分规范与落地文档写法

> 来源文档：`06_前哨材料与前哨站修复系统_修订版_废土生存法则.md`
> 本目录只做落地拆分，不删除、不覆盖原文档。

## 0. 执行总则

本目录所有拆分文档必须遵守：

```text
Doc/00_落地执行文档总则.md
```

前哨文档必须优先说明候选点、状态机、需求、材料刷新、提交读条、安全区注册、存储和撤离联动。

## 1. 拆分目标

06 模块要把“前哨站修复”拆成可直接施工的 Godot 功能块：

```text
候选点随机 -> 需求生成 -> 材料刷新 -> 玩家搬运材料 -> 修复读条 -> 前哨站激活 -> 安全区/存储/撤离联动
```

拆分文档不是复述原文，而是给后续 AI/MCP 直接开工用的任务说明。

## 2. 统一文档结构

每份施工文档尽量包含：

- 实现目标
- 设计依据
- 功能规则细化
- Godot 落地范围
- 数据结构建议
- 核心流程
- 信号与接口
- AI/MCP 开工提示词
- 验收标准
- 暂不实现

## 4. 施工硬约束

- 前哨站不是第二个家。
- 前哨材料不能依赖容器随机掉落作为主来源。
- 前哨材料不能太重，否则会破坏探索节奏。
- 修复前哨站必须是局内风险选择，不是局外永久成长。
- 修复完成后提供临时安全、临时存储和撤离联动，但不替代家。
- 多人只预留字段，V0.1 不实现同步。

## 5. 推荐开工顺序

1. 前哨站配置与候选点。
2. 前哨站状态机。
3. 前哨材料需求生成。
4. 材料刷新与拾取。
5. 修复交互与读条。
6. 激活安全区。
7. 临时存储。
8. 撤离和死亡结算。
9. Debug 与验证。
