# 06-02 候选点随机与互斥规则

> 来源文档：`06_前哨材料与前哨站修复系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

每局从地图候选点中选择前哨站位置，让路线有变化，同时避免候选点过近、互相遮挡或破坏安全区/撤离点布局。

## 2. 候选点基础规则

候选点应由地图数据提供，前哨站系统只负责选择。

候选点字段建议：

```gdscript
{
    "point_id": "outpost_candidate_01",
    "stage_group": "stage_1",
    "position": Vector2.ZERO,
    "weight": 10,
    "mutual_exclusion_group": "north_road",
    "min_distance_to_home": 80.0,
    "min_distance_to_extract": 60.0,
    "tags": ["street", "medium_risk"]
}
```

## 3. 随机流程

```text
读取阶段配置
-> 获取对应候选点组
-> 过滤非法点
-> 按权重随机选择
-> 标记互斥组
-> 生成前哨站实体
```

非法点包括：

- 距离家太近。
- 距离撤离点太近。
- 与已选前哨站互斥。
- 位于不可通行区域。
- 与关键交互点重叠。

## 4. 互斥规则

互斥用于防止一局中两个前哨站出现在同一条路线或过近区域。

实现建议：

```gdscript
var used_exclusion_groups: Dictionary = {}

func is_candidate_available(candidate: Dictionary) -> bool:
    var group := candidate.get("mutual_exclusion_group", "")
    return group == "" or not used_exclusion_groups.has(group)
```

被选择后：

```gdscript
used_exclusion_groups[group] = true
```

## 5. 信号与接口

```gdscript
signal outpost_candidate_selected(stage_index: int, point_id: String)
signal outpost_candidate_rejected(point_id: String, reason: String)
```

需要地图系统提供：

```gdscript
func get_outpost_candidates(stage_group: String) -> Array[Dictionary]
func is_position_walkable(position: Vector2) -> bool
func is_position_reserved(position: Vector2) -> bool
```

## 6. AI/MCP 开工提示词

```text
请实现前哨站候选点随机与互斥规则。

要求：
1. 从地图候选点数据中按 stage_group 读取候选点。
2. 过滤距离家太近、距离撤离点太近、不可通行、关键交互点重叠、互斥组已占用的点。
3. 按 weight 进行加权随机。
4. 支持 2 个阶段分别选择候选点。
5. 记录 rejected reason，供 Debug 面板显示。
6. 发出 outpost_candidate_selected 和 outpost_candidate_rejected 信号。
```

## 7. 验收标准

- 每局前哨站位置可以变化。
- 同互斥组不会同时出现多个前哨站。
- 非法点不会被选中。
- Debug 能看到候选点拒绝原因。

## 8. 暂不实现

- 动态运行时迁移前哨站。
- 玩家建造自定义前哨站。
