# 06-07 安全区与视野表现

> 来源文档：`06_前哨材料与前哨站修复系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

前哨站修复完成后生成局部安全区，并用视野和场景反馈告诉玩家“这里暂时安全，但不是家”。

## 2. 安全区效果

ACTIVE 前哨站应提供：

- 小范围敌对威胁屏蔽或降低。
- 容器不在安全区内刷新。
- 可使用临时存储。
- 可能启用撤离联动。

不应提供：

- 无限治疗。
- 永久安全。
- 全地图敌人停止。
- 局外仓库入口。

## 3. 安全区接口

前哨站激活时向安全区系统注册：

```gdscript
SafeZoneService.register_temporary_safe_zone({
    "source_id": outpost_id,
    "position": global_position,
    "radius": safe_radius,
    "type": "outpost"
})
```

失效或局结束时注销：

```gdscript
SafeZoneService.unregister_safe_zone(outpost_id)
```

## 4. 视野表现

建议表现：

- 修复前：破损、暗色、低亮度指示灯。
- 可修复：局部高亮、工具提示。
- 修复中：进度条和轻微灯光变化。
- 修复后：安全半径边界、照明恢复、图标变化。

安全边界不要过亮，避免像永久基地。

## 5. 信号

```gdscript
signal outpost_safe_zone_enabled(outpost_id: String, radius: float)
signal outpost_safe_zone_disabled(outpost_id: String)
```

## 6. AI/MCP 开工提示词

```text
请实现前哨站修复后的安全区与视野表现。

要求：
1. 前哨站 ACTIVE 时向 SafeZoneService 注册 temporary safe zone。
2. 安全区半径来自 OutpostDefinition。
3. 安全区内禁止容器刷新，敌对威胁降低或屏蔽。
4. 局结束或前哨站禁用时注销安全区。
5. 用灯光、图标、边界或材质变化表现修复前/可修复/修复中/已激活。
6. 不要把前哨站表现成永久家园。
```

## 7. 验收标准

- 前哨站 ACTIVE 后安全区生效。
- 容器刷新系统能识别该安全区。
- 视野表现能区分修复前后。
- 局结束后安全区清理。

## 8. 暂不实现

- 安全区升级。
- 安全区被攻击破坏。
- 大型基地视觉演出。
