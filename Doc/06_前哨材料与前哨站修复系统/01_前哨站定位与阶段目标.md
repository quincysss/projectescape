# 06-01 前哨站定位与阶段目标

> 来源文档：`06_前哨材料与前哨站修复系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

建立前哨站的设计边界：前哨站是局内临时安全节点、临时存储节点和撤离推进节点，不是家的替代品。

V0.1 推荐局内 2 个前哨站：

- 第一前哨站：帮助玩家从家园向中层推进。
- 第二前哨站：帮助玩家向外层与撤离点推进。

## 2. 功能定位

前哨站修复后提供：

- 小范围安全区。
- 临时存储。
- 撤离条件或撤离路线联动。
- 探索路线的中继点。

前哨站不提供：

- 永久安全。
- 局外仓库。
- 免费治疗全套功能。
- 主基地建筑升级。
- 稳定刷资源点。

## 3. 与家的区别

| 项目 | 家 | 前哨站 |
| --- | --- | --- |
| 安全性 | 高，默认安全 | 修复后局部安全 |
| 存储 | 可作为主要仓库 | 临时存储，容量小 |
| 位置 | 固定 | 局内候选点随机 |
| 成长 | 局外长期成长 | 局内临时收益 |
| 资源循环 | 归档和整理 | 风险路线中继 |

## 4. 阶段目标

第一前哨站应让玩家感到：

- 离家更远但仍可退回。
- 材料需求低。
- 修复后能缓解背包和撤离压力。

第二前哨站应让玩家感到：

- 进入更高风险区域。
- 材料需求更复杂。
- 修复后明显改善撤离或高价值探索能力。

## 6. AI/MCP 开工提示词

```text
请在 Godot 中建立前哨站定位和阶段配置。

要求：
1. V0.1 支持 2 个前哨站阶段。
2. 前哨站是局内临时中继点，不得实现为永久家园。
3. 每个前哨站配置包含阶段、候选点组、修复需求、安全区半径、临时存储容量、撤离联动开关。
4. 输出 OutpostDefinition/OutpostStageConfig/OutpostManager 的脚本或资源结构。
5. 不实现完整修复流程，只先建立配置和阶段边界。
```

## 7. 验收标准

- 工程中能配置两个前哨站阶段。
- 前哨站配置与家园系统分离。
- 前哨站不会被当作永久仓库。
- 后续模块可以读取阶段配置生成需求和候选点。

## 8. 暂不实现

- 局外前哨站升级。
- 前哨站防御建设。
- 多前哨站网络。
