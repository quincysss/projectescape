# 06-08 前哨站存储规则

> 来源文档：`06_前哨材料与前哨站修复系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

实现前哨站修复后的临时存储，让玩家能缓解背包压力，但不能把前哨站当作永久仓库。

## 2. 初始容量

V0.1 建议：

| 前哨站 | 存储容量 |
| --- | --- |
| 第一前哨站 | 小容量，约 6-10 格 |
| 第二前哨站 | 中小容量，约 8-12 格 |

具体容量需要与 07 背包系统对齐。

## 3. 存储规则

- 只有 ACTIVE 前哨站可打开存储。
- 玩家必须在交互范围内。
- 可以从背包转入前哨存储。
- 可以从前哨存储取回背包。
- 受背包容量和负重约束。
- 局结束时根据死亡/撤离规则处理。

## 4. 不能作为永久仓库

限制建议：

- 前哨存储只在本局有效。
- 死亡失败时前哨存储物品丢失。
- 撤离成功时按结算规则带回或转入局外仓库。
- 不允许局外直接访问前哨存储。

## 6. AI/MCP 开工提示词

```text
请实现前哨站临时存储。

要求：
1. 只有 ACTIVE 前哨站可以打开存储。
2. 存储容量来自 OutpostDefinition。
3. 支持背包到前哨、前哨到背包的转移。
4. 转移必须调用 07 背包/存储系统接口，不要单独写一套物品逻辑。
5. 前哨存储是局内临时存储，死亡失败时丢失，撤离成功时交给结算规则处理。
6. 局外不能直接访问前哨存储。
```

## 7. 验收标准

- 未修复前无法打开存储。
- 修复后可以转移物品。
- 背包容量不足时取回失败且物品不消失。
- 死亡/撤离结算能处理前哨存储内容。

## 8. 暂不实现

- 前哨存储扩容升级。
- 多人共享仓库权限。
