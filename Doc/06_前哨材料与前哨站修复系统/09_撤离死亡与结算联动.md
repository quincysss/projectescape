# 06-09 撤离、死亡与结算联动

> 来源文档：`06_前哨材料与前哨站修复系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

明确前哨站修复与撤离条件、死亡失败、撤离成功之间的物品归属和流程边界。

## 2. 撤离条件联动

前哨站可以参与撤离：

- 修复某个前哨站后解锁撤离点。
- 修复第二前哨站后降低撤离风险。
- 前哨站作为撤离路线中继。

V0.1 推荐采用简单规则：

```text
至少修复指定前哨站 -> 撤离点可用
```

## 3. 死亡后处理

玩家死亡失败时：

- 局内背包物品按死亡规则处理。
- 前哨存储物品丢失或按失败规则处理。
- 前哨站修复状态不带回局外永久进度。
- Debug 记录结算原因。

## 4. 撤离成功处理

撤离成功时：

- 背包中可带回物品进入局外结算。
- 前哨存储内容按规则转入结算或遗弃。
- 前哨站本局状态清理。
- 临时安全区注销。

## 5. 结算接口

```gdscript
func collect_outpost_settlement_payload() -> Dictionary:
    return {
        "activated_outposts": activated_ids,
        "temporary_storage_items": storage_items,
        "enabled_extractions": extraction_ids
    }
```

## 6. AI/MCP 开工提示词

```text
请实现前哨站与撤离/死亡结算的接口。

要求：
1. 前哨站修复后可解锁或影响撤离条件。
2. V0.1 可使用“指定前哨站 ACTIVE 后撤离点可用”的简单规则。
3. 玩家死亡失败时，前哨站修复状态和临时存储不得变成永久收益。
4. 撤离成功时，将前哨站临时存储内容交给结算系统处理。
5. 局结束时注销前哨安全区并清理临时状态。
```

## 7. 验收标准

- 未满足前哨条件时撤离点不可用或提示缺失条件。
- 修复前哨站后撤离状态更新。
- 死亡不会把前哨存储带回局外。
- 撤离成功结算能读取前哨存储内容。

## 8. 暂不实现

- 多路线复杂撤离策略。
- 前哨站永久解锁撤离点。
