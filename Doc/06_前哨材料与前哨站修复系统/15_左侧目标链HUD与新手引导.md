# 06-15 左侧目标链 HUD 与新手引导

> 来源文档：`06_前哨材料与前哨站修复系统_修订版_废土生存法则.md`
> 关联文档：`08_撤离与结算系统/02_撤离条件与激活显示.md`、`20_UI视觉规范与界面资源Sheet提示词_修订版_废土生存法则.md`
> 施工补充：本文件用于解决玩家不理解“菱形前哨材料 -> 修复前哨站 -> 解锁撤离”的学习成本问题。

## 1. 实现目标

将左侧金黄色框从“前哨站状态列表”改造为“当前目标链 HUD”，让玩家在局内始终理解当前主线目标：

```text
收集前哨站材料（菱形） n/N
-> 前哨站修复
-> 解锁撤离
```

目标不是增加长教程，而是在玩家行动过程中持续表达因果链。

## 2. 版式规则

左侧黄色框分三行，使用纵向目标链：

```text
当前目标：收集前哨站材料（菱形） 0/2
下一步：前哨站修复
解锁撤离：未解锁
```

推荐结构：

| 区域 | 内容 | 规则 |
| --- | --- | --- |
| 标题行 | `当前目标：收集前哨站材料（菱形） n/N` | 字号最大，用文字说明材料外观，不再使用小号菱形符号 |
| 第二行 | `下一步：前哨站修复` | 中等字号，前哨站可修复时高亮 |
| 第三行 | `解锁撤离：未解锁/已解锁` | 与顶部撤离状态一致 |

目标链三行均为单行显示，不允许自动换行；外框需要横向放宽、纵向收紧，保证“当前目标：收集前哨站材料（菱形） n/N”完整落在第一行。玩家在家中全图观察时外框保持高可读性；玩家离家进入近景探索后，目标链外框降低透明度，减少对场景、容器和材料信息的遮挡。

避免继续显示“I / II 未修复”作为主信息。前哨编号可以作为小角标或 Debug 信息保留，但不能压过目标链。

## 3. 状态文案

### 3.1 缺材料

```text
当前目标：收集前哨站材料（菱形） 0/2
下一步：前哨站修复
解锁撤离：未解锁
```

### 3.2 已收集足够材料但未修复

```text
当前目标：前往前哨站
下一步：长按修复前哨站
解锁撤离：未解锁
```

### 3.3 修复中

```text
当前目标：修复前哨站 45%
下一步：保持按住直到完成
解锁撤离：未解锁
```

### 3.4 修复完成

```text
当前目标：撤离已解锁
下一步：返回家中撤离
解锁撤离：已解锁
```

## 4. 数据来源

HUD 不直接计算任务状态，只读取系统状态：

```gdscript
OutpostManager.get_current_required_material_count()
OutpostManager.get_current_collected_material_count()
OutpostManager.get_current_outpost_state()
OutpostManager.get_repair_progress()
ExtractionController.is_extraction_unlocked()
```

建议输出统一视图模型：

```gdscript
{
    "title": "当前目标：收集前哨站材料（菱形） 1/2",
    "next_step": "下一步：前哨站修复",
    "extraction": "解锁撤离：未解锁",
    "stage": "collect_material",
    "progress": 0.5
}
```

## 5. 触发更新规则

以下事件必须刷新左侧目标链：

- 本局开始。
- 前哨需求生成。
- 玩家拾取菱形前哨材料。
- 玩家提交前哨材料。
- 前哨站进入 `REPAIRABLE`。
- 修复读条进度变化。
- 前哨站进入 `ACTIVE`。
- 撤离状态解锁。

拾取材料时必须出现短反馈：

```text
前哨材料已获得 1/2
```

该反馈 1.5-2 秒后消失，左侧目标链继续常驻。

## 6. 视觉规则

- 左侧目标链使用“前哨站材料（菱形）”文字说明材料外观；地图中的拾取物本体仍使用菱形图标。
- 当前阶段使用亮黄/白色。
- 未完成阶段使用灰黄。
- 已完成阶段可出现短暂绿色勾，但不要长期把左侧框变成绿色。
- 三行文字必须在黄色框内完整显示，不允许被黑色底条遮挡。
- 禁止在框内放长段解释文字。

## 7. 与撤离系统的关系

第三行必须与 08 撤离状态一致：

- 前哨未 ACTIVE：`解锁撤离：未解锁`
- 指定前哨 ACTIVE 且撤离点可用：`解锁撤离：已解锁`
- 玩家在家中且已解锁：可显示 `解锁撤离：可撤离`

如果玩家走到撤离点但未解锁，撤离 UI 必须反向提示：

```text
撤离未解锁：先修复前哨站
```

## 8. AI/MCP 开工提示词

```text
请将左侧金黄色前哨信息框改造成目标链 HUD。

要求：
1. 标题行显示“当前目标：收集前哨站材料（菱形） n/N”，n 来自已获得/已提交的当前前哨材料进度，N 来自当前前哨需求。
2. 第二行显示下一步：“前哨站修复”；当材料足够时改为“长按修复前哨站”；修复中显示修复百分比。
3. 第三行显示撤离状态：“解锁撤离：未解锁/已解锁/可撤离”，状态来自 ExtractionController。
4. HUD 不直接计算业务状态，只读取 OutpostManager 和 ExtractionController 的视图模型。
5. 玩家拾取菱形前哨材料时，短暂提示“前哨材料已获得 n/N”，并刷新左侧目标链。
6. 前哨站 ACTIVE 后，左侧目标链切换为“当前目标：撤离已解锁 / 下一步：返回家中撤离 / 解锁撤离：已解锁”。
7. 三行文字必须完整显示在黄色框中，不能被底图遮挡；不要显示长教程文本。
```

## 9. 验收标准

- 新玩家只看左侧框就能理解先收集菱形材料。
- 拾取材料后 n/N 立即更新。
- 材料足够后下一步明确指向修复前哨站。
- 修复完成后左侧框明确显示撤离已解锁。
- 第三行与顶部撤离状态不矛盾。
