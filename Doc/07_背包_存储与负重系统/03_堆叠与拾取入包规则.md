# 07-03 堆叠与拾取入包规则

> 来源文档：`07_背包_存储与负重系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `16_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

实现物品堆叠和拾取入包，保证容器、地图拾取、前哨材料拾取都能通过同一接口处理。

## 2. ItemStack

```gdscript
class_name ItemStack
extends Resource

@export var stack_id: String
@export var item_id: String
@export var count: int
@export var quality: String
@export var origin: Vector2i
@export var size: Vector2i
```

## 3. 堆叠规则

可堆叠条件：

- `item_id` 相同。
- `quality` 相同。
- 未达到 `stack_limit`。
- 特殊状态一致，V0.1 可忽略耐久/词缀。

堆叠流程：

```text
先合并已有堆叠
-> 剩余数量寻找空格
-> 能放多少放多少
-> 返回未放入数量
```

## 4. 拾取入包接口

```gdscript
func try_add_item(item_id: String, count: int, quality: String) -> Dictionary
```

返回：

```gdscript
{
    "accepted_count": 2,
    "rejected_count": 1,
    "reason": "no_space",
    "became_overweight": false
}
```

失败不能吞物品，调用方必须能把 rejected 部分留在容器或世界物中。

## 5. AI/MCP 开工提示词

```text
请实现背包堆叠与拾取入包接口。

要求：
1. ItemStack 包含 stack_id、item_id、count、quality、origin、size。
2. try_add_item 先合并同 item_id/quality 的未满堆叠，再寻找空位。
3. 支持部分成功，返回 accepted_count 和 rejected_count。
4. 背包满时 rejected_count 必须准确，物品不能消失。
5. 超重不阻止入包，只在返回值中标记 became_overweight。
6. 05 容器和 06 前哨材料都必须通过该接口拾取。
```

## 6. 验收标准

- 同类物品优先堆叠。
- 堆叠满后开新格。
- 空间不足时部分成功。
- 调用方能保留未拾取物品。
