# 07-08 容器残留与丢弃边界

> 来源文档：`07_背包_存储与负重系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `16_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

明确容器残留和丢弃边界，避免拾取失败吞物品，也避免 V0.1 因世界掉落引入刷物问题。

## 2. 容器残留

05 容器拾取时：

- 背包可接收的部分进入背包。
- 背包无法接收的部分保留在容器。
- 关闭容器面板不会销毁剩余物。
- 容器过期时剩余物随容器消失。

07 只提供入包结果，不直接管理容器生命周期。

## 3. 丢弃规则

V0.1 推荐：

- 玩家可以从背包丢弃。
- 丢弃意味着销毁物品。
- 不生成世界掉落。
- 需要二次确认的高品质物品可后续实现。

为什么不掉落：

- 避免刷物。
- 避免地图长期堆积物。
- 避免多人同步复杂度。

## 4. 接口

```gdscript
func discard_stack(stack_id: String, count: int = -1) -> Dictionary
```

返回：

```gdscript
{
    "discarded_count": 1,
    "removed_stack": true,
    "reason": "discarded"
}
```

## 5. AI/MCP 开工提示词

```text
请实现容器残留和丢弃边界所需的背包接口。

要求：
1. try_add_item 支持部分成功，并把 rejected_count 返回给 05 容器。
2. 背包满时不吞容器物品。
3. discard_stack 从背包删除物品，但 V0.1 不生成世界掉落。
4. 丢弃操作写日志，便于 Debug。
5. 不要让丢弃物重新被拾取。
```

## 6. 验收标准

- 容器中未拾取物品会保留。
- 背包丢弃会删除物品。
- 丢弃不会在世界生成可拾取物。

## 7. 暂不实现

- 世界掉落。
- 掉落物过期。
- 高品质丢弃确认。
