# 07-11 死亡撤离与局外仓库接口

> 来源文档：`07_背包_存储与负重系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `16_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

明确死亡失败、撤离成功、局外仓库之间的物品归属，供结算系统调用。

## 2. 死亡失败

死亡失败时：

- 局内背包按失败规则丢失或保留少量保险物。
- 前哨临时存储默认丢失。
- 家中存储不受影响。
- Debug 记录丢失列表。

V0.1 可采用简单规则：局内背包全部丢失，家中仓库保留。

## 3. 撤离成功

撤离成功时：

- 背包物品进入结算。
- 可选导入前哨临时存储。
- 结算后进入局外仓库或家中存储。
- 清空局内背包临时状态。

## 4. 局外仓库接口

```gdscript
func export_inventory_for_settlement() -> Array[Dictionary]
func import_items_to_home_storage(items: Array[Dictionary]) -> Dictionary

```

## 5. AI/MCP 开工提示词

```text
请实现背包系统的死亡/撤离结算接口。

要求：
1. 死亡失败时导出丢失物品日志，局内背包清理，家中存储不受影响。
2. 撤离成功时导出背包物品给结算系统。
3. 结算系统可以把物品导入家中存储。
4. 前哨临时存储由 06 提供 payload，07 负责接收物品列表并尝试入库。
5. 所有导入必须处理空间不足，不得吞物。
```

## 6. 验收标准

- 死亡后局内物品按规则清理。
- 撤离后物品进入家中存储或结算列表。
- 家中仓库不会因死亡被清空。
- 空间不足有明确返回。
