# 07-12 排序、一键入库与多人预留

> 来源文档：`07_背包_存储与负重系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `16_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

为背包整理体验和未来多人存储同步预留接口。V0.1 可以只做基础排序和本地版本号。

## 2. 排序规则

推荐排序键：

```text
item_type -> quality -> weight -> item_id
```

排序只改变显示或重新排布网格，必须保证不丢物。

## 3. 一键入库预留

一键入库规则见 `10_物品转移与一键入库.md`，本文件补充策略：

- 可按类型过滤。
- 可跳过收藏/锁定物品。
- 可优先存材料。
- 后续可支持一键取出配方材料。

## 4. 多人预留字段

```gdscript
var owner_player_id: String = "local_player"
var revision: int = 0
var last_modified_by: String = ""
var access_policy: String = "owner_only"
```

未来多人权威应覆盖：

- 背包物品增减。
- 存储转移。
- 丢弃。
- 装备栏变化。
- 结算导入。

## 5. AI/MCP 开工提示词

```text
请为背包/存储系统实现基础排序和多人预留字段。

要求：
1. 排序支持按 item_type、quality、weight、item_id。
2. 排序过程不得丢失物品或复制物品。
3. ItemStack/InventoryGrid/StorageContainer 包含 owner_player_id、revision、last_modified_by、access_policy。
4. V0.1 使用 local_player。
5. 每次物品变化 revision +1。
6. 不实现网络同步和权限 UI。
```

## 6. 验收标准

- 排序后物品数量一致。
- 本地单机流程正常。
- revision 能反映物品变化。

## 7. 暂不实现

- 多人同步。
- 权限管理 UI。
- 智能整理算法。
