# 07-09 背包与存储界面规则

> 来源文档：`07_背包_存储与负重系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `16_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

实现背包、家中存储、前哨存储的基础界面规则，让玩家能查看、转移、理解容量和负重。

## 2. 打开条件

背包：

- 玩家存活。
- 未被强制 UI 锁定。

存储：

- 家中存储需要在家中交互范围或局外整理界面。
- 前哨存储需要前哨 ACTIVE 且在交互范围。
- 家中存储不因进入家安全区自动打开，玩家需按 TAB 主动打开。
- 前哨存储保持前哨规则：未研究存储前不自动打开；研究激活存储后，进入已修复前哨站自动打开。

## 3. 显示内容

背包界面显示：

- 网格。
- 物品图标。
- 堆叠数量。
- 品质颜色。
- 当前负重/负重上限。
- 负重阶段提示。

存储界面显示：

- 左侧背包。
- 右侧存储。
- 选中物品详情。
- 转移按钮或双击转移。

家中界面补充：

- 玩家在家中时，顶部撤离信号 UI 下方提示“按 TAB 打开背包界面”。
- 玩家按 TAB 打开后显示背包 + 家中存储。
- 玩家离开家时，已打开的家中背包/存储界面自动关闭。

## 4. UI 状态

必须处理：

- 背包满。
- 存储满。
- 超重。
- 不可转移物品。
- 外部系统刷新物品列表。

## 5. AI/MCP 开工提示词

```text
请实现背包与存储基础 UI。

要求：
1. 背包显示网格、物品图标、堆叠数量、品质颜色、当前负重和阶段。
2. 存储界面使用左右双栏：背包 + 存储。
3. 支持单个转移和一键转移入口。
4. 失败时显示简短原因，如空间不足、不可存入。
5. 前哨存储 UI 必须通过 06 的 ACTIVE 条件打开。
6. UI 不直接改物品数组，必须调用 InventoryService/StorageContainer 接口。
```

## 6. 验收标准

- 玩家能打开背包。
- 家中和前哨存储能显示双栏。
- 转移失败有反馈。
- 负重变化会刷新 UI。
