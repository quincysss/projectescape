# 07-01 物品定义与品质规则

> 来源文档：`07_背包_存储与负重系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `16_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

建立统一 ItemDefinition，让容器、前哨材料、消耗品、装备和存储系统使用同一物品数据源。

## 2. 基础字段

```gdscript
class_name ItemDefinition
extends Resource

@export var id: String
@export var display_name: String
@export var item_type: String
@export var quality: String
@export var weight: float
@export var grid_size: Vector2i = Vector2i.ONE
@export var stack_limit: int = 1
@export var icon: Texture2D
@export var sellable: bool = false # 历史兼容字段，新经济优先使用 sale_good
@export var sell_currency_id: String = "mine_coin"
@export var sell_value: int = 0
@export var base_sale_value: int = 0
@export var tags: Array[String]
```

## 3. 物品类型

建议 V0.1 类型：

- `material`
- `outpost_material`
- `consumable`
- `equipment`
- `ammo`
- `quest`
- `misc`

前哨材料必须有 `outpost_material` 类型或标签，供 06 系统识别。

## 4. 品质规则

品质建议：

| 品质 | 用途 |
| --- | --- |
| C | 常见基础物资 |
| B | 普通有效物资 |
| A | 稀有高价值物资 |
| S | 极稀有物资 |
| SS | 红色超稀有物资，极低概率产出，主要用于收藏、炫耀和高价值售卖 |

品质影响：

- 容器掉落权重。
- UI 颜色。
- 结算价值。
- 制造物资等级和货台成交价值参考。
- 后续可影响装备属性。

V0.1 不要求所有物品都拥有复杂品质效果。

SS 品质规则：

- UI 主色为红色。
- 必须配置可用于高阶制造、收藏展示或后续特殊需求的标签。
- 如果作为可售物资，则应使用 `item_type=sale_good` 和 `base_sale_value > 0`。
- 建议包含 `ss_rare`、`showcase` 标签。
- 不得作为前哨核心修复材料。
- 不得作为撤离解锁条件。
- 不进入普通 C/B/A/S 圈层品质权重，掉落由 05 的 SS 专用规则控制。

首批 SS 道具：

| item_id 建议 | 显示名 |
| --- | --- |
| `ss_silverwing_engine_core` | 银翼发动机核心 |
| `ss_pink_star` | 粉星 |
| `ss_wanming_pocket_watch` | 万鸣怀表 |
| `ss_old_world_gold_bar` | 旧世金砖 |
| `ss_zero_master_control_board` | 零号主控板 |

## 5. AI/MCP 开工提示词

```text
请实现统一 ItemDefinition 资源。

要求：
1. 字段包含 id、display_name、item_type、quality、weight、grid_size、stack_limit、icon、sellable、sell_currency_id、sell_value、tags。
2. 支持 C/B/A/S/SS 品质，其中 SS 为红色超稀有品质。
3. 支持 material、outpost_material、consumable、equipment、ammo、quest、misc 类型。
4. 前哨材料必须能被 06 系统识别。
5. 05 容器掉落、06 前哨材料、07 背包都必须使用同一物品定义。
6. 原材料不再要求配置直接出售价格；可售物资使用 item_type=sale_good 和 base_sale_value。
7. SS 道具必须包含 ss_rare/showcase 标签，不得被当作普通 S 品质处理；如果作为可售物资，必须配置 base_sale_value>0。
8. 首批 SS 道具为银翼发动机核心、粉星、万鸣怀表、旧世金砖、零号主控板。
```

## 6. 验收标准

- 同一 item_id 可以被背包、容器、前哨系统读取。
- 物品重量和格子尺寸可配置。
- 品质能用于 UI 和掉落权重。
- sale_good 物品能被货台系统读取基础成交价值。
- SS 道具能以红色品质显示，并能进入仓库、图鉴和高阶制造链。

## 7. 暂不实现

- 装备随机词缀。
- 耐久度。
- 物品拆解配方。
- 多货币价格浮动。
