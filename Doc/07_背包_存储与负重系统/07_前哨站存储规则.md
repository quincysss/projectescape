# 07-07 前哨站存储规则

> 来源文档：`07_背包_存储与负重系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `16_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

为 06 前哨站提供临时存储实现接口，强调它与家中存储不同：局内有效、容量小、结算时处理。

## 2. 前哨容量

容量由 06 的 `OutpostDefinition.storage_capacity_slots` 或网格尺寸决定。

07 提供通用 `StorageContainer`，06 负责创建和销毁。

## 3. 存储规则

- 只有 ACTIVE 前哨站可访问。
- 需要玩家在交互范围内。
- 使用与背包相同的转移接口。
- 死亡失败时默认丢失。
- 撤离成功时交给结算系统。

## 4. 接口

```gdscript
func create_temporary_storage(owner_id: String, width: int, height: int) -> String
func get_storage(storage_id: String) -> StorageContainer
func destroy_temporary_storage(storage_id: String) -> Array[ItemStack]
```

## 5. AI/MCP 开工提示词

```text
请实现前哨站临时存储的背包系统接口。

要求：
1. 07 提供 StorageContainer，06 前哨站通过接口创建临时存储。
2. 前哨存储容量由 06 配置传入。
3. 支持与背包双向转移。
4. 死亡失败时可销毁并丢弃物品。
5. 撤离成功时可导出物品列表给结算系统。
6. 不要把前哨存储接入家中长期仓库。
```

## 6. 验收标准

- 06 可以创建前哨临时存储。
- 前哨存储能转移物品。
- 销毁时能返回剩余物品用于结算或丢弃。

## 7. 暂不实现

- 多人共享权限。
- 前哨存储扩容。
