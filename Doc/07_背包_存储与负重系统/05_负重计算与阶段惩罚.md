# 07-05 负重计算与阶段惩罚

> 来源文档：`07_背包_存储与负重系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `16_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

实现总负重计算、负重上限和阶段惩罚。负重应制造取舍，但不能用“禁止拾取”打断核心循环。

## 2. 负重定义

总负重包含：

- 背包内物品。
- 装备栏物品。
- 快捷栏引用的物品本体，避免重复计算时需注意引用关系。

```gdscript
func calculate_total_weight() -> float
```

## 3. 负重阶段

建议：

| 阶段 | 范围 | 效果 |
| --- | --- | --- |
| NORMAL | 0-70% | 无惩罚 |
| HEAVY | 70-100% | 轻微移速提示 |
| OVERWEIGHT | 100-130% | 明显移速惩罚 |
| CRITICAL | 130%+ | 强警告，可限制冲刺 |

## 4. 超重规则

- 超重不阻止拾取。
- 超重触发 UI 警告。
- 超重影响移动速度或冲刺能力。
- 极限阶段可限制部分动作，但 V0.1 不建议硬锁移动。

## 5. 接口

```gdscript
signal weight_changed(current: float, limit: float, stage: String)
func get_weight_stage() -> String
func get_speed_modifier() -> float
```

## 6. AI/MCP 开工提示词

```text
请实现负重计算和阶段惩罚。

要求：
1. 总负重来自背包物品和装备物品。
2. 实现 NORMAL、HEAVY、OVERWEIGHT、CRITICAL 四个阶段。
3. 超重不阻止拾取，只触发 UI 警告和移动惩罚。
4. 发出 weight_changed 信号。
5. 提供 get_speed_modifier 给移动系统读取。
6. 注意快捷栏如果只是引用背包堆叠，不要重复计算重量。
```

## 7. 验收标准

- 拾取物品后负重更新。
- 装备物品后负重更新。
- 超重仍可拾取。
- 移动系统能读取惩罚倍率。

## 8. 暂不实现

- 体力与负重深度联动。
- 受伤导致负重上限变化。
