# 07-10 物品转移与一键入库

> 来源文档：`07_背包_存储与负重系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `16_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

实现背包、家中存储、前哨存储之间的可靠转移和一键入库。

## 2. 转移规则

背包到存储：

- 检查目标存储空间。
- 可堆叠先堆叠。
- 成功后从背包移除对应数量。
- 失败时背包不变。

存储到背包：

- 检查背包空间。
- 成功后从存储移除。
- 可能导致超重，但不阻止。
- 失败时存储不变。

## 3. 一键存入

一键存入建议：

```text
先材料
-> 再普通资源
-> 再低品质
-> 跳过装备中物品、快捷栏锁定物品、任务锁定物品
```

一键存入必须支持部分成功，并报告剩余。

## 4. 原子性

转移必须避免半途失败导致复制或丢失。

推荐流程：

```text
模拟目标能接收多少
-> 从源容器扣除
-> 写入目标容器
-> 如果写入失败则回滚
```

## 5. AI/MCP 开工提示词

```text
请实现物品转移和一键入库。

要求：
1. 支持背包到存储、存储到背包。
2. 转移先模拟目标容量，避免复制或吞物。
3. 可堆叠物品优先合并。
4. 一键存入跳过装备中、快捷栏锁定、任务锁定物品。
5. 支持部分成功，并返回 moved_count、remaining_count、reason。
6. 取回导致超重时允许成功并触发 weight_changed。
```

## 6. 验收标准

- 空间足够时转移成功。
- 空间不足时部分成功或失败且不丢物。
- 一键入库不会移动锁定物品。
- 不会复制物品。
