## 1. 最小闭环

```text
物品定义
-> 拾取入背包
-> 网格占格/堆叠
-> 负重刷新
-> 背包 UI 展示
-> 家中/前哨存储转移
-> 死亡/撤离结算
```

## 2. 必须交付

- ItemDefinition。
- ItemDefinition 售价字段：sellable、sell_currency_id、sell_value。
- ItemStack。
- InventoryGrid。
- try_add_item。
- 堆叠和占格。
- WeightComponent。
- 背包 UI。
- StorageContainer。
- 家中存储。
- 前哨临时存储接口。
- 转移接口。
- 丢弃销毁。
- 死亡/撤离导出接口。
- Debug 面板。

## 3. 可以延后

- 物品旋转。
- 高品质丢弃确认。
- 世界掉落。
- 复杂装备属性。
- 多人同步。
- 智能整理。

## 4. 推荐施工顺序

1. ItemDefinition 和 ItemStack，包含商人售卖所需字段。
2. InventoryGrid 占格。
3. try_add_item 和堆叠。
4. WeightComponent。
5. StorageContainer。
6. 转移接口。
7. 背包/存储 UI。
8. 死亡撤离接口。
9. Debug 验证。

## 5. AI/MCP 开工提示词

```text
请根据 07_背包_存储与负重系统拆分文档，在 Godot 中完成 V0.1 最小闭环。

优先实现：
1. ItemDefinition / ItemStack，包含 sellable、sell_currency_id、sell_value。
2. InventoryGrid 的占格、放置、移除、查找空位。
3. try_add_item：堆叠优先、支持部分成功、不吞物。
4. WeightComponent：负重阶段和移动惩罚接口。
5. StorageContainer：家中存储和前哨临时存储。
6. 背包/存储之间转移，保证不复制、不丢物。
7. 丢弃销毁，不生成世界掉落。
8. 死亡/撤离结算接口。
9. Debug 面板。

约束：
- 05 容器和 06 前哨必须通过本系统接口处理物品。
- 超重不阻止拾取。
- 空间不足不能吞物。
- 前哨存储不是永久仓库。
```

## 6. 完成定义

- 玩家可以拾取物品进背包。
- 背包满时物品不消失。
- 负重随物品变化。
- 超重仍能拾取并产生惩罚提示。
- 家中和前哨存储能转移物品。
- 死亡/撤离能导出正确物品归属。
