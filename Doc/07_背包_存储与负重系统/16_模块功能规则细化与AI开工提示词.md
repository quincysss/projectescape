# 07 模块功能规则细化与 AI 开工提示词

> 来源文档：`07_背包_存储与负重系统_修订版_废土生存法则.md`
> 用途：作为 07 模块的 AI/MCP 总入口。

## 1. 总体实现目标

背包、存储与负重系统是局内资源循环底座：

```text
物品定义
-> 物品堆叠
-> 网格背包
-> 拾取入包
-> 负重阶段
-> 家中/前哨存储
-> 物品转移
-> 死亡/撤离结算
```

05 容器、06 前哨材料、06 前哨存储都必须通过本系统接口，避免物品逻辑分裂。

## 3. 核心规则总表

| 项目 | V0.1 规则 |
| --- | --- |
| 背包 | 网格占格 |
| 堆叠 | 同 item_id + quality 合并 |
| 价值 | ItemDefinition 提供品质、标签和可售物资基础成交价 |
| S 品质 | 当前最高常规品质，可收藏、可用于高阶制造，不参与前哨修复主线 |
| 拾取 | 堆叠优先，再找空格 |
| 拾取失败 | 返回 rejected_count，不吞物 |
| 超重 | 不阻止拾取，只提示/惩罚 |
| 家中存储 | 长期存储 |
| 前哨存储 | 局内临时存储 |
| 丢弃 | 直接销毁，不生成世界掉落 |
| 死亡 | 局内背包按失败规则处理 |
| 撤离 | 背包导出给结算/家中存储 |
| 多人 | 只预留 owner/revision 字段 |

## 4. 推荐核心接口

```gdscript
func try_add_item(item_id: String, count: int, quality: String) -> Dictionary
func remove_stack(stack_id: String, count: int = -1) -> Dictionary
func transfer_between(source_id: String, target_id: String, stack_id: String, count: int) -> Dictionary
func discard_stack(stack_id: String, count: int = -1) -> Dictionary
func calculate_total_weight() -> float
func export_inventory_for_settlement() -> Array[Dictionary]
```

返回值必须包含明确 reason，供 UI、05 容器和 Debug 使用。

## 5. 分阶段 AI 开工提示词

### 阶段一：物品与背包核心

```text
请读取 07 拆分文档中的 01、02、03、13。
实现 ItemDefinition、ItemStack、InventoryGrid 和 InventoryService.try_add_item。
要求支持网格占格、堆叠优先、部分成功、失败 reason、不吞物，并在 ItemDefinition 中包含品质、标签、stack_limit、base_sale_value。
品质必须支持 C/B/A/S；S 使用最高品质表现，主要用于收藏、高阶制造或高级可售物资配方。
05 容器和 06 前哨材料后续都要通过 try_add_item。
09 货台后续通过 sale_good 配置读取基础成交价，不在 UI 中硬编码。
```

### 阶段二：负重、装备和快捷栏

```text
请读取 07 拆分文档中的 04、05。
实现装备栏、消耗品快捷栏和 WeightComponent。
装备重量计入总负重，超重不阻止拾取，只提供 UI 警告和移动惩罚倍率。
```

### 阶段三：存储与转移

```text
请读取 07 拆分文档中的 06、07、09、10。
实现家中存储、前哨临时存储、背包/存储 UI 和可靠转移接口。
转移必须避免复制或丢物；一键存入跳过装备中、快捷栏锁定和任务锁定物品。
```

### 阶段四：结算、丢弃和 Debug

```text
请读取 07 拆分文档中的 08、11、12、14、15。
实现丢弃销毁、死亡/撤离导出、排序、多人与 revision 预留、Debug 面板。
V0.1 不生成世界掉落，不实现多人同步。
```

## 6. 总体验收标准

- 物品定义可被 05、06、07 共同使用。
- 背包占格和堆叠正确。
- 满包拾取不会吞物。
- 超重仍能拾取并触发提示/惩罚。
- 家中存储和前哨存储边界清楚。
- 转移不会复制或丢物。
- 死亡/撤离归属清楚。
- Debug 能复现满包、超重、转移失败。
- S 品质物品可以入包、入库、结算、图鉴展示和高阶制造。

## 7. 给后续 AI 的硬约束

```text
不要删除原始 07 文档。
不要让外部系统直接改背包数组。
不要在空间不足时吞物品。
不要用超重阻止拾取。
不要把前哨存储接成永久仓库。
不要在 V0.1 做世界掉落。
不要再新增独立于 C/B/A/S 的额外品质分支。
不要实现多人同步，只预留字段。
```
