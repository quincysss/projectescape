> 文档版本：v0.1  
> 所属项目：《废土生存法则》  
> 前置文档：13_数据配置表与TAB规范.md / 16_Godot工程结构与代码模块规划.md  
> 适用阶段：V0.3 / V0.4 数据驱动落地  
> 文档目标：定义 `.tab` 数据校验、调试面板、Godot 编辑器插件和错误报告规范，减少配置错误进入运行期。

---

## 1. 本文档一句话说明

数据校验工具负责在游戏运行前发现表格错误。

---

## 2. 设计目标

工具需要做到：

```text
读取所有 .tab。
检查字段缺失。
检查 ID 重复。
检查跨表引用。
检查数值范围。
检查资源路径。
生成清晰错误报告。
支持开发模式一键重载。
```

不追求：

```text
完整 Excel 编辑器。
复杂数据库管理。
在线表格同步。
策划权限系统。
```

---

## 3. 校验入口

推荐三个入口：

```text
游戏启动时自动校验。
DebugPanel 手动校验。
Godot EditorPlugin 菜单校验。
```

启动校验：

```text
开发模式：ERROR 阻止进入主流程。
发布模式：记录错误，禁用错误行，尽量不中断游戏。
```

手动校验：

```text
显示错误列表。
点击错误可复制文件名、行号、字段名。
```

---

## 4. 错误等级

| 等级 | 含义 | 处理 |
|---|---|---|
| ERROR | 会导致系统错误 | 开发模式阻止运行 |
| WARN | 可运行但表现或数据异常 | 允许继续 |
| INFO | 建议优化 | 只显示 |

ERROR 示例：

```text
重复 ID。
引用不存在。
必填字段为空。
数值类型错误。
装备扩展表 id 不存在于 items.tab。
```

WARN 示例：

```text
icon 为空。
description 为空。
enabled_version 为空。
掉落权重为 0。
```

## 5. 校验规则清单

## 5.1 表结构

```text
文件存在。
表头存在。
必需字段存在。
字段名无重复。
未知字段给 WARN。
空行跳过。
# 注释行跳过。
```

---

## 5.2 ID

```text
id 不为空。
id 全局或表内唯一。
id 只使用小写英文、数字、下划线。
id 不使用 test、new、temp 等临时名进入正式版本。
```

---

## 5.3 跨表引用

```text
equipment.id -> items.id
consumables.id -> items.id
materials.id -> items.id
recipes.required_conditions -> condition registry
recipes.output_item -> items.id
recipes.required_items -> items.id
research.required_items -> items.id
effect_id -> effects.id
unlock_recipe -> recipes.id
```

---

## 5.4 数值范围

```text
weight >= 0
stack_limit >= 1
durability_max >= 1
initial_durability_min <= initial_durability_max
cooldown_seconds >= 0
craft_time_seconds >= 0
drop weight >= 0
stability modifier > 0
```

---

## 5.5 资源路径

```text
icon 填写后必须存在。
appearance_scene 填写后必须存在。
audio_path 填写后必须存在。
路径必须以 res:// 开头。
```

## 6. DataValidator 设计

推荐接口：

```text
validate_all() -> ValidationReport
validate_table(table_name) -> ValidationReport
validate_references() -> ValidationReport
validate_resources() -> ValidationReport
```

ValidationIssue 字段：

```text
level
table_name
file_path
line
field
id
message
suggestion
```

示例：

```text
ERROR equipment.tab:12 field=id id=backpack_a message="items.tab 中不存在该 id"
```

---

## 7. DebugPanel 设计

功能：

```text
Reload Data。
Validate Data。
Show Errors。
Filter ERROR/WARN/INFO。
Copy Issue。
Open Resource Path。
Print Definition。
```

显示格式：

```text
[ERROR] equipment.tab:12 id=backpack_a items.tab missing base item
[WARN] items.tab:30 id=scrap_metal icon empty
```

---

## 8. Godot EditorPlugin 预留

后续可做插件：

```text
菜单：ProjectEscape / Validate Data
菜单：ProjectEscape / Reload Data
菜单：ProjectEscape / Export Data Snapshot
Dock：Data Validation Panel
```

插件不负责修改表格。

插件只负责：

```text
读取。
校验。
报告。
跳转资源。
导出快照。
```

## 11. 验证清单

```text
缺少 items.tab 能报错。
重复 id 能报错。
引用不存在能报错。
错误行号正确。
资源路径不存在能警告或报错。
一键重载后 DataRegistry 更新。
发布模式不会因 WARN 阻止进入游戏。
```

---

## 12. 本文档结论

数据驱动会提升维护效率，但也会制造新的错误入口。

因此 V0.3 开始接入 `.tab` 时，必须同步接入：

```text
DataValidator。
错误分级。
调试面板。
清晰报告。
```

先把错误拦在启动前，后续内容扩张才不会失控。
