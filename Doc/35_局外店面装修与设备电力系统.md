# 35_局外店面装修与设备电力系统

> 文档版本：v0.1
> 所属项目：《废土生存法则》
> 关联文档：`29_局外杂货店经营与界面重构路线图.md`、`30_昼夜局外流程_店铺营业与夜间出发准备.md`、`31_掉落物资与制造经济规划.md`、`33_资源类别与用途闭环规则.md`。
> 文档目标：定义店面装修作为矿币消耗口和经营构筑系统的边界，避免它变成单纯数值堆叠或重新制造材料卡点。

## 1. 系统定位

店面装修不是自由摆放模拟器，而是固定槽位设备系统。

它负责回答：

```text
钱赚来干什么。
店铺如何改变卖什么、留什么、加工什么。
店铺如何反哺下一次探索。
```

## 2. 获取方式

家具/设备主要通过购买获得。

```text
购买：主要路线，花矿币解决。
制作：省钱路线，需要材料，但不作为唯一入口。
任务/订单：解锁特殊设备或高级供应商。
```

不要让核心设备只依赖随机材料制作，否则会把资源循环重新卡死。

## 3. 固定槽位

第一版建议只开放 5-6 个槽位，后续扩展到约 10 个。

| 槽位 | 功能边界 | 第一版建议 |
|---|---|---|
| 发电机位 | 提供电力上限 | 必做 |
| 货架位 | 决定可上架数量和商品类型 | 必做 |
| 工作台位 | 解锁加工、组合商品、批量制作 | 必做 |
| 冷藏位 | 食物/药品库存和相关商品 | 建议 |
| 仓储位 | 增加局外库存或分类存储 | 建议 |
| 招牌位 | 影响顾客类型和需求热度 | 可选 |
| 情报位 | 提供地图资源提示和折扣情报 | 可选 |
| 照明位 | 影响营业效率、满意度、偷窃风险 | 后续 |
| 安保位 | 减少抢劫、损耗和事故 | 后续 |
| 休息位 | 员工效率或角色恢复 | 后续 |

## 4. 电力规则

发电机提供电力上限，设备消耗电力。

第一版规则：

```text
总耗电不能超过当前发电量。
超过时不能启用新设备。
玩家必须手动关闭其他设备或升级发电机。
关闭的设备不提供效果。
```

不建议第一版做随机跳闸。跳闸可以后续作为事件、劣质设备或天气事故。

示例：

```text
当前发电量：10
已启用耗电：8
新冰柜耗电：4
结果：无法启用，需先关闭其他设备或升级发电机
```

## 5. 设备功能原则

每类设备应改变经营方式，不只堆百分比。

| 设备类型 | 应改变的问题 | 不建议只做 |
|---|---|---|
| 货架 | 卖多少、卖哪类 | 全商品售价 +5% |
| 工作台 | 能做什么、能否批量 | 制作速度 +5% |
| 冷藏 | 食物药品如何经营 | 普通售价 +5% |
| 招牌 | 吸引谁来买 | 全顾客数量 +5% |
| 情报台 | 下一次去哪找 | 随机给一点钱 |
| 安保 | 损耗和风险 | 纯防御数值 |

## 6. 冷藏位第一版口径

如果暂时没有完整保质期系统，冷藏位先不要强行引入复杂腐坏。

第一版可做：

```text
增加食物/药品类局外库存上限。
允许制作或上架冷藏类高价商品。
提高食物/药品需求日的成交稳定性。
```

保质期、腐坏、冷藏损耗等系统后续再加。

## 7. 与研发关系

研发只消耗矿币，但可以要求设备作为条件。

示例：

```text
仓库整理 III 需要安装中级仓储架。
高级冷藏商品需要安装冰柜。
信号辨识 II 需要安装情报台。
```

设备作为前置条件，不作为材料消耗。

## 8. 数据配置建议

### 8.1 shop_fixture_slots.tab

用途：定义店铺固定槽位、解锁条件和允许安装的设备类型。

建议字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| slot_id | string | 槽位 ID，如 generator_slot / shelf_slot_1 |
| display_name | string | 槽位显示名 |
| allowed_fixture_types | list | 允许设备类型 |
| unlock_conditions | string | 章节、任务、店铺等级等条件 |
| first_version_required | bool | 是否首版必做槽位 |
| enabled_version | string | 启用版本 |

### 8.2 shop_fixtures.tab

用途：定义可购买或可制作的设备。

建议字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| fixture_id | string | 设备 ID |
| display_name | string | 设备名 |
| fixture_type | enum | generator/shelf/workbench/refrigerator/storage/sign/intel/security |
| slot_id | string | 默认安装槽位或槽位类型 |
| tier | int | 设备档位 |
| power_provided | int | 发电机提供电力，非发电设备为 0 |
| power_cost | int | 启用耗电 |
| coin_price | int | 矿币购买价格 |
| craft_recipe_id | string | 可选制作配方，允许为空 |
| unlock_conditions | string | 供应商、任务、店铺等级等解锁条件 |
| effect_type | enum | shelf_capacity/unlock_recipe/storage_limit/demand_bias/intel_hint/risk_reduce |
| effect_value | string | 效果参数 |
| enabled_version | string | 启用版本 |

### 8.3 fixture_vendor.tab

用途：定义设备商人上架内容，承接矿币消耗口。

建议字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| vendor_id | string | 商人或供应商 ID |
| fixture_id | string | 可售设备 ID |
| unlock_conditions | string | 解锁条件 |
| price_modifier | float | 价格修正 |
| stock_policy | enum | always/limited/rotation |
| enabled_version | string | 启用版本 |

## 9. Godot 模块建议

| 模块 | 职责 |
|---|---|
| ShopFixtureService | 查询、购买、安装、拆卸设备 |
| PowerBudgetService | 计算发电量、已启用耗电和是否允许启用 |
| FixtureVendorService | 根据解锁条件生成设备商人列表 |
| FixtureEffectService | 将设备效果转给店铺、制作、仓库、目标面板等系统 |

设备系统只发出能力和条件，不直接改掉落。比如情报台可以提高地图提示精度，但不应直接把某容器改成必出。

## 10. 校验规则

版本启用时必须检查：

```text
每个 fixture 必须能安装到至少一个允许槽位。
非发电设备的 power_cost 必须大于等于 0。
发电机 power_provided 必须大于 0。
启用设备时，PowerBudgetService 必须拒绝超过发电量的方案。
超过电力上限时不做随机停机，玩家必须主动关闭设备或升级发电机。
核心首版设备必须能通过矿币购买获得。
制作路线只能作为省钱或进阶路线，不能作为核心入口。
```

## 11. BDD 场景补充

Feature: 店面装修与设备电力

  Scenario: 电力不足时不能启用新设备
    Given 当前发电量为 10
    And 已启用设备总耗电为 8
    When 玩家尝试启用耗电为 4 的冰柜
    Then 系统应拒绝启用
    And 提示玩家关闭设备或升级发电机

  Scenario: 购买是核心设备的主入口
    Given 玩家拥有足够矿币
    When 玩家打开设备商人列表
    Then 基础发电机、基础货架和基础工作台应可通过矿币购买

  Scenario: 设备可作为研发前置条件
    Given 玩家已经安装中级仓储设备
    And 玩家拥有足够矿币
    When 玩家研究仓库整理 III
    Then 系统应允许研究
    And 不扣除材料
