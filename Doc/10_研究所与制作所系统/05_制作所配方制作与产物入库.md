# 10-05 制作所配方制作与产物入库

> 来源文档：`10_研究所与制作所系统_修订版_废土生存法则.md`  
> 目标：定义制造所/制作所如何读取配方、消耗仓库材料并生成产物。

## 1. 实现目标

制作所负责将局外仓库材料转化为可携带物品。

```text
玩家打开制造所
-> 查看配方列表
-> 选择已解锁配方
-> 校验材料、矿币、前置研究、配方可用条件和仓库空间
-> 确认制作
-> 原子扣除材料
-> 生成 ItemInstance
-> 产物进入局外仓库
```

## 2. 设计依据

- V0.2 首版建议即时制作，不做真实时间队列。
- 配方数据走 `recipes.tab`，脚本只解释规则。
- 制作只能消耗局外仓库物品，不能消耗结算待处理物、出发准备已装备物或待领取物。
- 制作产物第一版优先直接入库；仓库不足时阻止制作。

## 4. 配方字段

`recipes.tab` 建议字段：

```text
recipe_id	display_name	category	output_item_id	output_quantity	required_items	required_currency_id	required_currency_amount	required_research_ids	required_condition_id	craft_time_seconds	instant_craft	enabled	enabled_version	description
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `recipe_id` | 配方 ID |
| `category` | `consumable`、`equipment`、`backpack`、`module` 等 |
| `output_item_id` | 产物物品 ID |
| `required_items` | `item_id:count;item_id:count` |
| `required_currency_*` | 可选货币消耗，默认 `mine_coin` |
| `required_research_ids` | 前置研究列表，可空 |
| `required_condition_id` | 前置配方可用条件，可空 |
| `instant_craft` | V0.2 首版为 `true` |

## 5. 制作流程

```text
1. RecipeManager 加载 recipes.tab。
2. CraftingManager 查询配方可见性和可制作状态。
3. UI 展示产物、材料、拥有数量、前置条件。
4. 玩家点击制作。
5. 系统重新获取制作 quote，避免 UI 状态过期。
6. 满足条件时弹出确认。
7. 玩家确认。
8. WarehouseManager 原子扣除材料。
9. CurrencyWallet 原子扣除货币。
10. CraftingManager 生成产物 ItemInstance。
11. WarehouseManager 尝试入库。
12. 保存仓库和玩家档案。
```

## 6. 仓库空间规则

V0.2 首版采用保守规则：

```text
制作前预检查仓库空间。
空间不足时制作按钮置灰。
空间不足不扣材料、不扣货币、不生成产物。
```

后续可扩展待领取列表：

```text
材料已扣除但产物无法入库时，产物进入 pending_claim_items。
pending_claim_items 不可用于研究、制作或出发准备。
玩家整理仓库后手动领取。
```

如果实现待领取列表，必须新增专门验证，不允许静默丢失产物。

## 7. 推荐首批配方

| 配方 ID | 名称 | 分类 | 解锁 | 材料 | 产物 |
| --- | --- | --- | --- | --- | --- |
| `craft_field_bandage` | 临时绷带 | 消耗品 | 默认 | `cloth_dirty:2;medicine_powder:1` | `field_bandage x1` |
| `craft_stability_candy` | 安定糖 | 消耗品 | 默认 | `sugar_pack:1;medicine_powder:1` | `stability_candy x1` |
| `craft_backpack_small` | 小型加固背包 | 背包 | `research_backpack_1` | `cloth_dirty:6;duct_tape_roll:3;reinforced_strap:1` | `backpack_small_reinforced x1` |

如果某个材料当前 `items.tab` 尚未存在，首版实现应优先选用已有材料，不要为了配方新增大量未投放资源。

## 8. UI 规则

- 制造所未解锁时，显示 03 文档定义的锁定页。
- 解锁后显示配方列表。
- 未解锁配方可以隐藏或显示锁定来源，首版推荐显示锁定来源。
- 材料和产物图标 hover 使用局外 Tooltip。
- 制作确认弹窗必须显示消耗和产物。
- 制作成功后刷新仓库、材料拥有数和配方状态。

## 9. 验收标准

- 序章返回剧情前不进入白天制造所主流程；配方未解锁时不能制作。
- 未解锁配方不能制作。
- 材料不足时不能制作。
- 仓库空间不足时不能制作，且不扣资源。
- 确认前取消不扣资源。
- 确认后扣除材料并生成产物。
- 产物进入局外仓库并保存。
- 重启后产物仍在仓库中。

## 10. 暂不实现

```text
真实时间制作队列。
制作取消返还比例。
随机词条制作。
制作暴击或失败。
装备强化。
多产物配方。
复杂材料替代。
```
