# 10-01 研究所移动速度研究规则

> 本文补充 `10_研究所与制作所系统_修订版_废土生存法则`。  
> 目标：先落地研究所最小闭环，让玩家消耗矿币并满足前置条件，获得永久移动速度成长。

## 1. 规则边界

- 研究所负责永久成长，不直接生产物品。
- 当前首版只实现 `move_speed` 一条研究线，共 3 档。
- 研究即时完成，不做现实时间倒计时。
- 研究只消耗局外货币 `mine_coin`。
- 可配置前置条件，但当前 `move_speed` 首版可以只检查矿币和前置等级。
- 矿币不足或前置不足时不能研究，且不扣除任何资源。
- 研究完成后写入局外存档字段 `research_levels`，不会在单局结束时清空。

## 2. 配置表

路径：

```text
setting/research.tab
```

字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| research_id | string | 研究线 ID，同一研究线多档共用同一个 ID |
| display_name | string | 当前档位显示名 |
| category | string | 分类，移动速度使用 `mobility` |
| level | int | 当前档位 |
| max_level | int | 最大档位 |
| required_items | string | 历史字段；研究不再消耗材料，必须留空或由兼容层忽略 |
| required_currency_id | string | 货币 ID，当前使用 `mine_coin` |
| required_currency_amount | int | 货币消耗数量 |
| required_conditions | string | 可选前置条件，如任务、章节、设备或任务条件 |
| effect_type | string | 效果类型，移动速度使用 `player_move_speed_multiplier` |
| effect_value | float | 效果值；移动速度使用最终倍率，例如 `1.05` 表示 +5% |
| enabled | bool | 是否启用 |
| enabled_version | string | 启用版本 |
| description | string | 策划说明 |

## 3. 移动速度研究线

| 档位 | 名称 | 前置条件建议 | 货币消耗 | 最终移速倍率 |
|---:|---|---|---:|---:|
| 1 | 行动训练 I | 无 | 80 矿币 | 1.05 |
| 2 | 行动训练 II | 成功撤离 2 次 | 180 矿币 | 1.10 |
| 3 | 行动训练 III | 店铺等级 2 或章节进度 | 360 矿币 | 1.16 |

`effect_value` 表示最终倍率，不是每级叠乘倍率。完成 3 档后，玩家基础移动速度为未研究时的 `116%`。

## 4. 生效时机

- 研究完成后，局外存档立即记录等级。
- 已经开始的单局不 retroactively 修改玩家速度。
- 下一次进入探索时，RunScene 创建玩家并读取 `get_player_move_speed_multiplier()`，将倍率写入玩家基础移动速度。
- 负重惩罚仍通过玩家 `speed_multiplier` 叠加在基础速度之后。

## 5. UI 规则

- 研究所作为白天准备阶段功能入口启用。
- 列表展示研究名称、当前等级、最大等级、下一档效果与状态。
- 选中研究后展示矿币拥有量、矿币需求、前置条件和完成后效果。
- 研究所不展示材料需求图标；Tooltip 用于解释效果、矿币消耗和前置条件。
- 资源不足时研究按钮置灰。
- 完成研究后刷新列表、仓库、货币与研究结果提示。

## 6. 验收标准

- `research.tab` 能被配置表注册器读取。
- 矿币不足或前置条件不足时不能研究，且不扣任何资源。
- 矿币和前置条件满足时可以完成研究，并只扣除矿币。
- 完成 1、2、3 档后，`move_speed` 等级分别为 1、2、3。
- 玩家进入下一局时基础移动速度按研究倍率提升。
- 查看研究 Tooltip 不会扣除矿币或保存研究状态。

## 7. Debug 面板

局外 BaseScene 允许保留开发期 Debug 面板，用于快速验证局外成长闭环：

- `+500 矿币`：直接给局外货币，用于研究、货台扩展、出发补给等局外消耗测试。
- `满足研究前置`：写入测试用任务/章节/店铺等级条件。
- `加入可售物资`：向仓库加入 sale_good，用于货台成交和货币流转测试。
- `补齐研究资源`：按 `move_speed` 下一档配置补齐缺少矿币和前置条件。
- `模拟货台成交`：调用货台正式接口扣除上架物资并增加矿币。
- `完成下一档移速`：补齐下一档成本后直接调用正式研究接口完成研究。
- `重置研究`：清空研究等级，方便重复验证。

Debug 面板只能调用正式 `GameState` / `ShelfInventoryService` / `CurrencyWallet` / `ResearchManager` 接口，不允许绕过正式扣除和入库逻辑。
