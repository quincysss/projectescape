# 10-03 制造所默认开放与首次开店接口

> 来源文档：`10_研究所与制作所系统_修订版_废土生存法则.md`
> 关联文档：`25_主菜单玩家档案与剧情章节目标系统/04_制造所解锁与第一章结束.md`、`32_局外白天经营流程`

## 1. 实现目标

制造所是局外经营默认设施，负责把局内带回材料或序章周转物资加工为可售物资，支撑白天开店闭环。

最新第一章目标不是解锁制造所，而是：

```text
制造第一批可售物资
-> 将可售物资摆上货台
-> 完成第一次开店结算
```

文案层可称为“制造所”，系统模块可沿用已有“制作所”命名。

## 2. 开放条件

```text
PlayerProfile.first_return_dialogue_seen == true
PlayerProfile.shop_loop_unlocked == true
```

说明：

- 制造所设施在世界观上默认存在。
- 新档开场后、序章首次出发前，不进入白天经营循环，因此玩家暂时不会操作制造所。
- 序章探索结束返回局外后，无论撤离成功或失败，剧情发放一次性开店周转物资，并开启白天经营循环。

## 3. 白天准备入口状态

白天准备阶段功能入口：

```text
仓库 / 制造所 / 研究所 / 图鉴
```

制造所入口规则：

- `shop_loop_unlocked == false` 时，制造所不作为主入口展示，或展示为剧情未到的弱化状态。
- `shop_loop_unlocked == true` 时，制造所可点击。
- 未解锁配方显示来源条件，例如研究、配方开放、章节进度或地点产出。
- 制造所入口本身不显示固定矿币解锁进度。

制造所配方如果展示制作材料、产物预览或配方开放图标，必须接入 `Doc/09_局外仓库系统/18_局外道具详情Tooltip.md` 的局外 Tooltip。

## 4. 首次开店教学接口

制造所需要向章节目标系统发出事件：

```gdscript
signal sale_good_crafted(recipe_id: StringName, output_item_id: StringName, count: int)
```

当玩家第一次制造任意 `sale_good=true` 的产物时：

```text
ChapterProgressService.complete_objective("craft_first_sale_good")
```

该事件只负责推进“制造第一批可售物资”目标，不直接完成上架或开店结算目标。

## 5. 周转物资配方要求

序章周转包必须能在制造所中合成至少 1 种 C 级可售物资，并尽量支持 2-3 种选择。

配方最低要求：

- 至少 1 个 C 级可售物资配方默认可见且可制作。
- 配方消耗材料来自 `prologue_shop_starter_pack` 或序章可带回基础材料。
- 配方产出进入仓库，不自动上架。
- 配方失败不吞材料。

## 6. AI/MCP 开工提示词

```text
请实现制造所默认开放和首次开店教学接口。

要求：
1. 不实现固定矿币解锁制造所。
2. shop_loop_unlocked=false 时，不进入制造所白天主流程。
3. 序章返回剧情结束后，shop_loop_unlocked=true，制造所入口可点击。
4. 周转物资必须能在制造所合成至少一种 C 级可售物资。
5. 玩家制造 sale_good=true 的产物后，发出 sale_good_crafted 并推进 first_shop_tutorial 的第一步。
6. 未解锁配方显示来源条件，不显示固定矿币解锁页。
7. 不实现旧独立交易入口，不允许仓库原材料直接换钱。
```

## 7. 验收标准

- 序章返回剧情前，玩家不会进入白天制造所主流程。
- 序章返回剧情后，制造所入口可用。
- 序章失败且空包返回时，周转物资仍能支撑至少一个 C 级可售物资配方。
- 制作成功后产物进入仓库。
- 制作成功后能推进“制造第一批可售物资”目标。
- UI 中不出现固定矿币解锁制造所目标。

## BDD 场景补充
