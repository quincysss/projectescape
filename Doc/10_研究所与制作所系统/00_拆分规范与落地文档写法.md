## 1. 系统定位

10 模块负责局外成长：

```text
序章探索结束返回局外
-> 成功带回物资或领取一次性开店周转物资
-> 局外仓库保存
-> 制造所把原材料加工为可售物资
-> 白天开店营业，货台成交获得矿币
-> 研究所消耗材料/矿币获得永久成长
-> 图纸、研究或章节进度扩展装备/消耗品配方
-> 产物进入仓库
-> 夜晚出发准备读取仓库物品进入下一局
```

研究所负责永久成长，不直接生产物品。
制造所/制作所负责可售物资、装备和消耗品生产，不直接提供永久属性。
图纸负责永久解锁配方，本身是局外仓库物品。

## 4. 施工硬约束

- 研究和制作只能消耗局外仓库中已经入库的物品。
- 已配置到出发准备的物品不能被研究或制作消耗。
- 材料扣除必须原子化：任一材料或货币不足时，全部不扣。
- 研究完成后写入玩家档案，不随单局结束清空。
- 制作产物必须进入仓库；仓库空间不足时应阻止制作，或进入待领取列表。
- UI 不允许直接修改仓库数组、货币字典、研究等级、周转物资领取 flag 或首次开店教学 flag。
- 研究所和制造所中的道具图标必须复用局外 Tooltip 规则。
- 制造所默认开放、序章周转物资和首次开店教学由 `03_第一章制造所解锁规则.md` 和 25-04 共同约束。
- 不把 `repairmaterial.tab` 的局内前哨修复材料用于研究或制作消耗。

## 5. 冲突处理

如果母版、拆分文档和工程实现冲突，处理顺序为：

```text
1. 已稳定运行并有测试覆盖的工程事实。
2. 本目录更具体的拆分文档。
3. 10 系统母版。
4. 旧的专题描述或历史提示词。
```

发现冲突时，不直接删除旧内容；先在具体拆分文档中写明当前落地事实和后续修订方向。
