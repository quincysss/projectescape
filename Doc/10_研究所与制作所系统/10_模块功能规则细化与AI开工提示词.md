## 1. 总体实现目标

建立局外成长闭环：

```text
仓库材料 + 矿币
-> 研究所永久成长
-> 制造所加工可售物资
-> 图纸学习
-> 配方制作
-> 产物入库
-> 下一局出发准备
```

## 3. 分阶段 AI 开工提示词

### 阶段一：研究所基础

```text
请读取 10-00、10-01、10-02、10-06、10-07、10-08。
实现或修订 ResearchManager、setting/research.tab、BaseScene 研究所 UI 和研究确认流程。
研究材料和矿币必须从 WarehouseManager / CurrencyWallet 原子扣除。
仓库容量按未研究 80，最终 120 的当前口径落地。
完成后运行 research_manager_check、research_capacity_application_check、research_speed_application_check。
```

### 阶段二：制造所默认开放和首次开店接口

```text
请读取 10-03 和 25-04。
实现制造所默认开放和首次开店教学接口。
新档序章首次返回剧情前，不进入白天制造所主流程。
序章返回剧情后，shop_loop_unlocked=true，制造所入口可点击。
周转物资必须能制造至少一种 C 级 sale_good。
制造 sale_good 后发出 sale_good_crafted，并推进第一章“制造第一批可售物资”目标。
UI 不得直接改货币或 flag。
完成后运行 profile_chapter_flow_check。
```

### 阶段三：图纸学习

```text
请读取 10-04、10-06、10-07、09-18。
新增 blueprints.tab、BlueprintManager 和配方解锁存档字段。
图纸是局外仓库物品，学习图纸必须消耗 WarehouseManager 中的图纸实例。
重复图纸不能再次学习，不自动删除或分解。
学习成功后写入 learned_blueprint_ids 和 unlocked_recipe_ids。
完成后补 blueprint_learning_check。
```

### 阶段四：制作所配方制作

```text
请读取 10-05、10-06、10-07、10-08。
新增 recipes.tab、RecipeManager、CraftingManager 和制造所配方 UI。
V0.2 首版即时制作，不做制作队列。
制作前检查配方解锁、材料、货币、前置研究、前置图纸和仓库空间。
确认后原子扣除材料和货币，生成产物并放入仓库。
仓库空间不足时阻止制作，不扣资源。
完成后补 crafting_manager_check。
```

### 阶段五：整体验证和整理

```text
请读取 10-08、10-09。
跑完整局外成长链路验证：研究、周转物资、可售物资制造、图纸学习、制作、产物入库、保存重载、Tooltip。
如果发现工程事实与文档冲突，优先反写具体拆分文档，不直接删母版。
```

## 4. 硬约束

```text
不要让 UI 直接修改仓库、货币、研究等级、周转物资领取或配方解锁状态。
不要消耗 repairmaterial.tab 的局内前哨修复材料。
不要让研究失败吞材料或矿币。
不要让制作失败吞材料、矿币或产物。
不要让序章返回剧情前进入白天制造所主流程。
不要让重复图纸自动消失。
不要第一版就做真实时间研究或制作队列。
不要让 Tooltip 触发任何消耗、学习、制作或保存。
```

## 5. 总体验收标准

- 研究所能读取 `research.tab`。
- 研究可消耗材料和矿币并保存等级。
- 仓库容量按 `80 -> 120` 当前口径生效。
- 制造所默认开放，周转物资能制造第一批可售物资。
- 图纸可学习并永久解锁配方。
- 配方可制作并把产物放入仓库。
- 研究/制作中的道具图标有局外 Tooltip。
- 所有失败分支都不吞资源。
- 重启后研究、制造所、图纸、配方和仓库产物仍保留。
