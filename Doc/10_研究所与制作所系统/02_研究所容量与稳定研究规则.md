# 10-02 研究所容量与稳定研究规则

> 来源文档：`10_研究所与制作所系统_修订版_废土生存法则.md`

## 目标

把研究所从单一移动速度研究扩展为局外长期成长系统。

本阶段已接入以下研究线：

| research_id | 档数 | 未研究基线 | 每档效果 |
|---|---:|---:|---|
| `inventory_slots` | 3 | 背包 8 格 | 12 / 16 / 20 格 |
| `home_storage_slots` | 3 | 家中安全箱 1 格 | 2 / 3 / 4 格 |
| `outpost_storage_slots` | 2 | 前哨安全箱 0 格，未激活 | 1 / 2 格 |
| `max_stability` | 5 | 稳定值 100 | 120 / 140 / 160 / 180 / 200 |
| `warehouse_capacity` | 5 | 局外仓库 80 格 | 80 / 90 / 100 / 110 / 120 格 |

## 配置规则

- 所有档位写入 `setting/research.tab`。
- `effect_value` 使用最终值，不使用增量值。
- 研究消耗从局外仓库扣除，货币使用 `mine_coin`。
- 不同研究线尽量使用不同资源组合，避免所有成长都争夺同一种基础材料。
- 五档研究的 Lv.4-Lv.5 必须明显提高材料品级、数量和矿币成本。
- 仓库容量当前按工程口径记录：未研究为 80 格，`warehouse_capacity` Lv.1-Lv.5 为 80 / 90 / 100 / 110 / 120 格。Lv.1 与未研究容量相同，视为当前配置事实。

## UI 补充规则

- 每条研究线的材料需求必须显示道具图标和 `拥有/需求` 数量。
- 材料图标 hover 时显示局外道具详情 Tooltip，规则见 `Doc/09_局外仓库系统/18_局外道具详情Tooltip.md`。
- Tooltip 只负责展示道具基础信息，不负责作为材料不足的主提示；缺材料状态仍由研究按钮、材料数量和缺失标记承担。
- hover 查看 Tooltip 不得触发扣除材料、扣除矿币或保存研究状态。

## 前哨安全箱特殊规则

未研究 `outpost_storage_slots` 时：

- 已修复前哨仍然提供安全区和稳定值恢复。
- 前哨安全箱容量为 0。
- 不创建前哨存储容器。
- 玩家进入已修复前哨安全区时，不自动打开前哨存储界面和背包界面。

完成 Lv.1 后：

- 已修复前哨激活存储功能。
- 玩家进入已修复前哨安全区时自动打开背包和前哨存储界面。
- 容量为 1 格。

完成 Lv.2 后容量为 2 格。

## 生效时机

| 效果 | 生效时机 |
|---|---|
| 背包容量 | 下一局开始时写入 `InventoryComponent.max_slots` |
| 家中安全箱容量 | 下一局开始时写入 `HomeStorageComponent.max_slots` |
| 前哨安全箱容量 | 下一局开始时写入 `RunConfig`，前哨修复后按容量决定是否创建存储 |
| 稳定值上限 | 下一局开始时写入 `StabilityComponent.max_stability` |
| 局外仓库容量 | 研究完成后立即刷新 `WarehouseManager.max_slots` |

## 验证清单

- 未研究时家中安全箱只有 1 格。
- 未研究时修复前哨不会创建前哨存储。
- `outpost_storage_slots` Lv.2 后前哨安全箱容量为 2。
- `inventory_slots` Lv.3 后局内背包容量为 20。
- `home_storage_slots` Lv.3 后家中安全箱容量为 4。
- `max_stability` Lv.5 后稳定值上限为 200。
- `warehouse_capacity` Lv.5 后局外仓库容量为 120。
- 任意研究材料图标 hover 后能显示局外道具详情 Tooltip。
