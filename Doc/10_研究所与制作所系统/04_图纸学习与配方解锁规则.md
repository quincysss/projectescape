# 10-04 图纸学习与配方解锁规则

> 来源文档：`10_研究所与制作所系统_修订版_废土生存法则.md`  
> 目标：定义图纸作为仓库物品时，如何被学习、消耗并永久解锁制作配方。

## 1. 实现目标

图纸系统负责把局内/商人获得的图纸物品转化为永久配方解锁。

```text
图纸进入局外仓库
-> 玩家在制造所/图纸界面选择学习
-> 校验配方未解锁
-> 确认后消耗图纸物品
-> 写入 learned_blueprint_ids / unlocked_recipe_ids
-> 制造所刷新配方列表
```

## 2. 设计依据

- 图纸本身是 `items.tab` 中的局外仓库物品，类型建议为 `blueprint`。
- 图纸学习后解锁配方，配方状态跟随玩家档案保存。
- 图纸不应用来卡住基础仓库容量、基础负重、稳定值上限这类核心成长。
- 重复图纸第一版保留为物品，不自动分解。

## 3. Godot 落地范围

允许修改：

```text
res://setting/items.tab
```

如果首版暂不拆 `BlueprintManager`，也可以先由 `GameState` 委托内部服务完成，但 UI 仍不得直接消耗图纸。

## 4. 字段与数据结构

`blueprints.tab` 建议字段：

```text
blueprint_id	display_name	item_id	unlock_recipe_id	quality	consume_on_learn	duplicate_policy	enabled	enabled_version	description
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `blueprint_id` | 图纸逻辑 ID |
| `item_id` | 对应 `items.tab` 中的图纸物品 ID |
| `unlock_recipe_id` | 学习后解锁的配方 |
| `consume_on_learn` | V0.2 默认为 `true` |
| `duplicate_policy` | 第一版使用 `keep_as_item` |

玩家存档建议字段：

```text
learned_blueprint_ids: Array[String]
unlocked_recipe_ids: Array[String]
```

## 5. 学习流程

```text
1. 玩家打开制造所/图纸界面。
2. 系统列出仓库中可学习的图纸。
3. 玩家选择图纸。
4. 系统检查 `unlock_recipe_id` 是否已解锁。
5. 未解锁时显示确认弹窗。
6. 玩家确认。
7. WarehouseManager 消耗对应图纸物品 1 件。
8. RecipeManager 写入配方解锁。
9. BlueprintManager 写入已学习图纸。
10. 保存玩家档案和仓库。
```

## 6. 重复图纸规则

第一版规则：

```text
配方已解锁时，重复图纸不能再次学习。
重复图纸仍保留在仓库。
重复图纸可由玩家手动丢弃或出售，具体取决于 items.tab.sellable。
系统不得自动删除或自动分解重复图纸。
```

后续可扩展但暂不实现：

```text
重复图纸分解为研究点。
重复图纸作为高级研究材料。
重复图纸提高制作成功品质。
```

## 7. 交互与提示

- 图纸图标 hover 必须显示局外道具详情 Tooltip。
- 已学习图纸显示“已学习”，学习按钮置灰。
- 未学习且图纸在仓库中时显示“学习图纸”。
- 配方已解锁但仍有重复图纸时，显示“配方已解锁，可保留/出售/丢弃”。

## 8. 验收标准

- 仓库中没有图纸时，图纸界面不显示可学习项。
- 学习未解锁图纸会消耗 1 件图纸。
- 学习后对应配方永久解锁。
- 重启后已学习图纸和已解锁配方仍保留。
- 重复图纸不能再次学习。
- 学习失败时不扣图纸，不解锁配方。

## 9. 暂不实现

```text
图纸碎片合成。
重复图纸自动分解。
图纸品质影响制作产物品质。
图纸交易市场。
图纸学习倒计时。
```
