# 06_前哨材料搜集闭环

> 对应母版章节：5.4、9、13.4  
> 实现层级：主目标材料直出、点位池、拾取与需求追踪。
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“前哨材料搜集闭环”小节。

---

## 1. 实现目标

前哨材料是推进主目标的资源。它不同于容器奖励：玩家在家中可看到更明确的材料信号，靠近后直接识别名称、数量、重量和是否为当前前哨需求；拾取后进入背包，并用于修复前哨站。

---

## 2. 设计依据

前哨材料可在家中显示：

```text
当前前哨站所需材料名称
当前已拥有数量 / 需求数量
地图上的前哨材料大致信号位置
```

靠近材料后显示：

```text
材料名称
材料数量
材料重量
是否为当前前哨需求材料
F 键拾取提示
```

前哨材料是直出资源，不走容器随机奖励流程。

## 4. 数据结构

材料字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `material_id` | `StringName` | 材料类型 ID |
| `display_name` | `String` | 展示名 |
| `amount` | `int` | 数量 |
| `weight_per_unit` | `float` | 单个重量 |
| `target_outpost_id` | `StringName` | 可为空，表示通用材料 |
| `is_required_now` | `bool` | 是否命中当前需求 |

需求追踪字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `active_outpost_id` | `StringName` | 当前推进目标 |
| `requirements` | `Dictionary` | `{material_id: required_amount}` |
| `owned_amounts` | `Dictionary` | 背包和家中可用材料统计 |
| `is_requirement_met` | `bool` | 是否满足修复条件 |

---

## 5. 核心流程

生成：

```text
1. 新局初始化时确定当前前哨站需求。
2. MaterialSpawner 从点位池生成对应材料。
3. 材料信号在家中可见，但不暴露过多非必要奖励信息。
4. 如果玩家长时间未获得关键材料，后续可接保底规则。
```

靠近与拾取：

```text
1. Player 进入材料交互范围。
2. 显示材料名称、数量、重量、是否当前需求。
3. 玩家按交互键。
4. InventoryComponent 检查格子和负重。
5. 可拾取则加入背包，并销毁世界材料节点。
6. RequirementTracker 重新统计当前需求进度。
7. 如果需求满足，通知前哨站可修复。
```

保底规则的 V0.1 处理：

```text
先预留接口，不做复杂保底。
如果调试时关键材料没有生成，应输出错误。
```

## 6. 信号与接口

`OutpostMaterialPickup` 发出：

```gdscript
signal material_detected(material_id: StringName, amount: int)
signal material_pickup_requested(material_id: StringName, amount: int)
signal material_picked(material_id: StringName, amount: int)
signal material_pickup_failed(material_id: StringName, reason: String)
```

`OutpostRequirementTracker` 发出：

```gdscript
signal requirement_progress_changed(outpost_id: StringName, progress: Dictionary)
signal requirement_met(outpost_id: StringName)
```

依赖接口：

```text
InventoryComponent.can_accept_item(material_item)
InventoryComponent.add_item(material_item)
OutpostStation.can_repair(requirement_tracker)
MaterialSignalDisplay.refresh(active_requirements, visible_pickups)
```

## 8. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 家中能看到当前前哨所需材料和拥有/需求数量 |
| A2 | 靠近材料后能看到明确拾取提示 |
| A3 | 材料拾取不经过容器开箱流程 |
| A4 | 背包或负重不足时拾取失败并保留世界材料 |
| A5 | 拾取成功后世界材料消失，背包增加对应物品 |
| A6 | 需求数量满足后发出 `requirement_met` |
