# 02-13 局外天数推进规则

> 本文补充 `02_核心循环规则_修订版_废土生存法则` 与 `08_撤离与结算系统`。  
> 目标：为局外阶段建立统一的天数概念，作为商人库存、未来局外事件、研究制作进度等系统的时间基准。

## 1. 基础定义

- `current_day` 是局外存档级字段，默认值为 1。
- 玩家第一次点击“开始探索”进入单局时，当前探索属于第 1 天。
- 开始探索本身不推进天数；天数用于描述当前这一轮探索发生在哪一天。
- 天数只在一局探索结束并回到局外时推进。

## 2. 推进时机

任何正式结算回局外的结果都会让天数加 1：

- 撤离成功后回到局外：第 1 天结束，局外显示第 2 天。
- 探索失败、死亡、稳定值耗尽或超时后回到局外：同样推进到下一天。
- 再次开始探索时，探索发生在当前显示的天数；再次结算回局外后继续加 1。

示例流程：

```text
初始局外 current_day = 1
-> 第一次开始探索：仍为第 1 天
-> 探索成功或失败并回到局外：current_day = 2
-> 第二次开始探索：第 2 天
-> 再次回到局外：current_day = 3
```

## 3. 不推进天数的情况

- 玩家停留在局外整理仓库、买卖物品、查看研究所或制作所时，不推进天数。
- 点击开始探索但未形成正式结算结果的调试中断，不应推进天数。
- 直接切换场景、编辑器调试重载等开发操作不作为正式天数推进依据。

## 4. 系统联动

- 商人随机库存按天刷新：当天库存只生成一轮，购买后数量减少，买空不自动补货。
- 当探索结算回到局外并推进到下一天时，旧商人库存清空；下一次打开商人页时按当前商店等级懒生成新库存。
- 后续研究所、制作所、随机事件、局外 NPC 行程等系统如需“每日刷新”，应接入同一个 `current_day`，不要各自维护独立日历。

## 5. UI 规则

- 局外界面需要展示当前天数。
- 当前版本只要求在局外展示天数；局内 HUD 是否展示“第几天”留给后续 UI 版本决定。
- 天数展示文本统一为 `第 N 天`。

## 6. 实现接口

推荐由全局状态节点统一提供接口：

```gdscript
func get_current_day() -> int
func get_day_display_text() -> String
func advance_day_after_run(result: Dictionary = {}) -> int
func reset_day(day: int = 1) -> void
```

`advance_day_after_run()` 只应由正式探索结算流程调用。局外按钮、商人买卖、仓库整理等功能不得直接推进天数。
