# 02-13 局外天数与昼夜阶段推进规则

> 本文补充 `02_核心循环规则_修订版_废土生存法则`、`08_撤离与结算系统`、`29_局外杂货店经营与界面重构路线图`、`30_昼夜局外流程_店铺营业与夜间出发准备`。
> 最新口径：天数服务于“白天准备 -> 白天营业 -> 夜晚出发 -> 局内探索 -> 结算返回”的完整循环，不再服务旧随机库存刷新。

## 1. 基础定义

- `current_day` 是局外存档级字段，默认值为 1。
- 每个 `current_day` 包含白天和夜晚两个局外阶段。
- 白天分为准备阶段和营业阶段。
- 夜晚分为夜间计划和出发准备。
- 玩家真正进入局内探索时，探索属于当前显示的 `current_day`。

推荐阶段枚举：

```text
DAY_PREP
DAY_SHOP_OPEN
DAY_SHOP_SETTLEMENT
NIGHT_PLAN
NIGHT_LOADOUT
RUNNING
RUN_SETTLEMENT
```

## 2. 当日流程

```text
DAY_PREP
-> 玩家管理仓库、制造可售物资、研究升级、查看当日需求榜
-> 点击开店营业
DAY_SHOP_OPEN
-> 货台成交，持续 1 分钟
DAY_SHOP_SETTLEMENT
-> 展示本日卖出物资、数量、单项收益和总收益
-> 关闭结算面板
NIGHT_PLAN
-> 只显示出发探索
-> 点击后选择角色和地点
NIGHT_LOADOUT
-> 从仓库选择装备和消耗品
-> 确认出发
RUNNING
-> 进入局内探索
RUN_SETTLEMENT
-> 撤离或失败结算
-> 返回局外并推进到下一天 DAY_PREP
```

## 3. 推进时机

白天推进到夜晚：
- 开店营业 1 分钟结束。
- 玩家关闭本日营业结算面板。

夜晚推进到局内：
- 玩家在夜间计划选择角色和地点。
- 玩家在出发准备完成装备和消耗品配置。
- 玩家点击确认出发。

天数加 1：
- 正式探索结算完成并返回局外时推进。
- 撤离成功、探索失败、死亡、稳定值耗尽或超时都算正式探索结果。
- 返回后进入下一天 `DAY_PREP`，同时刷新当日需求排行榜。

示例流程：

```text
初始局外 current_day = 1，phase = DAY_PREP
-> 第 1 天白天准备和营业
-> 第 1 天夜晚出发
-> 探索成功或失败并回到局外：current_day = 2，phase = DAY_PREP
-> 第 2 天刷新需求排行榜
```

## 4. 不推进天数的情况

- 玩家停留在白天准备阶段整理仓库、制造物资、研究升级、查看图鉴。
- 玩家在白天营业阶段上架、撤下、补货或查看成交进度。
- 玩家在夜间计划界面更换角色或地点。
- 玩家在出发准备界面调整装备和消耗品。
- 点击出发但未进入正式局内的调试中断。
- 直接切换场景、编辑器调试重载等开发操作。

## 5. 系统联动

- 当日需求排行榜在每天 `DAY_PREP` 开始时刷新一次。
- 货台成交速度读取当天需求热度；排行榜靠前的可售物资成交间隔短，靠后的成交间隔长。
- 开店营业只持续当前白天，不跨天。
- 研究和制造不因为天数自动完成；V0.2 仍以即时完成或明确队列为准。
- 后续局外事件、难民来访、地点路线费用等系统如需每日刷新，应接入同一个 `current_day` 和 `phase`。

## 6. UI 规则

- 局外界面展示 `第 N 天` 和当前阶段。
- `DAY_PREP` 显示仓库、制造所、研究所、图鉴、当日需求、开店营业。
- `DAY_SHOP_OPEN` 显示货台、成交进度、当前营业倒计时。
- `DAY_SHOP_SETTLEMENT` 显示本日营业结算。
- `NIGHT_PLAN` 只显示出发探索入口。
- `NIGHT_LOADOUT` 显示角色装备、消耗品栏和仓库候选物。

## 7. 实现接口

推荐由全局状态节点统一提供接口：

```gdscript
func get_current_day() -> int
func get_day_display_text() -> String
func get_base_phase() -> StringName
func enter_day_prep() -> void
func start_shop_open() -> Dictionary
func finish_shop_open(settlement: Dictionary) -> Dictionary
func enter_night_plan() -> void
func enter_night_loadout(selection: Dictionary) -> void
func start_run_from_loadout(loadout: Dictionary) -> Dictionary
func advance_day_after_run(result: Dictionary = {}) -> int
func reset_day(day: int = 1) -> void
```

`advance_day_after_run()` 只应由正式探索结算流程调用。局外按钮、货台成交、仓库整理、研究制造和出发准备调整不得直接推进天数。

```gherkin
    And phase 为 DAY_PREP

    And current_day 仍为 1

  Scenario: 正式探索结算后推进到下一天
    Given current_day 为 1
    And phase 为 RUN_SETTLEMENT

    Then current_day 应变为 2
    And phase 应变为 DAY_PREP
    And 当日需求排行榜刷新
```
