# 05_容器搜集交互闭环

> 对应母版章节：5.3、8、11.1、11.2、11.3  
> 实现层级：容器识别、读条、锁定、产出、拾取。
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“容器搜集交互闭环”小节。

---

## 1. 实现目标

玩家在危险区靠近容器后，系统识别容器、显示交互提示、按住或点击交互开始开启读条。开启期间玩家承担稳定值下降风险；读条完成后容器产出随机收益，玩家再选择拾取哪些物品。

---

## 2. 设计依据

家中只能看到容器大致信号，不能看到具体奖励：

```text
可显示：大致位置、信号强弱、轮廓或类型倾向、异常预兆
不可显示：具体物品、品质、数量、配方可用条件、稀有物资
```

靠近容器后才进入具体交互：

```text
识别容器类型
显示 F 键开启提示
开始开启倒计时
完成后生成奖励
奖励受背包格子和负重约束
```

## 4. 数据结构

容器状态：

```gdscript
enum ContainerState {
    HIDDEN_SIGNAL,
    AVAILABLE,
    OPENING,
    OPENED,
    EMPTY
}
```

容器字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `container_id` | `StringName` | 本局唯一 ID |
| `container_type` | `StringName` | 普通、医疗、工具、异常等 |
| `signal_strength` | `int` | 家中观察用 |
| `open_duration` | `float` | 读条时间 |
| `state` | `ContainerState` | 当前状态 |
| `rewards` | `Array` | 开启后生成 |
| `map_instance_id` | `StringName` | 所属地图实例 |

## 5. 核心流程

靠近识别：

```text
1. Player 进入容器交互范围。
2. LootContainer 检查 state == AVAILABLE。
3. UI 显示容器类型倾向和 F 键提示。
4. 玩家离开范围后隐藏提示。
```

开启：

```text
1. 玩家按下交互键。
2. LootContainer 进入 OPENING。
3. 记录 is_locked_by_player = true。
4. 显示开启读条。
5. 读条期间玩家不能同时开启其他容器。
6. 玩家离开范围或取消时，读条中断，容器回到 AVAILABLE。
7. 读条完成后进入 OPENED。
8. LootRewardResolver 生成 rewards。
9. 显示拾取界面或掉落物。
```

拾取：

```text
1. 玩家选择物品。
2. InventoryComponent 检查格子和负重。
3. 可放入则进入背包。
4. 不可放入则提示背包/负重不足，物品仍留在容器结果中。
5. 容器结果清空后，容器可隐藏或标记为空。
```

实例残留规则：

```text
普通容器没有存在倒计时。
读条取消后，容器回到 AVAILABLE。
已开启但未拿空的容器在当前地图实例内保留残留。
容器拿空后进入 EMPTY，可隐藏或从交互列表移除。
离开地图实例后，容器残留不进入结算、不写入长期存档。
```

## 6. 信号与接口

`LootContainer` 发出：

```gdscript
signal container_detected(container_id: StringName)
signal container_open_started(container_id: StringName, duration: float)
signal container_open_cancelled(container_id: StringName)
signal container_opened(container_id: StringName, rewards: Array)
signal container_emptied(container_id: StringName)
```

依赖接口：

```text
InventoryComponent.can_accept_item(item) -> bool
InventoryComponent.add_item(item) -> bool
StabilityComponent 继续自然衰减，不因开箱暂停
RunStateMachine 必须处于 SCAVENGE 或 GREED_DECISION 才允许开箱
```

## 8. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 玩家只有靠近容器时才看到交互提示 |
| A2 | 开启读条完成前离开范围会取消开启 |
| A3 | 开启期间稳定值继续下降 |
| A4 | 读条完成后容器只生成一次奖励 |
| A5 | 背包格子不足或负重不足时不能拾取对应物品 |
| A6 | 已开启容器不会再次被开启 |
| A7 | 已开启但未拿空的容器只在当前地图实例内保留残留，离图后不结算 |
