# 09_前哨站推进与修复闭环

> 对应母版章节：9、12.2、13、14、15.1  
> 实现层级：主目标推进、临时安全区、撤离解锁。
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“前哨站推进与修复闭环”小节。

---

## 1. 实现目标

单局随机选出两个临时前哨站。玩家收集指定材料后可以修复前哨站，修复后的前哨站成为本局有效的临时安全区。修复两个前哨站后，撤离解锁，玩家进入“继续贪婪搜集”或“回家撤离”的决策。

---

## 2. 设计依据

前哨站规则：

```text
第一前哨站：从 3 个候选点中随机 1 个。
第二前哨站：从 4 个候选点中随机 1 个。
初始状态均为未修复。
修复需要消耗指定前哨材料。
修复后成为本局临时安全区。
本局结束后前哨站状态不永久保留。
两个前哨站都修复后，撤离按钮激活。
```

## 4. 数据结构

前哨状态：

```gdscript
enum OutpostState {
    HIDDEN,
    SELECTED_UNREPAIRED,
    REPAIRING,
    REPAIRED
}
```

前哨字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `outpost_id` | `StringName` | 前哨 ID |
| `outpost_order` | `int` | 第 1 或第 2 前哨 |
| `state` | `OutpostState` | 当前状态 |
| `required_materials` | `Dictionary` | 修复需求 |
| `repair_duration` | `float` | 修复读条 |
| `safe_zone` | `SafeZone` | 修复后激活 |
| `is_run_persistent` | `bool` | 固定为 false，本局有效 |

## 5. 核心流程

候选点抽取：

```text
1. RunInitializer 获取第一前哨候选点池。
2. 随机选 1 个作为第一前哨。
3. 获取第二前哨候选点池。
4. 随机选 1 个作为第二前哨。
5. 未选中的候选点保持隐藏或普通地图点位。
6. 两个已选前哨状态为 SELECTED_UNREPAIRED。
```

修复交互：

```text
1. 玩家靠近未修复前哨。
2. OutpostStation 查询 RequirementTracker。
3. 如果材料不足，显示缺少材料。
4. 如果材料满足，显示 F 键修复。
5. 玩家开始修复，进入 REPAIRING。
6. 修复期间可设置读条，稳定值继续按安全区状态决定。
7. 修复完成后消耗材料。
8. 前哨进入 REPAIRED。
9. 激活 SafeZone。
10. repaired_outpost_count + 1。
11. 如果数量达到 2，发出 extraction_unlocked。
```

修复后安全区：

```text
进入已修复前哨站时，稳定值停止下降并恢复。
前哨站可作为回收点，但 V0.1 不一定提供完整家中存储。
是否允许存储由 07 文档最终实现决定，建议 V0.1 可先不提供存储，只提供稳定值恢复。
```

## 6. 信号与接口

`OutpostStation` 发出：

```gdscript
signal outpost_selected(outpost_id: StringName, order: int)
signal outpost_repair_available(outpost_id: StringName)
signal outpost_repair_started(outpost_id: StringName)
signal outpost_repair_cancelled(outpost_id: StringName)
signal outpost_repaired(outpost_id: StringName)
```

依赖接口：

```text
OutpostRequirementTracker.has_required_materials(outpost_id)
InventoryComponent.consume_items(requirements)
SafeZone.set_active(true)
RunStateMachine.on_outpost_repaired(outpost_id)
ExtractionController.unlock_when_outposts_repaired(count)
```

## 8. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 新局会随机选出第一和第二前哨站 |
| A2 | 未修复前哨靠近后能提示所需材料 |
| A3 | 材料不足时不能开始修复 |
| A4 | 材料满足时可以读条修复 |
| A5 | 修复完成后消耗材料，前哨状态变为 `REPAIRED` |
| A6 | 修复前哨站激活安全区功能 |
| A7 | 修复两个前哨站后发出撤离解锁事件 |
| A8 | 本局结束后前哨修复状态不会写入永久存档 |
