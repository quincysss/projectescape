# 04_稳定值与视野闭环

> 对应母版章节：6.4、7、12.1、16.1  
> 实现层级：玩家压力计时器 + 可视范围控制 + 死亡入口。
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“稳定值与视野闭环”小节。

---

## 1. 实现目标

稳定值是玩家离开安全区后的核心压力。玩家在危险区稳定值持续下降，下降到不同阶段时视野变小并反馈增强；稳定值归零时触发死亡失败。进入家或已修复前哨站后，稳定值停止下降并快速恢复。

---

## 2. 设计依据

稳定值影响：

```text
玩家是否存活
玩家可视范围大小
玩家探索压力
玩家返程决策
玩家开容器时的停留风险
```

V0.1 建议：

```text
最大值：100
危险区下降速度：1 点/秒
安全区回复速度：10 点/秒
稳定值归零：死亡
```

稳定值不受玩家是否移动影响，站在危险区不动也会持续下降。

建议新增：

| 路径 | 职责 |
|---|---|
| `res://scripts/player/StabilityComponent.gd` | 稳定值数值、衰减、恢复、阶段判断 |

| `res://scripts/player/PlayerDeathController.gd` | 接收归零事件并通知状态机 |

## 4. 数据结构

稳定值阶段建议：

| 范围 | 阶段 | 视野表现 | 反馈表现 |
|---|---|---|---|
| 76-100 | `SAFE` | 标准危险区视野 | 轻微暗角 |
| 51-75 | `TENSE` | 视野略缩小 | UI 轻微提示 |
| 26-50 | `DANGER` | 视野明显缩小 | 心跳/闪烁提示 |
| 1-25 | `CRITICAL` | 视野很小 | 强烈警告 |
| 0 | `DEPLETED` | 不再探索 | 死亡 |

`StabilityComponent` 字段：

| 字段 | 默认值 | 说明 |
|---|---:|---|
| `max_stability` | `100` | 上限 |
| `current_stability` | `100` | 当前值 |
| `decay_per_second` | `1` | 危险区下降速度 |
| `recover_per_second` | `10` | 安全区恢复速度 |
| `is_decaying` | `false` | 是否下降 |
| `is_recovering` | `false` | 是否恢复 |
| `current_stage` | `SAFE` | 当前阶段 |

---

## 5. 核心流程

危险区衰减：

```text
1. RunDirector 收到 safe_zone_exited。
2. StabilityComponent.start_decay()。
3. 每帧或固定 tick 扣除 decay_per_second * delta。
4. 数值变化后重新计算阶段。
5. 阶段变化时通知 VisionController 和 HUD。
6. current_stability <= 0 时发出 stability_depleted。
7. RunDirector 通知状态机进入 FAILED。
```

安全区恢复：

```text
1. RunDirector 收到 safe_zone_entered。
2. StabilityComponent.stop_decay()。
3. StabilityComponent.start_recover()。
4. 每帧恢复 recover_per_second * delta，最多到 100。
5. 满值后可以继续保持恢复状态，但不再超过上限。
```

视野联动：

```text
SAFE：使用基础可视半径
TENSE：可视半径约 85%
DANGER：可视半径约 65%
CRITICAL：可视半径约 45%
DEPLETED：交给死亡流程
```

---

## 6. 信号与接口

`StabilityComponent` 发出：

```gdscript
signal stability_changed(current: float, max_value: float, stage: int)
signal stability_stage_changed(old_stage: int, new_stage: int)
signal stability_depleted()
```

`VisionController` 接收：

```text
set_darkness_enabled(enabled)
set_vision_stage(stage)
transition_to_radius(radius, duration)
```

`RunDirector` 接收：

```text
on_stability_depleted() -> player_dead("stability_depleted")
```

## 8. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 玩家离开安全区后，稳定值无论移动与否都持续下降 |
| A2 | 玩家进入安全区后，稳定值停止下降并恢复 |
| A3 | 稳定值不会超过 100，也不会低于 0 |
| A4 | 跨过 75、50、25 阈值时阶段变化只触发一次 |
| A5 | 阶段变化会驱动视野范围改变 |
| A6 | 稳定值归零后状态机进入 `FAILED` |
| A7 | 死亡后稳定值系统不再继续发出探索阶段变化 |
