# 10_撤离死亡结算闭环

> 对应母版章节：15、16、17  
> 实现层级：本局结束入口、死亡处理、收益进入结算池。
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“撤离死亡结算闭环”小节。

---

## 1. 实现目标

当两个前哨站修复后，玩家可以返回家中激活撤离。撤离读条完成后进入结算，背包和家中临时存储物品进入结算池。若玩家稳定值归零死亡，则进入失败流程，并按死亡规则处理背包、家中存储和前哨站状态。

---

## 2. 设计依据

撤离激活条件：

```text
两个本局临时前哨站均已修复。
玩家回到家中。
顶部撤离信号 UI 进入家中可长按状态。
```

死亡触发：

```text
稳定值归零。
后续可扩展敌人、异常、陷阱等死亡来源。
```

死亡后处理：

```text
背包物品按死亡规则处理，V0.1 可先全部丢失。
家中存储物品不丢失。
前哨站为本局临时有效，死亡后本局结束，状态清空。
```

## 4. 数据结构

撤离状态：

```gdscript
enum ExtractionState {
    LOCKED,
    UNLOCKED,
    CHANNELING,
    COMPLETED,
    INTERRUPTED
}
```

本局结果：

```gdscript
enum RunResult {
    NONE,
    EXTRACTED,
    DEAD,
    ABORTED
}
```

结算池字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `result` | `RunResult` | 撤离成功或死亡 |
| `backpack_items` | `Array` | 撤离成功时带回 |
| `home_storage_items` | `Array` | 撤离成功或死亡可保留 |
| `lost_items` | `Array` | 死亡丢失或溢出预留 |
| `run_duration` | `float` | 本局时长 |
| `repaired_outpost_count` | `int` | 用于结算展示 |

## 5. 核心流程

撤离成功：

```text
1. OutpostSystem 发出两个前哨站已修复。
2. ExtractionController 进入 UNLOCKED。
3. 玩家回到 HomeSafeZone。
4. 顶部撤离信号 UI 显示“长按 E 键进行撤离”。
5. 玩家按住 E 或按住撤离信号 UI。
6. ExtractionController 进入 CHANNELING。
7. 读条期间玩家离开家或取消，则 INTERRUPTED，再回到 UNLOCKED。
8. 读条完成后进入 COMPLETED。
9. RunStateMachine 进入 SETTLEMENT。
10. SettlementController 收集背包和家中存储。
11. 显示结算界面。
```

死亡失败：

```text
1. StabilityComponent 发出 stability_depleted。
2. PlayerDeathController 发出 player_dead("stability_depleted")。
3. RunStateMachine 进入 FAILED。
4. RunEndController 创建死亡结算数据。
5. 背包物品进入 lost_items。
6. 家中存储保留到结算池或按后续仓库规则处理。
7. 本局临时前哨状态清空。
8. 显示失败结算或返回局外。
```

一键入库预留：

```text
V0.1 可以先显示结算池，不真正写入局外仓库。
正式入库由 09_局外仓库系统 文档实现。
```

## 6. 信号与接口

`ExtractionController` 发出：

```gdscript
signal extraction_unlocked()
signal extraction_started(duration: float)
signal extraction_interrupted(reason: String)
signal extraction_completed()
```

`RunEndController` 发出：

```gdscript
signal run_finished(result: int, settlement_data)
```

依赖接口：

```text
HomeSafeZone.can_extract()
InventoryComponent.get_items()
HomeStorageComponent.get_items()
RunContext.elapsed_seconds
RunStateMachine.finish_run(result)
```

## 8. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 少于两个前哨修复时撤离按钮不可用 |
| A2 | 两个前哨修复后撤离解锁 |
| A3 | 只有玩家在家中时可以通过 E 或撤离信号 UI 开始撤离读条 |
| A4 | 撤离读条中离开家会中断 |
| A5 | 撤离读条完成后进入 `SETTLEMENT` |
| A6 | 撤离成功结算包含背包和家中存储 |
| A7 | 稳定值归零会进入死亡结果 |
| A8 | 死亡时背包物品丢失，家中存储不丢失 |
