## 1. 最小闭环定义

V0.1 最小可玩闭环只证明一件事：

```text
玩家出生在家
→ 观察信号
→ 离家进入危险区
→ 稳定值下降
→ 开容器或拾取前哨材料
→ 背包和负重产生约束
→ 回家或修复前哨站恢复
→ 修复两个前哨站
→ 返回家中撤离
→ 进入结算
```

不是 V0.1 目标：

```text
完整美术
完整音效
正式掉落权重
局外完整成长
多人模式
复杂异常事件
```

## 4. 调试按键建议

| 按键 | 用途 |
|---|---|
| `F` | 与容器、材料、前哨、撤离按钮交互 |
| `Tab` | 打开/关闭背包调试界面 |
| `R` | 重开新局 |
| `[` | 稳定值 -10，调试阶段变化 |
| `]` | 稳定值 +10，调试恢复 |
| `1` | 给予第一前哨材料 |
| `2` | 给予第二前哨材料 |
| `0` | 强制稳定值归零，测试死亡 |

## 5. 验收总清单

核心流程：

```text
[ ] 新局启动后玩家出生在家。
[ ] 家中为观察模式，黑暗遮罩关闭。
[ ] 离家后镜头拉近，黑暗遮罩开启。
[ ] 离家后稳定值开始下降。
[ ] 稳定值阶段能改变视野范围。
[ ] 稳定值归零会死亡。
[ ] 容器可以靠近、开箱、拾取奖励。
[ ] 前哨材料可以直接拾取。
[ ] 背包格子和负重会限制拾取。
[ ] 回家可以恢复稳定值并存储物品。
[ ] 材料满足后可以修复第一前哨站。
[ ] 材料满足后可以修复第二前哨站。
[ ] 修复后的前哨站提供临时安全区。
[ ] 两个前哨站修复后撤离解锁。
[ ] 玩家在家中可以撤离。
[ ] 撤离成功进入结算。
[ ] 结算能显示背包和家中存储物品。
```

工程质量：

```text
[ ] 状态机不直接控制具体节点，只发信号。
[ ] 容器系统不直接修改前哨系统。
[ ] 前哨材料不走容器掉落流程。
[ ] 背包和负重检查集中在 Inventory/Weight 组件。
[ ] 死亡和撤离都通过 RunEndController 收束。
[ ] 所有关键状态变化都有日志或调试 UI。
```
