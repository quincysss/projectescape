# 08_资源刷新与异常预兆编排

> 对应母版章节：8.3、8.6、9.3、9.4、9.5、9.6、11  
> 实现层级：局内刷新调度，不负责具体掉落内容。
> 施工补充：开工前必须读取 `12_模块功能规则细化与AI开工提示词.md` 的“资源刷新与异常预兆编排”小节。

---

## 1. 实现目标

建立局内资源生成框架：普通容器在进入地图实例时按地图资源生态、地点状态和点位池一次性生成；前哨材料按当前需求生成并带较长存在时间；S 级异常预兆作为高价值风险信号预留。该文档只定义生成编排，不定义正式掉落表。

---

## 2. 设计依据

核心约束：

```text
容器是随机收益来源，但不按局内计时轮询补刷。
前哨材料是主线推进资源，应围绕当前前哨需求生成。
前哨材料应比普通容器信息更明确。
普通容器没有存在倒计时；读条取消后回到可开启状态，拿空后移除。
S 级异常预兆用于贪婪决策阶段的高风险诱因。
```

## 4. 数据结构

点位字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `point_id` | `StringName` | 点位 ID |
| `ring` | `int` | 圈层，用于离家距离或风险层 |
| `tags` | `Array[StringName]` | 道路、建筑、边缘等 |
| `is_occupied` | `bool` | 是否已有资源 |
| `spawn_weight` | `float` | 被选中的权重 |

容器生成配置：

| 字段 | 说明 |
|---|---|
| `map_resource_profile` | 地图资源生态 |
| `location_state` | 富集/普通/贫瘠/恢复中 |
| `base_container_count` | 地图基础容器数量区间 |
| `state_multiplier` | 地点状态数量倍率 |
| `point_weights` | 房间/区域/点位语义权重 |

材料刷新配置：

| 字段 | 说明 |
|---|---|
| `required_material_id` | 当前前哨所需材料 |
| `required_amount` | 需求数量 |
| `min_visible_material_count` | 地图上至少存在的相关材料数量 |
| `material_lifetime` | 材料存在时间，默认 120 秒 |
| `respawn_on_expire` | 过期后是否立刻换点重刷 |

## 5. 核心流程

容器生成：

```text
1. 玩家进入某张地图实例。
2. 读取地图资源生态、地点状态、房间/区域语义和 LootSpawnPoint 点位池。
3. 按地图基础容器数量和地点状态倍率计算本次目标数量。
4. 从合法点位中抽取本次容器点位。
5. 生成 LootContainer，状态为 AVAILABLE。
6. 普通容器没有存在倒计时，局内不轮询补刷。
7. 离开地图时按玩家拿走资源比例回写地点状态。
```

前哨材料生成：

```text
1. 当前前哨需求确定后，MaterialSpawner 读取需求。
2. 从 MaterialSpawnPoint 中挑选点位。
3. 生成对应材料拾取物。
4. 材料开始 120 秒存在时间倒计时。
5. 倒计时归零且未拾取时，旧材料消失，并从同一前哨站材料点位池中选择另一个点位立刻重刷。
6. 需求满足后，下一前哨需求激活时刷新下一批材料。
```

异常预兆预留：

```text
1. 两个前哨站修复后，系统可允许 S 级异常预兆出现。
2. V0.1 先只实现信号对象和调试显示。
3. 不实现正式高价值奖励和风险事件。
```

## 6. 信号与接口

`ResourceRefreshDirector` 发出：

```gdscript
signal container_spawned(container_id: StringName, point_id: StringName)
signal container_removed(container_id: StringName, reason: String)
signal material_spawned(material_id: StringName, point_id: StringName)
signal anomaly_signal_spawned(signal_id: StringName)
```

依赖接口：

```text
RunContext 提供 seed
LootContainerSpawner 使用点位池
OutpostRequirementTracker 提供当前材料需求
SignalDisplay 获取活跃容器/材料/异常信号
```

## 7. MCP 开工指令

```text
请实现“资源生成与异常预兆编排”的最小版本。
创建 ResourceRefreshDirector，能在进入地图实例时按地图资源生态、地点状态和点位池生成本次容器清单，并按当前前哨需求生成带 120 秒存在时间的材料。
普通容器不实现倒计时、过期或局内轮询补刷；容器拿空或离开地图实例后清理。
材料过期后应移除旧点并立刻换点重刷。
S 级异常只做预留信号节点和日志，不做奖励。
```

## 8. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 进入地图实例时一次性生成本次容器清单 |
| A2 | 容器只生成在合法空闲点位上 |
| A3 | 容器数量受地图基础数量和地点状态倍率影响 |
| A4 | 普通容器没有倒计时、过期或局内轮询补刷 |
| A5 | 当前前哨材料按需求生成，120 秒过期后会换点重刷 |
| A6 | 两个前哨修复后可以触发异常预兆的预留信号 |
