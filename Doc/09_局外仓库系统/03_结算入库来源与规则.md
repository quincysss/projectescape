# 09-03 结算入库来源与规则

> 来源文档：`09_局外仓库系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

承接 08 结算系统传来的物品条目，并把可入库物品写入局外仓库。

## 2. 入库来源

V0.1 主要来源：

- 撤离成功背包物品。
- 撤离成功前哨临时存储物品。
- 目标奖励。

失败流程不能直接写仓库。

## 3. 可入库判断

条目必须满足：

- `can_store == true`
- `status == pending`
- item_id 存在于 ItemDefinition。
- 数量大于 0。

## 4. 接口

```gdscript
func try_store_settlement_items(items: Array[Dictionary]) -> Dictionary
```

返回：

```gdscript
{
    "stored": [],
    "overflow": [],
    "rejected": []
}
```

## 5. AI/MCP 开工提示词

```text
请实现 08 结算物品到 09 局外仓库的入库接口。
要求：
1. 只处理撤离成功的 pending 且 can_store 条目。
2. 校验 item_id、count 和仓库空间。
3. 支持部分成功，返回 stored、overflow、rejected。
4. 入库失败的条目不得消失。
5. 更新后的状态交回 08 SettlementUI 显示。
```

## 6. 验收标准

- 成功结算物能入库。
- 失败结算物不能直接入库。
- 部分成功时剩余物仍在结算界面。

## 7. 暂不实现

- 自动出售。
- 入库税或消耗。
