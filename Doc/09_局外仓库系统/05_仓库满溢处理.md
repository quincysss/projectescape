# 09-05 仓库满溢处理

> 来源文档：`09_局外仓库系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

处理仓库空间不足，让玩家整理后继续入库，而不是让系统替玩家丢弃。

## 2. 空间不足

空间不足时：

- 已入库条目保存。
- 未入库条目返回给 08，状态为 overflow 或 pending。
- UI 显示仓库空间不足。
- 允许打开仓库整理。

## 3. 整理后继续

玩家整理仓库后：

```text
返回结算界面 -> 再次尝试入库 overflow/pending 条目
```

## 4. AI/MCP 开工提示词

```text
请实现仓库满溢处理。
要求：
1. 仓库空间不足时不得删除结算物。
2. 返回 overflow 列表给 08。
3. 支持玩家整理仓库后再次尝试入库。
4. 已成功入库的物品不能重复入库。
5. Debug 能复现满仓入库失败。
```

## 5. 验收标准

- 满仓入库失败不吞物。
- 整理后可继续入库。
- 不会重复入库已 stored 物品。

## 6. 暂不实现

- 临时溢出邮箱。
- 自动扩容。
