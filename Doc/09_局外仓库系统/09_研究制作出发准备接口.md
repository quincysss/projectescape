# 09-09 研究、制作与出发准备接口

> 来源文档：`09_局外仓库系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

为研究所、制作所和出发准备提供稳定仓库查询/消耗接口，避免这些系统直接修改仓库数据。

局外商人也属于仓库外部消费者。商人可以查询可出售物品，但出售扣物必须走 WarehouseManager 原子接口。

## 2. 研究所接口

```gdscript
func has_items(requirements: Array[Dictionary]) -> bool
func consume_items_for_research(requirements: Array[Dictionary]) -> Dictionary
```

研究所只能通过接口消耗材料。

## 3. 制作所接口

```gdscript
func consume_items_for_crafting(recipe_id: String, requirements: Array[Dictionary]) -> Dictionary
func add_crafted_item(item_id: String, count: int, quality: String) -> Dictionary
```

## 4. 出发准备接口

```gdscript
func move_item_to_loadout(stack_id: String, count: int) -> Dictionary
func query_available_loadout_items(filters: Dictionary) -> Array[Dictionary]
```

从仓库取到局内背包时需要调用 07 负重/背包接口。

## 4.5 商人接口

```gdscript
func query_sellable_items(filters: Dictionary = {}) -> Array[Dictionary]
func sell_warehouse_item(warehouse_item_id: String, count: int, currency_id: String = "mine_coin") -> Dictionary
func sell_warehouse_items(requests: Array[Dictionary]) -> Dictionary
```

规则：

- query_sellable_items 只返回 items.tab 中 sellable = true 的仓库物品。
- sell_warehouse_item 必须先校验数量、售价、货币 ID，再扣除仓库物品并增加货币。
- 扣物和加币必须在同一事务结果中返回；失败时不能只扣物或只加币。

## 5. AI/MCP 开工提示词

```text
请实现 WarehouseQueryService，为研究所、制作所和出发准备提供接口。
要求：
1. 研究所可查询和消耗仓库材料。
2. 制作所可消耗材料并添加成品。
3. 出发准备可查询可带出物品，并移动到局内背包/loadout。
4. 商人可查询可出售物品并按数量出售。
5. 所有消耗和出售必须原子化，失败时不扣材料、不扣物品、不加错货币。
6. 外部系统不得直接修改 WarehouseData。
```

## 6. 验收标准

- 研究材料不足时不消耗。
- 制作失败不吞材料。
- 出发准备取物会检查背包空间。
- 商人出售失败不会扣物或加币。
