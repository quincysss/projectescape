# 09-09 研究、制作与出发准备接口

> 来源文档：`09_局外仓库系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

为研究所、制作所、白天货台和夜晚出发准备提供稳定仓库查询/消耗接口，避免这些系统直接修改仓库数据。

旧局外商人系统已废弃，不再作为仓库外部消费者。货币收入来自白天开店成交，成交扣货必须走货台/仓库原子接口。

局外道具详情 Tooltip 不直接消耗或修改仓库数据，但研究所、制作所、仓库、货台、图鉴和出发准备中的道具图标都要能读取同一套 ItemDefinition 展示信息；详细规则见 `18_局外道具详情Tooltip.md`。

## 2. 研究所接口

```gdscript
func has_items(requirements: Array[Dictionary]) -> bool
func consume_items_for_research(requirements: Array[Dictionary]) -> Dictionary
```

研究所只能通过接口消耗材料。

## 3. 制作所接口

```gdscript
func consume_items_for_crafting(recipe_id: String, requirements: Array[Dictionary]) -> Dictionary
func add_crafted_item(item_id: String, count: int, quality: String) -> Dictionary
```

## 4. 出发准备接口

```gdscript
func move_item_to_loadout(stack_id: String, count: int) -> Dictionary
func query_available_loadout_items(filters: Dictionary) -> Array[Dictionary]
```

从仓库取到局内背包时需要调用 07 负重/背包接口。

## 4.5 货台接口

```gdscript
func query_shelfable_sale_goods(filters: Dictionary = {}) -> Array[Dictionary]
func move_item_to_shelf(warehouse_item_id: String, count: int, shelf_slot_index: int) -> Dictionary
func return_shelf_item_to_warehouse(shelf_slot_index: int, count: int = -1) -> Dictionary
func consume_shelf_item_for_deal(shelf_slot_index: int, count: int = 1) -> Dictionary
```

规则：

- `query_shelfable_sale_goods` 只返回 `item_type=sale_good` 或 `tags` 包含 `sale_good` 的仓库物品。
- 原材料 `material` 不得进入货台候选池。
- 上架、撤下、成交扣货必须保持原子化；失败时不能丢货、不能重复回仓、不能重复记账。
- 营业结束时所有未成交货品必须回仓。

## 4.6 局外 Tooltip 查询接口

```gdscript
func build_item_tooltip_data(item_id: String, context: Dictionary = {}) -> Dictionary
func build_warehouse_item_tooltip_data(warehouse_item_id: String, context: Dictionary = {}) -> Dictionary
```

规则：

- Tooltip 查询只读，不允许触发仓库扣除、出售、制作或研究。
- 基础展示信息来自 items.tab / ItemDefinition。
- 售价和货币名来自 items.tab.sell_currency_id、items.tab.sell_value 和 currencies.tab。
- 研究/制作界面可以通过 context 传入 owned_count、required_count，但 V0.1 不强制 Tooltip 重复显示数量。
- 无效 item_id 在 Debug 模式记录 warning；正式模式不显示错误 Tooltip。

## 5. AI/MCP 开工提示词

```text
请实现 WarehouseQueryService，为研究所、制作所和出发准备提供接口。
要求：
1. 研究所可查询和消耗仓库材料。
2. 制作所可消耗材料并添加成品。
3. 出发准备可查询可带出物品，并移动到局内背包/loadout。
4. 货台可查询可上架物资、上架、撤下和成交扣货。
5. 所有消耗、上架、撤下和成交扣货必须原子化，失败时不扣材料、不扣物品、不加错货币。
6. 为局外道具详情 Tooltip 提供只读数据构建接口，仓库、货台、研究所、制作所、图鉴、出发准备可复用。
7. 外部系统不得直接修改 WarehouseData。
```

## 6. 验收标准

- 研究材料不足时不消耗。
- 制作失败不吞材料。
- 出发准备取物会检查背包空间。
- 货台成交失败不会扣物或加币。
- Tooltip 查询接口只读，hover 查看道具详情不会改变仓库、货币、研究或制作状态。

## 8. 研究材料来源兼容规则（V0.3）

研究所只关心局外仓库中物品的 `item_id` 和数量，不区分物品来源。

有效来源包括：
- 局内探索、容器、事件等带回仓库的物品。
- 制造所加工后进入仓库的可售物资、装备或消耗品。
- 后勤订购标准件进入仓库的物品。

实现约束：
- 局内带回的物品可以带有 `source = "run_loot"` 或容器来源字段。
- 后勤订购物品可以带有 `source = "logistics_order"`。
- `ResearchManager.get_research_quote()` 只能通过 `WarehouseManager.has_items(requirements)` 判断是否满足材料。
- `ResearchManager.complete_research()` 只能通过 `WarehouseManager.consume_items(requirements)` 扣除材料。
- 研究扣除逻辑不得按 `source`、`sellable`、`item_type` 做额外排除；否则会导致同 ID 道具因来源不同而不能升级科技。
- 例外：`outpost_material` 不进入局外仓库，不得作为研究所科技消耗材料。

配置约束：
- `research.tab.required_items` 引用的所有 `item_id` 必须存在于 `items.tab`。
- `research.tab.required_items` 引用的所有 `item_id` 必须能从局内掉落配置中获得，避免科技线被卡死。
- 普通 `material` 类型的研究材料主要通过局内掉落和制造/转化链获得；后勤订购只提供标准件兜底，不替代局内探索。
- `outpost_material` 只由局内菱形材料刷新系统产出，只用于局内修复前哨站，不进入容器掉落、货台售卖或研究消耗。
- 消耗品、装备、稀有物可作为研究材料，但不进入货台售卖池，仍主要通过局内探索或制造获得。
