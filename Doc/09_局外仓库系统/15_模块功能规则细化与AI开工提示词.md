# 09 模块功能规则细化与 AI 开工提示词

> 来源文档：`09_局外仓库系统_修订版_废土生存法则.md`
> 用途：作为 09 模块的 AI/MCP 总入口。

## 1. 总体实现目标

局外仓库承接成功撤离收益，并为局外成长系统提供物品数据：

```text
08 结算入库
-> 仓库长期保存
-> 制造所消耗原材料并产出 sale_good 可售物资、装备、消耗品
-> 白天货台上架 sale_good
-> 开店成交获得矿币
-> 研究所/制造所继续消耗
-> 夜晚出发准备取物
-> 图鉴保留曾经获得过的道具点亮记录
```

仓库是长期系统，任何删除、消耗、上架、撤下和成交扣货都必须清楚、可验证、可保存。

## 4. 分阶段 AI 开工提示词

### 阶段一：仓库核心

```text
请读取 09 拆分文档 01、02、08、10、12。
实现 WarehouseData、WarehouseItem、WarehouseManager、仓库网格、占格、堆叠和保存基础。
仓库不计算负重，stack_limit 从 07 ItemDefinition 读取。
```

### 阶段二：结算入库

```text
请读取 09 拆分文档 03、04、05。
实现 08 结算物入库、手动入库、一键入库优先级和满溢处理。
支持部分成功，未入库物品返回给 08，不能静默丢弃。
```

### 阶段三：仓库 UI、图鉴和 Tooltip

```text
请读取 09 拆分文档 06、07、18、19，以及 20 UI 文档。
实现白天准备阶段的仓库入口、仓库 UI、手动整理、排序、丢弃确认、局外道具详情 Tooltip 和局外道具图鉴入口。
白天准备阶段功能入口为：仓库、制造所、研究所、图鉴。
夜晚、夜间计划、出发准备和营业结算阶段按 30 文档控制入口显示。
仓库、制造所、研究所、图鉴、货台、出发准备中的道具图标 hover 0.15 秒后显示 Tooltip；拖拽、滚动、切换功能入口、弹窗时隐藏。
```

### 阶段四：货台接口和货币

```text
请读取 09 拆分文档 09、12，以及 Doc/29、Doc/30、Doc/31。
实现 ShelfInventoryService 和 CurrencyWallet。
货台只接受 item_type=sale_good 或 tags 包含 sale_good 的仓库物资。
原材料 material 不能直接上架。
上架、撤下、成交扣货、营业结束回仓必须原子化。
店铺营业结算统一调用 CurrencyWallet 增加 mine_coin。
```

### 阶段五：对外接口和 Debug

```text
请读取 09 拆分文档 09、11、13、14。
实现 WarehouseQueryService，给研究所、制造所、货台和出发准备提供查询/消耗/取物接口。

```

## 5. 总体验收标准

- 成功撤离物品能入库。
- 仓库空间不足不吞物。
- 仓库不影响负重。
- 仓库能保存和重载。
- 白天准备阶段能进入仓库。
- 仓库界面不提供旧式直接变现入口。
- 货台只允许 `sale_good` 可售物资上架。
- 成交成功扣货并在结算时增加 mine_coin。
- S 级稀有道具可以进入仓库、保留收藏或用于后续特殊系统，不自动变现。
- 货币按 currency_id 保存和读取。
- 局外仓库、货台、研究所、制造所、图鉴、出发准备中的道具图标 hover 后显示局外道具详情 Tooltip。
- 局外图鉴入口显示所有可展示道具，每行四张卡片，曾经获得过的道具永久点亮。
- 研究/制造消耗原子化。
- 出发准备取物检查背包容量。
- 外部系统不直接修改仓库数组。

## 6. 给后续 AI 的硬约束

```text
不要删除原始 09 文档。
不要静默丢弃玩家仓库物品。
不要让仓库计算局内负重。
不要让失败结算直接写仓库。
不要实现独立旧商人入口。
不要实现仓库物品直接变现。
不要让研究/制造/货台/出发准备直接改 WarehouseData.items。
不要让 UI 直接删除 WarehouseData.items 或直接改 currencies。
不要把货台成交价格写死在 UI；售价和需求修正来自 30/31 的经营配置。
不要把道具详情 Tooltip 接入局内背包、局内容器、拾取提示或结算界面；V0.1 仅局外 BaseScene 启用。
不要在 Tooltip 中写死道具名或描述；信息来自 items.tab / currencies.tab。
不要把图鉴当成仓库；图鉴只记录曾经获得过的 item_id，不显示或操作当前物品实例。
不要因为道具被成交、消耗、升级、制作使用、丢弃或仓库清空而取消图鉴点亮。
不要让图鉴展示、研究矿币/条件展示、制造预览本身触发点亮。
不要自动处理 S 级道具；S 级道具必须由玩家主动选择并确认。
不要实现复杂交易市场、旧买入列表、价格波动或多货币兑换。
不要实现多人同步，只预留字段。
```
