# 09 模块功能规则细化与 AI 开工提示词

> 来源文档：`09_局外仓库系统_修订版_废土生存法则.md`
> 用途：作为 09 模块的 AI/MCP 总入口。

## 1. 总体实现目标

局外仓库承接成功撤离收益，并为局外成长系统提供物品数据：

```text
08 结算入库
-> 仓库长期保存
-> 商人出售换取矿币
-> 研究所/制作所消耗
-> 出发准备取物
```

仓库是长期系统，任何删除和消耗都必须清楚、可验证、可保存。

## 4. 分阶段 AI 开工提示词

### 阶段一：仓库核心

```text
请读取 09 拆分文档 01、02、08、10、12。
实现 WarehouseData、WarehouseItem、WarehouseManager、仓库网格、占格、堆叠和保存基础。
仓库不计算负重，stack_limit 从 07 ItemDefinition 读取。
```

### 阶段二：结算入库

```text
请读取 09 拆分文档 03、04、05。
实现 08 结算物入库、手动入库、一键入库优先级和满溢处理。
支持部分成功，未入库物品返回给 08，不能静默丢弃。
```

### 阶段三：仓库 UI 和丢弃

```text
请读取 09 拆分文档 06、07、16，以及 20 UI 文档。
实现局外顶部页签、仓库 UI、手动整理、排序、丢弃确认和商人入口。
A/S 品质和关键材料丢弃需要确认；V0.1 不生成世界掉落。
顶部页签顺序必须是：仓库、商人、研究所、制作所；未开发完成页签置灰并禁止点击。
```

### 阶段四：商人和货币

```text
请读取 09 拆分文档 16、07-01、13 数据表规范。
实现 data/currencies.tab、CurrencyWallet、MerchantService 和 MerchantPanel。
玩家可从仓库物品中选择要卖的道具和数量，出售成功后扣除仓库数量并按 currency_id 增加货币。
V0.1 只有 mine_coin；保存数据只保存 currency_id -> amount，不保存货币显示名和图标。
```

### 阶段五：对外接口和 Debug

```text
请读取 09 拆分文档 09、11、13、14。
实现 WarehouseQueryService，给研究所、制作所和出发准备提供查询/消耗/取物接口。
补齐账号预留字段和 Debug 面板，验证保存/重载、满溢、接口消耗、商人出售和货币保存。
```

## 5. 总体验收标准

- 成功撤离物品能入库。
- 仓库空间不足不吞物。
- 仓库不影响负重。
- 仓库能保存和重载。
- 商人出售能选择物品和数量。
- 出售成功扣物并增加 mine_coin。
- SS 红色超稀有道具可以进入仓库、保留收藏或手动出售。
- 货币按 currency_id 保存和读取。
- 研究/制作消耗原子化。
- 出发准备取物检查背包容量。
- 外部系统不直接修改仓库数组。

## 6. 给后续 AI 的硬约束

```text
不要删除原始 09 文档。
不要静默丢弃玩家仓库物品。
不要让仓库计算局内负重。
不要让失败结算直接写仓库。
不要让研究/制作/出发准备直接改 WarehouseData.items。
不要让商人 UI 直接删除 WarehouseData.items 或直接改 currencies。
不要把售价写死在 MerchantUI；售价来自 items.tab / ItemDefinition。
不要自动出售 SS 道具；SS 必须由玩家主动选择并确认。
不要在 V0.1 实现复杂交易市场、买入商店、价格波动或多货币兑换。
不要实现多人同步，只预留字段。
```
