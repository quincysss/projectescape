# 09-10 数据字段与保存规则

> 来源文档：`09_局外仓库系统_修订版_废土生存法则.md`
> 最新口径：局外经济以“局内带回道具 -> 制造所产出可售物资 -> 白天开店成交 -> 夜晚出发准备”为主线。旧独立售卖入口不再作为实现依据。

## 1. 实现目标

定义仓库、货币、图鉴点亮、货台占用和保存时机，保证局外长期资源可持久化，并且不会因为物资被制造、成交、消耗或丢弃而熄灭图鉴。

## 2. WarehouseItem

```gdscript
{
    "warehouse_item_id": "uuid",
    "item_id": "scrap_metal",
    "count": 4,
    "quality": "C",
    "origin": Vector2i(0, 0),
    "size": Vector2i(1, 1),
    "tags": ["raw_material"]
}
```

规则：
- 同 `item_id + quality + bind_state + durability_state` 的原材料允许堆叠。
- 装备、耐久不同的消耗品、任务物品默认不与普通材料混堆。
- 可售物资也是 `item_id`，但 `item_type` 应标记为 `sale_good`，用于货台上架和店铺结算。

## 3. WarehouseData

```gdscript
{
    "width": 12,
    "height": 10,
    "items": [],
    "shelf_items": [],
    "currencies": {
        "mine_coin": 0
    },
    "collected_item_ids": {
        "scrap_metal": true
    },
    "revision": 0,
    "last_saved_at": 0
}
```

字段说明：
- `items`：仓库内长期持有的道具堆。
- `shelf_items`：已从仓库移到货台、等待白天营业成交的可售物资堆；初始货台 3 格。
- `currencies`：玩家持有货币字典，V0.2 至少包含 `mine_coin`。
- `collected_item_ids`：玩家曾经获得过的 `item_id` 字典，只保存 true，不保存 false。
- `revision`：仓库、货台、货币或图鉴记录发生持久化变更时递增。

## 4. 图鉴保存规则

- 玩家首次获得任意可收集道具时写入 `collected_item_ids[item_id] = true`。
- 获得来源包括：结算入库、制造所产物生成、后勤订购入库、系统奖励、调试添加。
- 制造消耗、研究消耗、货台成交、出发携带、装备穿戴、丢弃和仓库清空，都不得移除图鉴点亮记录。
- 图鉴展示数据来自 `items.tab / ItemDefinition`，存档不保存道具名、图标和描述。
- 仅出现在制造需求、研究需求、排行榜需求、图鉴卡片或后勤订购预览中的道具，不算“曾经获得”。

## 5. 保存时机

必须触发保存：
- 结算入库成功或部分成功。
- 玩家获得任何新道具并点亮图鉴。
- 仓库整理、堆叠合并、拆分、丢弃成功。
- 制造所消耗材料成功。
- 制造所产物生成并进入仓库成功。
- 货台上架、撤下或营业成交成功。
- 货币增加或消耗成功。
- 后勤订购标准件、装备或消耗品成功入库。
- 夜晚出发准备从仓库取出装备或消耗品成功。

不得触发保存：
- 仅预览制造配方。
- 仅查看道具详情 Tooltip。
- 仅查看难民需求排行榜。
- 仅打开或关闭图鉴、仓库、制造所、研究所界面。

保存失败时：
- 返回错误并显示可见提示。
- 保持内存状态不伪装为已保存。
- 不得静默丢物、静默扣币或静默熄灭图鉴。

## 6. AI 开工提示词

```text
请实现 09 仓库数据字段与保存规则。
要求：
1. WarehouseItem 包含 warehouse_item_id、item_id、count、quality、origin、size、tags。
2. WarehouseData 包含 width、height、items、shelf_items、currencies、collected_item_ids、revision、last_saved_at。
3. 玩家首次获得道具时写入 collected_item_ids；制造消耗、研究消耗、货台成交、出发携带、装备穿戴、丢弃和仓库清空不得移除图鉴点亮记录。
4. 入库、整理、丢弃、制造消耗、制造产物、货台上架/撤下/成交、货币变化、后勤订购、出发取物后保存。
5. 不实现旧独立售卖入口，不实现直接把仓库原材料换钱。
6. 保存失败必须返回错误并提示，不得假装成功。
```

## 7. 验收标准

- 重启后仓库物品、货台物品、货币数量和图鉴点亮记录保留。
- 制造、研究、货台成交、出发携带、丢弃之后，`collected_item_ids` 不被清除。
- 仅预览需求或排行榜不会误点亮图鉴。
- `revision` 随有效变更递增。
- 保存失败可见，并且不会造成静默丢物。
