# 09-10 数据字段与保存规则

> 来源文档：`09_局外仓库系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

定义 WarehouseItem、WarehouseData 和保存时机，保证跨局资源可持久化。

## 2. WarehouseItem

```gdscript
{
    "warehouse_item_id": "uuid",
    "item_id": "scrap_metal",
    "count": 4,
    "quality": "B",
    "origin": Vector2i(0, 0),
    "size": Vector2i(1, 1),
    "tags": ["material"]
}
```

## 3. WarehouseData

```gdscript
{
    "width": 12,
    "height": 10,
    "items": [],
    "currencies": {
        "mine_coin": 0
    },
    "revision": 0,
    "last_saved_at": 0
}
```

货币保存规则：

- 玩家持有货币按 currency_id 存储。
- V0.1 只有 `mine_coin`。
- 不在存档中保存货币显示名、图标或类型，这些静态定义来自 `data/currencies.tab`。
- 后续增加其他货币时，只在 currencies 字典中增加对应 currency_id 数量。

## 4. 保存规则

保存时机：

- 入库成功。
- 仓库整理完成。
- 丢弃成功。
- 商人出售成功。
- 研究/制作消耗成功。
- 出发准备取物成功。

## 5. AI/MCP 开工提示词

```text
请实现 WarehouseItem、WarehouseData 和保存规则。
要求：
1. WarehouseItem 包含 warehouse_item_id、item_id、count、quality、origin、size、tags。
2. WarehouseData 包含 width、height、items、currencies、revision、last_saved_at。
3. 每次仓库变更 revision +1。
4. 入库、整理、丢弃、商人出售、研究消耗、制作消耗、出发取物后保存。
5. 保存失败时返回错误并提示，不要假装成功。
```

## 6. 验收标准

- 重启后仓库物品保留。
- 重启后货币数量保留，并能按 currency_id 查询。
- revision 随变更更新。
- 保存失败可见。
