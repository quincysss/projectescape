# 09-10 数据字段与保存规则

> 来源文档：`09_局外仓库系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

定义 WarehouseItem、WarehouseData、图鉴获得记录和保存时机，保证跨局资源可持久化。

## 2. WarehouseItem

```gdscript
{
    "warehouse_item_id": "uuid",
    "item_id": "scrap_metal",
    "count": 4,
    "quality": "B",
    "origin": Vector2i(0, 0),
    "size": Vector2i(1, 1),
    "tags": ["material"]
}
```

## 3. WarehouseData

```gdscript
{
    "width": 12,
    "height": 10,
    "items": [],
    "currencies": {
        "mine_coin": 0
    },
    "collected_item_ids": {
        "scrap_metal": true
    },
    "revision": 0,
    "last_saved_at": 0
}
```

货币保存规则：

- 玩家持有货币按 currency_id 存储。
- V0.1 只有 `mine_coin`。
- 不在存档中保存货币显示名、图标或类型，这些静态定义来自 `data/currencies.tab`。
- 后续增加其他货币时，只在 currencies 字典中增加对应 currency_id 数量。

图鉴保存规则：

- 玩家曾经获得过的道具按 `collected_item_ids[item_id] = true` 保存。
- 只保存已获得过的 item_id，不保存 false 项。
- 新档默认 `{}`，旧档缺字段时补空字典。
- 获得道具时写入；出售、消耗、升级使用、制作使用、丢弃和仓库清空时不删除。
- 图鉴展示信息仍来自 `items.tab / ItemDefinition`，存档不保存道具名、图标和描述。

## 4. 保存规则

保存时机：

- 入库成功。
- 玩家获得任何可收集道具并点亮图鉴。
- 仓库整理完成。
- 丢弃成功。
- 商人出售成功。
- 商人购买成功。
- 研究/制作消耗成功。
- 制作所产物生成成功。
- 出发准备取物成功。

## 5. AI/MCP 开工提示词

```text
请实现 WarehouseItem、WarehouseData 和保存规则。
要求：
1. WarehouseItem 包含 warehouse_item_id、item_id、count、quality、origin、size、tags。
2. WarehouseData 包含 width、height、items、currencies、collected_item_ids、revision、last_saved_at。
3. 每次仓库变更 revision +1。
4. 玩家获得道具时写入 collected_item_ids；商人出售、研究消耗、制作消耗、丢弃和仓库清空不得移除图鉴点亮记录。
5. 入库、整理、丢弃、商人出售、商人购买、研究消耗、制作消耗、制作产物、出发取物后保存。
6. 保存失败时返回错误并提示，不要假装成功。
```

## 6. 验收标准

- 重启后仓库物品保留。
- 重启后货币数量保留，并能按 currency_id 查询。
- 重启后图鉴 collected_item_ids 保留，并能按 item_id 查询点亮状态。
- 出售、消耗、升级使用、制作使用、丢弃和仓库清空不会清除 collected_item_ids。
- revision 随变更更新。
- 保存失败可见。

## 7. 暂不实现

- 云存档。
- 多存档槽。
