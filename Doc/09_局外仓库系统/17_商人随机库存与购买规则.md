# 09-17 商人随机库存与购买规则

> 本文补充 `09-16 局外商人与货币系统`。  
> 目标：让局外商人除了收购玩家仓库中的道具外，也能按商店等级随机出售一批资源。

## 1. 规则边界

原有出售规则不变：

- 玩家可以把仓库中的可出售道具、医疗用品、装备、图纸或稀有物卖给商人。
- 材料和前哨材料仍然没有 `sell_value`，不会出现在玩家出售列表中。
- 玩家卖出物品的价格继续来自 `items.tab` 的 `sellable / sell_currency_id / sell_value`。

新增购买规则：

- 商人出售资源时使用独立的 `setting/shop_stock.tab`。
- 商人出售价格使用 `buy_price`，不复用材料的 `sell_value`。
- V0.3 商人只出售普通 `material`。
- `outpost_material` 是局内前哨修复专用资源，不会被带到局外，不会被商店售卖。
- 商人暂不出售装备、消耗品、图纸和稀有物。

## 2. 商店等级

商店等级由研究所升级决定。研究所系统未完成前，存档字段先保留为：

```text
merchant_shop_level
```

默认等级为 1，当前上限为 3。

| 等级 | 来源 | 单轮库存槽位 | 可出售范围 |
|---|---|---:|---|
| 1 | 初始商人 | 3 | 基础材料、少量电子材料 |
| 2 | 研究所升级 I | 4 | 等级 1 资源 + 医疗、工具、电力材料 |
| 3 | 研究所升级 II | 5 | 等级 2 资源 + 少量前哨维修材料 |

研究所完成后只负责写入等级；商人系统只读取等级，不直接依赖研究节点 ID。

## 3. shop_stock.tab

路径：

```text
setting/shop_stock.tab
```

字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| shop_id | string | 商店 ID，当前默认 `base_merchant` |
| min_shop_level | int | 进入随机池所需最低商店等级 |
| max_shop_level | int | 仍可出现的最高商店等级 |
| item_id | string | 出售资源 ID，必须存在于 `items.tab` |
| min_count | int | 单轮刷新最小数量 |
| max_count | int | 单轮刷新最大数量 |
| weight | int | 随机权重 |
| buy_currency_id | string | 购买消耗货币，默认 `mine_coin` |
| buy_price | int | 单件购买价格 |
| enabled | bool | 是否启用 |
| enabled_version | string | 启用版本 |
| description | string | 策划说明 |

## 4. 随机库存生成

流程：

```text
读取 merchant_shop_level
-> 过滤 shop_id / enabled / 等级范围 / 有效 item_id
-> 只保留 material
-> 按商店等级确定库存槽位数
-> 按 weight 无放回抽取资源
-> 按 min_count / max_count 随机数量
-> 写入当前商人库存
```

规则：

- 同一轮库存中，同一资源最多出现一次。
- 玩家购买后，该资源库存数量减少。
- 库存为 0 的资源从商人列表隐藏。
- V0.1.1 不做现实时间自动刷新；商人库存跟随局外天数刷新。
- 同一天内买空的资源不会自动补货；只有探索结束回到局外、天数推进后，才会清空旧库存，并在下一次打开商人页时懒生成新一轮库存。
- 调试工具仍可主动调用刷新接口，但正式流程以局外天数推进为准。

## 5. 购买原子性

购买必须通过商人服务完成：

```text
校验商人库存存在
-> 校验购买数量合法
-> 校验玩家矿币足够
-> 扣除矿币
-> 将资源加入局外仓库
-> 扣减商人库存
-> 返回购买结果
```

失败时禁止：

- 扣币成功但资源未入库。
- 资源入库但未扣币。
- 库存不足时部分购买。
- UI 直接修改矿币、仓库或商人库存。

## 7. 当前等级库存池（V0.3）

商人出售池只承担“补材料”的职责，不出售装备、消耗品、图纸或稀有结算物。研究所需要的非材料类物品仍然主要依赖局内探索掉落带回。

| 商店等级 | 资源池 | 说明 |
|---|---|---|
| 1 | `scrap_metal`、`cloth_dirty`、`rusted_bolts`、`duct_tape_roll`、`cracked_lens`、`wire_coil` | 基础材料池，包含少量电子材料，用于前期研究和基础仓库/背包升级。 |
| 2 | 等级 1 全部资源 + `battery_old`、`medicine_powder`、`tool_parts`、`reinforced_strap`、`pulse_battery`、`signal_resonator_coil` | 补齐医疗、工具、携行、电力、信号类材料，用于中期研究。 |
| 3 | 等级 2 全部资源 | 不新增前哨材料；只通过更多库存槽位提高补给稳定性。 |

维护规则：
- `shop_stock.tab` 内的 `item_id` 必须存在于 `items.tab`。
- 商人出售项必须是普通 `material`。
- `outpost_material` 不得进入 `shop_stock.tab`。
- 研究需求中的普通 `material` 应在 3 级商人池中可购买，用于给玩家提供兜底补材料路径。
- 前哨修复材料只通过局内菱形材料刷新系统出现，不通过商人、容器掉落或研究所消耗。
- 研究需求中的消耗品、装备、稀有物仍不进入商店池，必须通过局内容器/事件获得。

## 8. 与研究所材料来源的关系

商人购买成功后，物品会进入局外仓库，并标记：

```gdscript
source = "merchant_shop"
```

局内带回的物品通常会保留局内来源标记，例如 `run_loot` 或容器来源字段。

研究所升级只通过 `WarehouseManager.has_items()` 和 `WarehouseManager.consume_items()` 按 `item_id` 统计与消耗仓库物品，不读取 `source` 字段。因此只要物品已经进入局外仓库，并且 `item_id` 与 `research.tab.required_items` 匹配，局内获得和商人购买的同 ID 物品都可以用于研究升级。

例外：`outpost_material` 不会进入局外仓库，也不得作为 `research.tab.required_items`。它只在局内作为前哨站修复材料使用。
