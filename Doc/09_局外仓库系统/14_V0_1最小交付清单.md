## 1. 最小闭环

```text
08 结算物品
-> 09 尝试入库
-> 仓库占格/堆叠
-> 满溢返回
-> 玩家整理/丢弃
-> 仓库保存
-> 研究/制作/出发查询
-> 首次成功返回剧情解锁商人/研究所
-> 商人出售仓库物品获得矿币
-> 图鉴保留曾经获得过的道具点亮记录
```

## 3. AI/MCP 开工提示词

```text
请根据 09_局外仓库系统拆分文档，在 Godot 中完成 V0.1 最小闭环。
优先实现 WarehouseData、WarehouseManager、结算入库、满溢处理、仓库 UI、顶部页签、商人出售、CurrencyWallet、局外道具详情 Tooltip、局外道具图鉴、丢弃确认、堆叠合并、研究/制作/出发查询接口、保存和 Debug。
约束：不要静默丢物；仓库不计算负重；08 入库必须支持部分成功；外部系统不得直接改仓库数组；新档第一天商人和研究所未解锁不可点击；商人出售必须扣物和加币原子化；V0.1 货币为 mine_coin；局外道具 Tooltip 只在 BaseScene 启用，数据来自 items.tab 和 currencies.tab；图鉴记录曾经获得过的 item_id，出售、消耗、升级或丢弃后不熄灭。
```

## 4. 完成定义

- 撤离成功物品能入库。
- 仓库满时剩余物保留。
- 仓库能保存和重载。
- 首次成功返回剧情结束前，商人和研究所页签锁定不可点击。
- 首次成功返回剧情结束后，玩家可在商人页签选择出售物品和数量，获得矿币。
- 货币按 currency_id 保存，重启后保留。
- 仓库、商人、研究所、制作所中的道具图标 hover 后能显示局外道具详情 Tooltip。
- 图鉴页签显示所有可展示道具，每行 4 张卡片，获得过的道具永久点亮。
- 研究/制作能查询和消耗材料。
- 出发准备能从仓库取物到背包。
