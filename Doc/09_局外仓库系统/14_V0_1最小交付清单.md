## 1. 最小闭环

```text
08 结算物品
-> 09 尝试入库
-> 仓库占格/堆叠
-> 满溢返回
-> 玩家整理/丢弃
-> 制造所消耗原材料
-> 制造出 sale_good
-> sale_good 上架货台
-> 白天营业成交获得 mine_coin
-> 仓库/货台/货币/图鉴保存
-> 夜晚出发准备从仓库选择装备和消耗品
```

## 4. AI 开工提示词

```text

约束：
1. 不要静默丢物。
2. 仓库不计算局内负重。
3. 08 入库必须支持部分成功。
4. 外部系统不得直接改仓库数组。
5. 局外经济不走旧独立售卖入口；货币来自白天营业成交。
6. sale_good 上架货台后成交才加币。
7. 图鉴记录曾经获得过的 item_id，成交、消耗、升级或丢弃后不熄灭。
```

## 5. 完成定义

- 撤离成功物品能入库。
- 仓库满时剩余物保留。
- 仓库能保存和重载。
- 玩家可在白天准备阶段进入仓库、制造所、研究所、图鉴。
- 玩家可制造可售物资并上架到货台。
- 营业成交后扣除货台物资并增加货币。
- 货币按 `currency_id` 保存，重启后保留。
- 仓库、制造所、研究所、货台、出发准备中的道具图标 hover 后能显示局外道具详情 Tooltip。
- 图鉴显示所有可展示道具，每行 4 张卡片，获得过的道具永久点亮。
- 研究所只读查询矿币和前置条件；制造所能查询和消耗材料。
- 夜晚出发准备能从仓库取装备和消耗品。
