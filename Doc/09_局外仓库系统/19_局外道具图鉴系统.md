# 09-19 局外道具图鉴系统

> 来源需求：在局外白天准备界面新增“图鉴”入口，记录玩家曾经收集过的道具。
> 关联文档：`06_仓库界面与手动整理.md`、`10_数据字段与保存规则.md`、`12_Godot模块与数据配置.md`、`13_调试工具与验证清单.md`、`18_局外道具详情Tooltip.md`、`Doc/13_数据配置表与TAB规范_修订版_废土生存法则.md`、`Doc/20_UI视觉规范与界面资源Sheet提示词_修订版_废土生存法则.md`、`Doc/29_局外杂货店经营与界面重构路线图.md`、`Doc/30_昼夜局外流程_店铺营业与夜间出发准备.md`。

## 1. 实现目标

在局外白天准备阶段的功能入口中新增“图鉴”，与“仓库、制造所、研究所”平级。

注意：按 `29_局外杂货店经营与界面重构路线图.md` 的最新口径，局外不再保留旧交易页。图鉴位于白天准备阶段功能入口，不在夜晚主界面显示。

图鉴用于展示 `items.tab / ItemDefinition` 中所有可展示道具，并记录玩家是否曾经获得过该道具：

- 默认展示所有道具，不因为未获得而隐藏。
- 未获得道具保持可见，不做整卡灰黑遮罩；可以使用低亮度边框、低饱和状态字或“未获得”文案区分。
- 玩家只要获得过一次某个 `item_id`，该道具永久点亮。
- 点亮状态不受后续消耗、升级、制作、货台成交、丢弃或仓库清空影响。
- 图鉴只记录“是否曾经获得”，不记录当前拥有数量，不替代仓库。

## 2. 设计依据

图鉴承担长期收集反馈，不参与物品交易、容量、负重和研究消耗。

它解决两个玩家体验问题：

- 玩家能在局外回看目前游戏内所有道具信息。
- 稀有道具被卖出或用于升级后，玩家仍保留“曾经拥有过”的成就反馈。

图鉴和仓库的边界：

| 系统 | 负责内容 | 不负责内容 |
| --- | --- | --- |
| 仓库 | 当前持有的长期物品实例、容量、货台上架和消耗 | 曾经获得记录 |
| 图鉴 | item_id 维度的永久点亮记录和道具信息浏览 | 当前数量、交易、消耗、拖拽 |

## 4. 节点与界面结构

白天准备阶段功能入口顺序：

```text
1. 仓库
2. 制造所
3. 研究所
4. 图鉴
```

图鉴入口状态规则：

- V0.1 默认可点击，不受研究所和制造所解锁条件影响。
- 点击“图鉴”后隐藏仓库、制造所、研究所面板，只显示 CatalogPanel。
- 当前选中入口高亮。
- 切换到其他入口时，图鉴滚动位置可以保留；刷新存档或重新进入 BaseScene 时可回到顶部。
- 夜晚、夜间计划、出发准备和开店结算阶段不显示图鉴入口。

图鉴内容布局：

- 每个道具显示为小卡片。
- 每张卡片显示道具图标、道具名、道具描述，全部居中。
- 道具图标尺寸明显大于道具名和描述，视觉参考局外 hover 道具卡。
- 每行固定 4 张卡片。
- 横向不滚动；父容器宽度变化时，卡片宽度等分当前可用宽度。
- 纵向使用滚动条展示更多道具。
- 卡片底部显示状态文案：`(已获得)` 或 `(未获得)`。

## 5. 字段与数据结构

存档新增字段：

```gdscript
profile["collected_item_ids"] = {
    "scrap_metal": true,
    "old_chip": true
}
```

字段规则：

- key 为 `item_id`。
- value 固定保存为 `true`，不保存 false 项。
- 新档默认 `{}`。
- 旧档读取时如果缺少该字段，补默认空字典。
- 重置本地进度时清空该字段。

图鉴展示数据结构：

```gdscript
{
    "item_id": "old_chip",
    "display_name": "旧时代芯片",
    "description": "一枚仍带余温的旧时代芯片。",
    "icon": "res://...",
    "quality": "A",
    "collected": true
}
```

数据来源：

- 展示列表来自 `items.tab / ItemDefinition`。
- `display_name` 使用 `name`。
- `description` 使用 `description`，为空时显示 `暂无记录。`。
- `icon` 使用 `icon`，缺失时显示统一占位图标或文字占位。
- 默认不新增 `items.tab` 字段。
- 如存在 debug、内部测试、货币、不可展示占位物品，应由现有 item 类型或 tags 过滤；若当前表字段不足，V0.1 先展示全部 items.tab 道具并在文档缺口中记录。

## 6. 点亮触发规则

核心规则：

```text
任意系统让玩家“获得”某个 item_id
-> 调用 ItemCatalogService.mark_collected(item_id, source)
-> 若该 item_id 尚未点亮，则写入 collected_item_ids[item_id] = true
-> 保存玩家档案
-> 图鉴界面刷新点亮状态
```

必须触发点亮的来源：

- 局内拾取进入背包。
- 撤离结算物品进入结算列表或成功入库。
- 制造所产物生成成功。
- 后勤订购物资、装备或消耗品入库成功。
- Debug 添加测试物品时，Debug 模式可点亮，并在日志中标记 source 为 debug。

不应触发点亮的来源：

- 仅作为研究矿币/条件图标或制作材料需求图标展示。
- 仅作为后勤订购预览、制造配方预览、当日需求排行榜展示，但玩家尚未获得。
- 仅作为图鉴卡片本身展示。
- 配置表加载。

边界规则：

- 货台成交或后勤消耗道具不熄灭图鉴。
- 用于制作、升级或货台成交的材料不熄灭图鉴；研究不消耗材料。
- 丢弃或仓库清空不熄灭图鉴。
- 如果获得同一 item_id 多次，只保留一次点亮记录。
- 如果某个 item_id 后续从表中删除，旧存档可保留 key，但图鉴列表不展示无定义道具；Debug 可输出 orphan collected id。

## 7. 交互与信号接口

推荐接口：

```gdscript
func mark_collected(item_id: String, source: String = "") -> Dictionary
func mark_many_collected(item_ids: Array[String], source: String = "") -> Dictionary
func is_collected(item_id: String) -> bool
func get_collected_item_ids() -> Dictionary
func get_catalog_items(filters: Dictionary = {}) -> Array[Dictionary]
func clear_collected_debug_only() -> void
```

GameState 推荐对外转发：

```gdscript
func mark_item_collected(item_id: String, source: String = "") -> Dictionary
func query_catalog_items(filters: Dictionary = {}) -> Array[Dictionary]
func is_item_collected(item_id: String) -> bool
```

UI 禁止直接写 `profile["collected_item_ids"]`。

## 8. 视觉规格

卡片风格：

```text
背景：接近黑色，透明度 90%-96%
边框：细蓝绿色描边
已获得：图标正常亮度，边框可轻微发光，底部显示“(已获得)”
未获得：卡片仍可见，图标和文字略低亮度，不使用整卡灰黑遮罩，底部显示“(未获得)”
圆角：0-2 px，保持废土硬边 UI 气质
```

布局建议：

| 区域 | 建议 |
| --- | --- |
| 卡片比例 | 纵向卡片，接近 Tooltip 卡 |
| 图标 | 卡片上半段居中，尺寸约为卡片宽度 40%-50% |
| 道具名 | 图标下方居中，16-20 px |
| 分割线 | 名称与描述之间可用细蓝绿色线 |
| 描述 | 居中换行，最多 3-5 行，超出时截断或缩小字号 |
| 状态 | 底部小字，12-14 px |

卡片不得显示当前拥有数量，避免玩家误认为图鉴可操作仓库物品。

## 9. 执行步骤

1. 在存档默认值中加入 `collected_item_ids`。
2. 在 GameState 加载、保存、重置流程中同步该字段。
3. 新增 ItemCatalogService，绑定 collected_item_ids 和 DataRegistry。
4. 在获得物品的入口调用 mark_collected，不在出售、消耗和丢弃时移除记录。
5. 在 BaseScene 白天准备功能栏新增 CatalogButton。
6. 新增 CatalogPanel，使用 ScrollContainer + GridContainer 或等效布局，固定 4 列。
7. 卡片从 `items.tab` 构建，按 `collected` 状态渲染点亮/未获得表现。
8. 增加 Debug 按钮或测试入口，用于点亮全部、清空图鉴、点亮指定道具。
9. 增加保存重载验证，确认点亮状态跨重启保留。

## 11. 验证命令或验证场景

文本验证：

```text
rg -n "collected_item_ids|mark_item_collected|query_catalog_items|CatalogTabButton|CatalogPanel" scripts scenes tests
```

建议新增自动测试：

```text
tests/item_catalog_collection_check.gd
```

建议覆盖：

- 新档 `collected_item_ids` 默认为空。
- mark_collected 后保存字段为 true。
- 重复 mark_collected 不产生重复数据。
- shelf_deal/consume/discard 后点亮状态不消失。
- query_catalog_items 返回 items.tab 全量道具，并带 collected 状态。

手动验收：

1. 新建档案进入局外 BaseScene。
2. 点击“图鉴”，确认所有道具以 4 列卡片展示。
3. 用 Debug 添加一个测试道具或完成一次拾取/结算入库/后勤订购/制作。
4. 返回图鉴，确认该道具点亮并显示“(已获得)”。
5. 将该道具通过货台成交或用于消耗。
6. 再次查看图鉴，确认仍为点亮状态。
7. 重启游戏或重新加载档案，确认点亮状态保留。
