# 09-16 局外商人与货币系统

> 来源文档：`09_局外仓库系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`、`07_背包_存储与负重系统/01_物品定义与品质规则.md`、`13_数据配置表与TAB规范_修订版_废土生存法则.md`、`20_UI视觉规范与界面资源Sheet提示词_修订版_废土生存法则.md`。

## 1. 实现目标

建立局外商人和货币底座，让玩家把已带到局外、已进入仓库的道具出售给商人，获得货币。

V0.1 只有一种货币：

```text
mine_coin / 矿币
```

但数据结构必须支持后续多货币扩展。

## 2. 系统边界

商人处理：

- 查询仓库中可出售物品。
- 显示物品单价、数量和总价。
- 让玩家选择要卖的物品和数量。
- 出售成功后扣除仓库物品。
- 按 currency_id 增加玩家货币。

商人不处理：

- 买入商品。
- 价格波动。
- 多商人声望。
- 拍卖行。
- 自动出售。
- 结算界面直接出售。

## 3. 货币数据

静态货币定义来自：

```text
data/currencies.tab
```

V0.1 表头：

```text
id	name	icon	currency_type	sort_order	enabled_version	description
```

V0.1 初始行：

```text
mine_coin	矿币	res://assets/icons/currency/mine_coin.png	standard	10	v0.1	废土城镇间通用的矿区铸币。
```

玩家持有货币保存为字典：

```gdscript
{
    "mine_coin": 120
}
```

规则：

- 存档只保存 currency_id 和数量。
- 货币名称、图标、类型来自 currencies.tab。
- 货币 ID 一旦进入存档，不随意改名。
- 后续新增货币时，直接在 currencies 字典中新增 key。

## 4. 道具售价

道具基础售价来自 items.tab / ItemDefinition：

```text
sellable
sell_currency_id
sell_value
```

规则：

- sellable = false 的物品不出现在商人出售列表。
- sell_value 表示单个物品基础售价。
- sell_currency_id 默认 `mine_coin`。
- 出售总价 = sell_value * count。
- sell_value <= 0 的物品默认不可出售，除非文档明确允许。
- quest / key / debug 标签物品默认不可出售。
- SS 品质道具默认可出售，必须拥有较高 `sell_value`，并在商人界面以红色品质显示。
- SS 品质道具也允许玩家选择不出售，作为仓库收藏或炫耀物保留。

## 5. UI 页签

BaseScene 局外顶部页签顺序：

```text
1. 仓库
2. 商人
3. 研究所
4. 制作所
5. 图鉴
```

状态规则：

- 已开发页签可点击。
- 商人页签必须同时满足 `merchant_unlocked == true` 才可点击；新档第一天未首次返回前显示锁定/置灰状态。
- 研究所页签必须同时满足 `research_station_unlocked == true` 才可点击；新档第一天未首次返回前显示锁定/置灰状态。
- 未开发完成页签显示置灰状态并禁止点击。
- 图鉴页签默认可点击，显示规则见 `19_局外道具图鉴系统.md`。
- 当前选中页签高亮。
- 顶部或右上角显示玩家当前矿币数量。

V0.1 建议：

```text
仓库：可用
商人：首次成功返回剧情结束后可用
研究所：首次成功返回剧情结束后可用；未完成则继续置灰不可点击
制作所：可显示置灰，未完成则不可点击
图鉴：可用
```

首次进入局外时的门禁：

```text
merchant_unlocked=false
research_station_unlocked=false
```

此时商人和研究所可以保留页签位置，但不能进入功能界面。首次成功撤离返回后，播放 `first_return_chapter_1_dialogue`，对白结束时由 25 号流程文档设置：

```text
merchant_unlocked=true
research_station_unlocked=true
```

随后目标卡的“前往商人”按钮才允许切换到商人页签。

## 6. 商人界面

必须显示：

- 仓库中可出售物品列表。
- 物品图标。
- 物品名称。
- 当前拥有数量。
- 单价。
- 道具图标 hover 详情 Tooltip。
- 出售数量选择器。
- 本次出售总价。
- 玩家当前矿币。
- 出售按钮。
- 出售结果提示。

SS 道具显示补充：

- 品质文字显示为 `SS`。
- 品质色使用红色。
- 物品卡可显示 `超稀有` 或 `收藏品` 标签。
- 不允许自动出售 SS，道具必须由玩家主动选择数量并确认。

交互规则：

- 玩家必须选择物品。
- hover 可出售物品图标时，显示局外道具详情 Tooltip，规则见 `18_局外道具详情Tooltip.md`。
- 玩家必须选择出售数量，范围为 1 到当前拥有数量。
- 数量变化后实时更新总价。
- 出售按钮在未选择物品、数量非法、物品不可出售时禁用。
- 出售成功后刷新仓库数量和矿币显示。
- 出售失败必须显示 reason。

## 7. 推荐接口

```gdscript
func query_sellable_items(filters: Dictionary = {}) -> Array[Dictionary]
func get_sell_quote(warehouse_item_id: String, count: int) -> Dictionary
func sell_warehouse_item(warehouse_item_id: String, count: int) -> Dictionary
func get_currency_amount(currency_id: String) -> int
func add_currency(currency_id: String, amount: int, reason: String) -> Dictionary
```

出售返回：

```gdscript
{
    "ok": true,
    "sold_item_id": "scrap_metal",
    "sold_count": 5,
    "currency_id": "mine_coin",
    "amount_added": 15,
    "new_currency_amount": 135
}
```

失败返回：

```gdscript
{
    "ok": false,
    "reason": "not_enough_count"
}
```

## 8. 原子性要求

出售必须是单个事务：

```text
校验仓库物品存在
-> 校验可出售
-> 校验数量
-> 计算总价
-> 扣除仓库物品
-> 增加货币
-> 保存
-> 返回结果
```

禁止：

- UI 直接删除仓库物品。
- UI 直接修改货币数量。
- 扣物成功但加币失败后假装成功。
- 加币成功但扣物失败后保留货币。
- 出售失败时改变仓库或货币。

## 11. 验收标准

- [ ] currencies.tab 存在且包含 mine_coin。

- [ ] 总价随数量变化。

- [ ] 出售成功后 mine_coin 增加。
- [ ] 出售失败不扣物、不加币。
