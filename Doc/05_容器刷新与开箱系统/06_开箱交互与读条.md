# 06_开箱交互与读条

> 对应母版章节：9、16.5  
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“开箱交互与读条”小节。

---

## 1. 实现目标

实现玩家靠近容器后的交互：显示 F 键提示，长按 F 开始读条，离开、移动、松手或死亡中断；读条完成后通知容器生成/展示产物。

---

## 2. 设计依据

交互条件：

```text
玩家在交互触发半径内。
容器处于 AVAILABLE 或 OPENED_AVAILABLE。
玩家没有正在执行其他读条交互。
玩家未死亡。
```

V0.1 推荐：

```text
长按 F 开启。
进度中断直接清零。
开箱期间稳定值继续下降。
玩家不能移动。
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 触发条件 | 玩家进入 ContainerInteractionArea |
| 输入规则 | 长按 F 开始/维持读条，松开可中断 |
| 中断规则 | 离开触发器、移动、松手、死亡均中断 |
| 读条规则 | V0.1 中断后进度清零 |
| 已开规则 | OPENED_AVAILABLE 再交互直接打开面板，不读条 |
| 背包规则 | 不要求背包有空间才能开 |
| 失败规则 | 同时只能执行一个读条交互 |

---

## 3.1 可交互描边规则

可交互描边属于交互反馈，不按每种容器制作固定 PNG 边框。默认实现应由 `InteractionHighlightService` 根据容器状态和玩家距离输出高亮状态，再由容器视图使用 Shader 按 Sprite alpha 轮廓自动描边。

| 状态 | 描边反馈 |
|---|---|
| 可交互但玩家未进入半径 | 可选弱描边或不显示，避免画面噪音 |
| 玩家进入交互半径 | 显示清晰的紫蓝/高饱和描边，并显示 F 提示 |
| 正在读条 | 描边可轻微脉冲或流动，读条 UI 负责显示主要进度 |
| 已打开且仍有物品 | 二次交互描边弱于首次打开状态 |
| 倒计时临近归零 | 可叠加短促红色闪烁，提示即将消失 |
| REMOVED / 空容器移除 | 不显示描边，不再响应交互 |

实现边界：

```text
ContainerInteractionArea 只负责触发半径检测。
ContainerOpenInteraction 只负责输入、读条和中断。
InteractionHighlightService 负责把“是否可交互、是否进入半径、是否读条、是否即将过期”转成描边状态。
容器视图使用统一 ShaderMaterial 参数表现描边，不为纸箱、木箱、工具柜等容器分别切固定边框图。
```

## 5. 数据结构

字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `is_player_inside` | `bool` | 是否在范围 |
| `is_opening` | `bool` | 是否读条 |
| `open_progress` | `float` | 0-open_time |
| `current_container_id` | `StringName` | 当前容器 |
| `interrupt_reason` | `StringName` | 中断原因 |

---

## 6. 核心流程

```text
player_entered_area
→ 如果容器 AVAILABLE，显示 F 提示
→ 长按 F
→ state = OPENING
→ 每帧 progress += delta
→ progress >= open_time
→ open_completed
→ ContainerEntity 进入 OPENED/LOOTING
```

中断：

```text
player_left_area / moved / key_released / stability_depleted
→ cancel_open(reason)
→ progress = 0
→ ContainerEntity.open_cancelled(reason)
```

```text
禁止实现掉落生成、背包拾取、正式 UI 美术。
完成后验证成功开箱、离开中断、移动中断、二次交互免读条。
```
