# 05_容器实体生命周期

> 对应母版章节：8、9.6、11.4、11.5、16.4  
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“容器实体生命周期”小节。

---

## 1. 实现目标

实现 `ContainerEntity` 的完整生命周期：生成、可交互、开启中、已打开、拾取中、残留可再次打开、过期、移除，并正确释放刷新点。

---

## 2. 设计依据

状态建议：

```text
SPAWNING
AVAILABLE
OPENING
OPENED
LOOTING
OPENED_AVAILABLE
EXPIRED
REMOVED
```

核心流程：

```text
AVAILABLE 等待交互
→ OPENING 读条
→ OPENED 生成产物
→ LOOTING 打开面板
→ OPENED_AVAILABLE 有剩余物，可二次交互免读条
→ REMOVED 拿空或过期移除
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 生成 | 只确定类型、种子、生命周期、点位 |
| 生命周期 | AVAILABLE 时倒计时归零进入 EXPIRED |
| 已打开 | 读条成功后 has_been_opened=true |
| 残留 | 有剩余 loot 时进入 OPENED_AVAILABLE |
| 二次交互 | 同一个实体已打开后再次交互免读条 |
| 移除 | 拿空或过期后 REMOVED，并释放点位进入冷却 |
| 失败规则 | REMOVED 后不得再次交互或重复释放点位 |
| 视觉状态 | ContainerEntity 只暴露 state、remaining_lifetime、has_been_opened 等状态，不直接写死描边颜色 |

---

## 3.1 描边状态输出

容器实体需要提供足够状态给交互描边系统，但不应持有每种容器的固定边框资源。

```text
AVAILABLE：可由 InteractionHighlightService 判定是否显示可交互描边。
OPENING：可输出 opening 状态，让视图表现轻微脉冲描边。
OPENED_AVAILABLE：仍可交互，但描边弱于首次打开。
EXPIRED / REMOVED：必须关闭描边和交互提示。
remaining_lifetime <= near_expire_seconds：可输出 near_expire，用于红色短闪提示。
```

默认实现使用统一 `InteractableOutlineShader` 按容器 Sprite alpha 自动描边，避免纸箱、木箱、保险柜等不同尺寸容器分别维护固定 PNG 边框。

## 5. 数据结构

字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `container_id` | `StringName` | 唯一 ID |
| `container_type` | `StringName` | 类型 |
| `state` | `ContainerState` | 当前状态 |
| `remaining_lifetime` | `float` | 剩余存在时间 |
| `open_time` | `float` | 本实体开启时间 |
| `open_progress` | `float` | 读条进度 |
| `interaction_radius` | `float` | 触发半径 |
| `has_been_opened` | `bool` | 是否首次打开过 |
| `remaining_loot_entries` | `Array` | 剩余产物 |
| `spawn_point_id` | `StringName` | 来源点 |
| `loot_seed` | `int` | 掉落种子 |
| `highlight_state` | `StringName` | 给交互描边系统读取的轻量状态，可由服务计算或缓存 |

---

## 6. 核心流程

```text
initialize()
→ state = SPAWNING
→ 短暂表现后 AVAILABLE
→ _process(delta) 扣 remaining_lifetime
→ 如果 AVAILABLE 且 <=0，expire()
→ open_completed 后 OPENED/LOOTING
→ panel_closed 后：
   → loot empty -> remove()
   → loot remains and lifetime >0 -> OPENED_AVAILABLE
   → loot remains and lifetime <=0 -> remove()
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | AVAILABLE 容器会倒计时 |
| A2 | 未打开到期会移除 |

| A4 | 有剩余物进入 OPENED_AVAILABLE |
| A5 | 已开容器二次交互免读条 |
