# 05_容器实体生命周期

> 对应母版章节：8、9.6、11.4、11.5、16.4  
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“容器实体生命周期”小节。

---

## 1. 实现目标

实现 `ContainerEntity` 的生命周期：生成、可交互、开启中、已打开、拾取中、残留可再次打开、拿空、移除。普通容器没有存在倒计时，也不会因为时间流逝自然过期。

---

## 2. 设计依据

状态建议：

```text
SPAWNING
AVAILABLE
OPENING
OPENED
LOOTING
OPENED_AVAILABLE
EMPTIED
REMOVED
```

核心流程：

```text
AVAILABLE 等待交互
→ OPENING 读条
→ OPENED 生成产物
→ LOOTING 打开面板
→ OPENED_AVAILABLE 有剩余物，可二次交互免读条
→ EMPTIED 拿空
→ REMOVED 从本次地图实例移除
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 生成 | 只确定类型、种子、点位、开启时间和交互半径 |
| 无限时移除 | 普通容器不维护限时存在字段 |
| 已打开 | 读条成功后 `has_been_opened=true` |
| 残留 | 有剩余 loot 时进入 `OPENED_AVAILABLE` |
| 二次交互 | 同一个实体已打开后再次交互免读条 |
| 拿空移除 | 容器内无物品后进入 `EMPTIED/REMOVED` |
| 离图结算 | 玩家离开地图后，未拾取物品按放弃处理，不进入局外仓库 |
| 失败规则 | `REMOVED` 后不得再次交互或重复释放点位 |
| 视觉状态 | `ContainerEntity` 只暴露 state、has_been_opened、remaining_loot_entries 等状态，不直接写死描边颜色 |

---

## 3.1 描边状态输出

容器实体需要提供足够状态给交互描边系统，但不应持有每种容器的固定边框资源。

```text
AVAILABLE：可由 InteractionHighlightService 判定是否显示可交互描边。
OPENING：可输出 opening 状态，让视图表现轻微脉冲描边。
OPENED_AVAILABLE：仍可交互，但描边弱于首次打开。
EMPTIED / REMOVED：必须关闭描边和交互提示。
```

默认实现使用统一 `InteractableOutlineShader` 按容器 Sprite alpha 自动描边，避免纸箱、木箱、保险柜等不同尺寸容器分别维护固定 PNG 边框。

## 6. 核心流程

```text
initialize()
→ state = SPAWNING
→ 短暂表现后 AVAILABLE
→ open_started 后 OPENING
→ open_cancelled 后回到 AVAILABLE
→ open_completed 后 OPENED/LOOTING
→ panel_closed 后：
   → loot empty -> EMPTIED -> remove()
   → loot remains -> OPENED_AVAILABLE
→ leave_map 时未拾取物品按放弃处理
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 普通容器没有存在倒计时 |
| A2 | 未打开容器不会因时间流逝移除 |
| A3 | 打开后设置 `has_been_opened` |
| A4 | 有剩余物进入 `OPENED_AVAILABLE` |
| A5 | 已开容器二次交互免读条 |
| A6 | 拿空后 `REMOVED`，不可再交互 |
