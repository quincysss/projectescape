# 05-15 SS 超稀有道具刷新规则

> 来源文档：`05_容器刷新与开箱系统_修订版_废土生存法则.md`
> 关联文档：`07_背包_存储与负重系统/01_物品定义与品质规则.md`、`09_局外仓库系统/16_局外商人与货币系统.md`
> 施工补充：本文件定义 SS 档红色超稀有道具的跨日命中、局内名额和容器掉落规则。

## 1. 实现目标

新增一批 `SS` 品质道具：

- 红色品质。
- 极其稀有。
- 可带回局外仓库。
- 可出售给商人换取矿币。
- 主要提供炫耀价值、收藏价值和高额售卖价值。

SS 道具不参与前哨站核心修复材料闭环，不得让主线推进依赖 SS。

## 2. 双层概率结构

SS 刷新分两层：

```text
跨日/跨局命中层：本局是否开启 SS 掉落池
-> 局内容器层：开启 SS 池后，各容器是否产出 SS 道具
```

禁止把“当天进入局内 20%”和“纸箱 0.1%”合并成一个概率。前者决定本局有没有 SS 机会，后者只在本局命中后才生效。

## 3. 跨日命中概率

玩家每天第一次进入局内时，进行一次 SS 本局命中判定。

初始概率：

```text
20%
```

推荐配置化衰减表：

| 命中档位 | 本局 SS 命中概率 |
| --- | ---: |
| 0 | 20% |
| 1 | 10% |
| 2 | 5% |
| 3 | 2.5% |
| 4+ | 1% |

规则：

- 新玩家或恢复后从 20% 开始。
- 如果当天进入对局命中 SS，则本局开启 SS 池，并且下一次每日判定降低到下一档。
- 如果当天未命中，则当前概率不下降。
- 未命中后，当前概率维持 2 个每日判定机会。
- 如果接下来连续 2 次每日判定都未命中，则第三次每日判定前恢复到 20%。
- 如果维持期间任意一次命中，则立即按命中规则进入下一档，并重置未命中维持计数。

示例：

```text
Day 1：20% 命中 -> Day 2 概率变 10%
Day 2：10% 未命中 -> Day 3 仍为 10%
Day 3：10% 未命中 -> Day 4 恢复 20%
Day 3：如果命中 -> Day 4 概率变 5%
```

## 4. 本局 SS 名额

如果本局跨日判定命中：

```text
本局 SS 名额 = 1-2 个
```

规则：

- 本局最多生成 2 个 SS 道具。
- 本局至少预留 1 个 SS 名额。
- 名额只代表“本局存在 SS 掉落机会”，不保证玩家一定能找到或带出。
- 已生成 SS 数达到名额后，本局后续容器不再产出 SS。
- SS 生成必须写入 Debug 日志，但正式 UI 不提前暴露。

推荐实现：

```gdscript
var ss_run_active: bool
var ss_budget_total: int
var ss_budget_used: int = 0
```

## 5. 容器 SS 概率

只有 `ss_run_active == true` 且 `ss_budget_used < ss_budget_total` 时，容器才允许进行 SS 判定。

建议基础概率：

| 容器类型 | SS 判定概率 |
| --- | ---: |
| 纸箱 | 0.1% |
| 木箱 | 1% |
| 工具柜 | 1% |
| 医疗柜 | 1% |
| 小保险箱 | 5% |
| 大保险柜 | 10% |
| 异常箱 | 10% |

容器概率在开启完成生成 loot 时判定，未打开前不得显示。

## 6. 最少 1 个名额的落地方式

为了满足“本局命中后最少 1 个 SS”的设计意图，同时不提前暴露位置，推荐采用隐藏名额 Director：

```text
本局命中 -> 生成 1-2 个 SS 名额
-> 每次容器生成 loot 时按容器概率 roll
-> roll 成功则消耗 1 个 SS 名额
-> 如果本局已经进入后段且仍未消耗任何名额，则提高高价值容器的隐藏补偿权重
```

V0.1 可先使用简单保底：

- 当本局 SS 激活且 `ss_budget_used == 0`；
- 玩家已开启至少 8 个有效容器或本局剩余时间低于 30%；
- 下一次开启小保险箱/大保险柜/异常箱时，允许强制消耗 1 个 SS 名额。

注意：

- 强制消耗仍只在开启完成时发生。
- 强制消耗不显示给玩家。
- 如果玩家完全不打开容器，不需要凭空发放 SS。

## 7. SS 道具数据要求

SS 道具 ItemDefinition 必须包含：

```gdscript
quality = "SS"
item_type = "rare"
sellable = true
sell_currency_id = "mine_coin"
sell_value > 0
tags = ["ss_rare", "showcase", "sellable"]
```

首批 SS 道具池：

| item_id 建议 | 显示名 | 定位 |
| --- | --- | --- |
| `ss_silverwing_engine_core` | 银翼发动机核心 | 旧世高端载具/飞行器遗留核心，收藏与高价售卖 |
| `ss_pink_star` | 粉星 | 稀有晶体或旧世纪念物，偏炫耀收藏 |
| `ss_wanming_pocket_watch` | 万鸣怀表 | 旧世精密机械藏品，偏收藏与故事价值 |
| `ss_old_world_gold_bar` | 旧世金砖 | 高价值贵金属，偏售卖价值 |
| `ss_zero_master_control_board` | 零号主控板 | 旧世核心控制板，偏科技遗物与高价售卖 |

这 5 个道具都属于 SS 池，不进入普通 C/B/A/S 掉落表。

展示规则：

- UI 品质色为红色。
- 结算界面高亮显示，但不自动入库之外额外弹大段说明。
- 商人界面显示高价值售价。
- 仓库中可作为收藏物长期保存。

## 8. Debug 与日志

Debug 面板需要显示：

- 当前每日 SS 概率档位。
- 今日是否已判定。
- 本局是否命中 SS。
- 本局 SS 名额总数。
- 本局 SS 已生成数量。
- 当前未命中维持计数。
- 强制触发 SS 本局命中。
- 强制生成 SS 道具。

日志事件：

```gdscript
ss_daily_roll
ss_run_activated
ss_run_missed
ss_chance_decayed
ss_chance_recovered
ss_container_roll
ss_item_generated
ss_budget_exhausted
```

## 9. AI/MCP 开工提示词

```text
请实现 SS 红色超稀有道具刷新规则。

要求：
1. 新增 SSRunChanceDirector，负责每日第一次进入局内时判定本局是否开启 SS 池。
2. 初始命中概率为 20%；命中后下一次每日判定降为 10%，继续命中继续按配置表下降；未命中则当前概率维持 2 次每日判定，连续 2 次未命中后第三次恢复 20%。
3. 新增 SSLootDirector，负责本局 SS 名额和容器内 SS 判定。
4. 本局命中后，SS 名额为 1-2 个；本局最多生成 2 个 SS。
5. 只有本局命中且名额未用完时，容器才按类型概率判定 SS：纸箱 0.1%、木箱 1%、工具柜 1%、医疗柜 1%、小保险箱 5%、大保险柜 10%、异常箱 10%。
6. SS 道具 quality 必须为 SS，UI 使用红色，sellable=true，sell_currency_id=mine_coin，sell_value>0。
7. 首批 SS 道具池包含：银翼发动机核心、粉星、万鸣怀表、旧世金砖、零号主控板。
8. SS 不得作为前哨核心修复材料，也不得影响撤离条件。
9. 正式 UI 不提前暴露 SS 是否存在；Debug 可显示每日概率、本局命中、名额和生成日志。
```

## 10. 验收标准

- 每日第一次进入局内会进行 SS 本局命中判定。
- 命中后下一次每日概率下降。
- 连续未命中能按规则恢复到 20%。
- 未命中本局不会生成 SS。
- 命中本局最多生成 2 个 SS。
- SS 只从容器 loot 生成。
- SS 道具为红色品质，可入库，可出售。
