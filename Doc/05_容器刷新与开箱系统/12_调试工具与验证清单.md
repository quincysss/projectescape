# 05-12 调试工具与验证清单

> 来源文档：`05_容器刷新与开箱系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取源文档的“调试工具”和“验证清单”小节。

## 1. 实现目标

为容器刷新、开箱、战利品、异常箱和安全区过滤提供可观察、可强制触发、可重复验证的调试工具。

容器系统随机性较高，如果没有调试面板，会很难判断问题来自刷新点、圈层权重、生命周期、掉落表还是背包接口。

## 2. Debug 面板字段

Debug 面板至少显示：

| 字段 | 用途 |
| --- | --- |
| active_container_count | 当前活跃容器数量 |
| next_spawn_seconds | 下一轮刷新倒计时 |
| spawn_points_by_ring | 各圈层可用刷新点数量 |
| blocked_points_count | 被安全区或占用过滤的点数量 |
| selected_container_id | 当前选中容器 |
| selected_state | 当前状态 |
| selected_timer | 剩余生命周期 |
| selected_seed | 战利品种子 |
| generated_loot_count | 已生成战利品数量 |

## 3. Debug 操作按钮

建议提供以下按钮：

| 按钮 | 行为 |
| --- | --- |
| Force Spawn Round | 立即执行一轮刷新 |
| Clear All Containers | 清除所有容器 |
| Spawn Type | 在玩家附近生成指定类型容器 |
| Spawn Anomaly | 强制生成异常箱 |
| Show Spawn Points | 显示全部刷新点 |
| Show Legal Points | 只显示当前合法点 |
| Preview Loot | 使用当前配置预览掉落 |
| Expire Selected | 让选中容器立即过期 |

`Preview Loot` 只能出现在 Debug 模式，不得进入正式 UI。

## 4. 刷新点可视化

刷新点颜色建议：

| 状态 | 颜色含义 |
| --- | --- |
| 可用 | 绿色 |
| 被安全区阻挡 | 红色 |
| 被容器占用 | 黄色 |
| 冷却中 | 蓝色 |
| 圈层不匹配 | 灰色 |

每个点显示：

```text
point_id
ring_id
cooldown
blocked_reason
```

## 5. 日志事件

容器系统应记录结构化日志：

```gdscript
ContainerDebug.log_event({
    "event": "spawn_attempt",
    "point_id": point_id,
    "ring_id": ring_id,
    "result": "rejected",
    "reason": "safe_zone"
})
```

推荐事件：

- `spawn_round_started`
- `spawn_attempt`
- `container_spawned`
- `container_expired`
- `open_started`
- `open_interrupted`
- `open_completed`
- `loot_generated`
- `loot_pickup_attempt`
- `anomaly_premonition_started`
- `anomaly_container_spawned`

## 6. 验证清单

### 6.1 刷新验证

- [ ] 容器每 10-15 秒刷新一轮。
- [ ] 每轮刷新 1-3 个容器。
- [ ] 只选择合法空点。
- [ ] 已占用点不重复生成。
- [ ] 过期容器被移除。
- [ ] 点位冷却 5-10 秒后重新可用。

### 6.2 圈层验证

- [ ] 内层主要生成纸箱和木箱。
- [ ] 中层开始出现工具柜和医疗柜。
- [ ] 外层开始出现保险箱。
- [ ] 最外层能出现异常箱。
- [ ] 高价值容器不会出现在家园门口。

### 6.3 开箱验证

- [ ] 玩家靠近后可以长按开箱。
- [ ] 离开范围会中断读条。
- [ ] 松开按键会中断读条。
- [ ] 稳定值耗尽会中断读条。
- [ ] V0.1 中断后进度重置。
- [ ] 已开启容器再次打开不需要读条。
- [ ] 进入交互半径时容器描边状态正确变化。
- [ ] 不同尺寸和形状的容器共用同一 Shader 描边，并贴合自身 Sprite 轮廓。
- [ ] 工程中没有为每种容器维护固定 PNG 描边边框的依赖。

### 6.4 过期验证

- [ ] 未开启容器到时会消失。
- [ ] 开箱读条期间计时继续。
- [ ] 读条完成时即使容器已经到时，也允许打开结果结算。
- [ ] 读条中断后如果容器已过期，会立即消失。
- [ ] 已开启但有剩余物品的容器保留到计时结束。

### 6.5 掉落验证

- [ ] 战利品在开启完成时生成。
- [ ] 开启前不会预览掉落。
- [ ] 同 seed 结果可复现。
- [ ] 不同圈层品质权重不同。
- [ ] 异常箱不保底 S 品质。
- [ ] 核心修复材料不依赖容器。

### 6.6 背包验证

- [ ] 单个拾取可以成功。
- [ ] 一键拾取优先合并堆叠。
- [ ] 背包满时未拾取物品留在容器里。
- [ ] 超重仍可拾取。
- [ ] 超重后出现提示或速度惩罚。

## 7. AI/MCP 开工提示词

```text
请为 Godot 容器刷新与开箱系统实现 Debug 面板和验证辅助工具。

要求：
1. Debug 面板显示活跃容器数量、下一轮刷新倒计时、各圈层可用点数量、阻挡点数量、选中容器状态/计时/seed。
2. 提供强制刷新、清空容器、指定类型生成、异常箱生成、显示刷新点、掉落预览、选中容器过期等按钮。
3. 刷新点可视化需要区分可用、安全区阻挡、占用、冷却、圈层不匹配。
4. 所有关键流程写结构化日志，方便追踪刷新失败和拾取失败原因。
5. Debug 掉落预览不得接入正式 UI。
6. 输出最小验证步骤，覆盖刷新、圈层、开箱、过期、掉落、背包接口。

请优先做可运行的工程内调试工具，不要只写静态文档或伪代码。
```

## 8. 验收标准

- Debug 面板可显示实时容器状态。
- 可以强制生成指定类型容器。
- 可以显示刷新点合法性。
- 可以复现同 seed 掉落。
- 日志能定位刷新失败原因。
- 验证清单中的 V0.1 项目可逐条人工测试。

## 9. 暂不实现

- 自动化压力测试工具。
- 线上遥测后台。
- 正式玩家可见的掉落预测功能。
