# 12_调试工具与验证清单

> 对应母版章节：18、19
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“调试工具与验证清单”小节。

---

## 1. 实现目标

提供容器系统调试面板和验证清单，用于确认地图进入生成、地点状态衰减、点位合法性、掉落生成、拾取和负重接口是否正确。

---

## 2. 调试面板字段

| 字段 | 说明 |
|---|---|
| `map_id` | 当前地图 |
| `location_state` | 富集/普通/贫瘠/恢复中 |
| `target_container_count` | 本次目标容器数 |
| `actual_container_count` | 本次实际生成容器数 |
| `blocked_point_count` | 被安全区、不可达或类型限制阻挡的点位数 |
| `active_container_count` | 当前活跃容器数 |
| `selected_container_id` | 选中容器 |
| `selected_state` | 选中容器状态 |
| `selected_loot_seed` | 选中容器产物种子 |
| `selected_remaining_loot_count` | 选中容器剩余产物数 |
| `looted_ratio` | 本次地图已拿走资源比例 |

---

## 3. 调试按钮

| 按钮 | 行为 |
|---|---|
| Regenerate Map Containers | 按当前地图和地点状态重新生成本次容器清单 |
| Set Location Rich | 将地点状态切到富集 |
| Set Location Normal | 将地点状态切到普通 |
| Set Location Poor | 将地点状态切到贫瘠 |
| Set Location Recovering | 将地点状态切到恢复中 |
| Spawn Container Type | 在合法点生成指定容器类型 |
| Clear Map Instance Containers | 清空本次地图实例容器 |
| Show Spawn Points | 显示点位、房间/区域标签和阻挡原因 |
| Preview Loot | 仅调试模式预览选中容器产出 |
| Settle Map Loot | 模拟离开地图并结算地点状态 |

## 5. 验证清单

### 5.1 地图生成验证

- [ ] 进入地图时一次性生成本次容器清单。
- [ ] 容器数量符合地图基础数量和地点状态倍率。
- [ ] 诊所更容易出现医疗柜，工业区更容易出现工具柜，住宅更容易出现纸箱/木箱/小保险箱。
- [ ] 合法点不足时减少实际容器数量，并输出警告。
- [ ] 局内不会按计时器补刷普通容器。

### 5.2 地点状态验证

- [ ] 富集状态容器数量接近地图基础上限。
- [ ] 贫瘠状态容器数量明显下降，但仍保留少量基础资源。
- [ ] 连续进入同一地图并大量搜刮后，地点状态下降。
- [ ] 间隔数天或事件后，贫瘠地点可进入恢复中。
- [ ] 特殊资源不会靠自然恢复补回。

### 5.3 开箱验证

- [ ] 玩家靠近容器显示 F 键提示。
- [ ] 读条时间由容器类型决定，不由最终奖励决定。
- [ ] 读条中断后容器仍存在，可再次开启。
- [ ] 已开封且有残留的容器可二次免读条打开。
- [ ] 拿空后容器移除。

### 5.4 拾取验证

- [ ] 一键拾取优先堆叠材料。
- [ ] 背包不足时不会吞物品。
- [ ] 超重不阻止拾取，但会提示并减速。
- [ ] 关闭面板后有残留则本次进入内保留。
- [ ] 离开地图后残留物不进入局外仓库。

### 5.5 掉落验证

- [ ] 不同地图、房间和容器类型会生成不同语义小池。
- [ ] 地点状态会影响数量和品质倾向。
- [ ] 大容器不保底高品质。
- [ ] S 级只作为少量惊喜或高风险事件产出。
