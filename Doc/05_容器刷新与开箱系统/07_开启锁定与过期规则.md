# 07_开启锁定与残留规则

> 对应母版章节：8.3、11.4、11.5
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“开启锁定与残留规则”小节。

---

## 1. 实现目标

明确容器开启锁定、读条中断、拾取面板和残留容器之间的关系。普通容器没有过期倒计时，旧的“到时消失”规则废弃。

---

## 2. 设计依据

核心规则：

```text
玩家开始读条后，容器进入 OPENING。
读条中断后，容器回到 AVAILABLE。
读条完成后，容器进入 OPENED/LOOTING。
拾取面板关闭时，如果仍有物品，容器进入 OPENED_AVAILABLE。
拾取面板关闭时，如果已拿空，容器进入 EMPTIED/REMOVED。
离开地图时，未拾取的残留物按放弃处理，不进入局外仓库。
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 开启锁定 | `OPENING` 时该容器不能被重复交互或调试生成覆盖 |
| 读条中断 | 玩家移动、离开触发器、松开长按或稳定值归零时中断 |
| 中断结果 | 普通中断回到 `AVAILABLE`，可再次开启 |
| 面板打开 | 打开期间锁定玩家移动输入，稳定值继续变化 |
| 面板关闭 | 有剩余物则 `OPENED_AVAILABLE`，拿空则 `REMOVED` |
| 残留限制 | 不能向容器存入背包物品，不能跨地图实例保存残留 |
| 离图结算 | 离开地图时统计拿走比例，用于地点状态衰减 |

---

| 路径 | 职责 |
|---|---|
| `res://scripts/containers/ContainerEntity.gd` | 锁定和残留状态 |

| `res://scripts/ui/LootPanelUI.gd` | 面板关闭时回写剩余物 |

---

## 5. 数据结构

字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `is_open_locked` | `bool` | 是否处于开启锁定 |
| `has_been_opened` | `bool` | 是否已完成过首次读条 |
| `remaining_loot_entries` | `Array` | 剩余产物 |
| `looted_entry_count` | `int` | 本容器被拿走的条目数量 |
| `initial_loot_entry_count` | `int` | 本容器初始条目数量 |

---

## 6. 核心流程

```text
AVAILABLE
→ 玩家开始读条
→ OPENING + is_open_locked=true
→ 读条中断：is_open_locked=false，回到 AVAILABLE
→ 读条完成：has_been_opened=true，进入 LOOTING
→ 面板关闭：
   → remaining_loot_entries 为空 -> EMPTIED/REMOVED
   → remaining_loot_entries 非空 -> OPENED_AVAILABLE
→ 离开地图：残留物放弃，统计 looted_ratio
```

| A6 | 离开地图后残留物不进入仓库 |
