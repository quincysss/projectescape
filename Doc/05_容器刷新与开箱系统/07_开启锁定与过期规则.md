# 07_开启锁定与过期规则

> 对应母版章节：7.3、8.3、11.5、11.6  
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“开启锁定与过期规则”小节。

---

## 1. 实现目标

明确容器倒计时、开启锁定、读条中断、拾取面板和点位冷却之间的关系，防止容器在读条中突然消失，也防止玩家把残留容器当仓库。

---

## 2. 设计依据

开启锁定：

```text
容器倒计时归零不会强行打断正在进行且未被玩家主动中断的读条。
但只要该读条被中断，倒计时已归零的容器就立即消失。
```

界面倒计时：

```text
玩家开始与容器交互后，该容器倒计时暂停。
容器界面打开并选择道具期间，该容器倒计时继续暂停。
界面关闭后，如果容器仍未清空，倒计时从暂停值继续流逝。
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 锁定规则 | OPENING 时点位进入 OPEN_LOCKED，不被刷新覆盖 |
| 倒计时 | OPENING/LOOTING 时 remaining_lifetime 暂停，关闭界面后继续 |
| 读条完成 | 即使剩余时间 <=0，只要未中断，仍可完成 |
| 读条中断 | 如果 remaining_lifetime <=0，立即 EXPIRED/REMOVED |
| 面板打开 | 倒计时归零不强制关窗 |
| 面板关闭 | 归零或拿空后移除并释放点位 |
| 仓库限制 | 玩家不能把背包物品放回容器 |

## 5. 数据结构

字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `is_open_locked` | `bool` | 开启锁定 |
| `expired_while_opening` | `bool` | 读条中倒计时归零 |
| `expired_while_looting` | `bool` | 面板中倒计时归零 |
| `spawn_point_cooldown` | `float` | 5-10 秒 |

---

## 6. 核心流程

```text
OPENING 中 remaining_lifetime <=0
→ expired_while_opening = true
→ 读条继续

open_cancelled
→ 如果 expired_while_opening，remove()
→ 否则回 AVAILABLE

LOOTING 中 remaining_lifetime <=0
→ expired_while_looting = true
→ 面板保持

loot_panel_closed
→ 如果 loot empty，remove()
→ 如果 expired_while_looting，remove()
→ 否则 OPENED_AVAILABLE
```

| 编号 | 标准 |

| A5 | 面板中归零不强制关窗 |
| A6 | 关闭面板后过期容器移除 |
| A7 | 容器不能当仓库存入物品 |
