# 05-11 安全区、负重与多人预留边界

> 来源文档：`05_容器刷新与开箱系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取源文档的“安全区规则”“重量与资源循环限制”“多人预留接口”小节。

## 1. 实现目标

明确容器系统与安全区、背包负重、未来多人拾取之间的边界，避免容器刷新破坏安全据点、撤离点和核心资源循环。

V0.1 目标不是实现完整多人同步，而是先把接口和状态归属留好，避免后续重构容器生命周期。

## 2. 安全区规则细化

容器不得生成在以下区域内：

| 区域 | 规则 |
| --- | --- |
| 玩家家园安全区 | 禁止刷新任何容器 |
| 已修复前哨站安全区 | 禁止刷新任何容器 |
| 前哨站修复交互点 | 禁止刷新容器遮挡交互 |
| 撤离点 | 禁止刷新容器 |
| 高价值容器专属区 | 不得靠近家门或极低风险路线 |

安全区应由地图系统提供统一查询接口：

```gdscript
func is_position_container_blocked(world_position: Vector2) -> bool
func get_container_block_reason(world_position: Vector2) -> String
```

容器刷新管理器不得自己维护一套安全区副本。

## 3. 家园附近低价值容器规则

家园附近允许出现低价值容器，但必须满足：

- 只允许 `PAPER_BOX` 或低权重 `WOODEN_CRATE`。
- 与家园出口有最短移动距离限制，建议 5-12 秒路程。
- 不得出现在出生点视野正中心。
- 不得形成“站在家门口等箱子”的收益循环。

推荐实现：

```gdscript
const HOME_NEAR_MIN_TRAVEL_SECONDS := 5.0
const HOME_NEAR_MAX_TRAVEL_SECONDS := 12.0
const HOME_NEAR_ALLOWED_TYPES := ["PAPER_BOX", "WOODEN_CRATE"]
```

实际距离换算由移动系统速度决定，V0.1 可以先用地图距离近似。

## 4. 负重接口规则

容器系统不直接决定玩家是否能拾取，而是请求背包系统处理。

```gdscript
func try_add_loot_item(item_stack: Dictionary) -> Dictionary
```

返回示例：

```gdscript
{
    "accepted_count": 2,
    "rejected_count": 1,
    "became_overweight": true,
    "reason": "capacity_full"
}
```

拾取规则：

- 超重不阻止拾取。
- 超重只触发移动速度惩罚或 UI 警告。
- 空格不足时，未放入的物品必须保留在容器中。
- 堆叠物品优先合并已有堆叠。
- 一键拾取推荐按“高品质优先、可堆叠优先”排序。

## 5. 核心资源循环限制

容器可以提供补给和探索奖励，但不能成为修复前哨站的唯一来源。

禁止设计：

- 关键修复材料只从保险箱掉落。
- 玩家必须刷容器才能推进主循环。
- 高价值箱子稳定产出核心修复资源。

推荐做法：

- 修复材料主来源来自地图节点、任务、拆解或据点事件。
- 容器只提供少量替代来源或随机补偿。
- Debug 面板显示修复材料来源分布，用于检查偏斜。

## 6. 多人预留状态

V0.1 可以单机实现，但容器状态需要预留多人归属字段。

```gdscript
var opened_by_player_id: String = ""
var locked_by_player_id: String = ""
var loot_seed: int = 0
var generated_loot: Array[Dictionary] = []
var revision: int = 0
```

预留规则：

- `locked_by_player_id` 用于未来防止多人同时开箱。
- `opened_by_player_id` 记录首次开启者。
- `generated_loot` 必须由服务器权威或主机权威生成。
- `revision` 用于未来 UI 刷新和冲突检测。

V0.1 单机可以把 player_id 固定为 `local_player`。

## 7. 信号与接口

容器向外发出：

```gdscript
signal container_spawn_rejected(point_id: String, reason: String)
signal loot_pickup_failed(container_id: String, item_id: String, reason: String)
signal container_claimed(container_id: String, player_id: String)
```

需要调用：

```gdscript
SafeZoneService.is_position_container_blocked(position)
InventoryService.try_add_loot_item(item_stack)
PlayerStateService.apply_overweight_warning()
```

```text

要求：

3. 家园附近只允许低价值容器，并保留最短移动距离限制。

```

## 9. 验收标准

- 安全区内不会刷新容器。
- 家园附近只出现低价值容器。
- 背包满时物品不会消失。
- 超重可以拾取并产生提示。
- 容器状态包含多人预留字段。
- 容器系统不直接修改安全区数据。

## 10. 暂不实现

- 服务器权威同步。
- 多人抢箱结算。
- 拾取冲突仲裁。
- 掉落归属保护时间。
