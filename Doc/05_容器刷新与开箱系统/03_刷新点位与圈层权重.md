# 03_地图容器点位与区域权重

> 对应母版章节：6、7.4、13、16.3
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“地图容器点位与区域权重”小节。

---

## 1. 实现目标

建立地图容器点位与区域权重：容器只能生成在当前地图内合法、空闲、可达、不在安全区的点位上；不同地图和房间/区域有不同容器类型权重。

---

## 2. 设计依据

地图/区域与容器：

| 地图 | 房间/区域 | 主要容器 |
|---|---|---|
| 小诊所 | 前台/走廊 | 纸箱、木箱 |
| 小诊所 | 药房/病房 | 医疗柜、小保险箱 |
| 废弃工业区 | 维修间/仓库 | 木箱、工具柜 |
| 废弃工业区 | 管理室 | 工具柜、小保险箱、大保险柜 |
| 住宅楼 | 客厅/储物间 | 纸箱、木箱 |
| 住宅楼 | 卧室/办公室 | 小保险箱、木箱 |

类型权重示例：

| 地图/区域 | 纸箱 | 木箱 | 工具柜 | 医疗柜 | 小保险箱 | 大保险柜 | 异常箱 |
|---|---:|---:|---:|---:|---:|---:|---:|
| 诊所-药房 | 10 | 10 | 0 | 60 | 20 | 0 | 0 |
| 工业区-维修间 | 0 | 30 | 55 | 0 | 10 | 5 | 0 |
| 住宅-储物间 | 40 | 45 | 5 | 0 | 10 | 0 | 0 |
| 高风险事件点 | 0 | 0 | 15 | 10 | 20 | 40 | 15 |

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 点位条件 | 当前地图、当前房间/区域、空闲、可达、不在安全区、不重叠前哨修复点 |
| 占用状态 | 空闲、已生成容器、开启锁定 |
| 无冷却 | 点位没有局内冷却；下次进入地图时重新计算本次实例 |
| 权重规则 | 地图/区域权重只决定容器类型倾向，不直接决定奖励品质 |
| 失败规则 | 合法点不足时减少本次容器数量，不强行刷非法点 |
| 反馈规则 | 调试显示各地图/区域空闲、占用、阻挡点位数量 |

## 5. 数据结构

```gdscript
enum SpawnPointState {
    FREE,
    OCCUPIED,
    OPEN_LOCKED,
    BLOCKED
}
```

`ContainerSpawnPoint` 字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `spawn_point_id` | `StringName` | 点位 ID |
| `map_id` | `StringName` | 所属地图 |
| `room_tag` | `StringName` | 房间/区域标签 |
| `risk_zone` | `StringName` | 风险区标签 |
| `allowed_container_types` | `Array[StringName]` | 合法类型 |
| `state` | `SpawnPointState` | 当前状态 |
| `is_reachable` | `bool` | 验证结果 |
| `is_inside_safe_area` | `bool` | 必须 false |

---

## 6. 核心流程

```text
1. 从当前地图获取容器点位。
2. 验证点位合法性。
3. 进入地图时按地点状态确定容器数量。
4. 按房间/区域权重选择 FREE 点位。
5. 生成容器后点位进入 OCCUPIED。
6. 开启中进入 OPEN_LOCKED。
7. 容器拿空移除后点位在本次地图实例中释放。
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 点位有 map_id、room_tag 和状态 |
| A2 | 安全区内点位不合法 |
| A3 | 占用点不会重复刷 |
| A4 | 拿空移除后本次实例可释放点位 |
| A5 | 地图/区域权重可读取 |
| A6 | 合法点不足能报告 |
