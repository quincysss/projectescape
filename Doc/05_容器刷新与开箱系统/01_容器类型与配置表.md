# 01_容器类型与配置表

> 对应母版章节：5、9.3、10.2、16、17  
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“容器类型与配置表”小节。

---

## 1. 实现目标

建立容器系统的配置源：容器类型、存在时间、开启时间范围、交互半径、产出格数、外观/信号倾向。后续刷新、交互、掉落、UI 都从配置读取，不在逻辑里硬编码。

---

## 2. 设计依据

| 容器类型 | 常见区域 | 存在时间 | 开启时间 | 交互半径 | 定位 |
|---|---|---:|---:|---:|---|
| 纸箱 | 内层/中层 | 60秒 | 0-1.0秒 | 1.6 | 教学、基础材料 |
| 木箱 | 内层/中层 | 60秒 | 0.8-1.6秒 | 1.8 | 普通材料 |
| 工具柜 | 中层/外层 | 45秒 | 1.2-2.4秒 | 2.0 | 工具、制作相关 |
| 医疗柜 | 中层/外层 | 45秒 | 1.0-2.2秒 | 2.0 | 回复物品 |
| 小保险箱 | 外层 | 40秒 | 2.4-3.6秒 | 1.8 | 中高价值期待 |
| 大保险柜 | 外层/最外层 | 30秒 | 3.6-4.8秒 | 2.4 | 高风险高期待 |
| 异常箱 | 最外层 | 30秒 | 3.6-4.8秒 | 2.2 | 特殊收益 |

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 配置规则 | 容器类型、时间、半径、产出格必须可配置 |
| 开启时间 | 由容器类型/锁具难度决定，不由最终奖励决定 |
| 价值规则 | 大保险柜提高期待，但不保底高品质 |
| 失败规则 | 配置缺失时该容器不能生成，必须报告错误 |
| 反馈规则 | 调试面板显示容器配置摘要 |
| 边界规则 | V0.1 至少支持纸箱、木箱、工具柜、保险箱 |

## 4. Godot 落地范围

| 路径 | 职责 |
|---|---|
| `res://resources/containers/ContainerConfig.tres` | 容器配置 |
| `res://scripts/resources/ContainerConfig.gd` | Resource 定义与验证 |
| `res://scripts/resources/ContainerTypeDef.gd` | 单个类型定义，可选 |

---

## 5. 数据结构

`ContainerTypeDef` 字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `type_id` | `StringName` | `cardboard_box` 等 |
| `display_name` | `String` | 展示名 |
| `lifetime` | `float` | 存在时间 |
| `open_time_min` | `float` | 最短开启时间 |
| `open_time_max` | `float` | 最长开启时间 |
| `interaction_radius` | `float` | 交互触发半径 |
| `loot_slots_min` | `int` | 最少产出格 |
| `loot_slots_max` | `int` | 最多产出格 |
| `allowed_rings` | `Array[StringName]` | 合法圈层 |
| `signal_profile` | `StringName` | 信号表现倾向 |

---

## 6. 核心流程

```text
1. ContainerConfig 加载。
2. validate_config() 检查每个容器类型。
3. SpawnManager 通过 type_id 读取 lifetime/open_time/radius。
4. ContainerEntity 初始化时复制必要运行时参数。
5. LootGenerator 读取 loot_slots 和类型权重。
```

---

## 7. 信号与接口

配置接口：

```text
get_container_type(type_id)
get_allowed_types_for_ring(ring)
get_random_open_time(type_id, rng)
validate_config() -> errors
```
