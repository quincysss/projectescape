> 来源文档：`../05_容器刷新与开箱系统_修订版_废土生存法则.md`  
> 参考规范：`../02_核心循环规则/00_拆分规范与落地文档写法.md`

## 0. 执行总则

本目录所有拆分文档必须遵守：

```text
Doc/00_落地执行文档总则.md
```

容器文档必须优先说明刷新点、管理器、实体生命周期、交互读条、掉落生成、背包接口和验证方式。

---

## 1. 拆分原则

05 的核心是“未知收益 + 停留风险 + 路线规划”。拆分文档必须围绕可执行系统边界，而不是复述玩法描述。

每份文档必须回答：

```text
谁生成容器？
容器在哪生成？
容器如何过期？
玩家如何开？
什么时候生成奖励？
奖励如何显示和拾取？
稳定值、负重、安全区如何影响它？
如何调试和验证？
```

## 4. 核心命名约定

容器状态：

```gdscript
enum ContainerState {
    SPAWNING,
    AVAILABLE,
    OPENING,
    OPENED,
    LOOTING,
    OPENED_AVAILABLE,
    EXPIRED,
    REMOVED
}
```

圈层：

```gdscript
enum ContainerRing {
    INNER,
    MIDDLE,
    OUTER,
    FAR_OUTER
}
```

推荐 ID：

```text
container_id
container_type
spawn_point_id
run_seed
loot_seed
opened_by_player_id
locked_by_player_id
```

## 5. AI/MCP 提示词底线

每份提示词必须包含：

```text
任务目标
必须读取的文档
允许新增/修改范围
禁止实现范围
完成标准
验证方式
最终报告格式
```

提示词必须避免：

```text
把开箱做好
优化容器系统
按文档实现
做一个掉落系统
```

合格提示必须明确到组件、字段、验证项。
