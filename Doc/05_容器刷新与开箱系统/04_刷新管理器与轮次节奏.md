# 04_刷新管理器与轮次节奏

> 对应母版章节：7、16.2  
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“刷新管理器与轮次节奏”小节。

---

## 1. 实现目标

实现 `ContainerSpawnManager`：管理刷新计时、每轮刷新数量、合法点位选择、容器实例创建、活跃容器列表和点位释放回调。

---

## 2. 设计依据

V0.1 基础刷新：

```text
每 10 - 15 秒触发一轮容器刷新。
每轮刷新 1 - 3 个容器。
刷新点必须来自容器点位池。
被占用点位不重复刷新。
合法点不足时减少实际刷新数量。
```

阶段建议：

| 单局阶段 | 每轮数量 | 目的 |
|---|---:|---|
| 开局前60秒 | 1-2 | 降低信息量 |
| 中期 | 1-3 | 常规诱惑 |
| 第二前哨修复后 | 2-3 | 贪婪空间 |

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 触发条件 | 新局初始化后启动刷新计时器 |
| 执行规则 | 计时器到达后随机本轮数量，逐个选择合法点位生成 |
| 点位规则 | 只选 FREE 且合法点位 |
| 数量规则 | 合法点不足时实际生成数小于目标数 |
| 种子规则 | 每局使用 run_seed 派生容器和 loot_seed |
| 失败规则 | 无合法点位时跳过本轮并输出警告 |
| 反馈规则 | 调试面板显示下一轮倒计时、活跃数量、各圈层空闲点 |

---

## 6. 核心流程

```text
start_spawn_loop(run_seed)
→ 计算 next_spawn_time
→ 计时归零
→ target_count = rng.randi_range(1, 3)
→ for target_count:
   → 选择圈层/类型
   → 获取合法空点位
   → 若无点位，continue
   → instantiate ContainerEntity
   → 初始化 type/lifetime/open_time/loot_seed/spawn_point_id
   → 点位 mark_occupied
   → active_containers 添加
→ 重置下一轮计时
```
