# 04_地图进入容器生成管理器

> 对应母版章节：7、16.2  
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“地图进入容器生成管理器”小节。

---

## 1. 实现目标

实现 `ContainerSpawnManager`：在玩家进入某张地图时，一次性生成本次地图实例的容器清单，并在离开地图时根据搜刮比例回写地点资源状态。

---

## 2. 设计依据

基础规则：

```text
普通容器不在局内轮询补刷。
局内不会因为倒计时新增普通容器。
进入地图时，根据地图配置、地点状态和点位池生成本次容器。
容器不会因为时间流逝自然消失。
容器只会在被拾取清空后从本次地图实例中移除。
连续进入同一地图时，资源量会随地点状态逐渐下降。
```

地点状态数量倍率：

| 状态 | 数量倍率 | 说明 |
|---|---:|---|
| 富集 | 100% | 首次进入、事件补给或久未搜刮 |
| 普通 | 60%-80% | 常规搜刮后 |
| 贫瘠 | 10%-30% | 连续搜刮后 |
| 恢复中 | 50%-70% | 间隔数天或事件后部分恢复 |

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 触发条件 | 玩家进入地图实例时生成容器清单 |
| 地图规则 | 读取 `map_resource_profile`、房间/区域权重和当前地点状态 |
| 数量规则 | 基础数量区间乘以地点状态倍率 |
| 点位规则 | 只选当前地图、当前房间/区域、可达、未占用、允许该容器类型的点位 |
| 种子规则 | 每次进入地图使用 `run_seed + map_id + visit_index` 派生 `container_seed` 和 `loot_seed` |
| 失败规则 | 合法点不足时减少本次实际容器数量，并输出配置警告 |
| 结算规则 | 离开地图时按拿走资源比例降低或保持地点状态 |
| 反馈规则 | 调试面板显示地图 ID、地点状态、目标数量、实际数量、跳过原因 |

## 5. 数据结构

字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `map_id` | `StringName` | 当前地图 |
| `location_state` | `StringName` | 富集/普通/贫瘠/恢复中 |
| `base_count_min` | `int` | 地图基础容器下限 |
| `base_count_max` | `int` | 地图基础容器上限 |
| `state_multiplier` | `float` | 地点状态数量倍率 |
| `active_containers` | `Dictionary` | id -> ContainerEntity |
| `run_seed` | `int` | 本局种子 |
| `visit_index` | `int` | 该地图本轮进入次数 |

---

## 6. 核心流程

```text
enter_map(map_id, run_seed, visit_index)
→ 读取 map_resource_profile
→ 读取地点状态
→ target_count = random(base_min, base_max) * state_multiplier
→ 从当前地图点位池按房间/区域权重选择点位
→ 为每个点位选择合法容器类型
→ instantiate ContainerEntity
→ 初始化 type/open_time/loot_seed/spawn_point_id/map_id
→ active_containers 添加
→ 玩家离开地图时统计 looted_ratio
→ 根据 looted_ratio 回写地点状态
```


## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 进入地图时生成本次容器清单 |
| A2 | 容器数量受地点状态倍率影响 |
| A3 | 容器只生成在当前地图合法点位 |
| A4 | 局内不会按计时器补刷普通容器 |
| A5 | 离开地图时按搜刮比例回写地点状态 |
| A6 | 调试面板可切换地点状态并重新生成 |
