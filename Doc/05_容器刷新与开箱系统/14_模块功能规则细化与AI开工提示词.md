## 1. 模块定位

容器系统负责把地图探索转化为可规划但不完全确定的短周期收益：

```text
玩家选择地图
→ 地图资源生态给出方向
→ 进入地图时生成本次容器清单
→ 玩家根据房间和容器类型判断收益倾向
→ 开箱读条制造停留风险
→ 打开后揭示具体产物
→ 拾取进入背包并影响负重
→ 离开地图时结算地点状态衰减
```

明确废弃：

```text
局内轮询生成普通容器
按轮次补刷普通容器
普通容器限时存在
普通容器到时移除
点位在局内反复回收再用
跨地图实例保存容器残留
```

## 3. 核心规则快照

| 项 | 规则 |
|---|---|
| 生成时机 | 玩家进入地图实例时一次性生成 |
| 数量来源 | 地图基础数量区间 × 地点状态倍率 |
| 地点状态 | 富集 / 普通 / 贫瘠 / 恢复中 |
| 点位来源 | 当前地图预设点位池 |
| 类型倾向 | 地图、房间/区域、点位语义共同决定 |
| 产物生成 | 打开容器时按 `loot_seed` 生成 |
| 读条时间 | 由容器类型/锁具/装备修正决定，不由最终奖励决定 |
| 容器消失 | 只因拿空、离开地图实例或调试清空 |
| 残留 | 只在本次进入地图期间保留 |
| 离图结算 | 按拿走资源比例衰减地点状态 |

地点状态数量倍率建议：

| 状态 | 数量倍率 |
|---|---:|
| 富集 | 100% |
| 普通 | 60%-80% |
| 贫瘠 | 10%-30% |
| 恢复中 | 50%-70% |

## 5. AI 开工提示词

### 5.1 地图进入容器生成

```text
请实现 05 容器系统的“地图进入容器生成”。
读取 03_刷新点位与圈层权重.md、04_刷新管理器与轮次节奏.md、08_战利品生成器.md、12_调试工具与验证清单.md，以及 Doc/34_地图资源生态与地点再生规则.md。
进入地图时一次性生成本次容器清单；容器数量由地图基础数量和地点状态倍率决定；容器类型由地图、房间/区域和点位语义决定。
不要实现旧的局内轮询补刷、普通容器限时移除或点位反复回收。
```

### 5.2 容器状态与开箱

```text
请实现 05 容器系统的“容器状态与开箱”。
读取 05_容器实体生命周期.md、06_开箱交互与读条.md、07_开启锁定与过期规则.md、09_拾取面板与背包负重接口.md。
实现 SPAWNING/AVAILABLE/OPENING/OPENED/LOOTING/OPENED_AVAILABLE/EMPTIED/REMOVED。
读条中断后容器回到 AVAILABLE；打开后有残留可二次免读条查看；拿空后移除；离开地图后残留按放弃处理。
```

### 5.3 掉落生成

```text
请实现 05 容器系统的“战利品生成器”。
读取 08_战利品生成器.md、Doc/31_掉落物资与制造经济规划.md、Doc/34_地图资源生态与地点再生规则.md。
按地图大池、房间/区域中池、容器类型小池和地点状态修正生成 LootEntry。
资源按大类和 C/B/A/S 品质权重抽取，避免回到全局大随机池。
```

### 5.4 调试验证

```text
请实现 05 容器系统 Debug 面板。
显示地图 ID、地点状态、目标容器数、实际容器数、点位阻挡原因、选中容器状态和 loot_seed。
提供切换富集/普通/贫瘠/恢复中、重新生成本次地图容器、显示点位、预览掉落和模拟离图结算按钮。
不要提供旧补刷计时、限时移除或点位回收相关调试项。
```

## 6. 验收底线

```text
同一地图富集状态明显比贫瘠状态容器更多。
诊所、工业区、住宅的容器类型倾向不同。
局内等待不会新增普通容器。
容器不会因为时间流逝自然消失。
未拿空容器在本次进入内可二次打开。
离开地图后残留物不进仓库。
连续大量搜刮同一地图会降低地点状态。
```
