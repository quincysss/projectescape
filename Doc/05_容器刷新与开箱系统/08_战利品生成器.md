# 08_战利品生成器

> 对应母版章节：10、16.6、17  
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“战利品生成器”小节。

---

## 1. 实现目标

实现 `LootGenerator`：容器开启完成时，根据地图资源生态、房间/区域语义、容器类型、地点状态、种子和 C/B/A/S 品质权重生成标准 `LootEntry` 列表。容器生成时不向玩家暴露具体产物。

---

## 2. 设计依据

产出时机：

```text
容器生成时只确定容器类型和种子。
容器开启完成时，根据种子生成产物。
```

产出格数：

| 容器 | 产出格数 |
|---|---:|
| 纸箱 | 1-2 |
| 木箱 | 1-3 |
| 工具柜 | 1-3 |
| 医疗柜 | 1-2 |
| 小保险箱 | 1-3 |
| 大保险柜 | 2-4 |
| 异常箱 | 1-3 |

地图风险品质基础权重：

| 风险层 | C | B | A | S |
|---|---:|---:|---:|---:|
| 内层 | 85 | 14 | 1 | 0 |
| 中层 | 65 | 30 | 5 | 0 |
| 外层 | 35 | 45 | 18 | 2 |
| 最外层 | 20 | 40 | 32 | 8 |

语义掉落池：

```text
地图类型决定大池。
房间/区域类型决定中池。
容器类型决定小池。
品质只在小池内部修正价值和稀有度。
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 生成时机 | 读条完成时生成 loot，未打开前不得暴露 |
| 随机规则 | 使用 loot_seed 可复现结果 |
| 语义规则 | 地图大池 + 房间中池 + 容器小池 |
| 品质规则 | 地图风险基础权重 + 容器类型修正 + 地点状态修正 |
| 类型规则 | 先确定资源类别小池，再按品质和权重抽具体物品 |
| 主流程规则 | 核心前哨修复材料不依赖容器产出 |
| 失败规则 | 掉落表缺失时返回空列表并报错，不崩溃 |

## 5. 数据结构

`LootEntry` 字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `item_id` | `StringName` | 物品 ID |
| `item_type` | `StringName` | material/consumable/equipment/rare/sale_good |
| `quality` | `StringName` | C/B/A/S |
| `quantity` | `int` | 数量 |
| `weight` | `float` | 总重量或单位重量 |
| `stackable` | `bool` | 是否堆叠 |
| `source_container_id` | `StringName` | 来源容器 |

---

## 6. 核心流程

```text
generate_loot(container_entity)
→ rng = Random(loot_seed)
→ slots = random between loot_slots_min/max
→ resource_pool = resolve_pool(map_profile, room_profile, container_type, location_state)
→ for each slot:
   → category = roll_category(resource_pool)
   → quality = roll_quality(map_risk, container_type, location_state)
   → item = pick_item_from_pool(resource_pool, category, quality)
   → quantity = roll_quantity(item)
   → create LootEntry
→ return entries
```

---

## 7. 信号与接口

接口：

```text
generate_loot(container_entity) -> Array[LootEntry]
preview_loot_debug(container_entity) -> Array[LootEntry]
```

注意：`preview_loot_debug` 只能调试模式使用。


## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 读条完成时才生成正式 loot |
| A2 | 同 seed 结果可复现 |
