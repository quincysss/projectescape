# 08_战利品生成器

> 对应母版章节：10、16.6、17  
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“战利品生成器”小节。

---

## 1. 实现目标

实现 `LootGenerator`：容器开启完成时，根据容器类型、圈层、种子、品质权重和物品类型权重生成标准 `LootEntry` 列表。容器生成时不向玩家暴露具体产物。

---

## 2. 设计依据

产出时机：

```text
容器生成时只确定容器类型和种子。
容器开启完成时，根据种子生成产物。
```

产出格数：

| 容器 | 产出格数 |
|---|---:|
| 纸箱 | 1-2 |
| 木箱 | 1-3 |
| 工具柜 | 1-3 |
| 医疗柜 | 1-2 |
| 小保险箱 | 1-3 |
| 大保险柜 | 2-4 |
| 异常箱 | 1-3 |

圈层品质基础权重：

| 圈层 | C | B | A | S |
|---|---:|---:|---:|---:|
| 内层 | 85 | 14 | 1 | 0 |
| 中层 | 65 | 30 | 5 | 0 |
| 外层 | 35 | 45 | 18 | 2 |
| 最外层 | 20 | 40 | 32 | 8 |

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 生成时机 | 读条完成时生成 loot，未打开前不得暴露 |
| 随机规则 | 使用 loot_seed 可复现结果 |
| 品质规则 | 圈层基础权重 + 容器类型修正 |
| 类型规则 | 先决定品质，再决定物品类型 |
| 主流程规则 | 核心前哨修复材料不依赖容器产出 |
| 图纸规则 | V0.1 可少量加入，主要在工具柜/保险柜/异常箱 |
| 失败规则 | 掉落表缺失时返回空列表并报错，不崩溃 |

## 5. 数据结构

`LootEntry` 字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `item_id` | `StringName` | 物品 ID |
| `item_type` | `StringName` | material/consumable/equipment/blueprint/rare |
| `quality` | `StringName` | C/B/A/S |
| `quantity` | `int` | 数量 |
| `weight` | `float` | 总重量或单位重量 |
| `stackable` | `bool` | 是否堆叠 |
| `source_container_id` | `StringName` | 来源容器 |

---

## 6. 核心流程

```text
generate_loot(container_type, ring, loot_seed)
→ rng = Random(loot_seed)
→ slots = random between loot_slots_min/max
→ for each slot:
   → quality = roll_quality(ring, container_type)
   → item_type = roll_item_type(container_type)
   → item = pick_item_from_table(quality, item_type)
   → quantity = roll_quantity(item)
   → create LootEntry
→ return entries
```

---

## 7. 信号与接口

接口：

```text
generate_loot(container_entity) -> Array[LootEntry]
preview_loot_debug(container_entity) -> Array[LootEntry]
```

注意：`preview_loot_debug` 只能调试模式使用。
