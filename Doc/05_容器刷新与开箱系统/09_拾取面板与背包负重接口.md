# 09_拾取面板与背包负重接口

> 对应母版章节：11、14、16.7  
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“拾取面板与背包负重接口”小节。

---

## 1. 实现目标

实现轻量 `LootPanelUI` 的规则：显示容器产物、背包容量、负重，一键拾取、单项拾取、关闭/放弃。UI 不直接修改背包，而是通过背包/命令接口请求拾取。

---

## 2. 设计依据

面板显示：

```text
容器名称
产物列表
每个物品的品质、数量、重量
当前背包容量
当前负重/负重上限
一键拾取
关闭/放弃
```

打开面板期间：

```text
锁定玩家移动输入。
稳定值继续按照当前区域规则变化。
容器倒计时继续流逝。
```

负重规则：

```text
超重不阻止拾取。
显示负重过高提示。
移动速度惩罚立即生效。
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 一键拾取 | 先堆叠，再按品质高到低尝试新物品 |
| 单项拾取 | 只拾取指定项，失败提示原因 |
| 背包空间 | 空间不足不吞物品，未拾取项保留在容器 |
| 负重 | 超重允许拾取，但提示并应用惩罚 |
| 关闭 | 有剩余物则容器保留到倒计时结束 |
| 残留限制 | 不能把背包物品放回容器 |
| 强制关闭 | 玩家离开触发器或死亡时关闭面板 |

## 5. 数据结构

面板字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `container_id` | `StringName` | 当前容器 |
| `loot_entries` | `Array` | 剩余产物 |
| `is_open` | `bool` | 面板状态 |
| `is_player_locked` | `bool` | 是否锁移动 |
| `was_forced_closed` | `bool` | 是否强制关闭 |

背包接口期望：

```text
can_stack(entry)
can_add_new_entry(entry)
add_loot_entry(entry) -> result
get_capacity_summary()
get_weight_summary()
```

---

## 6. 核心流程

打开：

```text
open_panel(container)
→ lock_player_movement()
→ 显示 loot_entries 和背包/负重
→ 容器状态 LOOTING
```

一键拾取：

```text
先尝试堆叠
→ 剩余按品质高到低
→ 成功项从容器 loot_entries 移除
→ 失败项保留并显示原因
```

关闭：

```text
unlock_player_movement()
→ 如果 loot empty，container.remove()
→ 如果 loot remains，container.close_loot_panel()
```

## 7. 信号与接口

```gdscript
signal loot_pickup_succeeded(container_id: StringName, loot_entry)
```
