# 02_容器信息层级与信号

> 对应母版章节：4、12.1、13  
> 施工补充：开工前必须读取 `14_模块功能规则细化与AI开工提示词.md` 的“容器信息层级与信号”小节。

---

## 1. 实现目标

定义容器信息从“家中机会信号”到“靠近识别”再到“打开揭示”的三层显示规则。玩家能规划路线，但不能提前知道具体奖励。

---

## 2. 设计依据

家中观察层可显示：

```text
容器大致位置
信号强弱
容器轮廓倾向
是否有异常预兆
当前地图地点状态
```

靠近识别层可显示：

```text
容器名称
容器类型
开启提示
开启预计时间
容器是否已开封
```

打开揭示层才显示：

```text
物品图标、名称、品质、数量、重量
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 家中 | 只能显示机会，不显示具体产物 |
| 靠近 | 显示容器类型和预计开启时间，但不显示奖励 |
| 打开 | 读条完成后才展示实际产物 |
| 异常 | 异常预兆只表示特别机会，不显示奖励 |
| 失败规则 | 家中或靠近时泄露具体奖励属于失败 |
| 反馈规则 | 调试模式可显示真实 loot_seed/预览，正式模式隐藏 |

## 4. Godot 落地范围

| 路径 | 职责 |
|---|---|
| `res://scripts/containers/ContainerSignalView.gd` | 容器信号表现 |
| `res://scripts/containers/ContainerInfoPresenter.gd` | 信息层级转换 |
| `res://scripts/ui/ContainerPromptUI.gd` | 靠近提示 |

---

## 5. 数据结构

```gdscript
enum ContainerInfoLayer {
    HOME_SIGNAL,
    NEAR_IDENTIFIED,
    OPENED_REVEALED,
    DEBUG_FULL
}
```

字段：

| 字段 | 说明 |
|---|---|
| `signal_strength` | 家中信号强弱 |
| `signal_profile` | 轮廓/类型倾向 |
| `coarse_time_hint` | 粗略紧迫感 |
| `identified_type` | 靠近后可见类型 |
| `revealed_loot` | 打开后才可见 |

---

## 6. 核心流程

```text
容器生成
→ 注册 HOME_SIGNAL
→ 玩家在家中看到粗略信号
→ 玩家离家靠近容器
→ 切 NEAR_IDENTIFIED，显示 F 提示和开启时间
→ 读条完成
→ 切 OPENED_REVEALED，打开 LootPanel
```

---

## 7. 信号与接口

监听：

```text
CameraMode/HOME_OVERVIEW
Player entered ContainerInteractionArea
ContainerEntity.open_completed
```

发出：

```gdscript
signal container_info_layer_changed(container_id: StringName, layer: int)
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 家中看不到具体奖励 |
| A2 | 靠近前不能确认准确容器名称 |
| A3 | 靠近后可显示容器类型和开箱时间 |
| A4 | 打开后才显示产物 |
| A5 | 调试模式可显示隐藏信息 |
| A6 | 正式模式不泄露 loot_seed |
