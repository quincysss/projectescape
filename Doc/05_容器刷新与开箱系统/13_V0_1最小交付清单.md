## 1. V0.1 交付目标

V0.1 的目标是完成一个可玩的容器刷新与开箱闭环：

```text
地图刷新点 -> 按圈层刷新容器 -> 玩家靠近长按开箱 -> 生成战利品 -> 拾取进背包 -> 容器过期清理
```

不追求完整平衡，但必须可调试、可复现、可扩展。

## 2. 必须交付内容

| 模块 | 必须完成 |
| --- | --- |
| 容器类型 | 7 类基础容器配置 |
| 刷新点 | 圈层、点位合法性、占用、冷却 |
| 刷新节奏 | 10-15 秒一轮，每轮 1-3 个 |
| 生命周期 | SPAWNING 到 REMOVED 的状态流 |
| 开箱交互 | 靠近、长按、读条、中断、完成 |
| 交互描边 | 容器进入交互半径后显示 Shader 自适应描边 |
| 过期规则 | 倒计时、开启中到时、剩余物保留 |
| 战利品 | 按容器类型、圈层、品质权重生成 |
| 拾取面板 | 单个拾取、一键拾取、容量处理 |
| 安全区 | 禁止在家园/前哨站/撤离点刷新 |
| Debug | 强制刷新、清空、显示点位、预览掉落 |

## 3. 可延后内容

V0.1 不做：

- 多人同步。
- 容器被破坏。
- 开锁小游戏。
- 稀有演出动画。
- 玩家存放物品回容器。
- 复杂动态刷新导演。
- 服务器权威掉落。

## 7. 关键验收场景

### 7.1 普通探索

1. 玩家离开家园。
2. 中层刷新一个木箱或工具柜。
3. 玩家靠近并长按打开。
4. 战利品面板出现 1-3 个物品。
5. 玩家拾取，背包增加物品。
6. 容器在倒计时结束后清理。

### 7.2 中断与过期

1. 玩家开始开箱。
2. 容器倒计时继续减少。
3. 玩家松开按键中断。
4. 如果容器已过期，容器立即消失。
5. 如果未过期，可重新开始读条。

### 7.3 背包空间不足

1. 背包接近满容量。
2. 玩家打开容器。
3. 一键拾取部分物品。
4. 可放入的物品进入背包。
5. 放不下的物品保留在容器面板。

### 7.4 安全区过滤

1. 在家园安全区放置刷新点。
2. 执行强制刷新。
3. 该点被拒绝。
4. Debug 显示拒绝原因。

## 9. V0.1 完成定义

- 玩家可以在地图上看到容器。
- 容器会按时间和圈层刷新。
- 容器不会刷进安全区。
- 玩家可以长按打开容器。
- 玩家进入容器交互半径时能看到贴合容器轮廓的可交互描边。
- 打开后能看到战利品。
- 背包可以接收拾取物。
- 容量不足不会吞物品。
- 容器可以过期并清理。
- Debug 工具能定位刷新和掉落问题。
