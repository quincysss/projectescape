# 13_V0_1最小交付清单

> 对应母版章节：21
> 目标：定义 05 容器系统在 V0.1 必须交付的最小闭环。

---

## 1. V0.1 最小目标

```text
地图进入时生成容器清单
地图/房间/容器三层语义产出
地点状态：富集 / 普通 / 贫瘠 / 恢复中
容器开箱读条
轻量拾取面板
背包堆叠与负重接口
离图结算地点状态衰减
调试面板验证地图生成和地点状态
```

---

## 2. 必须交付

| 模块 | 最小内容 |
|---|---|
| 容器类型 | 纸箱、木箱、工具柜、医疗柜、保险箱 |
| 地图点位 | 至少支持诊所、工业区、住宅三类点位标签 |
| 生成管理 | 进入地图时按地点状态一次性生成容器 |
| 地点状态 | 富集、普通、贫瘠、恢复中影响容器数量 |
| 生命周期 | SPAWNING/AVAILABLE/OPENING/OPENED/LOOTING/OPENED_AVAILABLE/EMPTIED/REMOVED |
| 开箱 | 靠近显示 F，长按读条，移动/离开/稳定值归零中断 |
| 拾取 | 一键拾取、单项拾取、残留保留、拿空移除 |
| 战利品 | 按地图资源生态、房间/区域、容器类型和品质权重生成 |
| Debug | 切换地点状态、重新生成地图容器、显示点位、预览掉落 |

---

## 3. 明确不做

```text
局内轮询生成普通容器
普通容器限时存在
普通容器到时移除
点位在局内反复回收再用
容器跨地图实例保存残留
把容器当临时仓库存入玩家物品
```

---

## 4. 核心体验链路

```text
玩家选择地图
→ 地图显示主要产出类别和地点状态
→ 进入地图后生成本次容器清单
→ 玩家根据房间和容器类型判断收益方向
→ 长按开箱
→ 打开后看到具体物资
→ 拾取进入背包并影响负重
→ 离开地图时按搜刮比例结算地点状态
→ 连续进入同一地图时资源量下降
```

---

## 5. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 同一地图在富集状态生成数量明显高于贫瘠状态 |
| A2 | 诊所、工业区、住宅的容器类型倾向不同 |
| A3 | 局内等待不会新增普通容器 |
| A4 | 未拿空容器在本次进入内可二次打开 |
| A5 | 拿空容器会移除 |
| A6 | 离开地图后残留物不进仓库 |
| A7 | 连续大量搜刮同一地图会降低地点状态 |

---

## 6. AI/MCP 开工提示词

```text

重点是地图进入时容器生成、地点状态倍率、地图/房间/容器语义掉落、开箱读条和拾取负重。
不要实现旧的局内轮询补刷、普通容器限时移除、点位反复回收或跨地图残留仓库。
```
