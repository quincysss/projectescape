# 36_出门前目标面板与资源指引规则

> 文档版本：v0.1
> 所属项目：《废土生存法则》
> 关联文档：`29_局外杂货店经营与界面重构路线图.md`、`31_掉落物资与制造经济规划.md`、`33_资源类别与用途闭环规则.md`、`34_地图资源生态与地点再生规则.md`。
> 文档目标：定义玩家夜出前看到的目标、缺口、推荐地图和推荐容器，让局外经营目标能翻译成局内行动。

## 1. 设计目标

出门前目标面板解决的问题：

```text
我今天店里缺什么。
我当前能做哪些商品。
我还差什么资源。
哪些地图更可能产出这些资源。
去哪里更危险但收益更好。
```

它不是强制任务列表，而是把局外目标翻译成可执行探索计划。

## 2. 面板内容

第一版建议显示 4 块：

| 区块 | 内容 | 作用 |
|---|---|---|
| 今日需求 | 当日高热商品和顾客偏好类别 | 告诉玩家卖什么更赚 |
| 可制造缺口 | 当前库存还差哪些类别资源 | 告诉玩家找什么 |
| 研发/店铺目标 | 当前矿币目标和设备/章节前置 | 告诉玩家赚钱为了什么 |
| 推荐地点 | 2-3 个匹配地图和风险收益 | 告诉玩家去哪 |

## 3. 缺口生成

缺口优先基于“类别”而非具体材料。

示例：

```text
今日高热：医疗固定卷、净水滤芯
库存缺口：食物药品类 x3、基础材料 x2、零件类 x1
推荐地点：诊所、住宅区
推荐容器：医疗柜、储物箱、冰箱
```

当高级订单要求指定物时，可以显示具体来源提示：

```text
旧线圈常见来源：工业区、工具柜、电子箱
无菌材料包常见来源：诊所、手术室柜、上锁药柜
```

## 4. 推荐规则

推荐地图时计算：

```text
当前需求类别匹配度
地图状态
预计收益
风险等级
是否有未完成特殊区域
玩家当前装备和负重能力
```

推荐不应给唯一答案。第一版可以给 2-3 个选择：

```text
稳妥：住宅区，低风险，基础材料和食物
目标：诊所，中风险，医疗资源
赌博：检查站，高风险，武器装备和贵重物
```

## 5. 信息透明度

面板提供方向，不提供完整掉落答案。

允许显示：

```text
资源类别
常见来源
地图状态
风险收益等级
推荐容器类型
```

不显示：

```text
某个容器内的具体物品
精确掉落概率
必出承诺
完整最优路线
```

## 6. 与研究所关系

研发只消耗矿币，因此目标面板不再提示“研究缺材料”。

它应提示：

```text
当前想研究的项目还差多少矿币。
是否缺前置设备、章节、任务或任务条件。
哪些经营方式更适合快速赚钱。
```

示例：

```text
目标：背包改装 II
还差：矿币 160
前置：店铺等级 2 已满足
建议：今日医疗商品需求高，可优先搜诊所做医疗商品。
```

## 7. 数据来源

目标面板不单独维护一套设计数据，必须从现有系统汇总：

| 来源 | 用途 |
|---|---|
| 今日需求/顾客偏好 | 判断什么类别更值得卖 |
| 仓库库存 | 判断现有资源和缺口 |
| 可售商品配方 | 判断能做什么、还差什么类别 |
| research.tab | 判断目标研究的矿币差额和前置条件 |
| shop_fixtures.tab | 判断设备购买目标和店铺能力缺口 |
| map_resource_profiles.tab | 判断地图主要/次要产出 |
| location_state_rules.tab | 判断当前地点收益衰减 |
| 玩家背包/装备/负重 | 判断风险承受和可带回能力 |

## 8. 推荐算法口径

第一版推荐不追求复杂 AI，使用可解释评分即可。

建议评分：

```text
地图分 = 需求类别匹配分
      + 可制造缺口匹配分
      + 地点状态收益分
      + 特殊区域/任务分
      - 风险惩罚
      - 负重/装备不足惩罚
```

推荐结果应分成 2-3 个方向：

```text
稳妥：低风险，覆盖基础营业缺口。
目标：最高匹配当前商品/设备/订单缺口。
高风险：收益高但需要装备或承担损耗。
```

当研发目标缺矿币时，面板应推荐更适合赚钱的经营方向，而不是提示“去找研究材料”。

## 9. Godot 模块建议

| 模块 | 职责 |
|---|---|
| DepartureGoalPanelService | 汇总并输出目标面板 ViewModel |
| ResourceGapAnalyzer | 从库存和配方计算类别缺口 |
| ResearchGoalAnalyzer | 从 research.tab 计算矿币差额和前置条件 |
| FixtureGoalAnalyzer | 从设备槽位和商人表计算下一步店铺目标 |
| MapRecommendationService | 根据资源缺口、地图生态和地点状态生成推荐 |

目标面板只做“解释和推荐”，不直接改变库存、掉落和研究状态。

## 10. 校验规则

版本启用时必须检查：

```text
面板不能显示研究缺具体材料。
面板必须能在没有订单时仍给出基础营业缺口。
地图推荐必须来自 map_resource_profiles 和 location_state_rules。
推荐容器只显示容器类型，不承诺具体掉落。
至少给出 2 个方向，除非当前只解锁 1 张地图。
```

## 11. BDD 场景补充

Feature: 出门前目标面板

  Scenario: 玩家能看到当前资源缺口和推荐地图
    Given 今日需求榜偏向医疗商品
    And 玩家库存缺少食物药品类资源
    When 玩家进入夜间计划界面
    Then 目标面板应推荐诊所或其他医疗资源地图
    And 显示推荐容器类型为医疗柜或手术柜

  Scenario: 研发目标只提示矿币和前置条件
    Given 玩家追踪背包改装 II
    When 玩家查看目标面板
    Then 面板应显示所需矿币和前置条件
    And 不显示具体材料缺口

  Scenario: 推荐提供多个方向
    Given 玩家同时缺基础材料和医疗资源
    When 系统生成推荐地图
    Then 面板应至少提供一个稳妥选择和一个目标选择
