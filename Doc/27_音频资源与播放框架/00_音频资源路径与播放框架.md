# 00_音频资源路径与播放框架

> 所属系统：全局音频 / BGM / 局内交互音效  
> 当前阶段：V0.1 资源占位与播放框架  
> 目标：固定资源路径、事件 ID 和脚本接口，后续只需要把正式音频补到对应路径。

---

## 1. 落地边界

本阶段只做音频播放框架，不制作正式音频资源。

已落地：

- 固定 BGM 与音效目录。
- 固定资源命名规范。
- 新增 `data/audio/audio_manifest.json` 作为音频事件配置表。
- 新增全局 `AudioManager` 自动加载脚本。
- 接入局外安全屋、局内安全屋、局内探索、容器读条、容器打开完成、前哨站修复完成、撤离成功、死亡、濒临崩溃音效入口。

暂不落地：

- 音量设置 UI。
- 复杂混音器与动态 ducking。
- 耳机模块、歌单选择、信号提示联动。
- 正式音频资源导入。

## 5. Godot 模块

| 路径 | 职责 |
|---|---|
| `res://scripts/audio/audio_manager.gd` | 全局音频播放管理，读取 manifest，播放 BGM/音效 |
| `res://project.godot` | 自动加载 `AudioManager` |
| `res://scripts/base/base_scene.gd` | 局外安全屋 BGM 入口 |
| `res://scripts/run/run_scene.gd` | 局内 BGM 与关键音效入口 |

`AudioManager` 规则：

- 资源不存在时不报错、不阻塞流程，只发出 `audio_asset_missing` 信号。
- BGM 使用 `Music` bus。
- 音效使用 `SFX` bus。
- `Music` / `SFX` bus 如果不存在，由 `AudioManager` 在运行时创建。
- BGM 根据 manifest 的 `loop` 字段循环播放；即使 WAV 导入设置没有勾循环，脚本层也会兜底重播。
- 循环音效由 `start_loop_sfx` / `stop_loop_sfx` 管理。
- BGM 切换前必须先停止并清空旧 stream，避免旧 `AudioStreamPlayback` 在退出时继续持有资源。
- `AudioManager.shutdown()` 负责清理 BGM、循环音效和一次性音效播放器；自动化测试退出前使用 `shutdown_and_flush()` 等待音频线程释放播放对象，避免 headless 退出阶段误报资源泄露。

## 6. 当前触发规则

| 场景/事件 | 播放 |
|---|---|
| 进入局外基地 | `base_safe_house` |
| 进入局内，初始在家安全区 | `run_safe_house` |
| 离开局内家安全区 | `run_exploration` |
| 进入已修复前哨安全区 | `run_safe_house` |
| 离开已修复前哨安全区 | `run_exploration` |
| 开始容器读条 | `container_open_loop` |
| 容器读条取消 | 停止 `container_open_loop` |
| 容器读条完成 | 停止 `container_open_loop`，播放 `container_open_complete` |
| 前哨站修复完成 | `outpost_repair_complete` |
| 稳定值进入 CRITICAL 且未死亡 | `stability_critical_loop` |
| 稳定值脱离 CRITICAL 或死亡 | 停止 `stability_critical_loop` |
| 撤离成功 | `cue_extraction_success` |
| 玩家死亡 | `cue_player_death` |

## 7. 后续替换资源流程

1. 按本文件第 2 节把音频文件放入对应路径。
2. 保持文件名和后缀与 `audio_manifest.json` 一致。
3. 打开 Godot 让资源完成导入。
4. 运行局外/局内流程验证播放。

如果需要更换路径，必须同时修改：

```text
res://data/audio/audio_manifest.json
res://assets/audio/README.md
本文件
```
