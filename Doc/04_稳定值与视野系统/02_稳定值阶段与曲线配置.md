# 02_稳定值阶段与曲线配置

> 对应母版章节：5、6.3、15  
> 施工补充：开工前必须读取 `13_模块功能规则细化与AI开工提示词.md` 的“稳定值阶段与曲线配置”小节。

---

## 1. 实现目标

把稳定值从“连续数值”转成玩家可感知的阶段，并提供视野半径曲线配置。阶段变化负责驱动 UI、音频、遮罩和调试提示，但不直接绘制任何表现。

---

## 2. 设计依据

V0.1 推荐阶段：

| 稳定值范围 | 阶段名 | 内部枚举 | 设计作用 |
|---:|---|---|---|
| 100 - 60 | 稳定 | `STABLE` | 常规探索窗口 |
| 60 - 35 | 轻度失稳 | `UNSTEADY` | 提醒返程 |
| 35 - 15 | 危险 | `DANGER` | 明显压迫 |
| 15 - 0 | 濒临崩溃 | `CRITICAL` | 最后逃生段 |
| 0 | 耗尽 | `DEPLETED` | 死亡入口 |

推荐视野曲线：

| 稳定值 | 视野半径 |
|---:|---:|
| 100 | 20 |
| 80 | 19 |
| 60 | 18 |
| 45 | 16 |
| 35 | 14 |
| 25 | 11 |
| 15 | 8 |
| 5 | 4 |
| 0 | 0 |

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 触发条件 | `current_stability` 变化后重新计算阶段 |
| 执行规则 | 阶段按阈值判断，不使用全局线性视野压缩 |
| 执行规则 | 曲线配置用于视野目标半径，稳定值为 0 进入死亡，不走普通视野 |
| 失败规则 | 阈值重叠、断档、顺序错误时配置验证失败 |
| 反馈规则 | 阶段变化发信号，UI/音频/遮罩各自响应 |
| 边界规则 | 60、35、15 边界值归属必须稳定，不能来回抖动 |

## 5. 数据结构

```gdscript
enum StabilityStage {
    STABLE,
    UNSTEADY,
    DANGER,
    CRITICAL,
    DEPLETED
}
```

阶段配置：

| 字段 | 说明 |
|---|---|
| `id` | 阶段枚举或 StringName |
| `min_value` | 包含下限 |
| `max_value` | 上限 |
| `ui_color` | UI 建议色 |
| `feedback_level` | 反馈强度 |

曲线点：

| 字段 | 说明 |
|---|---|
| `stability` | 稳定值 |
| `radius` | 对应目标视野半径 |

---

## 6. 核心流程

```text
on_stability_changed(current)
→ resolve_stage(current)
→ 如果阶段变化，发 stability_stage_changed
→ resolve_vision_radius(current)
→ VisionController 使用曲线半径作为目标值
```

配置验证：

```text
1. 最大值应覆盖到 100。
2. 最小值应覆盖到 0。
3. 阶段区间不能重叠。
4. 曲线点稳定值应从高到低或低到高一致排序。
5. 0 点半径应为 0。
```

---

## 7. 信号与接口

```gdscript
signal stability_stage_changed(old_stage: int, new_stage: int)
signal stability_config_invalid(errors: Array)
```

提供：

```text
get_stage_for_value(value) -> StabilityStage
get_target_radius_for_value(value) -> float
```
