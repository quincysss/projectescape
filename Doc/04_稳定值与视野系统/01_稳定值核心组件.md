# 01_稳定值核心组件

> 对应母版章节：3、4、14.2、15  
> 施工补充：开工前必须读取 `13_模块功能规则细化与AI开工提示词.md` 的“稳定值核心组件”小节。

---

## 1. 实现目标

实现 `StabilityComponent`：负责保存稳定值、执行危险区下降、安全区回复、倍率修正、数值 clamp、阶段判断入口和归零事件。它是 04 系统的核心数据组件，但不负责 UI、音频、遮罩绘制和物品结算。

---

## 2. 设计依据

V0.1 基础数值：

| 参数 | 建议值 |
|---|---:|
| 最大稳定值 | 100 |
| 初始稳定值 | 100 |
| 危险区基础下降速度 | 1 / 秒 |
| 家中回复速度 | 10 / 秒 |
| 前哨站回复速度 | 8 / 秒 |
| 死亡阈值 | 0 |

危险区公式：

```text
每帧下降值 = 基础下降速度 × 区域倍率 × 装备倍率 × 物品倍率 × delta
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 触发条件 | 离开安全区后开始下降，进入安全区后停止下降并回复 |
| 前置条件 | 组件初始化时必须读取配置或使用默认配置 |
| 执行规则 | 数值每帧 clamp 到 `[0, max_stability]` |
| 执行规则 | 玩家不移动、开容器、打开背包时稳定值仍按危险区规则下降 |
| 执行规则 | 怪物碰撞可以造成一次性稳定值伤害，V0.1 为 -20 |
| 中断规则 | 稳定值归零后停止继续下降并只发一次 depleted |
| 失败规则 | 配置缺失时使用默认值并输出警告，不崩溃 |
| 反馈规则 | 每次数值变化发 `stability_changed`，阶段变化交给阶段模块 |
| 边界规则 | 安全区内正常情况下不应因稳定值死亡 |

## 5. 数据结构

字段：

| 字段 | 类型 | 默认值 |
|---|---|---|
| `max_stability` | `float` | `100` |
| `current_stability` | `float` | `100` |
| `base_decay_rate` | `float` | `1` |
| `current_recovery_rate` | `float` | `0` |
| `area_decay_multiplier` | `float` | `1` |
| `equipment_decay_multiplier` | `float` | `1` |
| `item_decay_multiplier` | `float` | `1` |
| `is_decaying` | `bool` | `false` |
| `is_recovering` | `bool` | `false` |
| `is_depleted` | `bool` | `false` |

---

## 6. 核心流程

```text
_process(delta)
→ 如果 is_depleted，返回
→ 如果 is_decaying，按公式扣稳定值
→ 如果 is_recovering，按 current_recovery_rate 回复
→ clamp 到 0-max
→ 如果数值变化，发 stability_changed
→ 如果 current <= 0，执行 depleted
```

入口：

```text
start_decay()
stop_decay()
start_recovery(rate)
stop_recovery()
apply_instant_recovery(amount)
apply_instant_delta(amount, source)
set_decay_multiplier(source, value)
reset_stability()
```

---

## 7. 信号与接口

```gdscript
signal stability_changed(current: float, max_value: float)
signal stability_depleted()
signal stability_recovery_started(rate: float)
signal stability_recovery_stopped()
signal stability_decay_started()
signal stability_decay_stopped()
```

被调用方：

```text
SafeArea2D：进入/离开安全区
InteractionStateController：开箱/修复/背包期间不暂停下降
StabilityItemEffect：即时回复或倍率修正
MonsterBasic：碰撞玩家时调用 apply_instant_delta(-20, "monster_contact")
PlayerDeathController：监听 depleted
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 初始稳定值为 100 |
| A2 | 危险区每秒下降 1 点 |
| A3 | 家中每秒回复 10 点 |
| A4 | 前哨站每秒回复 8 点 |
| A5 | 数值不会低于 0 或高于最大值 |
| A6 | 归零时只发一次 `stability_depleted` |
| A7 | 配置缺失有默认值和警告 |
| A8 | apply_instant_delta(-20, "monster_contact") 会正确扣稳定值并 clamp |
