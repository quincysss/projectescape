# 06_交互期间稳定值规则

> 对应母版章节：10、13.1  
> 施工补充：开工前必须读取 `13_模块功能规则细化与AI开工提示词.md` 的“交互期间稳定值规则”小节。

---

## 1. 实现目标

明确玩家在开容器、修复前哨站、打开背包等交互期间，稳定值仍然如何变化，以及稳定值归零时如何中断这些交互。

---

## 2. 设计依据

开容器期间：

```text
稳定值继续按危险区规则下降。
如果稳定值归零，立即进入死亡流程。
移动会中断开箱。
```

修复前哨期间：

```text
修复读条期间稳定值继续下降。
读条完成后前哨站才变为安全区。
```

背包打开期间：

```text
单人原型中，打开背包可以暂停移动，但不暂停稳定值。
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 开箱 | 处于危险区时，开箱不暂停稳定值下降 |
| 修复 | 未修复前哨站修复期间不提供安全效果 |
| 背包 | 危险区打开背包不暂停稳定值，是否暂停移动由背包系统决定 |
| 中断规则 | stability_depleted 立即中断开箱、修复、撤离等进行中交互 |
| 失败规则 | 任何交互不得自行把稳定值下降暂停，除非安全区或物品效果明确授权 |
| 反馈规则 | 调试显示 current_interaction_state 与是否允许衰减 |
| 边界规则 | 安全区内打开背包不下降，因为安全区状态优先 |

## 5. 数据结构

```gdscript
enum InteractionState {
    NONE,
    OPENING_CONTAINER,
    REPAIRING_OUTPOST,
    INVENTORY_OPEN,
    EXTRACTING
}
```

字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `current_interaction_state` | `InteractionState` | 当前交互 |
| `does_pause_stability` | `bool` | V0.1 大多数为 false |
| `interrupt_on_depleted` | `bool` | true |

---

## 6. 核心流程

```text
interaction_started(state)
→ 记录 current_interaction_state
→ 不修改 StabilityComponent 的 decaying 状态

stability_depleted
→ 如果 current_interaction_state != NONE
→ 发 interrupt_current_interaction("stability_depleted")
→ 清空交互状态
→ 交给死亡流程
```

---

## 7. 信号与接口

```gdscript
signal interaction_interrupted_by_stability(state: int, reason: String)
signal interaction_stability_rule_checked(state: int, does_pause: bool)
```

监听：

```text
LootContainer.container_open_started/cancelled/opened
OutpostRepairController.repair_started/cancelled/repaired
InventoryUI.opened/closed
StabilityComponent.stability_depleted
```

## 8. AI/MCP 开工提示词

```text
请实现“交互期间稳定值规则”。
读取 Doc/04_稳定值与视野系统/06_交互期间稳定值规则.md 和 13 中对应小节。
新增 InteractionStabilityBridge 或等价桥接逻辑，记录 OPENING_CONTAINER、REPAIRING_OUTPOST、INVENTORY_OPEN、EXTRACTING。
这些交互不能自行暂停危险区稳定值下降；stability_depleted 时必须中断当前交互并通知死亡流程。
禁止实现容器奖励、前哨材料消耗、背包完整 UI。
完成后验证开箱/修复/背包期间稳定值继续下降，归零会中断交互。
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 开容器期间稳定值继续下降 |
| A2 | 修复未完成前前哨不安全 |
| A3 | 背包打开期间稳定值继续下降 |
| A4 | 稳定值归零会中断当前交互 |
| A5 | 安全区状态优先于交互状态 |
| A6 | 调试面板显示当前交互状态 |
