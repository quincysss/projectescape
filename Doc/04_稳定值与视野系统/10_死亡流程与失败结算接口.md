# 10_死亡流程与失败结算接口

> 对应母版章节：13、14.4  
> 施工补充：开工前必须读取 `13_模块功能规则细化与AI开工提示词.md` 的“死亡流程与失败结算接口”小节。

---

## 1. 实现目标

稳定值归零后，统一触发死亡流程：锁定输入、中断交互、黑幕收束、播放死亡反馈、通知 02 的失败结算。稳定值系统不直接处理物品，只发死亡原因和事件。

---

## 2. 设计依据

死亡触发：

```text
稳定值 <= 0
```

可触发状态：

```text
EXPLORING
OPENING_CONTAINER
REPAIRING_OUTPOST
```

死亡流程：

```text
稳定值降至0
↓
玩家输入锁定
↓
容器/修复/撤离交互中断
↓
黑暗遮罩快速收束到玩家
↓
画面完全变黑
↓
播放死亡音效
↓
进入失败结算
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 触发条件 | `StabilityComponent.stability_depleted` |
| 执行规则 | 死亡事件只能触发一次 |
| 执行规则 | 先锁输入，再中断交互，再启动黑幕收束 |
| 结算规则 | 稳定值系统只通知死亡，不处理背包/仓库 |
| 中断规则 | 开箱、修复、撤离读条必须被中断 |
| 失败规则 | 死亡后不能继续开箱、修复、撤离或切回探索阶段 |
| 反馈规则 | 调试日志记录 death_reason、phase、interaction_state |

## 5. 数据结构

```gdscript
enum DeathReason {
    STABILITY_DEPLETED,
    DEBUG_FORCE,
    FUTURE_DAMAGE
}
```

死亡上下文：

| 字段 | 类型 | 说明 |
|---|---|---|
| `reason` | `DeathReason` | 死亡原因 |
| `stability_at_death` | `float` | 通常 0 |
| `run_phase` | `int` | 死亡时阶段 |
| `interaction_state` | `int` | 死亡时交互 |
| `position` | `Vector2` | 死亡位置 |

---

## 6. 核心流程

```text
on_stability_depleted()
→ 如果 already_dead，返回
→ already_dead = true
→ lock_player_input()
→ interrupt_current_interaction("death")
→ darkness_mask.start_death_collapse()
→ 等待 death_collapse_finished
→ run_end_controller.finish_run(DEAD, death_context)
```

---

## 7. 信号与接口

```gdscript
signal player_death_started(context)
signal player_death_visual_finished(context)
signal player_death_finished(context)
```

调用：

```text
InputController.lock()
InteractionStabilityBridge.interrupt_current()
DarknessMask.start_death_collapse()
RunEndController.finish_run(DEAD, context)
```
