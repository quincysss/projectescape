# 11_配置表与调试验证工具

> 对应母版章节：15、16、17  
> 施工补充：开工前必须读取 `13_模块功能规则细化与AI开工提示词.md` 的“配置表与调试验证工具”小节。

---

## 1. 实现目标

把稳定值核心数值、阶段、曲线、回复速度、遮罩参数做成可调配置，并提供调试面板和快捷键，方便快速验证离家压力、视野曲线、死亡和安全区回复。

---

## 2. 设计依据

调试面板显示：

```text
当前稳定值
当前阶段
当前视野半径
是否在安全区
当前下降/回复速度
区域倍率
装备倍率
距离家和前哨站的路径时间，可选
```

调试快捷键：

```text
加稳定值
减稳定值
切换阶段
开关黑暗遮罩
显示视野圆半径
显示安全区范围
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 配置规则 | 核心数值不硬编码，优先来自 StabilityConfig |
| 验证规则 | 配置加载时检查阶段和曲线是否合法 |
| 调试规则 | 面板显示当前值、阶段、半径、安全区、倍率 |
| 快捷键 | 支持加/减稳定值、强制归零、开关遮罩、显示半径 |
| 失败规则 | 配置错误必须输出具体字段，不允许静默使用错误配置 |
| 边界规则 | 调试功能不应在正式发布默认开启 |

## 6. 核心流程

```text
1. StabilityConfig 加载。
2. validate_config() 检查最大值、阶段、曲线。
3. DebugStabilityVision 创建玩家、家安全区、前哨安全区、遮罩和 HUD。
4. StabilityDebugPanel 监听稳定值和视野信号。
5. 快捷键修改稳定值或切换状态。
6. StabilityValidationRunner 执行基础功能、视野、节奏验证。
```

---

## 7. 信号与接口

```gdscript
signal stability_config_validated(is_valid: bool, errors: Array)
signal stability_debug_action_triggered(action_id: StringName)
signal stability_validation_finished(results: Array)
```
