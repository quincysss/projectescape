# 04_黑暗遮罩与死亡收束

> 对应母版章节：7、13.2、14.4  
> 施工补充：开工前必须读取 `13_模块功能规则细化与AI开工提示词.md` 的“黑暗遮罩与死亡收束”小节。

---

## 1. 实现目标

实现黑暗遮罩的状态与表现入口：家中关闭，离家淡入，危险区生效，回家淡出，前哨站变浅，死亡时黑圈收束到玩家并转入黑屏。

---

## 2. 设计依据

遮罩状态：

| 状态 | 触发 | 表现 |
|---|---|---|
| 关闭 | 家中全图观察 | 无黑暗遮罩 |
| 淡入 | 离开安全区 | 黑暗逐渐覆盖 |
| 生效 | 危险区探索 | 玩家周围圆形可见 |
| 淡出 | 回到家 | 黑暗消失 |
| 前哨休整 | 已修复前哨站 | 可选保留轻遮罩 |
| 死亡收束 | 稳定值归零 | 黑圈收束至玩家 |

死亡演出建议：

| 项目 | 建议值 |
|---|---:|
| 输入锁定 | 立即 |
| 黑幕收束 | 1.2 秒 |
| 黑屏停留 | 0.5 秒 |
| 进入结算延迟 | 0.5 - 1 秒 |

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 触发条件 | 离家、回家、进入已修复前哨、稳定值归零 |
| 执行规则 | 遮罩跟随玩家屏幕位置，半径由 VisionController 提供 |
| 淡入淡出 | 离家淡入 0.3-0.5 秒，回家淡出 0.3-0.5 秒 |
| 前哨规则 | 前哨站内遮罩变浅但不关闭，不切全图 |
| 死亡规则 | 归零后停止普通半径更新，进入死亡收束 |
| 失败规则 | UI 文字不能被黑圈遮到不可读 |
| 反馈规则 | 调试面板显示 mask_state、alpha、radius、player_screen_position |

## 5. 数据结构

```gdscript
enum DarknessMaskState {
    DISABLED,
    FADING_IN,
    ACTIVE,
    FADING_OUT,
    OUTPOST_REST,
    DEATH_COLLAPSE
}
```

字段：

| 字段 | 类型 | 默认值 |
|---|---|---|
| `state` | `DarknessMaskState` | `DISABLED` |
| `vision_radius` | `float` | `20` |
| `darkness_alpha` | `float` | `0` |
| `edge_softness` | `float` | 项目调试 |
| `noise_strength` | `float` | `0` |
| `fade_duration` | `float` | `0.4` |
| `death_collapse_duration` | `float` | `1.2` |

---

## 6. 核心流程

离家：

```text
safe_area_exited
→ DarknessMask.fade_in()
→ state = FADING_IN
→ 完成后 ACTIVE
```

回家：

```text
home_entered
→ DarknessMask.fade_out()
→ 完成后 DISABLED
```

前哨：

```text
repaired_outpost_entered
→ state = OUTPOST_REST
→ alpha 降低，半径目标由 VisionController 设为 20
```

死亡：

```text
stability_depleted
→ state = DEATH_COLLAPSE
→ 半径从当前值收束到 0
→ 黑屏停留
→ 发 death_collapse_finished
```

---

## 7. 信号与接口

```gdscript
signal darkness_state_changed(old_state: int, new_state: int)
signal darkness_fade_finished(state: int)
signal death_collapse_finished()
```

接收：

```text
VisionController.vision_radius_changed
SafeArea2D.safe_area_entered/exited
StabilityComponent.stability_depleted
Camera2D：用于换算 player_screen_position
```
