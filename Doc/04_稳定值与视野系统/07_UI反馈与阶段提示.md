# 07_UI反馈与阶段提示

> 对应母版章节：5.3、8、18.2  
> 施工补充：开工前必须读取 `13_模块功能规则细化与AI开工提示词.md` 的“UI反馈与阶段提示”小节。

---

## 1. 实现目标

实现稳定值的基础 UI 反馈：左上角稳定值条、阶段刻度、颜色变化、阶段切换短促提示和可选新手提示。UI 应帮助玩家理解风险，而不是用频繁弹窗打断探索。

---

## 2. 设计依据

稳定值条：

```text
左上角显示玩家头像/角色图标。
头像右侧或下方显示稳定值条。
稳定值条分段显示。
```

分段：

```text
100 / 60 / 35 / 15 / 0
```

阶段提示：

```text
60以下：稳定值条边缘轻微抖动。
35以下：稳定值条开始闪烁，加入心跳音。
15以下：稳定值条强闪，黑暗边缘脉冲。
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 触发条件 | stability_changed 更新数值，stage_changed 更新表现 |
| 执行规则 | UI 条显示当前值/最大值和阶段刻度 |
| 阶段规则 | 进入新阶段时短闪，不持续遮挡视线 |
| 新手提示 | 首次进入 DANGER 可显示一次短提示，后续不重复 |
| 失败规则 | UI 文本不能被黑暗遮罩遮住 |
| 反馈规则 | 调试可显示精确数值，正式 UI 可更简洁 |
| 边界规则 | 稳定值归零后 UI 可冻结或切死亡表现，不继续闪阶段 |

---

## 5. 数据结构

UI 状态：

| 字段 | 类型 | 说明 |
|---|---|---|
| `display_current` | `float` | 显示值 |
| `display_max` | `float` | 最大值 |
| `current_stage` | `StabilityStage` | 当前阶段 |
| `has_shown_danger_hint` | `bool` | DANGER 首次提示 |
| `is_death_locked` | `bool` | 死亡后冻结 |

---

## 6. 核心流程

```text
on_stability_changed(current, max)
→ 更新进度条比例
→ 更新数值文本或调试文本

on_stability_stage_changed(old, new)
→ 切换颜色
→ 播放短闪/抖动
→ 如果 new == DANGER 且未提示，显示一次短提示
```

---

## 7. 信号与接口

监听：

```text
StabilityComponent.stability_changed
StabilityStageResolver.stability_stage_changed
StabilityComponent.stability_depleted
```

可发：

```gdscript
signal danger_hint_shown()
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | UI 显示当前稳定值比例 |
| A2 | UI 有 100/60/35/15/0 分段感 |
| A3 | 阶段变化会改变颜色或表现 |
| A4 | 首次进入 DANGER 有一次提示 |
| A5 | 提示不会每次重复弹出 |
| A6 | UI 不被黑暗遮罩遮挡 |
