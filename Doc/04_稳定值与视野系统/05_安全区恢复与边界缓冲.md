# 05_安全区恢复与边界缓冲

> 对应母版章节：4.3、4.4、7.4、14.5  
> 施工补充：开工前必须读取 `13_模块功能规则细化与AI开工提示词.md` 的“安全区恢复与边界缓冲”小节。

---

## 1. 实现目标

实现家和前哨站安全区对稳定值系统的影响：家快速回复并关闭遮罩/全图观察；已修复前哨较慢回复但不提供全图；未修复前哨不安全。安全区进入和离开必须有短缓冲，避免边界抖动。

---

## 2. 设计依据

缓冲建议：

| 项目 | 建议值 |
|---|---:|
| 进入安全区确认时间 | 0.1 - 0.2 秒 |
| 离开安全区确认时间 | 0.2 - 0.3 秒 |
| 镜头过渡时间 | 0.5 - 0.6 秒 |
| 遮罩淡入/淡出 | 0.3 - 0.5 秒 |

安全区字段：

```text
safe_area_type: HOME / OUTPOST
recovery_rate
enable_overview_camera
enable_storage
is_active
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 触发条件 | 玩家进入/离开 SafeArea |
| 家规则 | `recovery_rate=10`，关闭遮罩，启用全图观察 |
| 前哨规则 | `recovery_rate=8`，不启用全图观察，修复前不激活 |
| 缓冲规则 | 进入确认 0.1-0.2 秒，离开确认 0.2-0.3 秒 |
| 中断规则 | 玩家在确认期间反向移动，应取消 pending 状态 |
| 失败规则 | 未修复前哨不应停止稳定值下降 |
| 反馈规则 | 调试显示当前安全区、pending_enter/pending_exit、回复速度 |

## 5. 数据结构

```gdscript
enum SafeAreaType {
    HOME,
    OUTPOST
}
```

字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `safe_area_id` | `StringName` | home/outpost id |
| `safe_area_type` | `SafeAreaType` | HOME/OUTPOST |
| `recovery_rate` | `float` | 10 或 8 |
| `enable_overview_camera` | `bool` | 家 true，前哨 false |
| `enable_storage` | `bool` | 可配置 |
| `is_active` | `bool` | 家 true，前哨修复后 true |
| `enter_confirm_time` | `float` | 0.15 |
| `exit_confirm_time` | `float` | 0.25 |

---

## 6. 核心流程

进入：

```text
body_entered player
→ 如果 !is_active，忽略安全效果
→ pending_enter 计时
→ 确认后发 safe_area_entered
→ StabilityComponent.stop_decay()
→ StabilityComponent.start_recovery(recovery_rate)
```

离开：

```text
body_exited player
→ pending_exit 计时
→ 确认后发 safe_area_exited
→ StabilityComponent.stop_recovery()
→ StabilityComponent.start_decay()
```

## 7. 信号与接口

```gdscript
signal safe_area_entered(area_id: StringName, area_type: int, recovery_rate: float)
signal safe_area_exited(area_id: StringName, area_type: int)
signal safe_area_pending_changed(area_id: StringName, pending_state: StringName)
```

接口：

```text
set_active(value)
is_player_inside_confirmed() -> bool
get_recovery_rate() -> float
```

| 编号 | 标准 |
|---|---|
| A1 | 家中停止下降并以 10/秒回复 |
| A2 | 已修复前哨停止下降并以 8/秒回复 |
| A3 | 未修复前哨不停止下降 |
| A4 | 进入/离开边界有防抖 |
| A5 | 前哨不触发全图观察 |
