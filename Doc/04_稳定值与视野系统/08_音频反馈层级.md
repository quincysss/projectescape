# 08_音频反馈层级

> 对应母版章节：9、8.3  
> 施工补充：开工前必须读取 `13_模块功能规则细化与AI开工提示词.md` 的“音频反馈层级”小节。

---

## 1. 实现目标

定义稳定值阶段对音频氛围的影响，并预留 `StabilityAudioController`。V0.1 可以先做简单音频事件或调试日志，但接口要清晰，避免后续音乐系统接入时返工。

---

## 2. 设计依据

音频层级：

| 阶段 | 音频表现 |
|---|---|
| 稳定 | 正常 BGM，环境音清晰 |
| 轻度失稳 | 加入轻微噪音或低频 |
| 危险 | 心跳/呼吸声出现，BGM 略压低 |
| 濒临崩溃 | 心跳急促，环境音变闷 |

低稳定值时不能盖住：

```text
敌人提示音
容器完成音
前哨站交互音
撤离提示音
```

---

## 3. 功能规则细化

| 分类 | 规则 |
|---|---|
| 触发条件 | stability_stage_changed |
| 执行规则 | 阶段改变时切换音频层级或发出音频事件 |
| 限制规则 | 低稳定音效不能掩盖关键交互提示音 |
| 失败规则 | 音频不能过于刺耳或频繁重复 |
| 反馈规则 | 没有音频资源时输出调试日志 |
| 边界规则 | 死亡收束时可播放死亡音效，但不处理结算 |

---

## 4. Godot 落地范围

| 路径 | 职责 |
|---|---|
| `res://scripts/audio/StabilityAudioController.gd` | 阶段音频控制 |
| `res://resources/audio/StabilityAudioConfig.tres` | 音频资源映射，可选 |

---

## 5. 数据结构

字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `current_stage` | `StabilityStage` | 当前阶段 |
| `heartbeat_intensity` | `float` | 心跳强度 |
| `noise_intensity` | `float` | 噪音强度 |
| `bgm_ducking_amount` | `float` | BGM 压低 |
| `critical_audio_lock` | `bool` | 防止重复触发 |

---

## 6. 核心流程

```text
on_stability_stage_changed(old, new)
→ 根据 new 设置 heartbeat/noise/bgm ducking
→ 播放一次阶段切换提示音
→ 如果无音频资源，输出 debug log

on_stability_depleted()
→ 播放死亡音效或发 death_audio_requested
```

---

## 7. 信号与接口

监听：

```text
stability_stage_changed
stability_depleted
```

发出：

```gdscript
signal stability_audio_layer_changed(stage: int)
signal death_audio_requested()
```

## 8. AI/MCP 开工提示词

```text
请实现“音频反馈层级”的预留控制器。
读取 Doc/04_稳定值与视野系统/08_音频反馈层级.md 和 13 中对应小节。
新增 StabilityAudioController，监听阶段变化并输出音频层级事件；如果没有音频资源，先用调试日志替代。
必须保证关键交互音效不被低稳定值音频完全盖住，接口上预留 ducking/priority。
禁止制作正式音乐、导入音频资源、实现完整音频混音系统。
完成后用阶段切换验证音频事件或日志。
```

## 9. 验收标准

| 编号 | 标准 |
|---|---|
| A1 | 阶段变化触发音频层级事件 |
| A2 | DANGER 阶段可请求心跳/呼吸反馈 |
| A3 | CRITICAL 阶段反馈更强 |
| A4 | 无音频资源时有调试日志 |
| A5 | 接口预留关键音效优先级 |
