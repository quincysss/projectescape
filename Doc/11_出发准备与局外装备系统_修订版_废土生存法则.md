# 《废土生存法则》出发准备与局外装备系统（修订版）

> 文档版本：v0.1  
> 所属项目：《废土生存法则》  
> 前置文档：07_背包、存储与负重系统.md / 09_局外仓库系统.md / 10_研究所与制作所系统.md  
> 适用阶段：V0.2 局外成长闭环  
> 文档目标：明确出发准备界面、装备携带、装备槽、装备属性、装备耐久、局内拾取装备带出、制造装备、废弃装备拆解与 Godot 实现接口，让玩家在局外对“带什么装备进局”做出清晰决策。

---

## 1. 本文档一句话说明

出发准备系统让玩家从局外仓库中选择装备和模块带入下一局。

局外装备系统让装备成为可消耗、可回收、可制造的长期资源，而不是一次获得永久无成本使用。

---

## 2. 系统设计目标

## 2.1 核心目标

出发准备与装备系统需要达成：

```text
让玩家在开局前做轻量配置。
让装备影响局内能力和负重。
让装备拥有耐久损耗，形成材料消耗口。
让局内找到的装备可以带回局外。
让制造装备有明确价值。
让废弃装备可拆解，回收部分材料。
为后续装备修复、词条、战斗装备预留接口。
```

---

## 2.2 不做什么

V0.2 初期不建议做：

```text
复杂装备词条。
随机强化。
装备修复。
装备升级失败。
装备套装效果。
装备交易。
装备镶嵌。
复杂比较面板。
```

核心原则：

```text
装备第一版要提供“选择、磨损、回收”的闭环，不急着做复杂养成。
```

---

## 3. 出发准备系统边界

出发准备系统处理：

```text
从仓库选择装备。
从仓库选择背包装备。
从仓库选择耳机模块，后续。
从仓库选择消耗品，由 12 文档定义。
显示当前负重与容量预估。
确认进入下一局。
取消出发时释放已配置物品。
```

出发准备系统不处理：

```text
研究升级。
制作装备。
装备拆解。
局内装备拾取。
消耗品具体效果。
```

---

## 4. 出发准备流程

```text
玩家进入局外主页。
↓
进入出发准备界面。
↓
系统读取仓库中的可装备物。
↓
玩家选择装备放入装备槽。
↓
系统锁定这些仓库物品为 CONFIGURED。
↓
显示预计负重、背包容量和关键属性。
↓
玩家确认开始探索。
↓
系统从仓库移出已配置物，生成局内携带数据。
↓
进入单局。
```

取消出发时：

```text
所有 CONFIGURED 物品回到仓库普通状态。
不扣耐久。
不改变物品位置。
```

## 5. 装备槽规则

## 5.1 V0.2 建议装备槽

建议第一版装备槽：

| 槽位 | 用途 | V0.2 建议 |
|---|---|---|
| HEAD | 头部装备 | 预留 |
| BODY | 身体装备 | 可选 |
| HAND | 手部工具 | 可选 |
| LEG | 腿部装备 | 预留 |
| BACK | 背包装备 | 必做 |
| SPECIAL | 特殊模块 | 预留 |

V0.2 最小实现建议：

```text
只做 BACK 背包装备。
可选做 BODY 或 HAND。
其他槽先显示为未开放。
```

---

## 5.2 装备槽限制

规则：

```text
每个槽位最多装备 1 件。
装备必须匹配槽位。
废弃状态装备不能装备。
待领取装备不能装备。
已配置到出发准备的装备不能被研究或制作消耗。
```

|---|---|

| WORN | 耐久大于 0 且小于等于 25% |
| BROKEN | 耐久为 0，废弃状态 |

## 7. 装备来源

装备来源分为：

| 来源 | 说明 | 初始耐久 |
|---|---|---|
| CRAFTED | 制作所制造 | 100% |
| LOOTED | 局内找到并带出 | 物品生成时定义，建议 50%-100% |
| REWARD | 系统奖励，后续 | 100% 或奖励定义 |

规则：

```text
制作所制造的装备耐久度为 100%。
局内找到的装备可以有随机耐久。
局内找到的装备如果穿戴后成功带出，需要扣除 1/8 最大耐久。
```

## 8. 装备耐久规则

## 8.1 耐久表现

V0.2 第一版建议用 0-100 的整数耐久。

```text
durability_max = 100
durability_current = 0 - 100
```

UI 显示：

```text
100% / 75% / 50% / 25% / 0%
```

也可以显示为耐久条。

---

## 8.2 局外带入装备的耐久扣除

V0.2 第一版使用固定扣除。

玩家从局外仓库配置装备并带入局内后，如果成功撤离并将该装备带回局外：

```text
装备减少 1/4 最大耐久。
如果 max 为 100，则减少 25 点。
```

示例：

```text
100 -> 75
75 -> 50
50 -> 25
25 -> 0
```

设计目的：

```text
让装备成为持续材料消耗口。
让玩家每次带装备出门都有成本。
让制造所和材料系统长期有价值。
```

---

## 8.3 局内找到装备并穿戴带出

V0.2 第一版使用固定扣除。

玩家在局内找到装备，并在本局穿戴过该装备，成功撤离带出时：

```text
装备减少 1/8 最大耐久。
如果 max 为 100，则减少 12.5 点。
第一版可四舍五入为 13 点，或用浮点保存后显示整数。
```

建议实现：

```text
内部耐久支持小数。
UI 显示四舍五入后的百分比。
扣除 12.5 后保存真实值。
```

如果该装备只是放在背包中带出，未穿戴过：

```text
不扣本次穿戴耐久。
保持拾取时的耐久。
```

---

## 8.4 后续版本：按穿戴时长扣耐久

V0.3/V0.4 建议将固定扣耐久升级为“按局内实际穿戴时长结算”。

核心公式：

```text
equipped_minutes = equipped_seconds / 60
loss_ratio = equipped_minutes * drain_rate_per_minute * source_multiplier * activity_multiplier * equipment_multiplier
loss_ratio = clamp(loss_ratio, min_loss_ratio, max_loss_ratio)
durability_loss = durability_max * loss_ratio
durability_current = max(0, durability_current - durability_loss)
```

推荐第一版参数：

```text
drain_rate_per_minute = 0.01
source_multiplier = 1.0
activity_multiplier = 1.0
equipment_multiplier = 1.0
min_loss_ratio = 0.0
max_loss_ratio = 0.25
```

示例：

```text
装备 A 穿戴 2 分钟：
loss_ratio = 2 * 0.01 = 0.02
耐久减少 2% 最大耐久

装备 B 穿戴 8 分钟：
loss_ratio = 8 * 0.01 = 0.08
耐久减少 8% 最大耐久
```

说明：

```text
这个公式只计算“实际穿在身上的时间”。
装备在背包里、仓库里、安全存储里不累计穿戴时长。
切换装备时，旧装备停止计时，新装备开始计时。
死亡失败时，如果身上装备丢失，可以不再结算耐久。
```

后续可扩展参数：

| 参数 | 用途 |
|---|---|
| source_multiplier | 区分局外带入装备、局内拾取装备、奖励装备 |
| activity_multiplier | 区分普通探索、战斗、异常区域、雨天等消耗强度 |
| equipment_multiplier | 不同装备自身耐久消耗系数 |
| max_loss_ratio | 防止单局时间过长导致装备一次直接报废 |

V0.3 建议配置：

```text
局外带入装备 source_multiplier = 1.0
局内拾取装备 source_multiplier = 0.8
背包装备 equipment_multiplier = 1.0
轻型装备 equipment_multiplier = 0.8
重型装备 equipment_multiplier = 1.2
```

---

## 8.5 局内找到装备并直接存入安全存储

如果局内找到装备后没有穿戴，而是放入家中存储或前哨存储并结算带回：

```text
不扣穿戴耐久。
按拾取时耐久进入结算和仓库。
```

---

## 8.6 死亡失败时装备归属

死亡失败时：

```text
玩家背包与身上装备按背包风险处理，进入丢失列表。
家中存储和已修复前哨站存储中的装备可进入带回列表。
带回的安全存储装备不扣穿戴耐久。
```

如果后续需要区分“身上已穿装备死亡是否损坏”，另开文档设计。

---

## 8.7 耐久归零

装备耐久归零时：

```text
装备状态变为 BROKEN。
BROKEN 装备不能配置到出发准备。
BROKEN 装备不能在局内穿戴。
BROKEN 装备仍占仓库格。
玩家可点击拆解。
```

V0.2 前几版不支持修复。

后续可增加：

```text
消耗材料修复装备。
修复上限衰减。
高品质装备需要稀有材料修复。
```

## 10. 局内装备规则

## 10.1 进入局内

开始对局时：

```text
出发准备中的装备转为局内装备实例。
装备重量计入玩家当前负重。
装备效果写入 PlayerStats。
仓库中对应物品暂时移除或标记为 IN_RUN。
```

---

## 10.2 局内替换装备

V0.2 第一版建议：

```text
允许在背包界面穿戴/卸下装备。
只能穿戴匹配槽位的装备。
穿戴后立即刷新负重和属性。
被替换装备进入背包。
如果背包无空间，不能替换。
```

如果实现成本高，第一版可以只允许局外配置，不允许局内替换。

---

## 10.3 装备穿戴记录

为了支持局内找到装备扣 1/8 耐久：

```text
装备实例需要记录 was_equipped_this_run。
局内穿戴过一次即标记 true。
后续时长公式需要记录 equipped_seconds_this_run。
结算时根据来源和该标记扣耐久。
```

后续按时长扣耐久时：

```text
每个装备实例记录本局累计穿戴秒数。
装备穿上时记录 equipped_start_time。
装备卸下、替换、死亡或结算时结算本段时长。
同一装备多次穿戴，累计 equipped_seconds_this_run。
结算完成后清空本局计时字段。
```

---

## 11. 撤离结算中的装备耐久处理

## 11.1 成功撤离

成功撤离时：

```text
从局外带入且成功带回的装备：扣 1/4 最大耐久。
局内拾取且穿戴过并成功带回的装备：扣 1/8 最大耐久。
局内拾取但未穿戴、仅带回的装备：不额外扣耐久。
安全存储中未穿戴装备：不额外扣耐久。
扣耐久后耐久 <= 0 的装备变为 BROKEN。
```

V0.3/V0.4 切换到时长公式后：

```text
不再使用固定 1/4、1/8 扣除。
根据每件装备本局 equipped_seconds_this_run 单独结算耐久。
未穿戴过的局内拾取装备 equipped_seconds_this_run = 0，不扣耐久。
```

---

## 11.2 失败结算

失败结算时：

```text
身上装备和背包装备丢失。
家中存储、已修复前哨站存储中的装备带回。
带回的安全存储装备不扣本次穿戴耐久。
```

---

## 12. 废弃装备拆解

## 12.1 拆解入口

BROKEN 装备在仓库中显示“拆解”按钮。

规则：

```text
玩家点击拆解。
显示拆解确认。
确认后销毁装备。
按概率生成材料。
材料尝试进入仓库。
仓库满时材料进入待领取列表，或阻止拆解。
```

V0.2 首版建议：

```text
仓库空间不足时不能拆解。
提示“仓库空间不足，无法接收拆解材料”。
```

---

## 12.2 拆解产出规则

拆解产出是概率性的，但第一版应简单。

建议：

| 装备品质 | 基础材料概率 | 稀有材料概率 | 产出数量 |
|---|---:|---:|---|
| C | 100% | 0% | 1-2 |
| B | 100% | 10% | 1-3 |
| A | 100% | 20% | 2-4 |
| S | 100% | 35% | 3-5 |

示例：

```text
C级背包拆解：旧布、胶带、废铁片中随机 1-2 个。
B级工具拆解：废铁片、弹簧、螺丝中随机 1-3 个，小概率电路板。
```

---

## 12.3 拆解设计目的

```text
给耐久归零的装备一个出口。
让装备磨损变成材料循环的一部分。
避免仓库被废弃装备永久占满。
让玩家对材料需求有更长期的规划。
```

拆解不能返还完整制造成本。

---

## 13. 装备修复预留

V0.2 前几版不支持修复。

原因：

```text
先让耐久成为材料消耗口。
如果立刻开放低成本修复，制造所和材料消耗压力会变弱。
```

后续修复可设计为：

```text
消耗材料恢复耐久。
不同品质需要不同材料。
每次修复降低最大耐久，后续可选。
修复需要研究所解锁。
```

## 15. 装备配置表规划

后续版本应将装备静态数据放入 `.tab` 配置表。

`.tab` 文件本质是以制表符分隔的 TSV 文件，能用电子表格打开，也方便 Godot 读取。

建议文件：

```text
data/equipment.tab
```

建议字段：

| 字段 | 说明 |
|---|---|
| id | 装备唯一ID |
| type | 装备类型，决定可放入哪个槽位 |
| slot | 装备槽位：HEAD/BODY/HAND/LEG/BACK/SPECIAL |
| name | 装备名 |
| quality | 品质：C/B/A/S |
| weight | 重量 |
| durability_max | 最大耐久 |
| initial_durability_min | 掉落时最小初始耐久 |
| initial_durability_max | 掉落时最大初始耐久 |
| durability_drain_multiplier | 后续时长公式中的装备消耗系数 |
| effect_ids | 装备效果ID列表 |
| effect_values | 装备效果参数 |
| appearance_scene | 局内外观场景路径 |
| icon | 图标路径 |
| description | 描述文本 |
| dismantle_group | 拆解掉落组 |
| drop_tags | 掉落标签 |
| enabled_version | 启用版本 |

示例表头：

```text
id	type	slot	name	quality	weight	durability_max	initial_durability_min	initial_durability_max	durability_drain_multiplier	effect_ids	effect_values	appearance_scene	icon	description	dismantle_group	drop_tags	enabled_version
```

示例行：

```text
backpack_small_reinforced	backpack	BACK	小型加固背包	C	1.0	100	100	100	1.0	add_carry_weight	3	res://assets/equipment/backpack_small.tscn	res://assets/icons/equipment/backpack_small.png	加固过的小型背包，可略微提高负重。	dismantle_cloth_basic	backpack;crafted	v0.2
```

规则：

```text
新增装备优先改 equipment.tab。
Godot 运行时读取配置生成 EquipmentDefinition。
装备实例只保存 instance_id、id、当前耐久、来源和局内计时等动态字段。
不要把每件装备的静态属性硬编码在脚本中。
```

## 16.2 LoadoutManager

## 16.3 EquipmentDurabilityService

职责：


## 20. 当前设计体检与建议

## 20.1 合理点

装备耐久能解决“产出口大于消耗口”的问题。

玩家带装备出门会更强，但每次成功带回都付出耐久成本，材料和制作所因此长期有价值。

---

## 20.2 需要注意的点

### 耐久损耗要足够清楚

玩家必须知道：

```text
带入装备成功带回会掉耐久。
局内拾取装备穿戴后带出也会掉耐久。
耐久为 0 后只能拆解，暂不能修复。
```

### 第一版装备效果不要太多

先做背包装备和负重加成最稳。

装备效果太多，会同时牵动稳定值、视野、交互速度、战斗等系统。

## 23. 本文档结论

出发准备与装备系统的第一目标不是复杂养成，而是建立“准备-使用-磨损-拆解-再制造”的材料循环。

V0.2 第一版应保持：

```text
装备能带入。
装备能影响局内。
装备会磨损。
装备坏了能拆解。
装备暂不能修复。
```

这样装备会成为玩家局外成长的重要决策，而不会变成永久无成本收益。
