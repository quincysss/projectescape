# 《废土生存法则》数值平衡表与掉落权重规范（修订版）

> 文档版本：v0.1  
> 所属项目：《废土生存法则》  
> 前置文档：05_容器刷新与开箱系统.md / 06_前哨材料与前哨站修复系统.md / 07_背包、存储与负重系统.md / 08_撤离与结算系统.md / 13_数据配置表与TAB规范.md  
> 适用阶段：V0.3 之后  
> 文档目标：定义材料产出、容器掉落、稀有物资、负重、研究矿币成本和制作消耗之间的平衡原则，避免产出过大、消耗口太小或玩家目标感断裂。

---

## 1. 本文档一句话说明

数值平衡的目标是让玩家每次出门都想多拿一点，但永远拿不完。

---

## 2. 平衡目标

核心目标：

```text
每局都有可带回的收益。
玩家经常面临背包取舍。
材料消耗口能跟上产出。
稀有物资保持惊喜。
前哨提交和绕路有策略价值。
装备耐久能形成长期材料消耗。
```

不追求：

```text
第一版精确经济模型。
复杂拍卖行价格。
动态市场。
极端刷子游戏产出。
```

---

## 3. 基础资源循环

资源流向：

```text
局内容器 -> 背包 -> 安全存储/撤离 -> 结算入库 -> 仓库
仓库 -> 研究/制作/出发装备 -> 局内能力提升
局内能力提升 -> 能走更远 -> 拿到更好资源
```

消耗口：

```text
研究。
制作。
前哨修复。
装备耐久损耗。
废弃装备拆解。
消耗品使用。
```

---

## 4. 掉落权重基础

掉落使用权重，不直接使用百分比。

示例：

```text
废金属 weight 100
脏布条 weight 80
旧电池 weight 30
稀有零件 weight 5
```

概率计算：

```text
item_probability = item_weight / total_weight
```

好处：

```text
新增物品时不用重算所有百分比。
不同容器可复用同一物品但给不同权重。
```

---

## 5. 容器掉落分层

容器建议分：

```text
small_common
medium_common
medical_box
tool_box
safe_box
rare_cache
outpost_supply
```

不同容器提供不同倾向：

| 容器 | 主要产出 | 风险 |
|---|---|---|
| small_common | 基础材料、小消耗品 | 低 |
| medium_common | 基础材料、少量装备 | 中 |
| medical_box | 医疗、稳定值消耗品 | 中 |
| tool_box | 工具、电子材料 | 中 |
| safe_box | 稀有物资、配方可用条件、装备 | 高 |
| rare_cache | 高价值物资 | 高 |
| outpost_supply | 前哨材料 | 中 |

---

## 6. 区域权重

距离安全屋越远，收益越高。

建议：

```text
近圈：安全、低收益、基础材料。
中圈：材料稳定、少量稀有、前哨推进。
远圈：高风险、高稀有、高重量。
```

倍率示例：

```text
near_loot_multiplier = 0.8
middle_loot_multiplier = 1.0
far_loot_multiplier = 1.4
far_rare_bonus = 1.5
```

注意：

```text
远圈不能只给更多垃圾。
远圈应提供更高价值密度，但也让撤离压力更强。
```

---

## 7. 价值密度

定义：

```text
value_density = item_value / item_weight
```

用途：

```text
决定玩家背包取舍。
低价值高重量：逼玩家放弃。
高价值低重量：制造惊喜。
高价值高重量：制造路线压力。
```

建议：

```text
基础材料：低价值低重量。
前哨材料：中价值中重量。
稀有物资：高价值低或中重量。
大型设备：高价值高重量。
```

---

## 8. 前哨材料平衡

前哨材料应满足：

```text
能在局内直接找到。
会占背包空间。
可以部分提交。
提交需要绕路和时间。
提交后释放背包空间。
```

设计意义：

```text
玩家自主选择：立刻提交释放空间，还是攒一波节省时间。
```

建议：

```text
每个前哨阶段需要 2-3 类材料。
单类需求数量 1-4。
基础前哨第一阶段应在 1-2 局内可完成。
```

---

## 9. 研究制作消耗

研究消耗目标：

```text
给玩家长期目标。
消耗常见材料。
引导玩家探索不同容器。
```

制作消耗目标：

```text
形成材料出口。
让装备和消耗品有成本。
避免仓库无限膨胀。
```

建议比例：

```text
基础研究：1-2 局可完成。
中级研究：3-5 局可完成。
稀有制作：需要至少 1 个低概率材料或配方可用条件。
```

---

## 10. 装备耐久消耗

装备耐久是长期材料出口。

当前 V0.2：

```text
局外带入装备成功带出扣 1/4。
局内找到装备穿戴带出扣 1/8。
```

后续时长公式：

```text
loss_ratio = equipped_minutes * drain_rate_per_minute * source_multiplier * activity_multiplier * equipment_multiplier
```

平衡目标：

```text
玩家愿意带装备。
装备不会永久保值。
材料有持续消耗口。
拆解废弃装备能回收一部分材料，但不能完全返还。
```

---

## 11. 稀有物资规则

稀有物资用于：

```text
高阶研究。
高阶制作。
配方可用条件兑换。
仓库收藏。
未来交易。
```

规则：

```text
稀有物资不应大量堆叠。
稀有物资应有明确用途。
稀有物资可以轻，但获取风险要高。
```

建议：

```text
稀有掉落主要出现在 safe_box 和 rare_cache。
近圈只给极低概率。
远圈给更高概率但不保证。
```

## 14. 验证清单

```text
近圈 1-2 个容器能给基础收益。
中圈能稳定提供前哨材料。
远圈稀有物资概率明显更高。
背包经常接近满载。
玩家需要丢弃或取舍物品。
研究矿币成本不会一局内全部毕业。
常规材料每次带回后至少有基础商品、转化、装备/家具制作或出售出口。
装备耐久让材料有长期消耗口。
拆解返还不会导致材料无限循环。
```

## 15. 当前设计体检与建议

合理点：

```text
前哨部分提交增加了玩家自主权。
装备耐久能修正产出口过大问题。
安全存储容量限制能制造撤离前取舍。
```

风险：

```text
如果普通材料太多，仓库会迅速膨胀。
如果稀有掉落太低，玩家会觉得无目标。
如果前哨需求太重，会变成搬砖。
如果装备损耗太高，玩家会不愿意带装备。
```

建议：

```text
先用小样本模拟，不要凭感觉调。
每新增一种材料，都要同时定义至少一个消耗口。
每新增一种稀有物资，都要定义至少一个明确用途。
```

---

## 16. 本文档结论

数值平衡要围绕一个问题：

```text
玩家这局多拿一个东西，是否会产生真实取舍？
```

如果答案是肯定的，背包、掉落、前哨、研究、制作和装备耐久就能形成闭环。

V0.3 最该优先做的是掉落模拟工具，而不是盲目加更多物品。
