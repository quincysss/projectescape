# 《废土生存法则》研究所与制作所系统（修订版）

> 文档版本：v0.1  
> 所属项目：《废土生存法则》  
> 前置文档：01_项目总纲.md / 07_背包、存储与负重系统.md / 08_撤离与结算系统.md / 09_局外仓库系统.md  
> 适用阶段：V0.2 局外成长闭环  
> 文档目标：明确研究所、配方可用条件解锁、制作所、产物入库、待领取、局外成长数值与 Godot 实现接口，让玩家带回仓库的材料能够转化为店铺经营、装备准备和下一局准备能力。

---

## 1. 本文档一句话说明

研究所负责“永久成长”，制作所负责“物品生产”。

当前权威口径：

```text
研发只消耗矿币，并受章节、店铺等级、任务条件、任务或前置研究条件限制。
制作消耗材料，并可少量消耗矿币作为加工费。
材料不再作为永久研发的主要门槛，避免玩家因为局内随机掉落卡住长期成长。
```

玩家从局内带回材料、情报/任务物和稀有物资后，主要通过制作所制造可售物资、装备和消耗品；通过开店获得矿币后，再用矿币在研究所解锁永久成长。

---

## 2. 系统设计目标

## 2.1 核心目标

研究所与制作所需要达成：

```text
消耗仓库中的材料。
让带回物资有长期用途。
提供局外成长目标。
解锁更多出发选择。
不破坏局内搜打撤主循环。
不让玩家在局外界面消耗过多时间。
为后续装备、消耗品、耳机模块、地图信息扩展预留接口。
```

---

## 2.2 不做什么

V0.2 初期不建议做：

```text
复杂科技树大图。
长时间真实倒计时研究。
复杂制作队列。
随机词条制作。
装备强化失败。
材料返还公式。
交易市场。
配方碎片合成大系统。
```

核心原则：

```text
局外成长要让玩家期待下一局，而不是把玩家困在菜单里。
```

---

## 3. 系统边界

## 3.1 研究所

研究所处理：

```text
永久数值升级。
系统功能解锁。
仓库容量升级。
局内临时存储容量升级。
负重上限升级。
信号显示增强。
制作配方可用条件。
```

研究所不直接生产物品。

研究所不消耗具体材料。研究所可以读取以下条件作为门禁：

```text
矿币数量
章节进度
店铺等级
已完成任务/订单
已学会任务条件
前置研究等级
已安装店面设备
```

这些条件用于控制节奏，不用于制造“缺一个随机材料无法研究”的卡点。

---

## 3.2 制造所 / 制作所

最新局外循环中，“制造所”承担两层功能：

```text
制造可售物资：默认开放，用于把局内带回的材料加工为店铺可售物资。
同类别资源转化：默认开放或随配方可用条件，用于把同类别多余材料转化为短缺材料，消耗必须大于产出。
扩展配方：通过研究、配方可用条件或后续章节逐步开放更高等级可售物资、装备、消耗品和后勤订购标准件。
```

制造所处理：

```text
可售物资制造。
同类别资源转化。
装备制作。
消耗品制作。
背包装备制作。
耳机模块制作，后续。
后勤订购标准件接入，后续。
```

制造所不提供永久数值升级。

第一章中，制造所不再作为矿币解锁目标。序章探索结束返回局外后，无论撤离成功或失败，剧情都会发放一次性开店周转物资；玩家用这些材料或自己带回的材料制造第一批可售物资，上架货台并完成首次营业结算。详细规则见 `Doc/10_研究所与制作所系统/03_第一章制造所解锁规则.md` 和 `Doc/25_主菜单玩家档案与剧情章节目标系统/04_制造所解锁与第一章结束.md`。

---

## 3.3 配方可用条件规则

配方可用性不是道具，也不通过拾取、入库、学习或消耗来完成。

配方是否开放由条件系统判断：

```text
默认开放。
研究完成。
章节进度。
任务完成。
店铺等级。
设备/家具启用。
商人供应。
地点事件。
```

制作所只显示“已满足条件的配方”和“可见但条件不足的配方”。不得再设计配方道具、学习配方道具或重复配方道具处理。

## 4. 局外成长总流程

玩家成功结算并入库后：

```text
进入局外主页。
↓
进入白天准备阶段。
↓
查看仓库、制造所、研究所、图鉴和当日需求排行榜。
↓
在基础手作台/制造所中，把带回材料制造为可售物资。
↓
将可售物资摆上货台。
↓
点击开店营业，货台成交获得矿币。
↓
白天结束，进入夜晚计划。
↓
选择出发角色和地点。
↓
进入出发准备界面，选择装备和消耗品。
↓
出发探索。
```

第一天特殊门禁：

```text
新档第一段剧情结束
-> 进入序章夜晚出发入口
-> 玩家完成首次地表探索并返回哨所
-> 无论撤离成功或失败，播放序章返回剧情
-> 发放一次性开店周转物资
-> 开放正式白天准备阶段
-> 开放仓库、制造所、图鉴和当日需求排行榜
-> 研究所按剧情需要锁定或开放
-> 第一章目标卡指引玩家完成制造、上架和首次营业结算
```

研究所是否在首次返回后开放由章节目标系统控制；但局外经济不再依赖旧交易入口。基础可售物资制造默认可用，后续更高等级配方通过研究、章节、任务、店铺等级、设备、商人供应或地点事件扩展，不再以第一章固定矿币目标解锁制造所。

白天准备阶段功能入口顺序由 09、29、30 文档定义：

```text
仓库
制造所
研究所
图鉴
```

如果研究所尚未满足剧情解锁门禁，则显示为锁定或隐藏；不得出现空白面板。

未解锁配方不应置灰成不可点击空状态，而是显示来源条件，例如需要研究、配方可用条件、章节进度或地点产出。制造所入口本身在序章返回剧情后可用。

## 5. 材料分类与用途

## 5.1 材料类型

建议将材料分为：

| 类型 | 用途 | 示例 |
|---|---|---|
| 基础材料 | 基础制作、低级商品、家具制作 | 废铁片、塑料块、旧布 |
| 电子材料 | 信号、耳机、照明相关 | 旧电池、破损电路板 |
| 医疗材料 | 恢复类消耗品制作 | 绷带、酒精棉、药瓶 |
| 工具材料 | 背包、工具、容器效率相关 | 螺丝、弹簧、胶带 |
| 稀有物资 | 高级制作、订单、收藏或剧情条件 | 稳定核心、异常晶片 |
| 情报/任务物 | 参与任务、地点事件、商人条件或配方可用条件 | 背包改装委托、照明模块任务线 |

---

## 5.2 材料消耗规则

当前版本中，材料消耗只适用于制作、转化、装备、消耗品和家具制作，不适用于研究所永久研发。

研究所永久研发规则：

```text
研发只扣除矿币。
研发可以要求前置条件，但不扣除具体材料。
研发失败时不得扣除矿币。
研发完成后立即写入玩家永久存档。
```

制作所材料消耗规则：

规则：

```text
只消耗已入库物品。
结算待处理物不能直接用于制作。
已配置到出发准备的物品不能被消耗。
待领取产物不能作为材料。
扣除材料必须通过 WarehouseManager 原子操作完成。
扣除失败时，制作不能完成。
```

---

## 5.3 材料优先扣除顺序

同一种材料存在多个堆叠时：

```text
优先扣除数量较少的堆叠。
再扣除仓库位置靠前的堆叠。
扣除后自动清理数量为 0 的堆叠。
```

原因：

```text
减少仓库碎片堆叠。
让玩家更容易理解材料减少。
```

## 6.5 研究解锁规则

研究节点可见分为三种状态：

| 状态 | 说明 |
|---|---|
| HIDDEN | 暂不显示，后续高级内容 |
| VISIBLE_LOCKED | 可看见，但前置条件未满足 |
| AVAILABLE | 前置满足，可研究 |
| COMPLETED | 已完成，效果生效 |

V0.2 可以不做复杂树状 UI。

最低实现：

```text
以列表形式显示研究项。
显示当前等级和下一等级效果。
显示矿币是否满足。
显示前置条件是否满足。
满足条件时按钮可点击。
```

## 6.6 研究完成方式

V0.2 建议研究即时完成。

流程：

```text
玩家选择研究节点。
系统检查前置研究。
系统检查仓库材料。
系统显示消耗确认。
玩家确认。
WarehouseManager 扣除材料。
ResearchManager 写入研究完成状态。
效果立即刷新。
保存局外存档。
```

暂不建议加入真实时间倒计时。

原因：

```text
当前项目重点是验证局内搜集与局外成长闭环。
真实时间研究会增加等待设计和离线收益设计。
```

研究效果分为：

| 效果类型 | 生效时机 |

## 7. 配方可用条件规则

## 7.1 定位

配方不是可拾取道具。玩家不会在局内捡到配方类物品，也不会在仓库里学习它们。

配方可用条件是制作系统的状态判断，由 `RecipeManager` 根据配置条件计算：

```text
default：默认开放
research：研究完成
chapter：章节进度
task：任务完成
shop_level：店铺等级
fixture：设备/家具启用
vendor：商人供应或声望
location_event：地点事件
```

---

## 7.2 配方字段

`recipes.tab` 建议字段：

```text
recipe_id
display_name
output_item
output_count
required_items
required_research
required_conditions
craft_time
quality_tier
enabled_version
```

`required_conditions` 只引用条件，不引用仓库物品。

---

## 7.3 开放流程

```text
玩家打开制作所。
↓
RecipeManager 读取 recipes.tab。
↓
检查 required_research 与 required_conditions。
↓
条件满足：配方可制作。
↓
条件不满足：按 UI 规则显示缺口或隐藏。
↓
条件变化时刷新制作所列表。
```

## 8. 制作所系统

## 8.1 制造所定位

制造所将仓库材料转化为店铺可售物资，并在对应配方、研究或配方可用条件开放后支持装备和消耗品制作。

可制作类型：

```text
可售物资，白天开店出售。
同类别资源转化，消耗大于产出。
消耗品
装备
背包装备
耳机模块，后续
工具模块，后续
```

---

## 8.2 配方字段

建议字段：

```text
recipe_id
display_name
output_item_id
output_quantity

required_materials
```

## 8.3 配方可用条件来源

配方可由以下方式解锁：

```text
默认解锁。
开放配方可用条件。
研究所节点解锁。
剧情或任务解锁，后续。
```

V0.2 初期建议：

```text
基础可售物资配方默认显示或默认解锁。
基础消耗品配方可以默认显示，但携带和使用仍受出发准备解锁限制。
消耗品携带与局内使用功能必须通过研究解锁。
背包装备和模块需要配方可用条件。
高级消耗品需要研究或配方可用条件。
```

## 8.5 制作完成方式

V0.2 建议制作即时完成。

流程：

```text
玩家选择配方。
系统检查配方是否解锁。
系统检查仓库材料是否足够。
系统检查产物入库空间。
显示制作确认。
玩家确认。
WarehouseManager 扣除材料。
CraftingManager 生成产物实例。
尝试放入仓库。
保存仓库和配方状态。
```

## 8.6 仓库满时的制作产物

推荐规则：

```text
制作前先预检查产物空间。
空间足够时允许制作。
空间不足时按钮置灰，并提示仓库空间不足。
```

但为了兼容未来制作倒计时或奖励领取：

```text
如果材料已扣除且产物生成时仓库满，产物进入待领取列表。
待领取物不占仓库格。
待领取物不可用于研究、制作或出发准备。
玩家整理仓库后可领取。
领取时再次尝试放入仓库。
```

V0.2 首版可以只做“空间不足不能制作”。

## 8.7 制作取消与材料返还

由于 V0.2 建议即时制作：

```text

材料扣除失败则制作失败，不生成产物。
```

## 17. 当前设计体检与建议

## 17.1 合理点

制作所能把“带回材料”变成明确经营目标，研究所能把“开店赚来的矿币”变成明确成长目标。

玩家会从“这一局拿什么值钱”进一步过渡到“我今天怎么把库存变成矿币，再把矿币投向哪项成长”。

这会让容器随机收益更有长期意义。

---

## 17.2 需要注意的点

### 不要让研究过早削弱局内压力

负重、仓库、存储容量都能降低局内取舍压力。

V0.2 第一版升级幅度要小，最好每级只给轻微提升。

### 不要让配方可用条件卡死核心成长

配方可用条件适合解锁可选配方，不适合卡住基础负重、仓库容量这种核心成长。

核心成长应主要依赖可规划材料。

### 制作产物不要太强

如果消耗品能大幅恢复稳定值，玩家会变成靠药硬顶黑暗。

V0.2 消耗品应是补救工具，不是重置风险的按钮。

## 20. 本文档结论

研究所与制作所是局外成长的第一层闭环。

V0.2 第一版应保持克制：

```text
研究即时完成。
制作即时完成。
配方可用条件永久解锁。
材料从仓库扣除。
产物进入仓库。
核心升级不依赖随机配方可用条件。
```

这样玩家每一局带回来的东西，都会自然转化为下一局更明确的目标和更强的准备能力。
