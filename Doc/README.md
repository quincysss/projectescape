# 《废土生存法则》文档导航与整理规则

> 本文件是 `Doc/` 的人工导航索引。  
> AI 开工必须先读 `00_AI开工入口.md`、`00_AI文档读取规则.md` 和 `系统文档索引表.md`，需求文档整理还必须读取 `00_BDD需求文档规范.md`，再进入具体系统文档。

## 2. 推荐阅读顺序

### 新任务开工

```text
1. 先读 Doc/00_AI开工入口.md
2. 再读 Doc/00_AI文档读取规则.md
3. 如果整理需求文档，读取 Doc/00_BDD需求文档规范.md
4. 查 Doc/系统文档索引表.md，定位目标系统
5. 读取目标系统目录下的 00_AI开工入口.md
6. 只读取系统入口要求的母版、拆分文档、验收清单和 AI 开工提示词
7. 对照工程现状后再执行代码、资源或文档修改
```

### 文档整理或修订

```text
1. 先确认要改的是“设计母版”还是“落地执行文档”
2. 检查 Doc/系统文档索引表.md 和目标系统 00_AI开工入口.md 是否需要同步
3. 需求文档整理按 BDD 写 `Feature / Scenario / Given / When / Then`
4. 母版只改长期规则、体验目标、系统边界
5. 拆分文档只改可执行路径、接口、字段、验收标准、AI 开工提示词
6. 如果工程已经稳定实现，以工程事实反写落地文档
7. 如果工程只是临时绕过，不直接推翻母版，先标注当前未实现和风险
```

## 4. 增删改原则

### 新增文档

- 新系统进入开发前，优先新增系统目录，而不是继续堆一个超长母版。
- 新增系统目录时，必须新增 `00_AI开工入口.md`，并同步 `系统文档索引表.md`。
- 新增或重整需求文档时，必须按 `00_BDD需求文档规范.md` 写清 BDD 场景。
- 新目录建议包含：

```text
00_系统总览与落地边界.md
01_核心规则或入口流程.md
02_数据配置与字段.md
03_Godot模块与接口.md
04_调试工具与验证清单.md
05_AI_MCP开工提示词.md
```

### 修改文档

- 修改母版时，写清“为什么改”，避免把工程临时实现误写成长期规则。
- 修改拆分文档时，优先补路径、字段、接口、验收标准。
- 任何影响 AI 读取路径的修改，都必须同步 `00_AI开工入口.md`、`系统文档索引表.md`、skill 和 doc-map。
- 如果发现同一规则在多个文档冲突，优先级为：

```text
已稳定运行的工程事实
更具体的拆分执行文档
系统设计母版
项目总纲
旧的专题说明或历史描述
```

### 删除或归档文档

- 不直接删除设计文档；先标记为归档候选。
- 可删除的对象通常只包括：
  - 明确重复且内容已被新文档完整吸收的旧稿。
  - 已确认不再使用的临时提示词。
  - 与工程事实冲突且已无设计价值的过期执行说明。
- 删除前至少记录：

```text
删除原因：
已被哪个文档替代：
是否影响 AI 开工提示词：
是否影响工程路径：
```

## 5. 后续整理路线

1. 先维护 `02` 至 `09` 的 V0.1 主线，确保每个系统都有“母版 + 拆分 + 验收 + AI 开工提示词”。

4. 最后再决定 `11` 至 `24` 哪些需要拆成目录，哪些保持专题母版。
5. 所有归档/删除动作放在最后一轮做，避免误删仍被代码或提示词引用的文档。
