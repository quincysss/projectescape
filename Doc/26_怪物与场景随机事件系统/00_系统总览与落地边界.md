# 26-00 怪物与场景随机事件系统总览

## 2026-05-13 怪物危机感调参说明

当前实现以 `setting/monster_defs.tab` 为准：怪物巡逻速度为 `168 px/s`，冲撞速度为 `1680 px/s`，视野半径为 `920 px`，玩家连续处于扇形视野内 `1 秒` 后触发冲撞。旧设计稿中出现的 `5 秒警戒` 已不再作为当前落地标准。

怪物成功攻击玩家并消失后，会在相同出生点等待 `30 秒` 补刷一只。补刷只由 `hit_player` 消失触发，不由普通清理、调试清空或帧循环触发；同一出生点同一时间最多挂一个待补刷任务。

> 目标：定义从第三日开始启用的场景随机事件系统，并落地怪物、时间变化、随机障碍三类事件的规则边界、生成点位、GM 调试和 AI/MCP 开工入口。

## 1. 系统定位

场景随机事件不是常驻系统，而是从第三日开始纳入局内探索的动态变化池。

它承担三件事：

- 让第三日之后的地表探索出现新的动态威胁。
- 让每局探索在时间压力、街道路线、怪物风险上出现可控变化。
- 为后续下雨、污染风暴、高收益窗口等事件建立统一框架。

## 2. 最新规则摘要

```text
第 1 日：不自然触发任何场景随机事件。
第 2 日：不自然触发任何场景随机事件。
第 3 日：场景随机事件池启用，怪物事件必出。
第 4 日及以后：按事件配置概率抽取。
GM 面板可强制指定事件。
```

当前事件：

| 事件 | 规则 |
| --- | --- |
| 怪物出现 | 第 3 日必出，第 4 日后 50%，10 个怪物点抽 4 个 |
| 超级时间 | 本局对局时长从 5 分钟改为 8 分钟 |
| 超短时间 | 本局对局时长从 5 分钟改为 3 分 30 秒 |
| 随机障碍 | 生成 2-4 个玩家不可通过、怪物可穿过的道路障碍 |

如果本局命中怪物事件：

```text
地图预先布置 10 个怪物街道刷新点。
本局从 10 个合法点中抽取 4 个。
每个被抽中的点生成 1 个怪物。
怪物只在街道上出现。
怪物只在出生点附近的街道巡逻。
```

怪物行为：

```text
默认巡逻。
怪物有可见扇形视野。
扇形默认淡绿色。
玩家进入扇形后开始 1 秒警戒计时。
警戒期间扇形由绿色逐渐变黄，黄色填充整个扇形。
满 1 秒后扇形变红，怪物冲向玩家。
怪物碰到玩家碰撞体后消失。
玩家稳定值立刻下降 20 点。
该怪物原出生点 30 秒后补刷 1 只新怪物。
```

如果本局命中时间事件：

```text
默认对局时间为 5:00。
超级时间改为 8:00。
超短时间改为 3:30。
两个时间事件互斥，同一局最多生效一个。
时间事件只影响当前局，不写回全局默认配置。
```

如果本局命中随机障碍事件：

```text
地图预先布置 10 个随机障碍候选点。
本局从合法点中抽取 2-4 个。
障碍使用现有地图资源生成。
玩家不能穿过障碍。
怪物可以穿过障碍。
生成后必须验证主流程路线仍可达。
```

## 4. 关键术语

| 术语 | 说明 |
| --- | --- |
| 场景随机事件 | 每次正式进入地表前，根据日期和概率抽取的局内环境事件 |
| 事件槽位 | 将事件按 threat/time_modifier/map_blocker 等类型分组，控制互斥与叠加 |
| 怪物事件 | threat 槽位事件，控制本局是否生成怪物 |
| 时间事件 | time_modifier 槽位事件，控制本局倒计时长度 |
| 随机障碍事件 | map_blocker 槽位事件，控制本局是否生成道路障碍 |
| 怪物刷新点 | 只允许放在街道可通行区域的点位 |
| 障碍候选点 | 用于随机生成障碍的街道点位 |
| 巡逻半径 | 怪物围绕出生点附近街道来回游荡的限制范围 |
| 扇形视野 | 怪物前方可见的世界 UI 区域，绿色/黄色/红色表现警戒进度 |
| 警戒计时 | 玩家持续处于扇形范围内的累计时间，满 1 秒进入冲撞 |
| 冲撞 | 怪物直接冲向玩家，碰撞后怪物消失并扣玩家稳定值 |

## 5. 硬约束

- 第 1 日和第 2 日不自然触发场景随机事件。
- 第 3 日开始启用场景随机事件池。
- GM 强制事件优先级高于自然概率。
- 同一局可以叠加不同槽位事件。
- 同一互斥组内最多只有一个事件生效。
- 时间事件只影响当前局倒计时。
- 随机障碍必须保留主流程可达路线。
- 随机障碍阻挡玩家，但不阻挡怪物。
- 怪物不在第 1 日和第 2 日自然出现。
- 第 3 日怪物事件必定触发。
- 第 4 日及以后怪物事件每日 50% 概率触发。
- 怪物刷新点必须位于街道可通行区。
- 怪物不能生成在家安全区、前哨安全区、建筑不可通行区、地图边界外。
- 本局怪物事件命中时，10 个刷新点中抽 4 个生成怪物。
- 怪物巡逻不能离开出生点附近的街道。
- 扇形视野必须显示给玩家。
- 玩家在扇形内累计 1 秒后，怪物进入冲撞。
- 冲撞碰到玩家后怪物消失，稳定值下降 20 点。

## 7. 当前落地状态（2026-05-12）

本次已落地 V0.1 怪物事件主链路：

- `monster_presence` 已接入场景随机事件框架：第 3 日必出，第 4 日及以后按 `setting/scene_random_events.tab` 的 50% 概率触发。
- `RunEventContext` 已写入 `RunContext.scene_events`，并记录 `monster_event_active`、`monster_spawn_point_ids`、`active_monster_ids`、`monster_hit_count` 与 `trigger_reasons`。
- `RunScene` 已读取 `WorldRoot/MapLayout/Points/MonsterSpawnPoints`，在怪物事件命中时从 10 个街道点抽 4 个生成怪物。
- 怪物实体使用 `scenes/entities/monsters/MonsterBasic.tscn`，主表现为 `BlackTideBoundaryEssence.tscn`。
- 怪物状态已落地为巡逻、警戒、冲撞、移除；扇形视野可见，警戒 1 秒后冲撞，命中玩家后怪物消失并让稳定值下降 20。
- GM 面板已补 `本日必出怪物`，通过 `GameState.debug_force_monster_presence_next_run()` 强制下一次出发触发怪物事件。

本次未落地时间事件与随机障碍事件的实体逻辑；它们仍按本目录 `06`、`07` 文档作为后续扩展。
