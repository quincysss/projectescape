# 26-03 怪物巡逻、视野警戒与冲撞

> 目标：定义怪物生成后的行为状态机、出生点附近巡逻、可见扇形视野、警戒填充、红色冲撞和稳定值伤害。

## 1. 当前落地状态

怪物是局内移动威胁，不是可击杀敌人。它的核心作用是压迫玩家走位、制造街道路线风险，并通过冲撞扣除稳定值。

当前实体：

| 路径 | 说明 |
| --- | --- |
| `res://scenes/entities/monsters/MonsterBasic.tscn` | 怪物逻辑场景 |
| `res://scenes/entities/monsters/BlackTideBoundaryEssence.tscn` | 黑潮边界残影表现资源 |
| `res://scripts/monsters/monster_basic.gd` | 巡逻、警戒、冲撞、命中移除 |
| `res://scripts/monsters/monster_spawn_controller.gd` | 根据场景随机事件生成怪物 |
| `res://scripts/monsters/monster_vision_cone.gd` | 扇形视野绘制 |
| `res://setting/monster_defs.tab` | 怪物数值配置 |

## 2. 状态机

```text
PATROL -> WARNING -> CHARGE -> REMOVED
```

| 状态 | 说明 |
| --- | --- |
| `PATROL` | 沿刷新点子层 `PatrolPath/Patrol_XX` 折返巡逻，扇形为淡绿色 |
| `WARNING` | 玩家处于当前扇形内，开始 2 秒警戒，扇形逐渐填充黄色 |
| `CHARGE` | 警戒满 2 秒，扇形变红，怪物高速冲向玩家 |
| `REMOVED` | 命中玩家后消失，并对稳定值造成一次伤害 |

## 4. 巡逻路径

怪物刷新点位于：

```text
RunScene/WorldRoot/MapLayout/Points/MonsterSpawnPoints
```

每个怪物刷新点推荐结构：

```text
monster_spawn_street_001
└── PatrolPath
    ├── Patrol_01
    ├── Patrol_02
    └── Patrol_03
```

规则：

- 如果刷新点下存在 `PatrolPath` 子节点，怪物优先按子节点全局坐标巡逻。
- 巡逻路径应放在街道内，不穿过街区块、家、前哨站或地图外区域。
- 路径点越远，怪物转向频率越低，视觉上更稳定。
- 如果没有有效 `PatrolPath`，系统使用 `patrol_radius_px` 沿出生点附近左右兜底巡逻。
- 怪物巡逻只负责表现和威胁，不参与复杂寻路。

## 5. 扇形视野与警戒

视野扇形是世界内 UI，跟随怪物移动和朝向。

规则：

- 默认颜色为淡绿色。
- 玩家进入扇形后进入 `WARNING`。
- 玩家持续处于扇形内时，`warning_timer` 累加。
- `warning_timer / warning_seconds` 控制黄色填充比例。
- 当前 `warning_seconds = 2.0`，即玩家连续暴露 2 秒后触发冲撞。
- 玩家短暂离开视野时保留 `0.18 秒` 缓冲，减少边缘抖动。
- 玩家离开超过缓冲时间后，警戒计时回落并恢复巡逻。

视野命中判断必须同时满足：

```text
玩家距离 <= vision_radius
玩家方向与怪物当前朝向夹角 <= vision_angle / 2
```

## 6. 冲撞与伤害

触发条件：

```text
warning_timer >= 2.0
```

冲撞规则：

- 怪物进入 `CHARGE` 后锁定当前玩家位置方向。
- 怪物按 `charge_speed = 1680 px/s` 冲刺。
- 怪物命中玩家碰撞体或进入 `hit_radius = 128 px` 内后消失。
- 命中后稳定值下降 `20`。
- 每只怪物只造成一次伤害。

## 7. 编辑器摆放注意

- 怪物刷新点只放在街道上。
- `PatrolPath` 子节点是可编辑轨迹线，后续调整巡逻长度时优先改这些点。
- 视野方向应以怪物资源中“眼睛看向的方向”为准。
- 怪物表现、视野扇形和命中半径需要一起调，避免画面看到碰到但逻辑没命中，或逻辑命中但视觉过远。
- 怪物不进入家、前哨站、街区块不可通行区域。
