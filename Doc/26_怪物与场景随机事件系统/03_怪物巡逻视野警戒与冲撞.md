# 26-03 怪物巡逻、视野警戒与冲撞

> 目标：定义怪物生成后的行为状态机、出生点附近巡逻、可见扇形视野、5 秒警戒、红色冲撞和稳定值伤害。

## 1. 实现目标

V0.1 怪物不是复杂战斗单位，而是“会巡逻的稳定值陷阱”。

玩家需要观察怪物扇形视野，避免在视野内停留太久。

## 2. 怪物状态机

```text
SPAWN
-> WANDER
-> WARNING
-> CHARGE
-> DISAPPEARED
```

| 状态 | 说明 |
| --- | --- |
| `SPAWN` | 初始化位置、巡逻范围、视野扇形 |
| `WANDER` | 在出生点附近街道来回游荡，扇形为淡绿色 |
| `WARNING` | 玩家处于扇形内，开始 5 秒警戒，扇形逐渐变黄 |
| `CHARGE` | 警戒满 5 秒，扇形变红，怪物冲向玩家 |
| `DISAPPEARED` | 碰到玩家后消失，或 Debug 强制移除 |

## 3. 巡逻规则

怪物只在出生点附近街道游荡。

默认建议值：

| 参数 | 值 |
| --- | ---: |
| 巡逻半径 | 180 |
| 巡逻速度 | 45 |
| 巡逻换向间隔 | 2-4 秒 |
| 巡逻目标点数量 | 2-4 个 |

巡逻限制：

- 怪物不能离开 `patrol_radius`。
- 怪物不能进入建筑碰撞。
- 怪物不能进入家安全区。
- 怪物不能进入已修复前哨安全区。
- 找不到合法巡逻目标时，原地等待 1 秒后重新取点。

## 4. 扇形视野规则

视野必须显示给玩家。

默认建议值：

| 参数 | 值 |
| --- | ---: |
| 扇形角度 | 70 度 |
| 扇形半径 | 220 |
| 警戒时长 | 5 秒 |
| 默认颜色 | 淡绿色 |
| 警戒颜色 | 黄色填充 |
| 冲撞颜色 | 红色 |

可视表现：

```text
WANDER：整个扇形淡绿色。
WARNING：淡绿色底上叠加黄色填充，黄色按警戒进度从 0% 增长到 100%。
CHARGE：整个扇形变红，怪物冲向玩家。
```

颜色建议：

| 状态 | 颜色 | 透明度 |
| --- | --- | ---: |
| WANDER | `#8BE07A` | 0.18 |
| WARNING | `#E5C84A` | 0.30-0.45 |
| CHARGE | `#D94A3A` | 0.45 |

## 5. 玩家进入扇形后的规则

当玩家进入扇形范围：

```text
进入 WARNING。
开始 warning_timer。
怪物移动路线不改变。
怪物不改成追逐移动。
视野扇形/身体朝向可以转向玩家，用于表现“盯住玩家”。
```

这里的“不会改变移动朝向，只会朝着玩家”落地解释为：

```text
怪物仍沿原巡逻路径移动，不因为玩家进入视野而改变巡逻目标。
但怪物的注意方向/扇形朝向会锁向玩家。
玩家离开扇形后，怪物恢复巡逻朝向。
```

警戒计时：

```text
warning_progress = warning_timer / 5.0
```

- 玩家持续在扇形内，计时增加。
- 玩家离开扇形，计时重置为 0。
- 为避免边界抖动，可允许 0.2 秒离开缓冲。
- 计时未满时，怪物不冲撞。
- 计时满 5 秒后，进入 `CHARGE`。

## 6. 冲撞规则

触发条件：

```text
warning_timer >= 5.0
```

冲撞行为：

```text
扇形变红。
怪物停止普通巡逻。
怪物直接冲向玩家当前位置。
```

默认建议值：

| 参数 | 值 |
| --- | ---: |
| 冲撞速度 | 180 |
| 最大冲撞时长 | 6 秒 |
| 最大离巢距离 | 480 |

冲撞结束：

| 条件 | 结果 |
| --- | --- |
| 碰到玩家碰撞体 | 怪物消失，玩家稳定值 -20 |
| 超过最大冲撞时长 | 回到 WANDER，警戒清零 |
| 超过最大离巢距离 | 回到出生点附近，警戒清零 |
| 玩家进入家安全区 | 怪物不进入安全区，回到 WANDER |

## 7. 稳定值伤害

怪物碰撞玩家时：

```text
StabilityComponent.apply_instant_delta(-20, "monster_contact")
```

规则：

- 每个怪物只造成一次稳定值伤害。
- 伤害后该怪物立即消失。
- 如果稳定值因此 <= 0，由 04 稳定值死亡流程处理。
- 稳定值伤害不直接处理结算，不直接清空背包。

## 11. 验收标准

- 怪物只在出生点附近街道巡逻。
- 怪物有可见扇形视野。
- 默认扇形为淡绿色。
- 玩家进入扇形后黄色填充开始增长。
- 玩家离开扇形后警戒计时重置。
- 玩家在扇形内满 5 秒后扇形变红。
- 红色后怪物冲向玩家。
- 怪物碰到玩家后消失。
- 玩家稳定值下降 20 点。
- 稳定值降到 0 时走现有死亡流程。

- 怪物表现默认缩放从 `0.45` 放大到 `0.68`。

## 13. 视觉转向与 EyeFocus 稳定规则（2026-05-12）

怪物转向时禁止翻转 `MonsterBasic/Visual` 根节点的 `scale.x`。

原因：
- `VisionCone` 的绘制锚点与判定锚点都来自表现资源内的 `EyeFocus`。
- 如果翻转整个 `Visual` 根节点，`EyeFocus` 会在左右两侧瞬移，玩家站在边界附近时会造成扇形抖动、警戒进度闪烁，视觉上像同时闪出两个怪物。

当前落地规则：
- `MonsterBasic` 只把 `face_direction` 传给表现资源。
- `black_tide_boundary_essence_visual.gd` 只翻转 `BodySprite.flip_h`。
- `EyeFocus.position.x` 按朝向显式镜像，始终贴近资源中眼睛所在一侧。
- `VisionCone.global_position` 每帧同步到 `get_vision_origin_global()`，即当前镜像后的 `EyeFocus.global_position`。

回归验证：
- `tests/monster_spawn_and_behavior_check.gd` 会强制怪物左右转身，确认 `Visual.scale.x` 保持稳定、`BodySprite.flip_h` 正常切换、`EyeFocus` 与扇形锚点不脱节。
