# 26-02 怪物街道点位与生成规则

> 目标：定义怪物刷新点的地图布置、每日生成、10 点出 4 个怪物、合法性检查和与地图点位池的关系。

## 1. 实现目标

在地图中布置 10 个怪物刷新点。

当本局 `monster_presence` 事件激活时：

```text
从 10 个合法怪物刷新点中随机抽取 4 个。
每个点生成 1 个怪物。
怪物只在该点附近街道巡逻。
```

## 2. 点位布置规则

怪物刷新点必须：

- 放在街道可通行区域。
- 远离家安全区。
- 不放在前哨安全区内。
- 不放在建筑区块碰撞内。
- 不放在容器、材料、撤离点正上方。
- 不堵住主路。
- 附近必须有足够街道空间供怪物来回游荡。

建议命名：

```text
monster_spawn_street_001
monster_spawn_street_002
...
monster_spawn_street_010
```

Group：

```text
monster_spawn_street
```

## 3. 生成规则

触发条件：

```text
RunEventContext.monster_presence.active == true
```

生成数量：

```text
total_spawn_points = 10
active_monster_count = 4
```

抽取规则：

```text
1. 读取 group=monster_spawn_street 的点位。
2. 过滤非法点位：不可达、被占用、安全区内、非街道。
3. 按 run_seed 洗牌。
4. 选前 4 个点位。
5. 每个点位生成 1 个怪物。
```

失败规则：

- 合法点位少于 4 个时，Debug 模式报错并高亮问题点。
- 正式运行中，合法点不足时只生成可用数量，但必须写日志。
- 合法点位为 0 时，不生成怪物，事件结果标记为 `spawn_failed_no_valid_points`。

## 4. 街道限定

怪物只允许在街道上生成。

合法性检查：

```text
point.position 位于 StreetWalkable。
point.position 不位于 BlockSolid。
point.position 不位于 SafeArea。
point.position 不位于 OutpostSafeArea。
```

怪物巡逻点也必须从该刷新点附近街道中选取，不能跨进建筑区块。

当前推荐编辑结构：

```text
monster_spawn_street_001
└── PatrolPath
    ├── Patrol_01
    └── Patrol_02
```

规则：

- `PatrolPath` 是编辑器整理节点，不参与点位计数。
- `Patrol_XX` 使用 `Marker2D`。
- 生成怪物时会读取刷新点子层中名称以 `Patrol` 开头的 `Node2D` 作为巡逻轨迹点。
- 怪物会在这些轨迹点之间折返巡逻；没有轨迹点时才回退到出生点附近随机取点。
- 巡逻轨迹点必须位于 `StreetWalkable`，不得落入 `BlockSolid`。

## 5. 与容器/材料点位的关系

怪物点位与容器、材料相似，但必须独立点位池：

```text
容器点位：container_spawn_*
材料点位：outpost_material_spawn_*
怪物点位：monster_spawn_street
```

禁止：

- 怪物和容器共用同一个点位节点。
- 怪物生成在前哨材料正上方。
- 怪物生成在家撤离点附近形成强制碰撞。

## 6. 数据结构

`MonsterSpawnPoint` 字段：

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `point_id` | `StringName` | 必填 | 如 `monster_spawn_street_001` |
| `point_type` | `StringName` | `monster` | 点位类型 |
| `patrol_radius` | `float` | `180` | 巡逻半径，像素或项目当前单位 |
| `street_only` | `bool` | `true` | 是否强制街道校验 |
| `min_day` | `int` | `3` | 最早自然启用日 |
| `tags` | `Array[StringName]` | `[]` | `street_edge/plaza/alley` |
| `is_enabled` | `bool` | `true` | 是否启用 |

## 9. 验收标准

- 地图中存在 10 个 `monster_spawn_street` 点位。
- 所有点位位于街道可通行区。
- 本局怪物事件未激活时不生成怪物。
- 第 3 日事件激活后生成 4 个怪物。
- 第 4 日后事件命中时生成 4 个怪物。
- 生成点不会与容器/材料/撤离点重叠。
- Debug Overlay 可显示 10 个怪物点和本局抽中的 4 个点。
