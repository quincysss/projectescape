## 3. GM 强制规则

GM 强制是一次性标记，只影响下一次正式进入地表。

推荐字段：

```gdscript
{
    "force_scene_events_next_run": {
        "monster_presence": true,
        "super_time": false,
        "short_time": false,
        "road_obstacle": true
    }
}
```

规则：

- 触发后自动清除。
- 第 1 日/第 2 日也可以被 GM 强制触发，用于测试。
- 正式玩家 UI 不显示 GM 标记。
- Debug 日志需要记录 `trigger_reason=gm_forced`。
- `super_time` 与 `short_time` 互斥，后设置的强制覆盖先设置的强制。

## 5. 验证清单

每日事件：

| 编号 | 标准 |
| --- | --- |
| E1 | 第 1 日自然不触发场景随机事件 |
| E2 | 第 2 日自然不触发场景随机事件 |
| E3 | 第 3 日场景随机事件池启用 |
| E4 | 第 3 日怪物必出 |
| E5 | 第 4 日以后按事件配置概率抽取 |
| E6 | GM 强制事件能无视自然概率命中 |

时间事件：

| 编号 | 标准 |
| --- | --- |
| T1 | 默认对局倒计时为 05:00 |
| T2 | 强制超级时间后倒计时为 08:00 |
| T3 | 强制超短时间后倒计时为 03:30 |
| T4 | 超级时间和超短时间不会同局同时生效 |
| T5 | 时间事件只影响当前局 |

怪物点位生成：

| 编号 | 标准 |
| --- | --- |
| M1 | 地图有 10 个怪物街道点位 |
| M2 | 本局命中后生成 4 个怪物 |
| M3 | 怪物不会生成到建筑/安全区/地图外 |
| M4 | Debug Overlay 能显示抽中点位 |

怪物行为：

| 编号 | 标准 |
| --- | --- |
| B1 | 怪物在出生点附近街道巡逻 |
| B2 | 视野扇形默认淡绿色 |
| B3 | 玩家进入扇形后黄色进度增长 |
| B4 | 玩家 5 秒内离开扇形会重置 |
| B5 | 玩家停留满 5 秒后扇形变红 |
| B6 | 红色后怪物冲向玩家 |
| B7 | 碰撞后怪物消失，稳定值 -20 |

随机障碍：

| 编号 | 标准 |
| --- | --- |
| O1 | 地图有 10 个随机障碍候选点 |
| O2 | 障碍事件命中后生成 2-4 个障碍 |
| O3 | 玩家不能穿过障碍 |
| O4 | 怪物可以穿过障碍 |
| O5 | 障碍不会堵死前哨/材料/撤离路线 |
| O6 | Debug Overlay 能显示障碍点和路径校验结果 |
