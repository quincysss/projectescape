# 26-07 随机障碍事件规则

> 目标：定义随机障碍作为场景随机事件的生成、碰撞、资源复用、路径校验和 AI/MCP 开工边界。

## 1. 实现目标

随机障碍是从第 3 日开始进入事件池的地图变化事件。

事件命中后，系统从地图预置的障碍点位中随机抽取若干点生成障碍。

障碍会阻挡玩家通行，但怪物可以穿过。

## 2. 事件定义

事件 ID：

```text
road_obstacle
```

默认建议：

| 参数 | 值 |
| --- | ---: |
| 最早出现日 | 第 3 日 |
| 每日概率 | 30% |
| 候选点位数量 | 10 |
| 本局生成数量 | 2-4 |
| 是否可与怪物叠加 | 是 |
| 是否可与时间事件叠加 | 是 |

概率和生成数量均为 V0.1 建议值，最终可通过配置表调整。

## 3. 可用资源

可直接复用现有资源：

```text
res://assets/map/decals/overlays/ground_decal_overlay_002.png
res://assets/map/props/placement/road_cone_01.png
res://assets/map/props/placement/broken_signboard_blank_01.png
```

资源使用建议：

| 资源 | 用法 |
| --- | --- |
| `ground_decal_overlay_002.png` | 地面裂痕/污染痕迹，需要配合不可见碰撞体 |
| `road_cone_01.png` | 小型路障，可组合 2-3 个形成阻挡 |
| `broken_signboard_blank_01.png` | 倒塌路牌，适合作为横向阻挡 |

`ground_decal_overlay_002.png` 本身只是贴图，不能只放图片。作为障碍使用时必须额外生成碰撞体。

## 4. 点位规则

随机障碍使用独立点位池：

```text
obstacle_spawn_001
...
obstacle_spawn_010
```

Group：

```text
random_obstacle_spawn
```

点位必须：

- 位于街道可通行区。
- 不在家安全区内。
- 不在前哨安全区内。
- 不在撤离点正上方。
- 不与容器、材料、怪物出生点完全重叠。
- 生成障碍后仍保留至少一条前往前哨和撤离点的可达路线。

## 5. 生成规则

当 `road_obstacle` 命中：

```text
1. 读取 random_obstacle_spawn 点位。
2. 过滤非法点位。
3. 按 run_seed 洗牌。
4. 在 2-4 之间抽取本局障碍数量。
5. 选取对应数量的点位。
6. 为每个点位随机选择一种障碍资源。
7. 生成 RandomObstacleEntity。
8. 运行路径校验；如果阻断主流程，回滚该障碍并换点。
```

障碍数量建议：

```text
obstacle_count_min = 2
obstacle_count_max = 4
```

如果合法点位不足：

- Debug 模式报错并高亮非法点位。
- 正式运行中只生成可用数量。
- 如果 0 个合法点位，事件结果标记为 `spawn_failed_no_valid_obstacle_points`。

## 6. 碰撞规则

核心规则：

```text
玩家不能穿过障碍。
怪物可以穿过障碍。
障碍不阻挡怪物巡逻、视野、冲撞。
障碍不被玩家拾取或破坏。
```

推荐碰撞分层：

| 对象 | 碰撞设置 |
| --- | --- |
| 玩家 | 与 `player_only_obstacle` 碰撞 |
| 怪物 | 不与 `player_only_obstacle` 碰撞 |
| 障碍 | 使用 `StaticBody2D` 或项目等价阻挡体 |

如果项目已有统一碰撞层命名，以现有命名为准，但必须保证“玩家阻挡、怪物忽略”。

## 7. 路径校验

每次障碍生成后必须验证：

- 玩家从家安全区能到达第一前哨候选点。
- 玩家从家安全区能到达第二前哨候选点。
- 玩家能到达撤离点。
- 前哨材料刷新区域仍可达。
- 没有障碍直接覆盖交互触发区域。

如果某个障碍导致不可达：

```text
移除该障碍。
将点位标记为本局不可用。
重新抽取下一个合法点。
最多重试 20 次。
```

重试失败时，减少本局障碍数量，不强行生成。

## 8. RunContext 字段

建议字段：

```gdscript
{
    "obstacle_event_active": true,
    "obstacle_spawn_point_ids": [
        "obstacle_spawn_003",
        "obstacle_spawn_007"
    ],
    "active_obstacle_ids": [],
    "obstacle_assets": {
        "obstacle_spawn_003": "road_cone_01",
        "obstacle_spawn_007": "broken_signboard_blank_01"
    }
}
```

## 9. Debug 表现

Debug Overlay 需要显示：

- 全部障碍候选点。
- 本局抽中的障碍点。
- 生成的障碍资源名。
- 因路径阻断被拒绝的点位。
- 玩家路径校验结果。
- 怪物是否忽略障碍碰撞。

```text

读取 Doc/26_怪物与场景随机事件系统/07_随机障碍事件规则.md 和 Doc/03_地图与安全区规则/15_随机障碍点位.md。
新增 road_obstacle 事件，从第 3 日开始进入场景随机事件池，建议自然概率 30%。
地图中读取 random_obstacle_spawn group 的 10 个候选点，事件命中后生成 2-4 个 RandomObstacleEntity。
障碍资源允许使用 ground_decal_overlay_002.png、road_cone_01.png、broken_signboard_blank_01.png。
玩家不能穿过障碍，怪物必须能穿过障碍。
生成后必须做路径校验，不能堵死家到前哨、材料点和撤离点的路线。
完成后提供 Debug 日志：事件是否命中、抽中点位、资源类型、路径校验、被拒绝点位。
```

## 11. 验收标准

| 编号 | 标准 |
| --- | --- |
| O1 | 第 1/2 日自然不会生成随机障碍 |
| O2 | 第 3 日后 `road_obstacle` 可自然命中 |
| O3 | 事件命中后从 10 个候选点中生成 2-4 个障碍 |
| O4 | 玩家无法穿过障碍 |
| O5 | 怪物可以穿过障碍 |
| O6 | 障碍不会完全堵死主流程路线 |
| O7 | Debug Overlay 能显示候选点、抽中点和拒绝原因 |
