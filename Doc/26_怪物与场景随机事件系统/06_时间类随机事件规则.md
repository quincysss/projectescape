# 26-06 时间类随机事件规则

> 目标：定义场景随机事件中的对局时长改写规则，让超级时间和超短时间只影响命中的单局，不污染全局配置。

## 1. 实现目标

时间类随机事件用于改变本局倒计时。

V0.1 默认对局时长仍为 5 分钟。

当本局命中时间类事件时，`RunTimerController` 使用事件结果中的 `run_duration_seconds` 初始化倒计时。

## 2. 事件列表

| 事件 ID | 显示名 | 效果 | 对局时长 |
| --- | --- | --- | ---: |
| `super_time` | 超级时间 | 本局探索时间变长 | 480 秒 |
| `short_time` | 超短时间 | 本局探索时间变短 | 210 秒 |

换算：

```text
默认时间：5:00 = 300 秒
超级时间：8:00 = 480 秒
超短时间：3:30 = 210 秒
```

## 3. 触发边界

时间类事件从第 3 日开始纳入场景随机事件池。

第 1 日和第 2 日不自然触发时间类事件。

GM 可以强制第 1 日或第 2 日触发，用于调试。

## 4. 互斥规则

`super_time` 和 `short_time` 属于同一个互斥组：

```text
conflict_group = run_duration
```

同一局最多只能命中一个时间类事件。

如果 Debug 或配置错误导致两个事件同时命中，按以下优先级处理：

```text
GM 强制事件 > 自然命中事件
后设置的 GM 强制 > 先设置的 GM 强制
自然命中冲突时按事件权重二次抽一个
```

最终 `RunEventContext` 只能写入一个 `active_time_event_id`。

## 5. 与其他事件叠加

时间类事件可以和这些事件叠加：

- 怪物出现。
- 随机障碍。
- 后续天气、污染、特殊资源事件。

时间类事件只改变本局倒计时，不直接改变资源产出、怪物数量或稳定值衰减。

如果后续想让超级时间提高收益，必须另开收益事件或在该事件中明确新增 `loot_multiplier` 字段，不能暗改现有掉落表。

## 6. RunContext 字段

建议字段：

```gdscript
{
    "base_run_duration_seconds": 300,
    "run_duration_seconds": 480,
    "active_time_event_id": "super_time",
    "time_event_payload": {
        "source_event_id": "super_time",
        "duration_seconds": 480
    }
}
```

无时间事件时：

```gdscript
{
    "base_run_duration_seconds": 300,
    "run_duration_seconds": 300,
    "active_time_event_id": ""
}
```

## 7. 初始化流程

```text
1. SceneRandomEventDirector 解析本局事件。
2. 检查是否命中时间类事件。
3. 如果没有命中，run_duration_seconds = 300。
4. 如果命中 super_time，run_duration_seconds = 480。
5. 如果命中 short_time，run_duration_seconds = 210。
6. RunInitializer 把 run_duration_seconds 写入 RunContext。
7. RunTimerController 使用 RunContext.run_duration_seconds 初始化倒计时。
8. UI 只显示实际倒计时，不显示默认时间。
```

## 8. UI 反馈

开局进入局内后，顶部倒计时直接显示最终时长：

```text
超级时间：08:00
超短时间：03:30
默认局：05:00
```

可选短提示：

```text
超级时间：本局地表信号稳定，探索窗口延长至 8 分钟。
超短时间：暗潮提前压近，本局探索窗口缩短至 3 分 30 秒。
```

提示只显示一次，不能遮挡玩家操作。

## 9. 数据配置建议

`scene_random_events.tab` 中建议字段：

```text
event_id	display_name	category	min_day	guaranteed_day	daily_chance	conflict_group	can_stack	gm_forceable	payload	notes
super_time	超级时间	time_modifier	3		0.10	run_duration	false	true	duration_seconds=480	本局对局时长改为8分钟
short_time	超短时间	time_modifier	3		0.10	run_duration	false	true	duration_seconds=210	本局对局时长改为3分30秒
```

概率是 V0.1 建议值，后续可通过表格调参。

## 11. 验收标准

| 编号 | 标准 |
| --- | --- |
| T1 | 第 1/2 日自然不会触发时间类事件 |
| T2 | 第 3 日后可抽取时间类事件 |
| T3 | `super_time` 命中后倒计时从 08:00 开始 |
| T4 | `short_time` 命中后倒计时从 03:30 开始 |
| T5 | 两个时间事件不会同局同时生效 |
| T6 | 时间事件只影响当前局 |
| T7 | Debug/GM 可以强制任一时间事件 |
