# 26-01 场景随机事件框架

> 目标：从第 3 日开始为每局地表探索抽取场景随机事件，并统一承载怪物、时间变化、随机障碍以及后续天气/特殊事件。

## 1. 实现目标

每次正式进入地表探索前，系统生成本局 `RunEventContext`。

`RunEventContext` 决定：

- 本局是否有怪物。
- 本局对局时长是否被改写。
- 本局是否生成随机障碍。
- 触发原因是自然概率、固定保底还是 GM 强制。
- 事件结果写入本局上下文，供 Loading、RunInitializer、RunTimer、MonsterSystem、ObstacleSystem 使用。

## 2. 开启日期

场景随机事件从第 3 日开始自然启用。

第 1 日和第 2 日不自然触发任何场景随机事件，保证玩家先完成基础学习。

因为 `surface_day` 在 loading 完成并按任意按钮进入局内后才 +1，所以抽事件时需要使用“即将进入的地表日”：

```text
run_day_index = profile.surface_day + 1
```

示例：

| 当前 profile.surface_day | 即将进入地表日 run_day_index | 场景随机事件池 |
| ---: | ---: | --- |
| 0 | 1 | 不启用 |
| 1 | 2 | 不启用 |
| 2 | 3 | 启用，怪物必出 |
| 3 | 4 | 启用，按配置抽取 |
| 4 | 5 | 启用，按配置抽取 |

GM 强制事件可以无视日期限制，用于测试。

## 3. 事件槽位

场景随机事件不是只能有一个。为了避免规则互相踩踏，V0.1 按槽位管理：

| 槽位 | 作用 | 当前事件 | 同局数量 |
| --- | --- | --- | --- |
| `threat` | 局内威胁 | `monster_presence` | 最多 1 个 |
| `time_modifier` | 对局时长改写 | `super_time` / `short_time` | 最多 1 个 |
| `map_blocker` | 地图通行变化 | `road_obstacle` | 最多 1 个 |

不同槽位可以叠加。

示例：

```text
怪物 + 超级时间：允许
怪物 + 随机障碍：允许
超级时间 + 随机障碍：允许
超级时间 + 超短时间：不允许
```

## 4. 当前事件列表

| 事件 ID | 槽位 | 最早自然日 | 默认概率 | 效果 |
| --- | --- | ---: | ---: | --- |
| `monster_presence` | `threat` | 3 | 第 3 日必出，第 4 日后 50% | 10 个怪物点抽 4 个生成怪物 |
| `super_time` | `time_modifier` | 3 | 10% | 本局对局时长变为 8 分钟 |
| `short_time` | `time_modifier` | 3 | 10% | 本局对局时长变为 3 分 30 秒 |
| `road_obstacle` | `map_blocker` | 3 | 30% | 从障碍点位中生成 2-4 个玩家不可通过的障碍 |

概率是 V0.1 建议值，最终以 `scene_random_events.tab` 为准。

## 5. 事件抽取顺序

```text
1. RunLoadingController 创建 run_context。
2. SceneRandomEventDirector 读取 profile.surface_day。
3. 计算 run_day_index = surface_day + 1。
4. 读取 GM 强制标记。
5. 如果 run_day_index <= 2 且没有 GM 强制，则不抽自然事件。
6. 按槽位评估事件池。
7. 处理互斥组与 GM 强制优先级。
8. 写入 RunEventContext。
9. RunInitializer 根据 RunEventContext 初始化局内实体、倒计时和地图变化。
```

## 6. 怪物事件规则

事件 ID：

```text
monster_presence
```

触发规则：

| 条件 | 结果 |
| --- | --- |
| GM 强制本日必出怪物 | 必定触发 |
| run_day_index <= 2 | 不自然触发 |
| run_day_index == 3 | 必定触发 |
| run_day_index >= 4 | 50% 概率触发 |

事件结果字段：

```gdscript
{
    "event_id": "monster_presence",
    "slot": "threat",
    "active": true,
    "run_day_index": 3,
    "trigger_reason": "guaranteed_day_3",
    "spawn_point_count": 10,
    "monster_count": 4,
    "seed": 12345
}
```

## 7. 时间事件规则

时间事件只影响当前局的倒计时。

```text
默认局：300 秒
super_time：480 秒
short_time：210 秒
```

`super_time` 和 `short_time` 互斥，同一局最多生效一个。

事件结果示例：

```gdscript
{
    "event_id": "super_time",
    "slot": "time_modifier",
    "active": true,
    "run_duration_seconds": 480,
    "trigger_reason": "daily_roll_success"
}
```

详细规则见 `06_时间类随机事件规则.md`。

## 8. 随机障碍事件规则

事件 ID：

```text
road_obstacle
```

命中后：

```text
读取 random_obstacle_spawn 点位。
从合法点位中抽取 2-4 个。
生成玩家不可通过、怪物可穿过的障碍。
生成后验证主流程路线仍可达。
```

事件结果示例：

```gdscript
{
    "event_id": "road_obstacle",
    "slot": "map_blocker",
    "active": true,
    "obstacle_count": 3,
    "spawn_point_ids": [
        "obstacle_spawn_002",
        "obstacle_spawn_005",
        "obstacle_spawn_009"
    ],
    "trigger_reason": "daily_roll_success"
}
```

详细规则见 `07_随机障碍事件规则.md`。

## 15. 验收标准

| 编号 | 标准 |
| --- | --- |
| A1 | `surface_day=0/1` 时不自然触发场景随机事件 |
| A2 | `surface_day=2` 时即将进入第 3 日，事件池启用 |
| A3 | 第 3 日 `monster_presence` 必定触发 |
| A4 | 第 4 日后怪物按 50% 概率抽取 |
| A5 | `super_time` 和 `short_time` 不会同局同时生效 |
| A6 | 时间事件能写入正确的 `run_duration_seconds` |
| A7 | `road_obstacle` 能写入障碍事件结果 |
| A8 | GM 强制事件优先于自然概率 |
| A9 | 所有事件结果写入 `RunEventContext` |
