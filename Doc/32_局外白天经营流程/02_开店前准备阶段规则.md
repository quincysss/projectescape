# 32-02 开店前准备阶段规则

> 开店前准备阶段对应 `DAY_PREP`。这是玩家安排本日经营策略的阶段，不计时，不自动成交，也不允许出发探索。

## 1. 阶段目标

让玩家在正式营业前完成三件事：

```text
决定今天卖什么
准备足够可售物资
理解今日难民需求热度
```

## 2. 界面结构

建议布局：

```text
顶部：第 N 天 / 白天 / 当前矿币 / 当前阶段：开店前

左侧功能入口：
- 仓库
- 制造所
- 研究所
- 图鉴

中间主面板：
- 当前选中功能内容

右侧固定面板：
- 今日需求榜
- 今日推荐可制造物资
- 当前可上架 sale_good 总数

底部右侧：
- 开店营业按钮
```

不显示：

- 出发探索按钮。
- 夜间计划入口。
- 出发准备入口。
- 旧独立买卖入口。
- 订单页签。

## 3. 可执行操作

### 3.0 开店前经营检查

`DAY_PREP` 的核心不是让玩家随便点页签，而是让玩家完成一次开店前检查。

系统应在界面右侧或底部给出轻量经营提示：

| 检查项 | 通过条件 | 未通过时提示 |
|---|---|---|
| 今日需求榜 | 已生成 `daily_demand_entries` | 今日需求尚未生成 |
| 可售库存 | 仓库存在至少 1 个 `sale_good` | 当前没有可上架货品 |
| 可制造货品 | 至少 1 个已解锁配方材料足够 | 可先制造今日热销物资 |
| 货台槽位 | 至少 1 个货台槽已解锁 | 货台不可用，请检查存档或配置 |
| 结算状态 | 上一日结算已处理，或无未处理结算 | 仍有未完成营业结算 |

未通过检查不一定阻止开店，但必须明确提示原因。只有“货台槽位不可用”和“上一日结算未处理”属于阻断项。

### 3.1 仓库管理

玩家可以：

- 查看仓库物品。
- 整理仓库。
- 丢弃物品。
- 查看 Tooltip。
- 查看哪些物品可用于制造可售物资。

玩家不能：

- 从仓库直接出售原材料。
- 从仓库直接把物资摆到货台成交。
- 把已标记为 `LOCKED_FOR_LOADOUT`、`IN_RESEARCH`、`IN_CRAFTING`、`ON_SHELF` 或 `PENDING_RETURN` 的物品作为普通可用库存处理。

仓库中每种可售物资在开店前应显示：

```text
item_id
display_name
quality
available_count
locked_count
sale_good=true/false
today_demand_rank
estimated_price
```

其中 `available_count` 是开店中可上架数量；`locked_count` 只用于说明为什么总数和可上架数不同。

### 3.2 制造所

玩家可以：

- 使用原材料制造 `sale_good` 可售物资。
- 进行同类别原材料转化。
- 查看配方需要的材料和产出数量。
- 查看未解锁配方的来源条件，例如研究、配方开放、章节进度或地点产出。

开店前制造出的 `sale_good` 进入仓库，等待开店中上架。

制造所需要给今日经营提供明确反馈：

| 配方状态 | UI 表现 | 说明 |
|---|---|---|
| 今日热销且可制造 | 推荐标记 | 优先展示在今日推荐可制造物资 |
| 今日热销但材料不足 | 缺料标记 | 显示缺少哪些原材料 |
| 非热销但可制造 | 普通可制造 | 可作为低收益备选货 |
| 非 `sale_good` 配方 | 不进入今日推荐 | 装备、消耗品和家具材料不作为货台推荐 |

如果配方产出多个同名可售物资，仓库应增加对应堆叠数量，而不是创建多个独立格子，除非该物品配置为不可堆叠。

### 3.3 研究所

玩家可以在开店前进行研究升级。

研究可影响：

- 仓库容量。
- 负重上限。
- 货台格子数量。
- 可售物资售价修正。
- 高阶需求榜权重。

研究消耗必须在 `DAY_PREP` 完成。进入 `SHOP_OPEN` 后不得研究。

### 3.4 今日需求榜

右侧需求榜必须展示：

| 字段 | 说明 |
|---|---|
| 排名 | 1 到 N |
| 可售物资图标 | 只显示 `sale_good` |
| 可售物资名 | 例如连轴齿轮 |
| 热度等级 | 高/中/低 |
| 预计成交间隔 | 用于提示卖得快慢 |
| 售价修正 | 例如 1.2x |

需求榜不展示原材料需求。

需求榜还应驱动两个派生信息：

```text
今日推荐可制造物资 = 需求榜中已解锁、材料足够或接近足够的 sale_good
当前可上架 sale_good 总数 = 仓库中 AVAILABLE 状态且 sale_good=true 的数量合计
```

如果需求榜商品未解锁，显示为“未知热销物资”或“需要解锁配方”，不得把未解锁物资加入可制造列表。

同一天内需求榜固定，不因玩家制造、上架、读档或进入 `SHOP_OPEN` 重新随机。

## 4. 开店营业按钮规则

按钮默认可点击，但需要给出状态提示。

| 条件 | 按钮状态 | 提示 |
|---|---|---|
| 仓库有至少 1 个 `sale_good` | 可点击 | 开店营业 |
| 仓库没有 `sale_good`，但可制造 | 可点击 | 仍可开店，但建议先制造货品 |
| 仓库没有 `sale_good`，也无可制造配方 | 可点击但二次确认 | 当前没有可售物资，开店后可立即提前结束 |
| 上一日结算未处理 | 不可点击 | 请先完成营业结算 |
| 货台槽位配置异常 | 不可点击 | 货台配置不可用 |

玩家允许空货开店，因为可能用于测试、剧情或后续随机事件。但系统必须在开店中提供“提前结束营业”。

点击开店前，如果存在可制造的今日热销货但仓库没有任何可售库存，系统应弹出二次确认：

```text
当前仓库没有可上架货品，但仍有可制造的今日需求货。是否继续开店？
```

玩家确认后仍可进入 `SHOP_OPEN`。

## 5. 点击开店营业

点击后执行：

```text
1. 锁定今日需求榜。
2. 创建 shop_session_id。
3. 初始化 shop_duration_seconds = 60。
4. 初始化 shop_elapsed_seconds = 0。
5. 初始化 shop_ended_by = ""。
6. 初始化 shop_sales_records = []。
7. 初始化 shop_total_earned = 0。
8. 初始化 shop_settlement_applied = false。
9. 初始化货台为空，或按工程配置恢复当天预设空槽。
10. 切换 outgame_phase = SHOP_OPEN。
11. 显示开店中营业界面。
```

不自动上架。玩家进入开店中后自己选择货台商品。

进入 `SHOP_OPEN` 时不得扣除仓库物品；只有玩家实际上架时，物品才从 `AVAILABLE` 转为 `ON_SHELF`。

## 6. 开店前失败与恢复

如果切换 `SHOP_OPEN` 过程中发生异常：

| 异常 | 处理 |
|---|---|
| 需求榜不存在 | 重新生成当日需求榜后再切换 |
| 货台配置缺失 | 阻止开店并提示配置错误 |
| 存档写入失败 | 保持 `DAY_PREP`，提示保存失败 |
| 旧结算未完成 | 跳转或恢复 `SHOP_SETTLEMENT` |

不得出现已经显示 `SHOP_OPEN` 但 `shop_session_id` 或需求榜缺失的半初始化状态。

## 7. BDD 场景

Feature: 开店前准备阶段

  Scenario: 开店前展示经营准备入口
    Given 当前阶段为 DAY_PREP
    When 玩家进入局外界面
    Then 界面应显示仓库、制造所、研究所、图鉴
    And 界面应显示今日需求榜
    And 界面应显示开店营业按钮
    And 界面不应显示出发探索按钮

  Scenario: 开店前制造可售物资
    Given 当前阶段为 DAY_PREP
    And 仓库中有 raw_scrap_metal 和 raw_dirty_cloth
    When 玩家在制造所制造 sale_good_simple_gear
    Then 仓库中应增加 sale_good_simple_gear
    And sale_good_simple_gear 可在开店中上架货台

  Scenario: 空货也可以开店但需要提示
    Given 当前阶段为 DAY_PREP
    And 仓库中没有 sale_good
    When 玩家点击开店营业
    Then 系统应显示“当前没有可售物资，开店后可提前结束”的确认提示
    When 玩家确认继续
    Then 当前阶段应变为 SHOP_OPEN

  Scenario: 上一日结算未处理时不能开店
    Given 当前阶段为 DAY_PREP
    And 上一日 shop_settlement_applied 为 false
    When 玩家查看开店营业按钮
    Then 开店营业按钮应不可点击
    And 系统应提示请先完成营业结算

  Scenario: 进入开店中时创建营业会话
    Given 当前阶段为 DAY_PREP
    And 今日需求榜已经生成
    When 玩家点击开店营业并确认
    Then 系统应创建 shop_session_id
    And shop_elapsed_seconds 应为 0
    And shop_sales_records 应为空
    And shop_settlement_applied 应为 false
