# 32-01 局外白天经营系统总览与阶段边界

> 本文细化 `Doc/29` 与 `Doc/30` 的白天经营部分。目标是把白天拆成“开店前”和“开店中”两个清晰阶段，后续 AI 可按本文直接实现状态机、界面和数据接口。

## 1. 设计定位

玩家白天不是逛商店，也不是直接把仓库物品卖掉，而是在避难所内经营自己的杂货店。

白天主循环：

```text
开店前准备
-> 制造可售物资
-> 查看当日需求
-> 整理仓库和货台策略
-> 点击开店营业
-> 开店中营业
-> 上架、补货、撤下、观察成交
-> 倒计时结束或提前结束
-> 今日营业结算
-> 夜晚
```

## 2. 阶段定义

| 阶段枚举 | 中文名 | 是否可操作 | 核心行为 |
|---|---|---:|---|
| `DAY_PREP` | 开店前准备 | 是 | 制造、研究、仓库、图鉴、需求榜、点击开店 |
| `SHOP_OPEN` | 开店中营业 | 是 | 货台上架、补货、撤下、自动成交、提前结束 |
| `SHOP_SETTLEMENT` | 营业结算 | 否，仅确认 | 展示本日收益，关闭后进入夜晚 |
| `NIGHT` | 夜晚 | 是 | 显示出发探索，进入夜间计划 |

白天“两个阶段”指 `DAY_PREP` 和 `SHOP_OPEN`。`SHOP_SETTLEMENT` 是结果面板状态，用于承接倒计时结束或提前结束后的结算，不算可经营阶段。

## 3. 阶段流转

```text
DAY_PREP
  点击开店营业
  -> SHOP_OPEN

SHOP_OPEN
  60 秒营业时间结束
  -> SHOP_SETTLEMENT

SHOP_OPEN
  玩家点击提前结束营业
  -> SHOP_SETTLEMENT

SHOP_SETTLEMENT
  玩家关闭结算面板
  -> NIGHT

NIGHT
  玩家点击出发探索
  -> NIGHT_PLAN
```

禁止流转：

```text
DAY_PREP -> NIGHT_PLAN
DAY_PREP -> LOADING_TO_RUN
SHOP_OPEN -> NIGHT_PLAN
SHOP_OPEN -> DAY_PREP
SHOP_SETTLEMENT -> DAY_PREP
```

## 4. 开店前和开店中的能力边界

| 功能 | DAY_PREP 开店前 | SHOP_OPEN 开店中 |
|---|---:|---:|
| 查看仓库 | 可用 | 只读或仅作为补货来源 |
| 整理仓库 | 可用 | 不可用 |
| 制造可售物资 | 可用 | 不可用 |
| 同类别资源转化 | 可用 | 不可用 |
| 研究升级 | 可用 | 不可用 |
| 查看图鉴 | 可用 | 可用，只读 |
| 查看当日需求榜 | 可用 | 可用 |
| 货台上架 | 不可用或仅预览 | 可用 |
| 货台补货 | 不可用 | 可用 |
| 货台撤下 | 不可用 | 可用 |
| 自动成交 | 不运行 | 运行 |
| 提前结束营业 | 不显示 | 显示 |
| 出发探索 | 不显示 | 不显示 |

## 5. 一天内的需求榜生命周期

当日需求榜在进入 `DAY_PREP` 时生成，并在同一天内固定。

```text
DAY_PREP 生成并展示今日需求榜
SHOP_OPEN 使用同一份需求榜计算成交速度和售价修正
SHOP_SETTLEMENT 记录本日需求榜和成交结果
NIGHT 不刷新需求榜
下一天 DAY_PREP 才刷新新需求榜
```

## 6. 数据持久化边界

需要保存：

- 当前 `outgame_phase`。
- 当前 `surface_day`。
- 今日需求榜 `daily_demand`.
- SHOP_OPEN 已经过的营业秒数。
- 货台中的物资、数量和来源仓库堆叠。
- 本日已成交记录。
- 已获得矿币。

读档时：

- 如果保存于 `DAY_PREP`，回到开店前准备。
- 如果保存于 `SHOP_OPEN`，恢复倒计时、货台、成交记录和可提前结束按钮。
- 如果保存于 `SHOP_SETTLEMENT`，恢复结算面板，防止重复加币。
- 如果保存于 `NIGHT`，显示夜晚出发入口。

## 7. BDD 场景

Feature: 局外白天阶段边界

  Scenario: 开店前只能进入开店中
    Given 当前阶段为 DAY_PREP
    When 玩家点击开店营业
    Then 当前阶段应变为 SHOP_OPEN
    And 界面显示 60 秒营业倒计时
    And 界面不显示出发探索按钮

  Scenario: 结算关闭后进入夜晚
    Given 当前阶段为 SHOP_SETTLEMENT
    And 今日营业收益已经结算
    When 玩家关闭营业结算面板
    Then 当前阶段应变为 NIGHT
    And 界面应显示出发探索按钮
    And 界面不显示开店营业按钮

  Scenario: 白天准备不能直接出发
    Given 当前阶段为 DAY_PREP
    When 玩家查看主操作按钮
    Then 界面不应显示出发探索
    And 任意白天按钮不得直接进入 Loading
