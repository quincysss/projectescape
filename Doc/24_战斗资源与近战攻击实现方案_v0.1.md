# 《废土生存法则》战斗资源与近战攻击实现方案

> 文档版本：v0.1  
> 适用阶段：Godot 原型 / Codex + GodotMCP 执行  
> 目标功能：基于现有序列帧角色，实现“移动中可攻击”的三段式近战战斗原型  
> 重点原则：角色身体动画、武器表现、攻击判定、武器数据分离，不为每把武器生成完整角色攻击序列帧。

---

## 0. 本文档解决的问题

当前角色资源是整张序列帧，例如：

```text
player_male_run_left_8f_sheet_01.png
```

这类资源适合做：

```text
idle
walk / run
基础方向切换
```

但如果继续用“整个人物烘焙序列帧”的方式做战斗，会出现资源爆炸：

```text
4方向 × 静止/移动 × 3段攻击 × 多种武器
```

例如木棍、铁棍、棒球棍如果每个都画一套完整角色攻击动画，后续维护成本会非常高。

本文档采用的方案是：

```text
角色身体序列帧
+
武器独立 Sprite
+
手部携带挂点 CarrySocket
+
攻击旋转轴 AttackPivot
+
挥击特效 SlashVFX
+
攻击判定 AttackHitbox
+
武器数据表
```

---

## 1. 核心结论

### 1.1 不制作“每把武器一套角色攻击精灵图”

禁止的制作方式：

```text
player_run_attack_wood_stick_left
player_run_attack_iron_rod_left
player_run_attack_baseball_bat_left
player_idle_attack_wood_stick_down
……
```

推荐的制作方式：

```text
角色基础动画：idle / run
武器独立图：wood_stick / iron_rod / baseball_bat
攻击动画：由 WeaponPivot 旋转武器完成
伤害判定：由 AttackHitbox 完成
连段节奏：由 CombatController 管理
```

### 1.2 移动攻击不额外生成“跑步攻击图”

移动攻击时：

```text
角色身体继续播放 run 动画
武器从 CarrySocket 切换到 AttackPivot
AttackPivot 播放三段攻击动画
AttackHitbox 在有效帧开启
攻击结束后武器回到 CarrySocket
```

也就是说，移动攻击不是一套新角色图，而是：

```text
跑步身体动画 + 独立武器攻击动画
```

### 1.3 后续正式版可以补“持武器跑步身体姿态”

第一版可以直接用普通 run 动画。

如果后续觉得武器和手臂不够贴合，可以追加：

```text
armed_idle_down
armed_idle_up
armed_idle_side
armed_run_down
armed_run_up
armed_run_side
```

注意：这套图中仍然不要画具体武器，只画角色身体处于“持武器准备姿态”。

## 2. 功能目标

本次实现目标是做一个最小可玩的近战原型：

```text
玩家可以装备木棍。
玩家静止时可以攻击。
玩家移动时也可以攻击。
攻击为三段连击。
每段攻击有前摇、有效期、后摇。
攻击有效期内开启 Hitbox。
敌人或测试靶进入 Hitbox 时受到伤害。
木棍、铁棍、棒球棍共享一手钝器攻击逻辑。
不同武器通过数据表改变伤害、速度、击退、范围。
```

## 3. 节点结构设计

### 3.1 Player 推荐节点结构

建议在现有玩家节点下增加战斗相关节点：

```text
Player
├── BodySprite                 # AnimatedSprite2D，播放 idle / run
├── CombatRoot                 # Node2D，战斗表现根节点
│   ├── CarrySocket            # Marker2D，平时武器挂点
│   ├── AttackPivot            # Node2D，攻击时武器旋转中心
│   │   ├── WeaponSprite        # Sprite2D，当前武器图
│   │   ├── SlashVFX            # AnimatedSprite2D，可选，挥击特效
│   │   └── AttackHitbox        # Area2D，攻击判定
│   │       └── CollisionShape2D
├── AnimationPlayer            # 播放武器旋转、Hitbox开关等动画
├── PlayerMovement             # 现有移动逻辑，可保留
└── CombatController           # 新增，处理攻击输入、连段、武器数据
```

如果当前项目已有 Player 节点结构，不要强行重构整个玩家。优先以“最小侵入”的方式添加：

```text
CombatRoot
AnimationPlayer
CombatController
```

### 3.2 节点职责

| 节点 | 职责 |
|---|---|
| `BodySprite` | 播放角色身体动画，不负责伤害 |
| `CarrySocket` | 非攻击状态下，武器跟随的位置 |
| `AttackPivot` | 攻击状态下，武器旋转和位移的父节点 |
| `WeaponSprite` | 显示当前装备武器的 PNG |
| `SlashVFX` | 播放挥击残影、弧光，可后续补 |
| `AttackHitbox` | 造成伤害的 Area2D |
| `AnimationPlayer` | 播放 attack_1 / attack_2 / attack_3 |
| `CombatController` | 管理攻击状态机、连击窗口、冷却、武器数据 |

## 6. 方向与挂点设计

### 6.1 方向枚举

建议统一使用四方向：

```gdscript
enum Facing {
	DOWN,
	UP,
	LEFT,
	RIGHT
}
```

也可以兼容项目现有方向字符串：

```text
"down"
"up"
"left"
"right"
```

### 6.2 CarrySocket 平时挂点

第一版使用方向级固定挂点，不做逐帧挂点。

假设玩家单帧参考尺寸为 128×128，角色原点在脚底或中心时需要按项目实际调整。下面是以“角色视觉中心附近”为参考的初始值：

```gdscript
var carry_socket_offsets := {
	"down": Vector2(10, 22),
	"up": Vector2(-8, 2),
	"left": Vector2(-24, 12),
	"right": Vector2(24, 12)
}
```

如果当前角色是 256×256，可按比例放大：

```text
128资源：offset × 1
256资源：offset × 2
```

### 6.3 AttackPivot 攻击挂点

攻击时不要完全跟随跑步手部挂点，而是切到更稳定的攻击旋转轴。

```gdscript
var attack_pivot_offsets := {
	"down": Vector2(8, 24),
	"up": Vector2(-8, 0),
	"left": Vector2(-22, 10),
	"right": Vector2(22, 10)
}
```

### 6.4 Z Index 遮挡规则

```text
朝下攻击：武器显示在身体前面，WeaponSprite.z_index 高于 BodySprite。
朝上攻击：武器显示在身体后面，WeaponSprite.z_index 低于 BodySprite。
朝左/右攻击：武器显示在身体前侧，WeaponSprite.z_index 高于 BodySprite。
```

建议：

```gdscript
func update_weapon_z_index(facing: String) -> void:
	if facing == "up":
		weapon_sprite.z_index = body_sprite.z_index - 1
	else:
		weapon_sprite.z_index = body_sprite.z_index + 1
```

## 7. 战斗状态机

### 7.1 状态枚举

```gdscript
enum CombatState {
	IDLE,
	WINDUP,
	ACTIVE,
	RECOVERY
}
```

### 7.2 连段变量

```gdscript
var combat_state := CombatState.IDLE
var combo_index := 0
var combo_timer := 0.0
var buffered_attack := false
var current_weapon_id := "wood_stick"
var current_facing := "down"
```

### 7.3 输入规则

```text
按下攻击键：
- 如果当前是 IDLE：立刻开始第 1 段攻击。
- 如果当前是 WINDUP：不切段，可记录 buffered_attack。
- 如果当前是 ACTIVE：可记录 buffered_attack。
- 如果当前是 RECOVERY 且处于可连段窗口：记录 buffered_attack 或直接进入下一段。
- 如果超过连段窗口：重置为第 1 段。
```

### 7.4 移动规则

攻击时允许移动，但降低速度：

```text
非攻击：移动速度 = 正常速度 × 负重倍率
攻击中：移动速度 = 正常速度 × 负重倍率 × weapon.move_speed_multiplier_while_attacking
```

不要在第一版攻击时完全锁死移动，否则“移动中攻击”的目标无法验证。

## 8. 攻击时间轴

### 8.1 三段攻击默认参数

| 段数 | 前摇 | 有效期 | 后摇 | 设计定位 |
|---|---:|---:|---:|---|
| 第1段 | 0.12s | 0.08s | 0.20s | 快速试探 |
| 第2段 | 0.14s | 0.09s | 0.22s | 稍大范围 |
| 第3段 | 0.22s | 0.12s | 0.36s | 重击、击退强 |

### 8.2 Hitbox 开启规则

```text
windup：Hitbox 关闭。
active：Hitbox 开启。
recovery：Hitbox 关闭。
```

每次攻击每个敌人只命中一次，需要记录本段已命中目标：

```gdscript
var hit_targets_this_swing: Array = []
```

进入新一段攻击时清空：

```gdscript
hit_targets_this_swing.clear()
```

## 9. Hitbox 方向换算

### 9.1 基础配置

数据表中的 hitbox 可以先按“右方向”配置：

```text
hitbox_offset = [48, 0]
hitbox_size = [72, 48]
```

然后根据方向换算：

```gdscript
func get_directional_hitbox_offset(base_offset: Vector2, facing: String) -> Vector2:
	match facing:
		"right":
			return base_offset
		"left":
			return Vector2(-base_offset.x, base_offset.y)
		"down":
			return Vector2(0, abs(base_offset.x))
		"up":
			return Vector2(0, -abs(base_offset.x))
		_:
			return base_offset
```

### 9.2 Hitbox 形状

第一版使用 RectangleShape2D：

```text
left / right：横向矩形
up / down：纵向矩形
```

方向换算：

```gdscript
func apply_hitbox_shape(size: Vector2, facing: String) -> void:
	var shape := RectangleShape2D.new()

	if facing == "up" or facing == "down":
		shape.size = Vector2(size.y, size.x)
	else:
		shape.size = size

	attack_hitbox_shape.shape = shape
```

## 10. 武器旋转动画

### 10.1 第一版可以不用逐帧图

武器攻击表现使用代码或 AnimationPlayer 旋转 `AttackPivot` / `WeaponSprite`。

### 10.2 推荐角度

右方向：

```text
attack_1：-50° → 55°
attack_2：55° → -45°
attack_3：-80° → 80°
```

左方向可以镜像：

```text
attack_1：50° → -55°
attack_2：-55° → 45°
attack_3：80° → -80°
```

下方向：

```text
attack_1：-70° → 35°
attack_2：35° → -55°
attack_3：-90° → 70°
```

上方向：

```text
attack_1：70° → -35°
attack_2：-35° → 55°
attack_3：90° → -70°
```

### 10.3 武器回收

攻击结束后：

```text
WeaponSprite 回到 CarrySocket。
AttackPivot.rotation_degrees = 0。
AttackHitbox 关闭。
SlashVFX 停止。
```

## 12. Codex 执行步骤

### Step 1：检查现有 Player 节点

让 Codex 先读取当前项目中的 Player 场景和脚本，确认：

```text
Player 场景路径。
AnimatedSprite2D 节点名。
移动脚本路径。
当前方向变量如何记录。
当前动画如何切换。
当前输入映射在哪里配置。
```

不要直接假设节点名。

### Step 2：创建 CombatRoot 结构

在 Player 场景中新增：

```text
CombatRoot
CarrySocket
AttackPivot
WeaponSprite
SlashVFX
AttackHitbox
CollisionShape2D
AnimationPlayer
```

要求：

```text
WeaponSprite 初始隐藏或装备后显示。
AttackHitbox 初始 monitoring = false。
CollisionShape2D 初始 disabled = true。
```

### Step 3：新增输入映射

新增输入：

```text
attack
```

建议按键：

```text
鼠标左键
J
```

如果项目已有输入配置文件，按项目规范写入。

### Step 4：新增 weapons.json

创建：

```text
res://data/combat/weapons.json
```

至少配置：

```text
wood_stick
```

### Step 5：新增 CombatController

实现：

```text
equip_weapon("wood_stick")
start_attack()
update_combo_timer(delta)
set_hitbox_active(true/false)
apply_directional_offsets()
get_attack_move_speed_multiplier()
```

### Step 6：接入移动系统

移动系统需要读取：

```gdscript
combat_controller.get_move_speed_multiplier()
```

攻击中移动速度降低，但不禁止移动。

### Step 7：接入 BodySprite 方向

CombatController 需要知道当前方向：

```text
down / up / left / right
```

优先从现有 PlayerMovement 或 Player 脚本读取。

如果现有项目没有统一方向变量，新增：

```gdscript
var facing := "down"
```

移动输入改变方向时更新。

### Step 8：实现测试靶

新增一个简单测试靶：

```text
res://scenes/debug/combat_test_dummy.tscn
```

节点：

```text
CombatTestDummy
├── Sprite2D / ColorRect
├── Hurtbox
│   └── CollisionShape2D
└── Label，可选，显示 HP
```

功能：

```text
初始 HP = 100。
被攻击命中时扣血。
打印日志。
播放闪烁反馈。
HP <= 0 时隐藏或销毁。
```

### Step 9：放到 RunScene 测试

在玩家出生点附近放置 1 个测试靶，验证：

```text
静止攻击可命中。
移动中攻击可命中。
三段攻击能连续触发。
第三段伤害更高。
Hitbox 只在 active 时间开启。
同一段攻击不会多次命中同一目标。
攻击期间移动速度变慢。
```

### Step 10：清理 Debug

测试通过后：

```text
测试靶可以保留在 DebugRoot。
默认不影响正式局内流程。
战斗输入和武器节点可保留。
```

## 16. 验收标准

### 16.1 功能验收

必须满足：

```text
玩家装备 wood_stick 后，武器可显示在角色身上。
玩家跑步时，武器能跟随 CarrySocket。
玩家按攻击键后，武器进入 AttackPivot 挥击。
玩家静止时可以攻击。
玩家移动中可以攻击。
攻击中移动速度降低，但不完全停止。
三段连击可以触发。
Hitbox 只在 active 阶段开启。
测试靶能被攻击扣血。
同一段攻击不会重复命中同一测试靶。
攻击结束后武器回到 CarrySocket。
```

建议满足：

```text
武器不会长时间明显漂浮在角色外。
朝上攻击时武器不会错误盖住身体正面。
朝下攻击时武器能显示在身体前侧。
左右攻击能镜像成立。
攻击节奏能看出第一段快、第二段衔接、第三段重。
```

### 16.3 工程验收

必须满足：

```text
不破坏现有移动系统。
不破坏现有 RunScene。
不破坏稳定值、容器、前哨、背包等已有系统。
武器数据不硬编码在大量分散脚本里。
Hitbox 初始关闭。
Debug 测试靶不影响正式流程。
脚本无报错。
Godot 运行主场景无阻塞错误。
```

## 18. 当前版本结论

本方案的重点不是一次性做出完整战斗系统，而是先验证：

```text
序列帧角色是否能通过挂点系统拿武器。
独立武器 Sprite 是否能复用多种武器。
移动中攻击是否能成立。
三段连击的节奏是否可玩。
Hitbox 判定是否稳定。
```

只要这套方案跑通，后续加入更多武器、敌人、装备数值、受击反馈和掉落系统，都可以在同一套结构上扩展，不需要反复重做角色精灵图。
