# 30_昼夜局外流程_店铺营业与夜间出发准备

> 文档版本：v0.1  
> 所属项目：《废土生存法则》  
> 前置文档：`29_局外杂货店经营与界面重构路线图.md`、`31_掉落物资与制造经济规划.md`、`10_研究所与制作所系统_修订版_废土生存法则.md`、`11_出发准备与局外装备系统_修订版_废土生存法则.md`、`20_UI视觉规范与界面资源Sheet提示词_修订版_废土生存法则.md`。  
> 文档目标：把“白天准备-白天开店-营业结算-夜间计划-出发准备-Loading”的流程拆成可实现界面、状态、数据和验收规则。

## 1. 状态机

局外主状态机：

```text
DAY_PREP
-> SHOP_OPEN
-> SHOP_SETTLEMENT
-> NIGHT
-> NIGHT_PLAN
-> LOADOUT
-> LOADING_TO_RUN
```

状态流转：

| 当前状态 | 触发 | 下一状态 |
| --- | --- | --- |
| DAY_PREP | 点击开店营业 | SHOP_OPEN |
| SHOP_OPEN | 60 秒倒计时结束 | SHOP_SETTLEMENT |
| SHOP_SETTLEMENT | 关闭结算面板 | NIGHT |
| NIGHT | 点击出发探索 | NIGHT_PLAN |
| NIGHT_PLAN | 点击准备 | LOADOUT |
| LOADOUT | 点击出发 | LOADING_TO_RUN |
| LOADING_TO_RUN | Loading 完成 | 局内场景 |

禁止流转：

- DAY_PREP 不允许直接进入 NIGHT_PLAN 或 LOADING_TO_RUN。
- SHOP_OPEN 不允许进入研究升级。
- NIGHT 不允许进入制造、仓库、研究和图鉴主面板。
- NIGHT_PLAN 不允许直接进入局内，必须经过 LOADOUT。

## 2. 白天准备界面

白天准备界面是局外主操作区。

建议主布局：

```text
顶部：日期、阶段、矿币、今日需求榜摘要
左侧：功能入口
中部：当前功能面板
右侧：今日需求榜
底部右侧：开店营业按钮
```

功能入口：

```text
仓库
制造所
研究所
图鉴
```

不显示：

```text
出发准备
出发探索
```

制造所职责：

- 使用局内带回的原材料制作可售物资。
- 使用原材料制作装备或消耗品。
- 支持同类别资源转化。

同类别资源转化规则：

```text
基础材料类：任意 2 个同类可转化材料 -> 1 个指定基础材料
零件/组件类：任意 3 个同类可转化材料 -> 1 个指定零件或组件
食物药品类：任意 3 个同类可转化材料 -> 1 个指定低阶食物或药品材料
武器装备、烟酒珠宝、S 级收藏：默认不开放通用转化
```

示例：

```text
目标：废金属 C x1
消耗：基础材料类 x2
或：同类低阶材料 x2
```

## 3. 今日需求榜

需求榜生成时机：

- 进入新一天白天准备阶段时生成。
- 同一天内不重复随机。
- 读取存档时如果已有 `today_demand_ranking`，直接复用。

候选池：

```text
所有已解锁配方中 sale_good=true 的制造产物
```

若候选不足 5 个：

- 展示已有候选。
- 不强制加入未解锁商品。
- 可在 UI 上显示“商品种类不足，解锁更多配方可扩大经营选择”。

推荐字段：

```text
demand_entry_id
good_item_id
rank
demand_label
sale_interval_min
sale_interval_max
price_multiplier
customer_weight
```

难民类型修正：

| 难民类型 | 偏好标签 | 说明 |
| --- | --- | --- |
| 伤员 | medical、bandage、painkiller | 医疗物资权重提高 |
| 家庭 | food、warmth、clean_water | 生活物资权重提高 |
| 巡逻者 | tool、battery、light | 工具和照明权重提高 |
| 工匠 | hardware、filter、repair | 维修类物资权重提高 |

当日可随机 1-2 个难民类型作为主需求来源。

## 4. 开店界面

开店界面建议布局：

```text
顶部：60 秒倒计时、今日收入、未满足人数
左侧：货台 3 格
中部：难民来访区
右侧：今日需求榜
底部：仓库可售物资快捷栏 / 成交日志 / 提前结束营业按钮
```

货台格：

```text
shelf_slot_index
good_item_id
stack_count
is_locked
```

交互：

- 点击空货台：打开可售物资选择。
- 点击已有货台：弹出补货、撤下、查看详情。
- 拖拽仓库物资到货台可作为后续增强，V0.2 可先用点击选择。
- 货台售空后显示空格状态。
- 营业全程显示“提前结束营业”按钮；没货时按钮高亮。

难民来访：

```text
refugee_id
preferred_tags
target_good_id 或 target_demand_rank
patience_seconds
matched_shelf_slot
```

V0.2 可不显示完整 NPC，只显示需求气泡和来访提示。

## 5. 店铺成交模拟

核心目标：

```text
高需求货品卖得快，低需求货品卖得慢，错误上架会浪费营业时间。
```

每个货台商品独立计算下一次可能成交时间：

```text
rank_interval = 今日需求榜给定区间
next_deal_time = current_shop_time + random(rank_interval.min, rank_interval.max)
```

成交校验：

```text
货台商品数量 > 0
商品在当前难民需求中匹配
营业时间未结束
```

成交成功：

```text
货台数量 -1
商品销售记录 +1
矿币收入计入当日临时账本
生成下一次该商品可能成交时间
```

未成交：

```text
missed_customer_count +1
不扣除货品
不增加收入
```

## 6. 营业中的玩家操作

营业 60 秒内，玩家应保持轻度操作压力。

核心操作：

- 观察右侧需求榜，决定 3 个货台格子的商品组合。
- 高需求商品售空后补货。
- 发现低需求商品长时间没人买时撤下换货。
- 根据难民提示临时换上匹配商品。
- 没有可售物资时提前结束营业并进入结算。

营业中限制：

- 不允许打开制造所临时制造新货。
- 不允许进行研究升级。
- 不允许整理仓库。
- 不允许出发探索。

该设计让玩家在 60 秒内玩“预测、补货、换货”，而不是站着等系统结算。

## 7. 结算面板

结算数据结构：

```text
shop_day
sold_goods: Array
  good_item_id
  sold_count
  unit_price
  price_multiplier
  subtotal
missed_customer_count
total_income
unsold_returned_goods
```

界面要求：

- 标题显示“今日营业结算”。
- 每一行显示物资名、卖出数量、单价、小计。
- 总收入在面板底部居中显示。
- 如果由提前结束触发，标题显示“今日营业结算（提前结束）”或在副标题标记提前结束。
- 关闭按钮在右下或底部。
- 关闭后进入 NIGHT。

## 8. 夜晚主界面

夜晚主界面只承担一个决策：

```text
是否出发探索
```

显示：

- 当前日期/夜晚阶段。
- 简短的夜晚氛围文案。
- 出发探索按钮。

不显示：

- 仓库入口。
- 制造所入口。
- 研究所入口。
- 开店营业按钮。

## 9. 夜间计划界面

夜间计划界面用于选择角色和地点。

参考结构：

```text
左侧：角色卡列表
中部：地点地图/地点点位
右侧：地点详情
右下：准备按钮
```

角色卡字段：

```text
character_id
display_name
role_title
status
carry_capacity
traits
is_available
```

地点字段：

```text
location_id
display_name
description
risk_level
expected_loot_tags
expected_loot_labels
unlock_conditions
is_available
```

V0.2 默认：

```text
character_id = owner
location_id = abandoned_house
```

但 UI 必须使用列表和选择状态，不得把唯一角色和唯一地点硬编码成静态按钮。

地点右侧描述应包含：

- 地点名。
- 地点描述。
- 可能产出。
- 危险程度。
- 特殊规则预留。

## 10. 出发准备界面

点击夜间计划的“准备”后进入出发准备界面。

布局：

```text
左侧：角色人形装备区
右侧：仓库可带物资列表
底部右侧：出发按钮
```

装备槽：

```text
HEAD
BODY
HAND
FOOT
```

人形布局要求：

```text
      [头部]

      [身体]

[手持]     [手持预留/副手预留]

      [脚部]

[消耗品1] [消耗品2 灰] [消耗品3 灰] [消耗品4 灰]
```

V0.2 规则：

- 头部、身体、手持、脚部均可显示槽位。
- 没有可用装备时槽位仍显示为空。
- 消耗品共 4 格，只解锁第 1 格。
- 未解锁消耗品格可见但置灰，不可点击。
- 仓库列表只显示可携带装备和可携带消耗品。
- 已装备或已放入消耗品栏的物品不能被制造、研究或售卖消耗。

点击“出发”校验：

- 已选择角色。
- 已选择地点。
- 装备槽物品合法。
- 消耗品槽物品合法。

校验通过后：

```text
写入本次出发配置
-> 锁定对应仓库物品
-> 进入当前 Loading
-> 进入局内场景
```

## 11. UI 与代码预留

即使 V0.2 只有一个角色和一个地点，也必须保留：

- `characters` 列表。
- `locations` 列表。
- 当前选择 ID。
- 地点详情刷新接口。
- 角色详情刷新接口。
- 出发配置对象。

不要在 UI 里写死：

- 唯一角色名字。
- 唯一地点名字。
- 唯一地点描述。
- 装备槽数量。
- 消耗品解锁数量。

## 12. AI/MCP 开工提示词

```text
请实现局外昼夜流程重构。

必须先读取：
- Doc/29_局外杂货店经营与界面重构路线图.md
- Doc/30_昼夜局外流程_店铺营业与夜间出发准备.md
- Doc/10_研究所与制作所系统_修订版_废土生存法则.md
- Doc/11_出发准备与局外装备系统_修订版_废土生存法则.md
- Doc/28_工程结构重构路线图.md

任务：
1. 从 BaseScene 中拆出 outgame_flow_controller，支持 DAY_PREP、SHOP_OPEN、SHOP_SETTLEMENT、NIGHT、NIGHT_PLAN、LOADOUT。
2. 白天准备阶段隐藏出发探索，只保留开店营业主按钮。
3. 制造所产物中增加 sale_good 标记，只有 sale_good=true 的物品可摆上货台。
4. 实现 daily_demand_service，按当前天数生成今日需求榜并保存。
5. 实现 shop_open_panel，包含 3 个货台格、60 秒倒计时、补货/撤货、自动售卖和提前结束营业。
6. 实现 shop_settlement_panel，展示卖出物资、数量、单价、小计和总收入。
7. 结算关闭后进入 NIGHT。
8. 夜晚点击出发探索进入 night_plan_panel，不直接 Loading。
9. night_plan_panel 使用角色列表和地点列表，即使当前只有一个角色和一个地点。
10. 点击准备进入 loadout_panel，按人形布局显示 HEAD/BODY/HAND/FOOT 和 4 个消耗品格。
11. 点击出发后复用现有 Loading 进入局内。

完成标准：
- 玩家能完成一轮：白天制造 -> 开店售卖 -> 结算 -> 夜晚计划 -> 出发准备 -> 进入局内。
- 原材料不能被直接摆上货台。
- 高需求商品比低需求商品更快售出。
- 出发准备只作为夜晚计划后的二级界面，不出现在白天功能入口中。
```

## 13. BDD 场景

```gherkin
Feature: 昼夜局外流程落地
  作为 避难所杂货店店主
  我希望 白天经营店铺，晚上选择地点并配置装备出发
  以便 局外经营和局内探索形成清晰节奏

  Scenario: 白天开店营业进入 60 秒倒计时
    Given 当前阶段为 DAY_PREP
    When 玩家点击开店营业
    Then 当前阶段应变为 SHOP_OPEN
    And 界面应显示 60 秒倒计时
    And 界面应显示 3 个货台格

  Scenario: 原材料不能被摆上货台
    Given 仓库中有 item_scrap_metal
    And item_scrap_metal 的 sale_good 为 false
    When 玩家点击空货台
    Then item_scrap_metal 不应出现在可上架列表中

  Scenario: 制造产物可以被摆上货台
    Given 仓库中有 good_water_filter
    And good_water_filter 的 sale_good 为 true
    When 玩家点击空货台并选择 good_water_filter
    Then 该货台应显示 good_water_filter

  Scenario: 关闭结算后进入夜晚
    Given 当前阶段为 SHOP_SETTLEMENT
    When 玩家关闭今日营业结算
    Then 当前阶段应变为 NIGHT
    And 界面应显示出发探索按钮

  Scenario: 没货时提前结算
    Given 当前阶段为 SHOP_OPEN
    And 开店剩余时间为 35 秒
    And 所有货台数量均为 0
    And 仓库中没有可上架 sale_good
    When 玩家点击提前结束营业并确认
    Then 当前阶段应变为 SHOP_SETTLEMENT
    And 界面应显示今日营业结算
    And 不需要等待剩余 35 秒

  Scenario: 夜晚出发探索进入夜间计划
    Given 当前阶段为 NIGHT
    When 玩家点击出发探索
    Then 当前阶段应变为 NIGHT_PLAN
    And 界面应显示角色列表、地点列表和地点详情

  Scenario: 夜间计划点击准备进入出发准备
    Given 当前阶段为 NIGHT_PLAN
    And 玩家已选择角色 owner
    And 玩家已选择地点 abandoned_house
    When 玩家点击准备
    Then 当前阶段应变为 LOADOUT
    And 界面应显示人形装备槽和消耗品槽

  Scenario: 出发准备点击出发进入 Loading
    Given 当前阶段为 LOADOUT
    And 装备和消耗品配置合法
    When 玩家点击出发
    Then 系统应保存本次出发配置
    And 系统应进入现有 Loading 流程
```

验收方式：

- 文档验收：本文存在明确状态机、界面布局、数据字段、BDD 场景。
- Debug 验收：可强制切换 DAY_PREP、SHOP_OPEN、SHOP_SETTLEMENT、NIGHT、NIGHT_PLAN、LOADOUT。
- 手动验收：完整走通一次白天营业和夜晚出发。
- 回归验收：原有 Loading 进入局内逻辑不被破坏。
