# 34_地图资源生态与地点再生规则

> 文档版本：v0.1
> 所属项目：《废土生存法则》
> 关联文档：`03_地图与安全区规则_修订版_废土生存法则.md`、`05_容器刷新与开箱系统_修订版_废土生存法则.md`、`31_掉落物资与制造经济规划.md`、`36_出门前目标面板与资源指引规则.md`。
> 文档目标：定义多地图资源生态、重复搜刮收益衰减和有限再生规则，让玩家能规划“去哪里找什么”，同时保留废土地点被消耗的生存感。

## 1. 核心定位

地图不是同一掉落池的不同皮肤。每张地图必须代表一个资源生态。

```text
诊所更容易产出医疗。
工业区更容易产出零件和组件。
住宅区更容易产出食物、布料和烟酒。
商业街更容易产出杂货、奢侈品和包装材料。
检查站更容易产出武器、装备和高风险贵重物。
```

## 2. 地图资源身份

| 地点 | 主要产出 | 次要产出 | 特殊产出 |
|---|---|---|---|
| 诊所 | 药品、酒精、绷带、无菌材料 | 布料、清洁剂 | 医疗配方开放、稀有药剂 |
| 工业区 | 零件、工具、电子组件、线缆 | 废金属、燃料 | 机器核心、设备任务条件 |
| 住宅区 | 食物、布料、生活用品 | 烟酒、低级贵重物 | 家庭储藏、剧情物 |
| 商业街 | 包装材料、杂货、烟酒珠宝 | 食物、工具 | 高价值商品、商人事件 |
| 检查站 | 武器部件、弹药、护具 | 工具、医疗 | 军用任务条件、高风险订单物 |

## 3. 地点状态

地点不采用“每天全刷新”，也不采用“完全一次性掏空”。

推荐状态：

| 状态 | 资源量 | 说明 |
|---|---:|---|
| 富集 | 100% | 首次进入或事件后刷新，含特殊资源 |
| 已搜刮 | 30%-50% | 短期重复进入，基础资源仍可找到 |
| 贫瘠 | 10%-30% | 连续搜刮后，收益明显下降 |
| 恢复中 | 50%-70% | 间隔数天或事件后，低级资源恢复 |
| 事件占据 | 变量 | 新敌人、商人、封锁、塌方或剧情改变收益 |

## 4. 一次性与再生

资源分三层：

```text
固定一次性资源：配方开放、关键设备、剧情物、特殊订单物。
地区倾向资源：该地图长期更容易出的类别。
日常再生资源：低级材料、基础商品燃料、少量补给。
```

规则：

```text
特殊资源拿走后不自然刷新。
基础资源可有限再生。
重复搜刮收益递减。
事件可以让旧地图重新有意义。
```

## 5. 容器产出关系

地图状态和容器系统的关系：

```text
地图类型决定大池。
房间类型决定中池。
容器类型决定小池。
地点状态修正数量和稀有概率。
```

示例：

```text
已搜刮诊所的药柜仍可能出绷带或空药瓶，但不应高频再出稀有抗生素。
恢复中的工业区仍有废金属和螺栓，但机器核心只通过事件或首次探索出现。
```

## 6. 地图 UI 信息

地图选择界面至少给出：

```text
主要产出类别
当前状态
预计收益等级
风险等级
是否存在未开启特殊区域
```

示例：

```text
诊所
主要产出：药品 / 酒精 / 布料
状态：已搜刮
预计收益：低
风险：中
特殊：地下药房尚未打开
```

## 7. 数据配置建议

### 7.1 map_resource_profiles.tab

用途：定义每张地图的资源身份和推荐权重。

建议字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| map_id | string | 地图 ID |
| display_name | string | 地图名称 |
| primary_categories | list | 主要资源类别 |
| secondary_categories | list | 次要资源类别 |
| special_tags | list | 医疗配方开放、军用物资、商人事件等 |
| base_risk | int | 基础风险等级 |
| default_state | enum | rich/searched/poor/recovering/event |
| enabled_version | string | 启用版本 |

### 7.2 room_resource_profiles.tab

用途：定义房间对地图资源池的二次筛选。

建议字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| room_profile_id | string | 房间资源配置 ID |
| room_type | string | 房间类型，如 clinic_room / warehouse / bedroom |
| category_bias | list | 房间倾向资源类别 |
| container_bias | list | 常见容器类型 |
| rare_tag_bias | list | 稀有标签倾向 |
| enabled_version | string | 启用版本 |

### 7.3 location_state_rules.tab

用途：定义重复搜刮、恢复和事件改变地点状态的规则。

建议字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| state_id | enum | rich/searched/poor/recovering/event |
| quantity_multiplier | float | 数量修正 |
| rare_multiplier | float | 稀有权重修正 |
| regen_days | int | 自然恢复所需天数，0 表示不自然恢复 |
| unique_drop_policy | enum | allow_once/blocked/event_only |
| ui_label | string | 地图界面显示文本 |

## 8. Godot 模块建议

| 模块 | 职责 |
|---|---|
| LocationResourceProfileService | 查询地图、房间、容器的资源倾向 |
| LocationStateService | 保存地点状态、搜刮次数、恢复天数和事件覆盖 |
| LootPoolResolver | 将 map_profile + room_profile + container_type + state 解析为可用掉落池 |
| MapRecommendationDataProvider | 向出门前目标面板提供地图类别、状态和风险收益 |

容器系统不应直接写死“诊所掉医疗”，而应通过 profile 解析。这样后续新增地图或临时事件时，只改配置表。

## 9. 校验规则

版本启用时必须检查：

```text
每张可进入地图必须有 map_resource_profiles 记录。
每种可刷容器必须能通过地图、房间和容器类型解析到至少一个掉落池。
unique_drop_policy 为 allow_once 的资源不能被自然恢复重复投放。
贫瘠状态仍应保留少量基础资源，不能把常规容器全部变成空容器。
地图 UI 显示的主要类别必须来自 map_resource_profiles，不允许手写另一套口径。
```

## 10. BDD 场景补充

Feature: 地图资源生态与地点再生

  Scenario: 玩家根据资源缺口选择地图
    Given 玩家缺少医疗类资源
    When 玩家查看地图选择界面
    Then 诊所应显示医疗类为主要产出
    And 推荐信息不应把工业区标为医疗主要来源

  Scenario: 重复搜刮地点收益衰减但不归零
    Given 玩家已经搜刮过诊所
    When 玩家短期内再次进入诊所
    Then 诊所仍可产出少量医疗基础资源
    And 稀有医疗资源概率应明显降低

  Scenario: 特殊资源一次性
    Given 玩家已经取得某地点的唯一任务条件
    When 玩家再次进入同一地点
    Then 该任务条件不应自然刷新
