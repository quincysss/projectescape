# 08-10 本局统计与结束重置

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

收集本局统计并在结算返回哨所流程中重置局内状态，保证下一局从干净状态开始。

## 2. 统计字段

建议：

```gdscript
{
    "run_seconds": 0,
    "items_collected": 0,
    "containers_opened": 0,
    "outposts_repaired": 0,
    "distance_traveled": 0.0,
    "end_type": "EXTRACTION_SUCCESS"
}
```

## 3. 重置内容

结算画面点击 `返回哨所` 后，进入 `ReturnToBaseLoadingScreen`。以下重置内容在返回 loading 的 `clear_run` 阶段执行一次：

- 清理局内背包。
- 清理临时容器。
- 清理前哨临时存储。
- 注销临时安全区。
- 重置地图临时状态。
- 保存本次结算后的局外仓库、货币和章节目标状态。
- 推进局外日刷新天数：成功或失败回到局外前，`current_day += 1`，并触发按天刷新的局外系统。
- 不推进 `surface_day`。`surface_day` 只在正式出发进入地表时增加，用于第三日怪物、场景随机事件等地表规则。
- 返回 loading 读到 100% 后，只显示 `按下任意按钮继续`，等待玩家输入再切回局外界面。
- 按任意按钮切回局外时不再重复保存、清理或推进天数。

## 4. AI/MCP 开工提示词

```text
请实现本局统计和结束重置。
要求：
1. RunResult 包含 run_seconds、items_collected、containers_opened、outposts_repaired、distance_traveled、end_type。
2. 成功和失败都生成统计。
3. 玩家在结算界面点击“返回哨所”后，进入 ReturnToBaseLoadingScreen。
4. 在返回 loading 的 clear_run 阶段清理局内背包、容器、前哨临时存储和安全区。
5. 重置只执行一次，并在提交成功后设置 settlement_return_committed。
6. 返回局外界面前保存局外仓库、货币和章节目标状态。
7. 返回局外界面前推进 current_day += 1，并触发按天刷新的局外系统。
8. 不要在返回 loading 中推进 surface_day；surface_day 只属于出发进入地表流程。
9. loading 读到 100% 后停留在完成态，提示“按下任意按钮继续”，玩家输入后再切回局外界面。
```

## 5. 验收标准

- 结算界面能显示统计。
- 下一局不会继承上一局临时状态。
- 仓库保存不受重置影响。
- 成功和失败结算都会在返回 loading 提交阶段让局外天数推进 1 天。
- 返回 loading 不会推进 `surface_day`。
- loading 100% 后不会自动回局外，必须等待玩家按任意按钮。
