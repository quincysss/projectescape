# 08-10 本局统计与结束重置

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

收集本局统计并在结算完成后重置局内状态，保证下一局从干净状态开始。

## 2. 统计字段

建议：

```gdscript
{
    "run_seconds": 0,
    "items_collected": 0,
    "containers_opened": 0,
    "outposts_repaired": 0,
    "distance_traveled": 0.0,
    "end_type": "EXTRACTION_SUCCESS"
}
```

## 3. 重置内容

结算完成后：

- 清理局内背包。
- 清理临时容器。
- 清理前哨临时存储。
- 注销临时安全区。
- 重置地图临时状态。
- 返回局外界面。

## 4. AI/MCP 开工提示词

```text
请实现本局统计和结束重置。
要求：
1. RunResult 包含 run_seconds、items_collected、containers_opened、outposts_repaired、distance_traveled、end_type。
2. 成功和失败都生成统计。
3. 玩家处理完结算后，清理局内背包、容器、前哨临时存储和安全区。
4. 重置只执行一次。
5. 返回局外界面前保存局外仓库状态。
```

## 5. 验收标准

- 结算界面能显示统计。
- 下一局不会继承上一局临时状态。
- 仓库保存不受重置影响。
