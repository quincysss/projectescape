# 08-02 撤离条件与激活显示

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

实现 V0.1 固定撤离条件，并让玩家能从撤离点、按钮和家中提示理解当前是否可撤离。

## 2. V0.1 撤离条件

建议固定规则：

- 玩家存活。
- 撤离点已激活。
- 玩家处于撤离范围。
- 没有正在进行互斥交互。

如果 06 前哨站作为撤离条件，则：

```text
指定前哨站 ACTIVE -> 撤离点激活
```

该条件必须同步给左侧目标链 HUD：

```text
前哨未 ACTIVE -> 解锁撤离：未解锁
前哨 ACTIVE 且玩家在家外 -> 解锁撤离：已解锁
前哨 ACTIVE 且玩家在家中/撤离范围 -> 解锁撤离：可撤离
```

## 3. 显示规则

撤离信号 UI 状态：

| 状态 | 表现 |
| --- | --- |
| 未激活 | 灰色，显示缺失条件 |
| 已激活但玩家在家外 | 高亮，显示“返回家中可撤离” |
| 已激活且玩家在家中 | 高亮，显示“长按 E 键进行撤离” |
| 撤离中 | 在撤离信号 UI 内显示增长进度 |
| 已结束 | 禁用 |

家中或安全区提示：

- 当前撤离条件。
- 已修复前哨站对撤离的影响。
- 家中时在撤离信号 UI 下方提示“按 TAB 打开背包界面”。
- 撤离入口使用顶部撤离信号 UI，不在家中存储面板旁另放撤离按钮。
- 左侧目标链 HUD 第三行必须与撤离状态一致，显示“解锁撤离：未解锁/已解锁/可撤离”。
- 玩家尝试撤离但未解锁时，提示“撤离未解锁：先修复前哨站”。

## 4. AI/MCP 开工提示词

```text
请实现 V0.1 撤离条件与 UI 激活显示。
要求：
1. 撤离点激活后，玩家在范围内才可按住撤离。
2. 如果 06 前哨站接入，则指定前哨站 ACTIVE 后激活撤离点。
3. UI 显示未激活、家外可返回、家中可长按、撤离中、已结束五种状态。
4. 未激活时显示简短缺失条件。
5. 撤离条件查询不要写死在 UI 内，提供 ExtractionController 接口。
6. 将撤离解锁状态同步给左侧目标链 HUD 的第三行。
```

## 5. 验收标准

- 未满足条件时不能撤离。
- 条件满足后 UI 更新。
- 前哨站激活能刷新撤离状态。

## 6. 暂不实现

- 多个随机撤离点。
- 额外强制任务链。
