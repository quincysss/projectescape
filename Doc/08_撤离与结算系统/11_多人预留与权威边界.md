# 08-11 多人预留与权威边界

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

V0.1 不做多人，但撤离和结算需要预留玩家 ID、队伍 ID 和权威字段。

## 2. 预留字段

```gdscript
var extracting_player_id: String = "local_player"
var squad_id: String = ""
var run_result_revision: int = 0
var settlement_confirmed_by: String = ""
```

## 3. 未来权威

未来多人时应由服务器/主机权威决定：

- 撤离条件是否满足。
- 撤离读条是否完成。
- 哪些玩家成功撤离。
- 结算物品来源。
- 入库确认。

## 4. AI/MCP 开工提示词

```text
请为撤离与结算系统预留多人字段，但不实现网络同步。
要求：
1. 撤离交互记录 extracting_player_id。
2. RunResult 包含 player_id、squad_id、revision。
3. 入库确认记录 settlement_confirmed_by。
4. V0.1 使用 local_player。
5. 状态变化 revision +1。
```

## 5. 验收标准

- 单机流程正常。
- RunResult 带玩家字段。
- 后续多人可扩展。

## 6. 暂不实现

- 队伍撤离等待。
- 分玩家结算。
