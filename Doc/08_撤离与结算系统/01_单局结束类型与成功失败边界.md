# 08-01 单局结束类型与成功失败边界

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

定义单局结束类型，并明确成功与失败对物品归属、统计展示和局内状态重置的影响。

## 2. 结束类型

```gdscript
enum RunEndType {
    EXTRACTION_SUCCESS,
    STABILITY_FAILED,
    TIMEOUT_FAILED,
    DEBUG_ABORT
}
```

## 3. 成功与失败区别

成功：

- 背包物品进入结算候选。
- 前哨临时存储可按规则进入结算候选。
- 显示获得物品、统计和入库入口。
- 局外仓库可接收物品。

失败：

- 局内背包按失败规则丢失。
- 前哨临时存储通常丢失。
- 显示损失说明。
- 不允许把失败局内收益静默入库。

## 4. 接口建议

```gdscript
func end_run(end_type: RunEndType, reason: String) -> void
func is_success(end_type: RunEndType) -> bool
```

## 5. AI/MCP 开工提示词

```text
请实现单局结束类型和成功/失败边界。
要求：
1. 定义 EXTRACTION_SUCCESS、STABILITY_FAILED、TIMEOUT_FAILED、DEBUG_ABORT。
2. 成功才允许进入可入库结算流程。
3. 失败必须生成损失说明，不得静默把局内物品入库。
4. end_run 必须只触发一次，防止重复结算。
5. 输出状态字段、信号和最小验证步骤。
```

## 6. 验收标准

- 同一局只能结算一次。
- 成功/失败流程分离。
- 失败不会错误入库。

## 7. 暂不实现

- 断线重连结算。
- 中途投降收益保留。
