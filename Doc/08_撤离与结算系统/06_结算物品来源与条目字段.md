# 08-06 结算物品来源与条目字段

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

统一结算物品来源和条目字段，让背包、前哨、任务奖励都能进入同一结算列表。

## 2. 来源类型

```text
inventory
outpost_storage
objective_reward
debug
```

## 3. 条目字段

```gdscript
{
    "settlement_item_id": "uuid",
    "item_id": "scrap_metal",
    "count": 3,
    "quality": "B",
    "source": "inventory",
    "weight": 1.5,
    "sellable": true,
    "sell_currency_id": "mine_coin",
    "sell_value_per_unit": 3,
    "can_store": true,
    "status": "pending"
}
```

状态：

- `pending`
- `stored`
- `discarded`
- `overflow`

## 4. 来源排序

建议显示顺序：

1. 任务/目标奖励。
2. 背包物品。
3. 前哨临时存储。
4. Debug/其他。

## 5. AI/MCP 开工提示词

```text
请实现 RunResultBuilder 的结算物品来源汇总。
要求：
1. 支持 inventory、outpost_storage、objective_reward、debug 来源。
2. 每个条目包含 settlement_item_id、item_id、count、quality、source、weight、sellable、sell_currency_id、sell_value_per_unit、can_store、status。
3. 条目默认 pending。
4. 结算 UI 按来源排序显示。
5. 失败结算也可生成损失条目，但不能标记为可入库。
6. 售卖价值来自 07 ItemDefinition / 13 items.tab，不在结算系统里硬编码。
```

## 6. 验收标准

- 背包和前哨物品能汇总。
- 来源字段正确。
- 条目状态可更新。
- 售卖字段能传递到局外商人，但结算界面不直接完成出售。
