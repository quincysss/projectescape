# 08-08 一键入库与手动入库

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

为结算物提供一键入库和手动入库入口，但最终处理必须尊重玩家选择，并通过 09 局外仓库接口。

结算界面只负责把可带回物品送入局外处理流程。玩家想出售物品时，应先完成入库或进入局外商人页签，再通过 09 商人系统选择出售数量。

## 2. 一键入库

一键入库流程：

```text
读取 pending 且 can_store 条目
-> 按 09 仓库优先级排序
-> 调用 WarehouseManager.try_store_items
-> 更新条目状态
-> 显示未入库原因
```

## 3. 手动入库

手动入库允许：

- 选择单个物品入库。
- 选择数量。
- 查看仓库空间。
- 处理空间不足。

## 4. AI/MCP 开工提示词

```text
请实现结算界面的一键入库和手动入库入口。
要求：
1. 一键入库只处理 pending 且 can_store 的结算条目。
2. 入库必须调用 09 WarehouseManager/SettlementTransferService。
3. 手动入库支持选择单个物品和数量。
4. 空间不足时保留 pending 或标记 overflow，不得吞物品。
5. 入库结果同步刷新结算条目状态。
```

## 5. 验收标准

- 一键入库能把可入库物品送入仓库。
- 空间不足时有剩余状态。
- 手动入库不会绕过仓库容量。

## 6. 暂不实现

- 自动出售。
- 入库推荐算法 UI。
- 结算界面内嵌商人。
