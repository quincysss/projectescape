# 08-03 撤离交互与按住进度

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

实现撤离点按住交互，让撤离成为一个可被风险打断的最终动作。

## 2. 触发方式

玩家必须：

- 在撤离范围内。
- 满足撤离条件。
- 按住撤离键 E，或按住顶部撤离信号 UI。
- 没有其他读条或 UI 锁定。

## 3. 撤离进度

```gdscript
@export var extraction_hold_seconds: float = 3.0
```

流程：

```text
按下 E/撤离信号 UI -> 撤离信号 UI 内进度增长 -> 持续满足条件 -> 进度满 -> 撤离成功
```

## 4. 中断条件

- 离开家安全区。
- 松开按键。
- 受到关键打断。
- 稳定值归零。
- 对局时间归零。
- 撤离点状态失效。

V0.1 中断后进度重置。

## 5. AI/MCP 开工提示词

```text
请实现撤离按住交互。
要求：
1. 玩家在撤离范围内且条件满足时可按住撤离键。
2. 默认按住 3 秒完成撤离。
3. 离开范围、松开按键、稳定值归零、时间归零会中断。
4. 中断后进度重置。
5. 进度完成后调用 end_run(EXTRACTION_SUCCESS)。
6. 防止重复触发撤离成功。
7. 撤离进度显示在顶部撤离信号 UI 内，家中存储面板不再承载撤离按钮。
```

## 6. 验收标准

- 按住进度正常增长。
- 中断后不撤离。
- 完成后进入成功结算。
