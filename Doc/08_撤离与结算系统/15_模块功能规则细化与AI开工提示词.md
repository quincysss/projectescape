# 08 模块功能规则细化与 AI 开工提示词

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 用途：作为 08 模块的 AI/MCP 总入口。

## 1. 总体实现目标

撤离与结算系统负责把一局探索转换为明确结果：

```text
撤离成功 -> 物品可入库 -> 本局收益成立
失败结束 -> 显示损失 -> 本局收益不成立或按失败规则处理
结算画面 -> 点击返回哨所 -> 返回 loading -> 按任意按钮回局外
```

## 4. 分阶段 AI 开工提示词

### 阶段一：撤离条件和交互

```text
请读取 08 拆分文档 01、02、03、12。
实现 ExtractionController 和 ExtractionPoint。
支持撤离条件查询、UI 激活状态、按住 3 秒撤离、中断和防重复结算。
同时把撤离解锁状态输出给左侧目标链 HUD：未解锁、已解锁、可撤离。
```

### 阶段二：成功失败结算

```text
请读取 08 拆分文档 04、05、06、10。
实现成功流程、稳定值失败、超时失败、RunResultBuilder、本局统计和结束重置。
成功从 07 背包和 06 前哨 payload 收集物品；失败显示损失说明。
```

### 阶段三：结算 UI 和入库

```text
请读取 08 拆分文档 07、08、09、16。
实现 SettlementUI、一键入库、手动入库、溢出处理、丢弃和返回哨所入口。
入库必须调用 09 仓库接口，空间不足不能吞物，离开前必须确认未处理物品。
成功结算主文案为“成功返回404哨所”，失败结算主文案为“你已被黑潮吞噬，将在404重塑躯体。”。
```

### 阶段三点五：返回哨所 loading

```text
请读取 08 拆分文档 16 和 25 的出发 loading 文档。
实现 ReturnToBaseLoadingController 和 ReturnToBaseLoadingScreen。
玩家在结算界面点击“返回哨所”后进入返回 loading，执行结算提交、仓库保存、局内清理、局外 UI 预热。
loading 到 100% 后显示“按下任意按钮继续”，玩家按任意按钮才回到局外。
返回 loading 不增加 surface_day，不启用局内控制，不走出发装备提交逻辑。
```

### 阶段四：Debug 和多人预留

```text
请读取 08 拆分文档 11、13、14。
补齐 player_id/revision 字段和 Debug 面板。
Debug 要能强制成功、强制失败、构建测试结算、测试满仓溢出。
```

## 5. 总体验收标准

- 撤离条件满足后可撤离。
- 撤离读条可中断。
- 成功/失败只结算一次。
- 成功能进入入库流程。
- 失败能解释损失。
- 仓库满时结算物不消失。
- 结算完成后局内临时状态清理。
- 撤离解锁状态与左侧目标链 HUD 第三行一致。
- 成功/失败结算都显示返回哨所按钮。
- 返回哨所 loading 100% 后等待任意按钮才回局外。

## 6. 给后续 AI 的硬约束

```text
不要删除原始 08 文档。
不要重复结算。
不要静默丢弃结算物。
不要绕过 09 仓库接口入库。
不要让失败流程进入成功入库。
不要让结算界面直接切回局外，必须经过返回 loading。
不要让返回 loading 增加 surface_day。
不要让顶部撤离状态和左侧目标链的撤离状态互相矛盾。
不要实现多人同步，只预留字段。
```
