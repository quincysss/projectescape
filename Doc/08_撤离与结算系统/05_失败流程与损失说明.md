# 08-05 失败流程与损失说明

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

实现稳定值归零和对局时长归零的失败流程，先全黑进入失败结算画面，再展示可带出的物资和返回哨所入口。

## 2. 失败类型

稳定值归零：

- 原因显示为精神/稳定崩溃。
- 局内背包收益失败处理。
- 显示丢失物品摘要。

对局时长归零：

- 原因显示为时间耗尽。
- 局内收益失败处理。
- 可显示未撤离提示。

## 3. 损失说明

失败界面必须显示：

- 失败原因。
- 主文案：`你已被黑潮吞噬，将在404重塑躯体。`
- 本局带出的物资。
- 无可带出物资时显示：`没有带回任何物资`。
- 返回哨所按钮。

不要只显示“失败”两个字。

失败流程：

```text
稳定值/时间归零
-> 锁定玩家控制
-> 全屏黑屏
-> 构建失败 RunResult
-> 打开失败 SettlementResultScreen
-> 展示“你已被黑潮吞噬，将在404重塑躯体。”
-> 展示“本局带出的物资”
-> 玩家点击“返回哨所”
-> 进入 ReturnToBaseLoadingScreen
```

## 4. AI/MCP 开工提示词

```text
请实现失败结算流程。
要求：
1. 稳定值归零调用 end_run(STABILITY_FAILED)。
2. 对局时间归零调用 end_run(TIMEOUT_FAILED)。
3. 失败后锁定玩家控制。
4. 打开失败结算画面，主文案为“你已被黑潮吞噬，将在404重塑躯体。”。
5. 展示本局带出的物资；没有时显示空状态。
6. 失败不允许把局内背包收益当作成功收益入库。
7. 点击返回哨所后进入返回 loading，100% 后等待任意按钮。
8. 防止失败和撤离成功同时结算。
```

## 5. 验收标准

- 稳定值归零会失败。
- 时间归零会失败。
- 失败界面能解释损失。
- 失败不会打开成功入库流程。
- 失败画面有返回哨所按钮，并能进入返回 loading。

## 6. 暂不实现

- 死亡尸体包。
- 保险物品保留。
