# 08-05 失败流程与损失说明

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

实现稳定值归零和对局时长归零的失败流程，并在结算界面清楚解释损失。

## 2. 失败类型

稳定值归零：

- 原因显示为精神/稳定崩溃。
- 局内背包收益失败处理。
- 显示丢失物品摘要。

对局时长归零：

- 原因显示为时间耗尽。
- 局内收益失败处理。
- 可显示未撤离提示。

## 3. 损失说明

失败界面必须显示：

- 失败原因。
- 丢失物品数量或摘要。
- 保留物品规则。
- 返回局外按钮。

不要只显示“失败”两个字。

## 4. AI/MCP 开工提示词

```text
请实现失败结算流程。
要求：
1. 稳定值归零调用 end_run(STABILITY_FAILED)。
2. 对局时间归零调用 end_run(TIMEOUT_FAILED)。
3. 失败后锁定玩家控制。
4. 生成损失说明，包括失败原因、丢失物品摘要、保留规则。
5. 失败不允许一键入库局内收益。
6. 防止失败和撤离成功同时结算。
```

## 5. 验收标准

- 稳定值归零会失败。
- 时间归零会失败。
- 失败界面能解释损失。
- 失败不会打开成功入库流程。

## 6. 暂不实现

- 死亡尸体包。
- 保险物品保留。
