# 08-09 确认入库、溢出与丢弃

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

处理仓库空间不足、玩家确认入库和主动丢弃结算物，避免系统静默替玩家丢东西。

## 2. 溢出处理

当仓库空间不足：

- 已成功入库的条目标记 `stored`。
- 未入库条目标记 `overflow` 或保持 `pending`。
- 显示需要整理仓库。
- 允许返回仓库整理后继续入库。

## 3. 确认规则

离开结算界面前，如果仍有 pending/overflow 物品：

- 必须提示玩家。
- 允许继续整理。
- 允许确认丢弃。
- 不得静默丢弃。

## 4. 丢弃结算物

丢弃结算物：

- 只影响结算条目。
- 不生成世界掉落。
- 高价值物品建议二次确认。

## 5. AI/MCP 开工提示词

```text
请实现结算入库确认、溢出和丢弃处理。
要求：
1. 仓库空间不足时未入库条目不能消失。
2. 离开结算界面前如果还有 pending/overflow，必须提示玩家确认。
3. 玩家可选择整理仓库后继续入库。
4. 玩家确认丢弃后条目标记 discarded。
5. V0.1 丢弃不生成世界掉落。
```

## 6. 验收标准

- 仓库满时不吞结算物。
- 离开前会提示未处理物品。
- 丢弃有明确状态。

## 7. 暂不实现

- 结算物临时保险箱。
- 丢弃撤销。
