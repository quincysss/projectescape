# 08-16 结算返回哨所加载流程

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`  
> 目标：补齐撤离成功/失败后的结算画面，以及玩家点击“返回哨所”后通过 loading 条回到局外的完整流程。

## 1. 实现目标

单局结束后不能直接切回局外界面。

必须先进入结算画面，让玩家明确本局结果和带回物资，然后由玩家主动点击“返回哨所”。

点击后进入返回哨所 loading，loading 读完后显示“按下任意按钮继续”，玩家按任意按钮才回到局外仓库/哨所界面。

## 2. 成功流程

```text
玩家完成撤离读条
-> 局内输入锁定
-> 屏幕淡入全黑
-> 构建成功 RunResult
-> 打开成功结算画面
-> 玩家查看本次获得物资
-> 玩家点击“返回哨所”
-> 打开 ReturnToBaseLoadingScreen
-> 提交结算、保存仓库、清理局内
-> loading 到 100%
-> 显示“按下任意按钮继续”
-> 玩家按任意按钮
-> 返回局外哨所界面
```

成功结算画面文案：

```text
成功返回404哨所
```

物资标题：

```text
本次获得的物资
```

按钮：

```text
返回哨所
```

## 3. 失败流程

失败来源：

- 稳定值归零。
- 对局时间归零。
- 后续其他失败原因。

失败流程：

```text
失败条件触发
-> 局内输入锁定
-> 屏幕立即或快速淡入全黑
-> 构建失败 RunResult
-> 打开失败结算画面
-> 玩家查看本局带出的物资
-> 玩家点击“返回哨所”
-> 打开 ReturnToBaseLoadingScreen
-> 提交失败结算、保存可保留物品、清理局内
-> loading 到 100%
-> 显示“按下任意按钮继续”
-> 玩家按任意按钮
-> 返回局外哨所界面
```

失败结算画面文案：

```text
你已被黑潮吞噬，将在404重塑躯体。
```

物资标题：

```text
本局带出的物资
```

如果没有任何可带出物资：

```text
没有带回任何物资
```

按钮：

```text
返回哨所
```

## 4. 画面布局

成功和失败结算画面都使用全黑背景。

布局从上到下：

```text
全屏黑色背景
中心偏上：结果主文案
中部：物资列表标题
中部偏下：物资展示区
底部偏中：返回哨所按钮
```

成功画面：

- 主文案使用灰白或偏暖白。
- 可以有很轻的暗色呼吸光，但不要做胜利庆典。
- 物资列表强调“带回来了什么”。

失败画面：

- 主文案使用灰白，允许轻微红黑边缘。
- 不显示夸张死亡特效。
- 重点表达黑潮吞噬和 404 重塑躯体。
- 物资列表只显示失败规则允许带出的物资。

## 5. 物资展示规则

成功结算展示：

- 玩家背包中成功撤离带回的物品。
- 家安全存储中的物品。
- 已修复前哨站安全存储中的物品。
- 任务奖励或章节奖励，如有。

失败结算展示：

- 家安全存储中的物品。
- 已修复前哨站安全存储中允许带出的物品。
- 失败规则明确保留的物品。

失败时不展示“已丢失物品”作为主要物资列表。丢失摘要可以放在 Debug 或折叠说明里，避免画面信息过重。

物品卡片字段：

```text
icon
display_name
count
quality
source
status
```

## 6. 返回哨所 loading

返回哨所 loading 复用出发 loading 的机制：

```text
任务权重 + 显示平滑
真实任务未完成不能显示 100%
100% 后停留在 loading 界面
显示“按下任意按钮继续”
玩家按任意按钮后才切回局外
```

但文案和任务不同。

推荐阶段：

| 阶段 | 进度区间 | 玩家可见文案 | 系统任务 |
| --- | ---: | --- | --- |
| lock_settlement | 0%-10% | 封存本局记录... | 锁定结算按钮，防止重复提交 |
| transfer_items | 10%-35% | 整理带回物资... | 将可保留物资写入结算转移队列 |
| save_warehouse | 35%-55% | 同步哨所仓库... | 写入仓库、安全存储、货币和统计 |
| clear_run | 55%-75% | 关闭地表通道... | 清理局内实体、计时器、事件、临时背包 |
| load_base | 75%-92% | 返回404哨所... | 加载局外白天准备界面、仓库、制造所、研究所、图鉴和需求榜 |
| prewarm_base_ui | 92%-98% | 点亮哨所终端... | 等待局外首帧和 UI 准备完成 |
| ready_to_continue | 98%-100% | 按下任意按钮继续 | 等待玩家确认回到局外 |

返回 loading 完成态：

```text
标题：正在返回404哨所
阶段：按下任意按钮继续
进度：100%
提示：物资已同步至哨所记录。
```

失败时提示可以改为：

```text
提示：躯体重塑完成，保留物资已同步。
```

## 7. 提交与保存时机

推荐提交点：

```text
玩家点击“返回哨所”
-> 创建 ReturnToBaseLoadContext
-> loading 执行结算提交和保存
-> loading 到 100% 后等待任意按钮
-> 玩家按任意按钮
-> GameFlowController 切回 BaseHub/Warehouse
```

结算提交必须只执行一次。

防重复字段：

```gdscript
{
    "settlement_commit_started": true,
    "settlement_committed": false,
    "return_loading_ready": false
}
```

如果保存失败：

- 停留在返回 loading 错误态。
- 不切回局外。
- 显示：`哨所记录同步失败，请重试。`
- Debug 模式输出失败原因。

## 8. 与入库规则的关系

V0.1 可以采用简化策略：

```text
结算画面只展示本次获得/带出的物资。
玩家点击“返回哨所”后，系统按现有 08/09 入库规则提交。
如果仓库容量足够，自动写入仓库。
如果仓库容量不足，必须先进入溢出处理，不允许静默吞物。
```

如果后续保留手动入库/一键入库：

- 手动入库区仍在结算画面内。
- `返回哨所` 是最终离开按钮。
- 存在未处理物资时，点击 `返回哨所` 必须弹出确认或进入溢出处理。

推荐模块：

```text
systems/extraction/settlement_ui.gd

systems/extraction/return_to_base_loading_controller.gd
systems/extraction/return_to_base_load_context.gd
systems/flow/game_flow_controller.gd
```

推荐场景：

```text
scenes/ui/SettlementResultScreen.tscn
scenes/ui/ReturnToBaseLoadingScreen.tscn
```

可复用：

```text
RunLoadingScreen 的进度条组件
RunLoadingController 的任务权重和平滑显示逻辑
```

不要复用：

```text
出发 loading 的提交 surface_day +1 逻辑
出发 loading 的局内输入启用逻辑
```

```text
请实现结算后返回哨所流程。
读取 Doc/08_撤离与结算系统/16_结算返回哨所加载流程.md、04_撤离成功流程.md、05_失败流程与损失说明.md、07_结算界面与物品展示.md，以及 Doc/25_主菜单玩家档案与剧情章节目标系统/07_出发进入对局加载进度条.md。

要求：
1. 撤离成功后先全黑，再显示成功结算画面，主文案为“成功返回404哨所”。
2. 失败后先全黑，再显示失败结算画面，主文案为“你已被黑潮吞噬，将在404重塑躯体。”。
3. 成功画面展示“本次获得的物资”，失败画面展示“本局带出的物资”。
4. 两种结算画面底部都有“返回哨所”按钮。
5. 点击“返回哨所”后进入 ReturnToBaseLoadingScreen。
6. 返回 loading 使用任务权重和平滑进度；100% 后显示“按下任意按钮继续”，不能自动回局外。
7. 玩家按任意按钮后才切回局外仓库/哨所界面。
8. 返回 loading 不增加 surface_day，不启用局内控制，不走出发装备提交逻辑。
9. 结算提交、仓库保存、局内清理只能执行一次。
10. 保存失败时停留在 loading 错误态，并允许重试。
```

## 11. 验收标准

| 编号 | 标准 |
| --- | --- |
| A1 | 撤离成功后进入全黑结算画面 |
| A2 | 成功标题为“成功返回404哨所” |
| A3 | 成功画面展示本次获得物资 |
| A4 | 失败后进入全黑失败结算画面 |
| A5 | 失败标题为“你已被黑潮吞噬，将在404重塑躯体。” |
| A6 | 失败画面展示本局带出的物资或空状态 |
| A7 | 两种结算都有“返回哨所”按钮 |
| A8 | 点击按钮后进入返回 loading |
| A9 | 返回 loading 到 100% 后等待任意按钮 |
| A10 | 按任意按钮后才回到局外 |
| A11 | 返回 loading 不增加 `surface_day` |
| A12 | 结算提交和保存不会重复执行 |
