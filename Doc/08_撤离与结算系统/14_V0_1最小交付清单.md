# 08-14 V0.1 最小交付清单

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 最小闭环

```text
撤离条件满足
-> 玩家按住撤离
-> 撤离成功
-> 汇总结算物
-> 玩家入库或丢弃
-> 保存仓库
-> 重置本局
```

失败闭环：

```text
稳定值/时间归零
-> 失败结算
-> 显示损失
-> 重置本局
```

## 2. 必须交付

- 结束类型。
- 撤离条件。
- 撤离点按住读条。
- 撤离成功流程。
- 稳定值/超时失败。
- RunResultBuilder。
- SettlementUI。
- 一键入库和手动入库。
- 溢出处理。
- 丢弃结算物。
- 本局统计。
- 结束重置。
- Debug 面板。

## 3. AI/MCP 开工提示词

```text
请根据 08_撤离与结算系统拆分文档，在 Godot 中完成 V0.1 最小闭环。
优先实现撤离条件、按住撤离、成功/失败结算、物品来源汇总、结算 UI、入库/溢出/丢弃、本局重置和 Debug。
约束：不要静默丢弃物品；不要重复结算；入库必须通过 09 仓库接口；物品来源从 07 背包和 06 前哨 payload 读取。
```

## 4. 完成定义

- 玩家可以成功撤离。
- 稳定值/时间失败会进入失败结算。
- 成功结算能入库。
- 仓库满不吞物。
- 未处理物品有确认。
