## 1. 最小闭环

```text
撤离条件满足
-> 玩家按住撤离
-> 撤离成功
-> 汇总结算物
-> 显示成功返回404哨所
-> 展示本次获得物资
-> 若有未处理/溢出物资，先处理入库或丢弃
-> 点击返回哨所
-> 返回 loading
-> 提交结算并保存仓库
-> 按任意按钮回局外
-> 重置本局
```

失败闭环：

```text
稳定值/时间归零
-> 失败结算
-> 全黑显示黑潮吞噬文案
-> 展示本局带出的物资
-> 点击返回哨所
-> 返回 loading
-> 按任意按钮回局外
-> 重置本局
```

## 2. 必须交付

- 结束类型。
- 撤离条件。
- 撤离点按住读条。
- 撤离成功流程。
- 稳定值/超时失败。
- RunResultBuilder。
- 结算条目售价字段。
- SettlementUI。
- 一键入库和手动入库。
- 溢出处理。
- 丢弃结算物。
- 本局统计。
- 结束重置。
- 成功/失败结算结果画面。
- 返回哨所 loading。
- Debug 面板。

## 3. AI/MCP 开工提示词

```text
请根据 08_撤离与结算系统拆分文档，在 Godot 中完成 V0.1 最小闭环。
优先实现撤离条件、按住撤离、成功/失败结算、物品来源汇总、结算 UI、返回哨所 loading、入库/溢出/丢弃、本局重置和 Debug。
约束：不要静默丢弃物品；不要重复结算；入库必须通过 09 仓库接口；物品来源从 07 背包和 06 前哨 payload 读取；售价字段来自 07 ItemDefinition / 13 items.tab；结算界面不直接出售物品；结算后不能直接回局外，必须走返回 loading 且 100% 后等待任意按钮。
```

## 4. 完成定义

- 玩家可以成功撤离。
- 稳定值/时间失败会进入失败结算。
- 成功结算能入库。
- 仓库满不吞物。
- 未处理物品有确认。
- 成功/失败结算都能点击返回哨所进入 loading。
- 返回 loading 完成后等待任意按钮才回局外。
