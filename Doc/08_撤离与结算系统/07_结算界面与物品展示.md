# 08-07 结算界面与物品展示

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

结算界面让玩家看清本局结果、获得/带出物资和返回哨所入口。V0.1 结算画面以全黑背景和中心文案为主，不直接把玩家扔回局外。

## 2. 必须显示

- 全黑背景。
- 成功或失败主文案。
- 物资列表标题。
- 结算物品列表或空状态。
- 每个物品的名称、数量、品质、来源。
- 返回哨所按钮。

成功主文案：

```text
成功返回404哨所
```

失败主文案：

```text
你已被黑潮吞噬，将在404重塑躯体。
```

## 3. 建议显示

- 探索时间。
- 拾取总量。
- 修复前哨数量。
- 丢失物品摘要，失败时可折叠显示。
- 一键入库/手动入库入口，若当前版本仍保留手动处理。

## 4. 物品展示

物品卡片字段：

```text
icon / display_name / count / quality / source / status
```

品质颜色应与 07 物品系统一致。

成功物资列表标题：

```text
本次获得的物资
```

失败物资列表标题：

```text
本局带出的物资
```

失败空状态：

```text
没有带回任何物资
```

底部按钮：

```text
返回哨所
```

点击 `返回哨所` 后不直接回局外，而是进入 `ReturnToBaseLoadingScreen`。

## 5. AI/MCP 开工提示词

```text
请实现 SettlementUI。
要求：
1. 使用全黑背景显示结算结果。
2. 成功主文案为“成功返回404哨所”，失败主文案为“你已被黑潮吞噬，将在404重塑躯体。”。
3. 成功展示“本次获得的物资”，失败展示“本局带出的物资”。
4. 物品展示包含名称、数量、品质、来源、状态。
5. 底部显示“返回哨所”按钮。
6. 点击返回哨所后请求 ReturnToBaseLoadingScreen，不直接切回局外。
7. UI 只调用 SettlementTransferService/SettlementReturnController，不直接写仓库。
```

## 6. 验收标准

- 成功结算展示可入库物品。
- 失败结算展示损失原因。
- 物品来源和状态清楚。
- 返回哨所按钮会进入返回 loading。

## 7. 暂不实现

- 评分系统。
