# 08-07 结算界面与物品展示

> 来源文档：`08_撤离与结算系统_修订版_废土生存法则.md`
> 施工补充：开工前必须读取 `15_模块功能规则细化与AI开工提示词.md`。

## 1. 实现目标

结算界面让玩家看清本局结果、获得/损失物品、统计数据和可执行操作。

## 2. 必须显示

- 成功或失败状态。
- 结束原因。
- 结算物品列表。
- 每个物品的名称、数量、品质、来源。
- 一键入库入口。
- 手动入库入口。
- 丢弃入口。
- 返回局外入口。

## 3. 建议显示

- 探索时间。
- 拾取总量。
- 修复前哨数量。
- 撤离点状态。
- 丢失物品摘要。

## 4. 物品展示

物品卡片字段：

```text
icon / display_name / count / quality / source / status
```

品质颜色应与 07 物品系统一致。

## 5. AI/MCP 开工提示词

```text
请实现 SettlementUI。
要求：
1. 显示成功/失败、结束原因、物品列表和本局统计。
2. 物品展示包含名称、数量、品质、来源、状态。
3. 成功时显示一键入库、手动入库、丢弃。
4. 失败时显示损失说明，不显示成功入库主流程。
5. UI 只调用 SettlementTransferService，不直接写仓库。
```

## 6. 验收标准

- 成功结算展示可入库物品。
- 失败结算展示损失原因。
- 物品来源和状态清楚。

## 7. 暂不实现

- 结算动画。
- 评分系统。
