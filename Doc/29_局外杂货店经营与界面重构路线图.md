# 29_局外杂货店经营与界面重构路线图

> 文档版本：v0.2
> 所属项目：《废土生存法则》  
> 关联文档：`02_核心循环规则`、`09_局外仓库系统`、`10_研究所与制作所系统`、`11_出发准备与局外装备系统_修订版_废土生存法则.md`、`19_数值平衡表与掉落权重规范_修订版_废土生存法则.md`、`20_UI视觉规范与界面资源Sheet提示词_修订版_废土生存法则.md`、`23_世界观包装_废土生存法则.md`、`28_工程结构重构路线图.md`、`30_昼夜局外流程_店铺营业与夜间出发准备.md`、`31_掉落物资与制造经济规划.md`。
> 文档目标：明确局外阶段从“页签堆叠”重构为“白天开店经营、夜晚出发探索”的大循环，并约束 BaseScene 拆分、制造所、售卖、需求榜、夜间出发准备和装备携带的落地边界。

## 1. 最新口径

局外不再以“仓库、商人、研究所、制作所、出发准备”并列页签作为主流程。

玩家的身份是：

```text
末世避难所中经营杂货店的店主
```

玩家夜晚去危险区域带回原材料，白天在避难所内把原材料加工成可售卖物资，再通过开店把物资卖给前哨所难民，获得矿币与后续成长资源。

权威循环为：

```text
夜晚出发探索
-> 局内搜集原材料
-> 撤离结算入库
-> 白天准备：制造、转化、仓库整理、研究升级、查看需求榜
-> 白天开店：摆货、补货、匹配难民需求、完成 1 分钟营业
-> 店铺结算：获得矿币和经营结果
-> 进入夜晚
-> 夜间计划：选择角色和地点
-> 出发准备：配置装备和消耗品
-> Loading 进入局内
```

## 2. 废弃旧口径

以下旧设计不再作为局外主流程目标：

- 不再有独立“商人页签”作为核心卖货入口。
- 不再把“订单页签”作为白天经营主界面。
- 不再让“出发准备”常驻在白天页签中。
- 不再允许白天直接点击“出发探索”进入 Loading。
- 不再把局内带回的原材料直接作为难民需求货品。

商人相关旧文档只作为废弃说明和重定向入口保留，不再作为后备经济方案；新实现必须按本文件和 `Doc/30_昼夜局外流程_店铺营业与夜间出发准备.md` 执行。

## 3. 资源口径

局内带回的是“原材料”，白天售卖的是“制造所产出的可售物资”。

```text
局内原材料
-> 制造所配方
-> 可售物资 / 装备 / 消耗品 / 研究材料
```

难民需求榜只面向可售物资，不面向原材料。

示例：

| 类型 | 示例 | 能否被难民直接需求 |
| --- | --- | --- |
| 原材料 | 废金属、脏布条、旧电池 | 否 |
| 可售物资 | 简易炊具包、止血包、净水滤芯、保暖毯 | 是 |
| 装备 | 头盔、护甲、手持工具、靴子 | 默认否，除非配置为可售物资 |
| 消耗品 | 止痛针、镇定剂、照明棒 | 可按配置决定 |

## 4. 白天阶段

白天分为两个阶段。

### 4.1 准备阶段

准备阶段是白天的自由整理期。

玩家可做：

- 在制造所把原材料合成为可售物资、装备或消耗品。
- 在制造所进行同品级资源转化。
- 管理仓库。
- 进行研究升级。
- 查看当日难民需求物资排行榜。
- 查看图鉴。

玩家不可做：

- 不显示“出发探索”按钮。
- 不进入夜间计划。
- 不直接 Loading 进入局内。

准备阶段主行动按钮只有：

```text
开店营业
```

### 4.2 开店阶段

点击“开店营业”后进入开店阶段。

开店阶段持续 60 秒。

玩家可做：

- 从仓库中选择可售物资摆放到货台。
- 对空货台补货。
- 将低需求货品撤下并换成高需求货品。
- 查看当日需求榜和营业中的难民需求提示。
- 当货台和仓库都没有可上架可售物资时，点击“提前结束营业”直接进入结算。

玩家不可做：

- 不可进行研究升级。
- 不可在营业中制造新物资或整理仓库。
- 不可进入夜间计划。
- 不可提前出发探索。

开店阶段详细规则见 `Doc/32_局外白天经营流程/03_开店中营业阶段规则.md` 和 `Doc/32_局外白天经营流程/04_结算与提前结束规则.md`。

## 5. 货台规则

货台是白天开店的核心操作区。

V0.2 默认：

```text
初始货台格子数 = 3
每个格子可摆放 1 种可售物资
每个格子记录 item_id 和 stack_count
```

规则：

- 只有 `sale_good=true` 的制造产物可摆放。
- 原材料默认不可摆放。
- 货台中的物资仍属于玩家库存，但处于 `ON_SHELF` 状态。
- 已摆上货台的物资不能被研究、制造或出发准备消耗。
- 营业结束后，未售出的货品自动回到仓库。
- 后续研究可增加货台格子数。

## 6. 难民需求榜

每天生成一份难民需求物资排行榜。

排行榜用途：

- 告诉玩家今日最高需求的可售物资。
- 决定开店阶段中不同货品的出售速度。
- 给制造所和仓库整理提供白天目标。

榜单只包含可售物资。

推荐展示：

```text
今日需求榜
1. 净水滤芯        火热
2. 止血包          很缺
3. 保暖毯          稳定
4. 简易炊具包      偶尔有人问
5. 粗制工具包      需求较低
```

随机逻辑建议：

```text
daily_seed = profile_id + current_day + demand_version
candidate_pool = 已解锁且 sale_good=true 的制造产物
base_weight = 商品基础需求权重
archetype_weight = 当日难民类型权重
event_weight = 当前章节/天气/剧情修正
anti_repeat_weight = 近 1-2 天高排名商品降权
final_weight = base_weight * archetype_weight * event_weight * anti_repeat_weight
```

榜单生成规则：

- 至少生成 5 个候选排名；候选不足时展示全部。
- 排名第 1-3 位应明显影响营业收益。
- 同一个商品不应连续多日稳定霸榜，除非剧情事件强制指定。
- 若玩家尚未解锁足够可售物资，系统可从基础商品中保底生成。

## 7. 售卖速度

开店阶段不做“直接点一下卖掉”，而是按需求榜模拟难民购买。

售卖间隔建议：

| 需求排名 | 难民购买间隔 | 售价修正 | 体验含义 |
| --- | --- | --- | --- |
| 第 1 名 | 4-6 秒 | 1.30x | 很快卖出，玩家需要补货 |
| 第 2 名 | 7-10 秒 | 1.15x | 稳定卖出 |
| 第 3 名 | 11-16 秒 | 1.00x | 正常卖出 |
| 第 4-5 名 | 18-28 秒 | 0.90x | 偶尔有人买 |
| 未上榜 | 30-45 秒或无人购买 | 0.75x | 可能整天卖不出去 |

每次销售事件：

```text
从当前货台中抽取可购买商品
-> 按需求排名计算购买概率和间隔
-> 扣除货台数量
-> 记录 sold_count / unit_price / subtotal
-> 增加营业收入
```

若难民需求与货台不匹配：

- 难民离开。
- 不扣货。
- 可记录 `missed_customer_count`。
- V0.2 不做惩罚，后续可影响店铺信誉。

## 8. 1 分钟营业玩法

1 分钟开店不能只是等待。核心玩法定位为：

```text
短时补货与需求匹配
```

玩家在 60 秒内需要做的事：

- 根据今日需求榜决定先上什么货。
- 观察到访难民的需求气泡或需求标签。
- 在 3 个货台格子有限空间内调整货品。
- 高需求货品卖空后及时从仓库补货。
- 若发现货台摆错，可撤下低需求货品换成高需求货品。
- 如库存不足，只能从仓库现有可售物资补货、调整货台，或提前结束营业；V0.2 不允许营业中临时制造。

V0.2 不需要做复杂服务小游戏。先把压力放在“货台格子有限、需求榜变化、补货是否及时”上。

后续可扩展：

- 难民临时加价需求。
- 多人排队和耐心值。
- 破损货台维修。
- 店员/助手自动补货。
- 特殊客人带来剧情或稀有图纸。

## 9. 结算面板

60 秒结束后弹出本日开店结算。若玩家在营业中点击“提前结束营业”，也立即弹出同一结算面板，不强制等待倒计时归零。

面板内容：

```text
今日营业结算

物资名        卖出数量    单价    小计
净水滤芯      4           35      140
止血包        2           28      56
保暖毯        1           24      24

未满足需求人数：3

            总收入：220
```

规则：

- 结算面板打开时暂停局外输入。
- 关闭结算面板后，白天结束，进入当日夜晚。
- 未售出的货品回仓库。
- 当日需求榜归档为历史记录，可用于 Debug 或经营日志。

## 10. 夜晚阶段

夜晚不开放白天页签堆叠。

夜晚主界面只有一个主按钮：

```text
出发探索
```

点击后进入夜间计划界面。

夜间计划界面参考《这是我的战争》的信息结构：

- 左侧：出发角色列表。
- 中间：地点地图或地点列表。
- 右侧：当前地点描述、危险等级、物资产出情况。
- 右下：准备按钮。

V0.2 只有：

```text
角色：店主
地点：废弃小屋 / 默认探索点
```

但 UI 和数据结构必须预留多角色、多地点扩展。

## 11. 出发准备

玩家在夜间计划中选择角色和地点后，点击“准备”，进入出发准备界面。

出发准备界面由仓库和角色装备区组成。

装备区要求：

- 左侧使用人形布局，不允许单纯排成一行。
- 从上到下依次为：头部、身体、手持、脚部。
- 人形图下方显示消耗品栏。
- 消耗品栏共 4 格。
- V0.2 只解锁第 1 格，第 2-4 格显示但置灰。

确认装备后，右下角点击：

```text
出发
```

随后进入现有 Loading，再进入局内逻辑。

## 12. BaseScene 重构目标

当前 `scripts/base/base_scene.gd` 承担了过多局外职责。新局外大流程落地前，需要拆成“流程控制 + 子面板 + 业务服务”。

目标：

```text
BaseScene 只做局外场景壳层和阶段切换
业务规则进入 Service/Manager
每个主要界面一个 Panel Controller
UI 不直接改仓库、货币、需求榜、货台、装备配置
```

推荐模块：

```text

scripts/base/outgame_flow_controller.gd
```

## 14. V0.2 最小交付

第一批实现目标：

1. BaseScene 支持局外阶段：白天准备、开店、结算、夜晚、夜间计划、出发准备。
2. 白天准备不显示出发探索，只显示开店营业。
3. 制造所能产出至少 3 种可售物资。
4. 每日需求榜可按天随机生成并显示。
5. 开店阶段有 3 个货台格子、60 秒倒计时、自动售卖、补货、撤货和提前结束营业操作。
6. 结算面板展示卖出物资、数量、单价、小计和总收入。
7. 结算关闭后进入夜晚。
8. 夜晚只有出发探索按钮。
9. 点击出发探索进入夜间计划，选择角色和地点。
10. 点击准备进入装备/消耗品出发准备界面。
11. 点击出发沿用当前 Loading 进入局内。

暂不做：

- 多角色真实差异。
- 多地点真实掉落差异。
- 店员系统。
- 复杂排队服务小游戏。
- 难民关系惩罚。
- 商人独立买卖页签。

## 15. BDD 场景

```gherkin
Feature: 局外昼夜循环与避难所杂货店经营
  作为 避难所杂货店店主
  我希望 白天制造并售卖可售物资，晚上准备出发探索
  以便 局内搜集、局外加工、店铺售卖和再次出发形成稳定循环

  Background:
    Given 玩家已经创建本地档案
    And 玩家可以进入局外 BaseScene
    And 仓库、制造所、研究所和出发准备基础数据已加载

  Scenario: 白天准备阶段没有出发探索按钮
    Given 当前局外阶段为 DAY_PREP
    When 玩家进入局外主界面
    Then 界面应显示开店营业按钮
    And 界面不应显示出发探索按钮

  Scenario: 玩家只能把制造产物摆上货台
    Given 当前局外阶段为 SHOP_OPEN
    And 仓库中有原材料 item_scrap_metal
    And 仓库中有可售物资 good_water_filter
    When 玩家打开货台选择列表
    Then good_water_filter 应可选择
    And item_scrap_metal 不应作为可上架货品出现

  Scenario: 高需求物资更快售出
    Given 今日需求榜第 1 名为 good_water_filter
    And good_water_filter 已摆上货台
    When 开店计时推进 10 秒
    Then 系统应至少尝试一次 good_water_filter 的购买事件

  Scenario: 营业结束后进入结算
    Given 当前局外阶段为 SHOP_OPEN
    And 开店剩余时间为 0 秒
    When 系统结算营业
    Then 应弹出今日营业结算面板
    And 面板应显示每种卖出物资的数量、单价、小计和总收入

  Scenario: 结算关闭后进入夜晚
    Given 今日营业结算面板正在显示
    When 玩家关闭结算面板
    Then 当前局外阶段应变为 NIGHT
    And 界面应只显示出发探索主按钮

  Scenario: 没货时提前结束营业
    Given 当前局外阶段为 SHOP_OPEN
    And 所有货台数量均为 0
    And 仓库中没有可上架 sale_good
    When 玩家点击提前结束营业
    Then 应弹出今日营业结算面板
    And 玩家不需要等待 60 秒倒计时自然结束

  Scenario: 夜晚出发探索先进入夜间计划
    Given 当前局外阶段为 NIGHT
    When 玩家点击出发探索
    Then 界面应进入 NIGHT_PLAN
    And 玩家应看到角色选择、地点选择和地点描述
    And 系统不应直接进入 Loading

  Scenario: 准备界面按人形布局显示装备槽
    Given 玩家在夜间计划中选择了角色和地点
    When 玩家点击准备
    Then 界面应进入 LOADOUT
    And 左侧应按头部、身体、手持、脚部的人形关系展示装备槽
    And 人形图下方应显示 4 个消耗品格子
    And 只有第 1 个消耗品格子可用
```

验收方式：

- 文本验收：检查本文是否保留 `Feature / Scenario / Given / When / Then`。
- 文档验收：检查旧“商人页签/订单页签/白天出发探索”不再作为新局外主流程目标。
- 实现验收：使用 `rg -n "outgame_phase|DAY_PREP|SHOP_OPEN|NIGHT_PLAN|LOADOUT|daily_demand|shop_shelf" scripts scenes data Doc` 检查实现命名。
- 手动验收：新档进入局外后，玩家应能理解“白天开店、晚上出发”的节奏，而不是面对一排无主次页签。
