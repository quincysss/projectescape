# 31_掉落物资与制造经济规划

> 文档版本：v0.1
> 所属项目：《废土生存法则》
> 关联文档：`05_容器刷新与开箱系统`、`07_背包_存储与负重系统`、`10_研究所与制作所系统`、`29_局外杂货店经营与界面重构路线图.md`、`30_昼夜局外流程_店铺营业与夜间出发准备.md`。
> 文档目标：重新规范局内掉落物 `item` 表，建立“局内带回资源 -> 保底商品化或组合制造 -> 白天开店售卖 -> 获得矿币 -> 消耗矿币强化下一次出发和店铺”的物资经济闭环。

## 1. 核心判断

当前掉落物最大问题不是名字不够好，而是用途链不够硬。

如果玩家捡到“底图残片”但不知道它能做什么，它就不是一个好掉落。局内掉落物必须满足：

```text
看到它时知道大概类别和用途
带回去后至少有一个保底出口
常规资源最好有 2-4 个制造或经营去向
在白天经营中能直接或间接变成钱
在装备、家具、消耗品或出发投入中也可能产生取舍
```

本轮重构后，局内掉落物不再以“固定配方材料”为主要价值，而是以“可商品化、可组合、可投入下一轮”为主要价值。

最高原则：

```text
玩家每次夜出不一定拿到最优资源，但不能拿到完全无用的资源。
单个常规局内道具带回后，必须能直接制作出至少一种低利润可售物资。
组合配方只负责提高利润和目标感，不能作为基础营业门槛。
```

## 2. 参考框架转译

这里只参考框架，不参考世界观和命名。

### 2.1 来自《这是我的战争》的框架

可借鉴：

- 局内带回的大多是低解释成本材料。
- 同一材料会被多个设施、升级和制作配方争夺。
- 稀缺感来自“我知道它有用，但今天必须决定先用在哪里”。

不照搬：

- 不做纯以物换物。
- 不把每种材料只绑定一条线。

### 2.2 来自《潜水员戴夫》的框架

可借鉴：

- 局内收集不是终点，白天加工和售卖才是收入来源。
- 需求热度影响玩家当天的生产选择。
- 高价值商品需要更高质量材料和更复杂组合。

不照搬：

- 不做餐厅服务题材。
- 不要求玩家每次都手动完成复杂服务小游戏。

## 3. 新物资分层

本项目物品分为 5 层。

| 层级 | 表内类型 | 说明 | 是否局内主要掉落 | 是否白天直接上货台 |
| --- | --- | --- | --- | --- |
| 原材料 | `material` | 局内主要产出，用于制造可售物资、装备、消耗品、家具或转化 | 是 | 否 |
| 可售物资 | `sale_good` | 制造所产物，用于白天货台售卖 | 否 | 是 |
| 出发消耗品 | `consumable` | 夜晚出发携带，局内使用 | 少量可掉落，主要制造 | 否 |
| 装备/装备部件 | `equipment` / `equipment_part` | 出发准备装备槽使用 | 少量可掉落，主要制造 | 否 |
| 情报/核心/收藏 | `intel` / `core` / `collectible` | 地点解锁、研究、高级制造、收藏 | 少量 | 否 |

规则：

- 白天难民需求榜只需求 `sale_good`。
- 局内普通容器优先掉落 `material`。
- 装备和消耗品可以少量掉落，但不应成为主要经济来源。
- 旧的“地图残片”类道具必须变成有明确系统用途的 `intel`，否则移出常规掉落池。
- 研究所永久研发不消耗 `material`，只消耗矿币并检查前置条件。
- `material` 必须按玩家可理解的类别组织，不以品质阶级作为主要识别语言。

## 3.1 玩家可见资源类别

玩家在局内和局外优先看到资源类别，而不是品级。

第一章推荐只让玩家稳定接触 8-12 个常规资源，配置表中可以预留更多资源，但不要全部进入早期掉落池。

| 类别 | 设计作用 | 示例 | 主要来源 |
| --- | --- | --- | --- |
| 基础材料 | 最低级商品、包装、修补 | 废金属、脏布条、塑料片 | 住宅、普通箱、垃圾点 |
| 零件类 | 维修、工具、设备 | 螺栓、工具零件、管材 | 工业区、工具柜 |
| 组件类 | 电子、照明、信号 | 旧线圈、旧电池、电子组件 | 工业区、电子箱 |
| 食物药品类 | 生存、医疗、冷藏商品 | 药粉、绷带、罐头、酒精 | 诊所、住宅、冰箱 |
| 武器装备类 | 夜出能力和高价值商品 | 刀具、护具、弹药、装备部件 | 检查站、上锁箱 |
| 烟酒珠宝类 | 交易品、奢侈品、顾客偏好 | 香烟、酒、珠宝、怀表 | 商业街、住宅、保险柜 |

品质保留为后台参数，用于掉落权重、售价、结算展示和稀有惊喜；它不再承担“玩家该捡什么”的主要说明责任。

## 3.2 商品化层级

可售物资采用四层结构：

```text
单一道具 -> 基础直制商品，保证捡到就能变现
单类别若干 -> 基础批量商品，保证同类资源有更好出口
多资源组合 -> 进阶商品，提高利润
指定稀有材料或订单 -> 高价值商品，提供目标感
```

同一种资源应有多种去向：

```text
废金属 x1 -> 废料五金包
脏布条 x1 -> 简易包扎卷
废金属 + 布料 -> 修补工具包
废金属 + 电子组件 + 任务条件 -> 简易电器
废金属若干 -> 商品升级、批量产出或家具制作
```

基础直制商品可以利润低、产量低、需求热度低，但不能不存在。它是防止玩家“一趟白跑”的保底出口。

这条规则优先级高于“配方丰富度”。如果玩家只带回 `scrap_metal x1`，也应该能做出某个 `sale_good`，只是收益不如组合商品。

## 4. item 表规范

`items.tab` 仍是所有道具的基础定义表，但不承担配方表职责。

推荐字段：

```text
id
item_type
material_category
name
quality
stackable
stack_limit
weight
icon
base_sale_value
crafting_tags
sale_good_recipe_tags
description
tags
enabled_version
```

新增语义约定：

| 字段/标签 | 用途 |
| --- | --- |
| `item_type=material` | 局内原材料 |
| `material_category=basic/part/component/food_medical/weapon_gear/luxury/outpost` | 玩家可见资源大类，也是地图产出和目标面板推荐的主依据 |
| `item_type=sale_good` | 白天可售物资 |
| `item_type=intel` | 路线、地点、需求情报 |
| `item_type=equipment_part` | 装备核心部件 |
| `tags` 包含 `raw` | 可进入制造配方 |
| `tags` 包含 `sale_good` | 可摆上货台 |
| `tags` 包含 `purchase_core` | 主要通过后勤采购获得的标准化核心材料 |
| `tags` 包含 `route_cost` | 可参与地点解锁、路线费用折扣或远行准备 |

落地要求：

- `material` 默认 `stackable=true`。
- `sale_good` 默认 `stackable=true`，但只存在于局外仓库和货台，不作为局内掉落主项。
- `equipment` 默认不可堆叠。
- `collectible` 不进入核心制造链，避免玩家以为必须刷收藏品才能推进经营。

## 7. 可售物资配方表

新增表建议：

```text
setting/sale_good_recipes.tab
```

表头：

```text
good_id	good_name	quality	category	unlock_day_min	demand_tier	base_price	output_count	ingredient_1	ingredient_1_count	ingredient_2	ingredient_2_count	ingredient_3	ingredient_3_count	ingredient_4	ingredient_4_count	ingredient_5	ingredient_5_count	craft_coin_cost	required_station_level	description
```

规则：

- 基础直制商品必须允许只消耗 1 个具体局内道具。
- 每个常规 `material` 至少配置 1 条基础直制配方。
- 单类别批量商品可以消耗同类别 2-3 个资源或通用类别需求。
- 进阶商品才主要使用 2-3 种道具组合。
- 每个配方需要 1-5 种道具。
- 少数高价值配方需要 4-5 种道具。
- 配方可产出多个同名可售物资，解决背包 8 格无法支撑开店的问题。
- 早期配方优先吃资源类别，后期配方再要求指定材料。

建议增加字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| recipe_tier | enum | direct_basic/category_basic/combo/order |
| ingredient_mode | enum | item/category/mixed |

其中 `direct_basic` 必须是“单一道具 -> 可售物资”，不能写成两种材料组合。

品质规则退到后台：

```text
单个低级资源 -> 基础直制商品
同类别低级资源 -> 基础批量商品
同类别高价值资源 -> 同类高价商品或更多产出
跨类别组合 -> 进阶商品
稀有指定物 -> 订单、高级商品或剧情商品
S 级不进入普通可售物资链，除非是特殊剧情、收藏展示或高价事件
```

## 10. 局内掉落调整

背包只有 8 个格子时，如果所有道具都不可堆叠，白天经营一定供不上货。

新规则：

```text
材料可堆叠
可售物资不在局内常规掉落
装备和核心部件不可堆叠
负重比格子更重要
```

推荐初始局内携带：

| 项目 | V0.2 建议 |
| --- | --- |
| 背包格子 | 8 格保留 |
| C 级材料堆叠上限 | 20 |
| B 级材料堆叠上限 | 6-12 |
| A 级材料堆叠上限 | 3-5 |
| 初始负重上限 | 10.0 |
| 轻度超重 | 10.0-13.0，移动速度轻微下降 |
| 严重超重 | 13.0-16.0，移动/战斗明显受影响 |
| 硬上限 | 16.0，禁止继续拾取或必须丢弃 |

一次成功撤离的期望带回：

```text
6-8 个背包格被占用

其中 C 级约 65%-75%
B 级约 20%-30%
```

## 11. 需求热度随天数推进

白天难民需求不应一开始就大量需要 A 级物资。

| 天数 | 需求结构 | 设计目标 |
| --- | --- | --- |
| Day 1-3 | C 级占 80%，B 级占 20%，不刷 A | 教玩家基础制造与货台补货 |
| Day 4-7 | C 级占 55%，B 级占 40%，A 级占 5% | 让玩家开始追 B 级材料 |
| Day 8-12 | C 级占 30%，B 级占 55%，A 级占 15% | 让中级地点和装备准备变重要 |
| Day 13+ | C 级占 15%，B 级占 50%，A 级占 30%，S 特殊需求 5% | 后期货币缺口和高级地图闭环 |

需求榜生成时：

- 前期高热物资优先是 C 级生活、净水、维修类。
- 中期高热物资逐渐转向 B 级医疗、照明、工具类。
- 后期高热物资增加 A 级路线、净水、医疗、扫描类。

## 12. 制造所与装备/消耗品

推荐方案：

```text
制造所可以制造可售物资、装备、消耗品。
可售物资主要消耗局内原材料，不额外收取大量货币。
装备和消耗品消耗局内原材料 + 标准化核心材料 + 货币。
```

理由：

- 如果制造可售物资也大量花钱，玩家会被卡在“没钱所以不能赚钱”的死循环。
- 装备和消耗品服务于夜晚探索，是天然的货币消耗口。
- 装备/消耗品消耗货币，能形成“今天赚的钱要不要投入下一次风险”的决策。

包装：

```text
店主能用废料做货品。
但战斗装备、医疗消耗品和高级工具需要稳定零件、药剂基底、标准接口、电力校准和设备工时。
这些成本以货币体现。
```

不建议做：

- 不建议把装备和消耗品完全改成只能购买。
- 不建议新增独立交易主界面来承接装备和消耗品。
- 不建议让后勤采购直接卖成品强装备，否则制造所价值会被稀释。

建议做：

- 在制造所中增加“缺少标准件时可后勤订购”的入口。
- 标准件只作为配方材料，不作为玩家主要刷取目标。
- 订购标准件消耗货币，并按天数或设施等级限制种类。

标准件示例：

| standard_part_id | 名称 | 用途 | 获取方式 |
| --- | --- | --- | --- |
| `std_fastener_box` | 标准紧固件盒 | 装备、货台升级、工具类消耗品 | 后勤订购 |
| `std_medical_base` | 药剂基底 | 医疗消耗品 | 后勤订购 |
| `std_power_socket` | 标准电力接口 | 电力装备、照明装备 | 后勤订购 |
| `std_filter_case` | 滤芯外壳 | 净水、面罩、过滤装备 | 后勤订购 |
| `std_armor_lining` | 防护内衬 | 身体装备、头部装备 | 后勤订购 |

## 13. 货币消耗口

货币必须不只用于研究。建议按“日常经营成本 + 出发成本 + 长期成长”三类消耗。

### 13.1 出发成本

第一个地点免费。

后续地点需要支付远行费用，包装为坐车、路线通行、燃料和向导成本。

| 地点等级 | 示例地点 | 费用 | 说明 |
| --- | --- | --- | --- |
| 0 | 废弃小屋 | 0 | 教学和保底 |
| 1 | 旧居民区 | 20-35 | 低风险，basic / food_medical / luxury 的 C/B 材料 |
| 2 | 废弃诊所 | 45-70 | food_medical 权重高，少量 basic 和 luxury |
| 2 | 修理厂 | 50-80 | part / component 权重高 |
| 3 | 信号塔 | 90-130 | component / outpost 权重高，A 级概率上升 |
| 4 | 封存设施 | 160-240 | component / weapon_gear / outpost 的 A/S 权重高，高风险 |

路线情报残页可用于：

- 解锁新地点。
- 降低下一次远行费用。
- 提前显示地点产出权重。

### 13.2 制造和维护成本

| 消耗项 | 用途 |
| --- | --- |
| 装备制造工时费 | 制作头部、身体、手持、脚部装备 |
| 消耗品加工费 | 制作药剂、照明、稳定类道具 |
| 装备维修费 | 修复耐久，价格低于重做但需要材料 |
| 货台维护费 | 扩展货台格、提高陈列效率 |
| 仓库整理费 | 扩容、分类、自动回收 |

### 13.3 经营成本

| 消耗项 | 用途 |
| --- | --- |
| 今日需求情报 | 提前看到更完整的需求榜或下一日趋势 |
| 货台扩建 | 3 格 -> 4 格 -> 5 格 |
| 店铺照明 | 提高营业期间高需求客流概率 |
| 包装材料 | 提高指定可售物资售价或产出数量 |
| 临时助手 | 营业期间自动补 1 个货台格 |

### 13.4 长期成长

| 消耗项 | 用途 |
| --- | --- |
| 研究升级 | 背包、负重、仓库、稳定值、货台 |
| 地点解锁 | 支付远行资格、路线图整理 |
| 设施解锁 | 制造所等级、后勤订购等级 |
| 风险抵消 | 购买保险、撤离救援、夜间交通 |

## 14. 装备和战斗经济

装备/消耗品要成为战斗和探索的“花钱理由”。

装备方向：

| 部位 | 经济作用 | 战斗/探索作用 |
| --- | --- | --- |
| 头部 | 高级镜片、热成像模块、信号线圈 | 视野、异常提示、弱点提示 |
| 身体 | 防护内衬、加固背带、布料 | 抗伤、负重、稳定损失降低 |
| 手持 | 工具零件、电力接口、线圈 | 开箱效率、照明、战斗安全 |
| 脚部 | 金属件、胶带、背带 | 移速、噪音、负重惩罚降低 |

消耗品方向：

| 类型 | 经济作用 | 战斗/探索作用 |
| --- | --- | --- |
| 医疗 | 药粉、无菌材料、药剂基底 | 恢复/止血 |
| 稳定 | 糖、药粉、化学材料 | 稳定值补救 |
| 照明 | 电池、线圈、镜片 | 黑暗区域探索 |
| 工具 | 工具零件、紧固件、电力接口 | 快速开箱/拆障碍 |

建议：

- 装备有耐久，成功带回也会扣耐久。
- 维修需要货币和材料。
- 高级地图最好需要一定装备或消耗品准备，否则钱只会堆积。


## 16. V0.2 最小交付

V0.2 不需要一次做完整大表。

最小可玩版本：

- 12 个原材料。
- 至少 16 个可售物资，其中必须包含 8-12 个单件直制基础商品。
- 每个 C/B 常规原材料至少有 1 个单件直制 `sale_good` 配方。
- 每个 C/B 常规原材料至少覆盖 3 个制造或经营去向。
- C 级材料可堆叠到 20。
- B 级材料可堆叠到 8-12。
- 一次撤离能带回 18-30 个材料单位。
- 前 3 天需求榜只刷 C/B 低级可售物资。
- 第一个地点免费，第二个地点开始收远行费用。
- 装备和消耗品制造需要少量货币和标准件。

## 17. BDD 场景

```gherkin
Feature: 掉落物资与制造经济规划
  作为 避难所杂货店店主
  我希望 局内掉落物能稳定转化为多种可售物资、装备和消耗品
  以便 每次探索、制造、开店和出发都形成明确取舍

  Scenario: 常规原材料至少覆盖多个制造去向
    Given item_id 为 scrap_metal
    When 系统读取 sale_good_recipes.tab
    Then 至少应存在 1 个 direct_basic 配方只消耗 scrap_metal x1
    And 至少应存在 3 个制造或经营去向引用 scrap_metal

  Scenario: 单个局内带回道具可以直接制作可售物资
    Given 玩家夜间只带回 scrap_metal x1
    When 玩家回到局外打开制造所
    Then 系统应显示至少 1 个可制作的 sale_good
    And 该配方不应要求第二种材料

  Scenario: 局内普通容器产出可堆叠材料
    Given 玩家打开普通纸箱
    When 系统生成掉落
    Then 掉落应优先为 material 类型
    And C 级 material 应允许堆叠

  Scenario: 原材料不能直接成为难民需求
    Given 今日需求榜正在生成
    When 候选池筛选道具
    Then item_type=material 的道具不应进入需求榜
    And item_type=sale_good 的道具可以进入需求榜

  Scenario: C 级配方产出多个可售物资
    Given 玩家拥有 scrap_metal x1
    And 玩家拥有 rusted_bolts x1
    When 玩家制造 good_gear_bundle
    Then 系统应扣除对应原材料
    And 系统应产出 good_gear_bundle x3

  Scenario: 需求热度随天数推进
    Given 当前天数为 Day 2
    When 系统生成今日需求榜
    Then C 级可售物资应占主要权重
    And A 级可售物资不应成为高热需求

  Scenario: 后续地点需要远行费用
    Given 玩家已经解锁旧居民区
    And 旧居民区不是免费教学地点
    When 玩家在夜间计划中选择旧居民区
    Then 出发准备前应校验远行费用
    And 货币不足时不允许确认出发

  Scenario: 装备制造消耗货币和标准件
    Given 玩家准备制造头部装备
    When 配方需要 std_power_socket 和 mine_coin
    Then 系统应同时校验标准件、局内材料和货币
```

验收方式：

- 文档验收：本文包含 item 表规范、可售物资配方表、覆盖矩阵、掉落调整、货币消耗口和 BDD。
- 表格验收：每个 C/B 常规原材料至少有 1 条单件直制配方，并至少被 3 个制造或经营去向引用。
- 体验验收：玩家看到任意常规掉落时，都能在制造所找到一个立刻可做的低利润商品，以及若干更高利润组合方向。
- 经济验收：白天赚到的货币至少可被地点费用、装备/消耗品制造、设施升级三类系统消耗。
